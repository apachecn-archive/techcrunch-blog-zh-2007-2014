# Foursquare 正在测试他们 API 的版本 2。“快了 1000 倍”

地理定位战争正在进行中。如果你需要任何证据，只要看看 Quora 上的这个帖子就知道了。一家初创公司发布了一个问题，想知道他们应该为他们的新签到应用程序使用哪个位置数据库。回应是什么？四方:我们。戈瓦拉:我们。SimpleGeo:我们。事实:我们。地点:美国。

这些公司中的每一家，以及其他几家没有在 Quora 上实践自我推销艺术的大公司(谷歌、脸书、Skyhook)都在进行一场旷日持久的战斗，以成为潜在的位置层，让其他 1000 家创业公司在其上蓬勃发展。

在过去的几个月里，看起来 Foursquare 在这方面做得非常好。其他几家热门创业公司(如 [Instagram](https://web.archive.org/web/20230202225529/http://instagr.am/) 和 [Foodspotting](https://web.archive.org/web/20230202225529/http://foodspotting.com/) )实际上都在使用 Foursquare 的位置数据库作为其应用的关键组成部分。但接下来是脸书地名。现在，初创公司似乎急于实现这一点，以便他们可以利用社交网络的 4 亿多用户和扩展能力。但是 Foursquare 并没有坐在那里静静地看着这一切发生。

正如 Foursquare 联合创始人纳文·塞尔瓦杜莱在 Quora thread 上的[回答](https://web.archive.org/web/20230202225529/http://www.quora.com/Where-can-I-get-place-data-for-a-check-in-app/answer/Naveen-Selvadurai)中透露的，该公司目前正在 alpha 测试他们 API 的第二个版本。虽然它还没有公开发布，但 Selvadurai 指出，你可以给他们的团队发电子邮件来获得它。

那么最新版本有什么新内容呢？我问过联合创始人丹尼斯·克劳利，他说这是对最初 API 的完全重写。他说，现在速度快了 1000 倍，也更灵活了。他还指出，它预示着一些即将到来的新功能，并与特价商品等东西进行了更紧密的集成。负责 API 的[谷歌团队对 v2 有更多的期待。](https://web.archive.org/web/20230202225529/http://groups.google.com/group/foursquare-api/topics)
