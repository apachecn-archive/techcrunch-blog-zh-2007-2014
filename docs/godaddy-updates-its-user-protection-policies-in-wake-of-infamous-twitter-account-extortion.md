# GoDaddy 在臭名昭著的 Twitter 账户敲诈事件后更新了用户保护政策

GoDaddy 已经更新了其账户安全政策，以应对现在臭名昭著的 Twitter 账户敲诈事件。正如 TechCrunch [之前报道的](https://web.archive.org/web/20230130234420/https://techcrunch.com/2014/01/29/why-hasnt-twitter-just-given-n-his-name-back/)，一名黑客声称从 PayPal 获得了 Twitter 用户的最后四个信用卡数字，然后用来说服 GoDaddy 重置他们的账户。

被入侵的 GoDaddy 账户——及其必要的域名集合——被用作勒索用户优秀的 Twitter 账户@N 的杠杆。在黑客攻击以及随之而来的对安全松懈的愤怒、对罪责的否认之后，TechCrunch [大声质问](https://web.archive.org/web/20230130234420/https://techcrunch.com/2014/01/29/why-hasnt-twitter-just-given-n-his-name-back/)为什么 Twitter 自己没有让@N 完整。

事后，我们采访了@N，也就是众所周知的广岛直树，他详细说明了 GoDaddy 应该做的一些事情，以加强其安全性，这些方法可能有助于保护他的帐户:

> 广岛表示:“[双因素认证]无法防止这种情况再次发生。”。“GoDaddy 允许这个家伙通过电话重置一切。只要一家公司只使用一张[信用卡]的最后 4 位数字来验证[身份]，这种情况就会不断发生。他们应该问多个问题。”

GoDaddy 采取了反映广岛所需要的措施。在今天的一条[推文中，该公司声明如下:](https://web.archive.org/web/20230130234420/https://twitter.com/GoDaddy/status/429607819147739137)

> @ N _ is _ stocked 就可以了。我们现在需要 8 位卡号，尝试 3 次后锁定，并以不同方式处理双因素认证帐户。美制细牙螺纹(national fine)

要求更多的信用卡数字很重要。如果有问题的黑客被要求提供这么多信息，这个骗局可能会过早地被揭穿:黑客声称 PayPal 给了他们广岛信用卡的最后四位数字。如果 GoDaddy 的门槛再高一点，我们现在就不会讨论这个问题了。

GoDaddy 能够以上述方式受到威胁令人失望，但新的安全政策应该会降低其客户的未来风险，我也是其中之一。

我已经联系 GoDaddy，要求其解释安全政策的变化，当我得到回复时，我会更新这篇文章。
