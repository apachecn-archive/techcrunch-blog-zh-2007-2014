# 谷歌的无限书柜:无限图书馆的抽象浏览器

不久前，我写了一篇关于互联网最终变得美丽的必要性的文章。问题是，在信息环境中，我们认为世界上的事物是美丽的——杂志和书籍的布局，排版，等等。—必须提供有限的信息。正是这种限制，已知的数量方面，让设计师有效地工作。

那么，你应该如何设计一些东西，通过一个相当有限的媒介(比如一个网络浏览器)有效地呈现无限的信息(比如世界上所有的书)？谷歌有一个想法。把它们放在[一个巨大的螺旋](https://web.archive.org/web/20230203055721/http://googleblog.blogspot.com/2011/10/designing-infinite-digital-bookcase.html)上。

【YouTube http://www.youtube.com/watch?v = 6 gqhjdpi-Ug & feature = player _ embedded w = 640]

正如你所看到的，他们已经建立了这个基于 WebGL 的演示程序，并运行了大约 10，000 本被 Google Books 编入索引的书籍。它是按流派划分的，你可以在流派之间缩放，产生惊心动魄的效果。让我想起了巴别塔:

[![](img/d42f3dc59aaed606316e66ec91317261.png "tower_of_babel.170113154")](https://web.archive.org/web/20230203055721/https://techcrunch.com/wp-content/uploads/2011/10/tower_of_babel-170113154.jpg)

但这真的是人们想要导航的东西吗？大概不会。人们喜欢在他们的数字目录中有类似的东西，这一个似乎有点太离谱了。当然，有成排的书。但它也是一个延伸到天空的螺旋。人们习惯于看到他们的书书脊向外排列——这不一定是世界上最好的事情，但它是一种优越的信息密度。我想知道是否把人放在里面比放在外面更好？

总之，[这是一个有趣的小实验，你可以在这里尝试一下](https://web.archive.org/web/20230203055721/http://workshop.chromeexperiments.com/bookcase)。给 Mac 笔记本电脑用户的提示:小心你滑动的方式，否则你可能会不小心离开页面或者调用一些神秘的手势。
