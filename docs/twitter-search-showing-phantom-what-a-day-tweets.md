# 谁在搞乱 Twitter 搜索？

![](img/97ef7717fe3d11926f5e4f3e400b06bb.png)

一名线人今天早上告诉我们，Twitter 的搜索引擎(在今年夏天被收购之前的名字是 Summize)一直在搜索结果中显示 Twitter 上并不存在的推文。

这些“幽灵”推文都包含“多么美好的一天:)”的信息。星期三，2008 年 10 月 8 日”加上一个军事格式的时间附在最后。你可以通过搜索[“多好的一天 10 月”](https://web.archive.org/web/20221007161220/http://search.twitter.com/search?q=What+a+day+Oct)或者简单的[“多好的一天:)”](https://web.archive.org/web/20221007161220/http://search.twitter.com/search?q=%22what+a+day+%3A)%22)来找到它们。这些推文太多了，以至于“10 月的一天”一词一度成为主页上的热门推文。

我们怎么知道这些推文实际上不存在？只需点击任何一个列出的用户帐户，你就会看到他们不会出现在用户的档案中。

尚不清楚是什么导致了这些不准确的结果。如果搜索被黑客入侵，这将引发人们对 Twitter API 安全性的担忧。如果不是，这些推文实际上以某种方式起源于 Twitter(并非不可能，特别是考虑到许多受影响的帐户看起来是多么虚假)，那么它只是表明垃圾 Twitter 搜索是多么容易。
