# Mixpanel 旨在帮助开发者通过新的应用内通知传递有针对性的信息

移动分析公司 [Mixpanel](https://web.archive.org/web/20221005151736/http://www.mixpanel.com/) 今天发布了一项功能，联合创始人 Suhail Doshi 表示，这项功能将为开发者提供一种与用户沟通的新方式。

Mixpanel 的客户现在将能够向他们的用户提供“[应用内通知](https://web.archive.org/web/20221005151736/https://mixpanel.com/inapp-notifications/)”——Doshi 将它们比作推送通知(T4 其他创业公司提供支持的领域)，除非用户实际上正在看应用，而不仅仅是浏览他们的主屏幕。

例如，游戏开发商可以使用通知来推广新的级别或提供提示，或者电子商务应用程序可以推广特定的交易或产品。基本上，这和你使用推送通知的目的是一样的，但是希望不要那么烦人，并且在用户更愿意接受你的消息的情况下。

为什么开发者不能在他们的应用中加入这种功能呢？Doshi 建议，拥有大型移动团队的资金充足的公司可以做到这一点，但对于较小的组织或独立开发人员来说，这就更难了。他还表示，基于 Mixpanel 的分析工具构建它可以提供非常有针对性的信息，比如说，如果你想通过特定的应用版本或基于特定城市的用户来获得信息。事实上，Doshi 表示，由于 Mixpanel 已经建立了个人用户的档案，开发者可以“无限深入”他们的目标——迄今为止，它已经有 1.6 亿个档案，并且每个月还会增加 2000 万个。

Mixpanel 的分析还将允许开发人员查看他们的通知是否有效，Doshi 建议，在未来，他们还可以 A/B 测试不同的通知，然后切换出不起作用的变化。

我一直说这将被开发人员使用，但是 Doshi 指出这也可以被“非技术”人员使用。他演示了一个通知编辑器，它允许客户在没有任何编程的情况下调整通知的视觉设计和措辞——事实上，Doshi 演示的较小变化只需点击几下。

这不是 Mixpanel 第一次推出非分析产品。去年秋天，它推出了移动调查。Doshi 说，更广泛的目标是建立在公司的数据和现有技术基础上，创建一套更广泛的开发工具，如网络上的“Marketo 和 Eloqua 和 Salesforce”提供的工具:“这些东西在移动生态系统中不存在。”
