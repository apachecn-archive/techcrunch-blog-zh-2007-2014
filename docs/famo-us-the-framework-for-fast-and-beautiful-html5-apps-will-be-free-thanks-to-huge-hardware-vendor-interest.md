# Famo.us 是一个快速而漂亮的 HTML5 应用程序框架，由于“巨大的硬件供应商兴趣”,它将是免费的

> 原文：<https://web.archive.org/web/https://techcrunch.com/2013/04/01/famo-us-the-framework-for-fast-and-beautiful-html5-apps-will-be-free-thanks-to-huge-hardware-vendor-interest/>

专为在纯 HTML5 中构建丰富(但速度超快的)界面而设计的 JavaScript 框架 Famo.us 今天上午在旧金山举行的 HTML5 开发者大会上发布了两项声明:由于一些“大型硬件供应商”的兴趣，该框架将对开发者免费，并且它将获得一个物理引擎。

在之前，我们曾经写过一两次关于 [Famo.us 的文章，但对于那些错过它的人来说:Famo.us 是由史蒂夫·纽康在 2011 年创立的，就在他的语言处理公司](https://web.archive.org/web/20221206102738/https://beta.techcrunch.com/tag/famo-us/) [Powerset](https://web.archive.org/web/20221206102738/http://www.crunchbase.com/company/powerset) 被微软以 1 亿美元收购并整合到 Bing 中的三年后。当 Famo.us 在 TechCrunch Disrupt SF 2012 上首次亮相时，它让一些评委感到困惑，但仅仅六个月后，famo . us 就筹集了 400 万美元的首轮融资。

简单地说:Famo.us 耍了一大堆聪明的花招[让网络开发者可以利用设备(无论是电脑、智能电视、平板电脑还是手机)的 GPU 进行计算，不需要插件。对开发人员来说，这意味着能够构建更丰富、更快速的界面。对最终用户来说，这意味着超级时髦的用户界面，无需安装任何插件。](https://web.archive.org/web/20221206102738/https://beta.techcrunch.com/2012/10/16/famo-us-reveals-more-details-about-its-html5-turbo-charger/)

有了几百万美元的资金，Famo.us 将如何开始赚钱呢？

他们可以向开发商收费。有偿框架并非闻所未闻，尽管它们往往是例外。不过，这不是 Famo.us 的计划。截止到今天早上，Famo.us 已经确认他们的框架将对开发者免费开放，“你想要多少应用程序，你想要多少用户。”

相反，Famo.us 依赖于一些大型硬件供应商的兴趣，这些供应商希望 Famo.us 在未来的设备上支持他们的 UI。他们将构建(或帮助构建)ui，然后向供应商收取许可费。此外，Famo.us 将提供可选的企业插件(想想分析，或者记录/回放用户会话的能力，以查看他们如何导航您的设计)。

确切地说，哪些硬件供应商对此感兴趣目前还处于严格的保密状态，但我们肯定可以缩小范围。只有那么多“巨大的”硬件公司有足够的财力让这样的事情变得有价值。苹果现在主要关注本机代码(尽管他们通过 WebKit 为 HTML5 做了更多的贡献)，所以可能不是他们。与此同时，三星和 LG 都在为他们即将推出的硬件向专注于 HTML5 的操作系统(分别是 Tizen 和 webOS)投入资金。

最后，Famo.us 还宣布他们的框架，到目前为止只专注于渲染，已经在这个过程中获得了一个物理引擎。史蒂夫说，他们开始寻找一个符合三个标准的物理引擎:它必须很快，必须在移动设备上工作，所有的数据必须以一种谷歌友好的方式呈现给[文档对象模型](https://web.archive.org/web/20221206102738/http://en.wikipedia.org/wiki/Document_Object_Model)。当他们找不到符合上述所有条件的产品时，他们决定自己生产。

对于一个主要用于界面以提供物理功能的框架来说，这似乎很奇怪——毕竟它不是一个游戏引擎——但它是有意义的:当你在空间(无论是 2D 还是 3D 空间)中处理物体时，能够处理与现实世界平行的力是非常有利的，如质量、重力和阻力。

另外，这与史蒂夫上个月向安东尼·哈[解释 Famo.us 的方式非常吻合:](https://web.archive.org/web/20221206102738/https://beta.techcrunch.com/2013/03/03/famous-series-a/)

"我们建立了一个低劣的游戏引擎，基本上是有史以来最好的应用程序引擎."

Famo.us 目前处于测试注册模式，史蒂夫说他们有大约 27，000 名开发人员等待开始建设。感兴趣的开发者可以在这里找到[报名页面。](https://web.archive.org/web/20221206102738/http://famo.us/)
