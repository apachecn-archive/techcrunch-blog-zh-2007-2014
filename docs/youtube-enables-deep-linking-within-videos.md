# YouTube 支持视频内部的深度链接

![](img/6c0a919cd077ec4c328a9debb2867563.png)

这不是一个大的新功能，但它肯定会派上用场: [YouTube](https://web.archive.org/web/20230308110341/http://www.youtube.com/) 现在可以让你通过在视频的 URL 末尾添加一个短标签，将用户发送到视频中的特定点。令人惊讶的是，这项功能没有更早推出，因为谷歌视频在两年前推出了同样的功能[。YouTube 用户被迫依赖第三方服务，如](https://web.archive.org/web/20230308110341/http://googlevideo.blogspot.com/2006/07/new-feature-link-within-video_19.html) [Splicd](https://web.archive.org/web/20230308110341/http://splicd.com/) 来做同样的事情。

要指定一个点，请使用以下语法在视频链接的末尾附加一个标记:“#t=1m45s”(您可以更改“m”和“s”前面的数字，分别编辑分钟和秒钟。

这里有一个例子:
[http://www.youtube.com/watch?v=1bibCui3lFM#t=1m45s](https://web.archive.org/web/20230308110341/http://www.youtube.com/watch?v=1bibCui3lFM#t=1m45s)

作为另一个很好的接触，YouTube 还会自动检测何时有人在视频评论中提到时间，并在视频中添加指向该点的超链接。

该功能刚刚上线，如果不手动添加标签，没有任何简单的方法可以通过 YouTube 用户界面完成(YouTube 也没有在其官方博客上提到它)。

YouTube 团队也很有幽默感。正如本月早些时候[所报道的](https://web.archive.org/web/20230308110341/http://www.mattcutts.com/blog/youtube-adds-read-comment-aloud-feature-from-xkcd/)，YouTube 评论者现在会看到一个按钮，询问他们是否愿意在发表评论之前听到他们的评论被大声朗读，希望以此防止意外愚蠢的情况发生(该功能是对[这部现已闻名的漫画](https://web.archive.org/web/20230308110341/http://xkcd.com/481/)的回应)。
