# Brightcove 的新播放器旨在提高 HTML5 视频兼容性

在三种不同的设备上观看网络视频内容可能会产生三种不同的结果。即使有可能让整个视频正常播放，在播放器如何报告自己，如何请求和缓存视频，以及如何过滤或显示视频方面也经常存在不一致。这对用户来说已经够糟了，但对广告来说可能是致命的。如果不能保证内容，怎么保证广告？

Brightcove 正在用一组新的兼容 HTML5 的“智能播放器”和一个试图识别和解释 Webkit 渲染器、操作系统等之间差异的生态系统来正面解决这个问题。如果你感兴趣的话，他们为你准备了一本关于 T2 的小册子。

本质上，他们已经为 HTML5 视频创建了一个中间层，类似于他们多年来为 Flash 容器创建的中间层。Brightcove 抽象层将充当视频容器和请求它的任何东西之间的中间人。通过这种方式，Brightcove 可以消除浏览器之间已知的不一致，充当某种解释器，确保视频正确加载，跟踪和广告正确显示。来自 Flash 容器的定制选项现在也扩展到了 HTML5。

自然，影响标签的碎片化也是各种浏览器和操作系统需要解决的问题。但是，如果你是一个广告客户，并且现在就想做好应对未来的准备，你不会想等 Mozilla 或苹果公司把他们的代码升级到代码。
