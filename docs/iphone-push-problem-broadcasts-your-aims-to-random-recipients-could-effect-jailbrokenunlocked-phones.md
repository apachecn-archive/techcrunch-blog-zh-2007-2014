# iPhone 推送问题向随机接收者广播你的目标，可能会影响越狱/解锁的手机

开发公司 [Equinux](https://web.archive.org/web/20221007104503/http://www.equinux.com/us/start/index.html) 的创始人[![18630461](img/f6214c448425a8ed157b7a5172de5b7c.png "18630461")](https://web.archive.org/web/20221007104503/https://beta.techcrunch.com/wp-content/uploads/2009/07/18630461.jpg)
[Till Schadde](https://web.archive.org/web/20221007104503/http://twitter.com/TillSchadde)发现了一个漏洞——实际上是一个广播错误——在你不知情或未经同意的情况下，将你的 AIM 消息发送给随机的接收者。这个问题似乎发生在解锁/越狱的 iPhones 上，并导致收件人的主屏幕上出现警告，显示您的信息。

蒂尔通过使用 iChat 从 OS X 桌面向他的 iPhone 发送 AIM 来测试这项服务。然后他收到了一个随机收件人的回复。很明显，这是消息寻址中的一个推送问题——每部 iPhone 都被分配了自己的标识符，并从苹果公司运营的中央服务器接收消息——尽管这可能会改变。

没有关于此漏洞影响哪些 iPhones 的信息。

这个集中式系统显然在解锁和越狱的 iPhones 上遇到了麻烦。也许是激活过程中的某些东西导致了 id 的交叉？我们很快就会与通话，但在此之前，要避免通过 AIM 发送太多的个人身份信息，反正你也一直在这么做，对吧？

众所周知，你不应该在任何网络应用上说任何你不会在拥挤的房间里大喊大叫的话。
