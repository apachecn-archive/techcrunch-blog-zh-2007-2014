# 浏览器枪战显示了性能上的微小变化——这仍然是个人喜好的问题

浏览器大战目前正处于紧张的暂停状态。一个曾经显而易见的优点和另一个的缺点不能像一年前那样指望了，快速变化的标准和交互方法产生了一种令人不安的缓和，而每个人都在等待相当于曼哈顿计划的浏览器将他们推入原子时代。

汤姆的硬件公司刚刚对 Windows 7 和 Ubuntu 上可用的浏览器做了一次彻底的检查，结果真是喜忧参半。以前是火狐一直赢，我们都可以取笑 IE。然后 Chrome 来了，赢了所有的速度基准。然后是歌剧。现在很乱。你如何选择最适合你的浏览器？简单:你抛硬币。

基准测试就是你所期望的:符合标准、HTML5 的东西、Javascript 速度、硬件加速。各方的胜利和失败混杂在一起，你几乎无法得出任何结论。最终，赢家(Firefox)远非决定性的，往往在亚军 Chrome 擅长的事情上“较弱”。反之亦然。与此同时，Opera 取得了一些胜利，IE 在内存管理方面是最好的(但其他方面没什么)，Safari 在页面加载时间方面也是如此(但其他方面没什么)。

![](img/f16e377e087379cc146ba9275765df27.png "mixy")

为什么我说你可以抛硬币呢？因为首先不要用 Safari。它还不够好，除非你被迫使用它来达到某种目的，否则它不会比其他东西更好，反而有很多缺点。并且不要使用 IE(测试除外)，因为尽管有一些有趣的功能，它仍然很难正确渲染。你可以使用 Opera，但是如果你这样做了，你就已经做出了决定，并且不太可能在短时间内反悔。但是对于那些想要使用流行插件的“普通”用户来说，确保与各种网页和应用程序兼容，并确保非常规律的更新时间表，Firefox 和 Chrome 确实是你唯一的选择。

你应该选择哪一个？如果你真的不知道，抛硬币。如果你试图在它登陆火狐时反驳它，那就安装 Chrome。不喜欢就装火狐。或者两者都用。事实就是现在，对于普通用户来说，没多大关系，两个浏览器都很棒。我使用 Firefox 的原因是微不足道的，但却比性能或幕后差异更重要。但除了我自己，我不会向任何人推荐 Chrome。

不要误解我的意思:浏览器之间有*和*的差异，如果浏览器是你工作中的一个关键任务，或者你非常担心某一类型的性能，那么结果是值得一试的。当然，在几个月内，所有这些位置可能会颠倒过来。所以你必须再把硬币拿出来。
