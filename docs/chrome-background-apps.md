# 谷歌详细介绍了 Chrome 的后台功能:无需查看即可运行网络应用

谷歌最近的一项任务是在他们的 Chrome 博客上概述 Chrome 的一些更酷的、[更新的](https://web.archive.org/web/20230203024626/https://techcrunch.com/2010/03/01/chrome-experimental-apis/)功能。今天带来了迄今为止最大的一款:[后台应用](https://web.archive.org/web/20230203024626/http://blog.chromium.org/2011/02/amping-up-chromes-background-feature.html)。

这个特性已经在各种版本的 Chrome/Chromium 上存在了一段时间了。但是谷歌并没有吹捧它，也不清楚它将如何使用和/或有用。今天，很明显。同样，非常酷。从本质上讲，基于 Chrome 的网络应用将能够一直打开，但隐藏在后台。

为什么会有人想在没有看到 web 应用的情况下使用它呢？有几个原因。首先，这允许浏览器实时通知你某些更新，比如聊天请求或新消息。第二，这可能允许浏览器预先呈现任何页面，这样当你打开它时，它会立即加载。

显然，这种权衡是你在后台运行一个应用程序所带来的内存消耗，但可以推测这些可以被编码为对内存使用产生最小的影响。

从本质上说，这给了网络应用一个“推送”通知功能，就像苹果为 iPhone 和 iPad 所做的那样。这有点不同，因为该系统使用苹果自己的服务器来 ping 你的手机，根本不需要打开应用程序，但效果是一样的。用户不会意识到应用程序是打开的，但仍会收到提醒。

当然，这存在安全风险。但是谷歌是这样解决的:

> 为了保护我们用户的隐私，我们已将此功能仅提供给应用和扩展；正规网站将无法打开后台窗口。开发者还需要在他们的应用程序中声明“后台”功能。

这比安全问题更重要。它可以让安装网络应用变得非常有用。到目前为止，Chrome 网络商店中的大多数应用程序都与网络上已经存在的应用程序相似。但是对于这个系统，你必须“安装”一个应用程序来让这个系统工作。
