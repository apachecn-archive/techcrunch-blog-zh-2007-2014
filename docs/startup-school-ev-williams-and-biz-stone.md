# 创业学校:Ev Williams 和比兹·斯通承认，甚至 Twitter 一开始也认为 Twitter 很蠢 

> 原文：<https://web.archive.org/web/https://techcrunch.com/2009/10/24/startup-school-ev-williams-and-biz-stone/>

![](img/07d8a50d848869a0c9e6677b07931199.png)

Ev Williams 和比兹·斯通刚刚登上创业学校的舞台，在那里他们将回答观众的许多问题。您可以通过推文提交一个类似“@ poll _ _ _ _ _ _ _ _ _ _ _”(其中空格是您的问题)的问题。我会在博客上直播这次会议。

问:Twitter 背后的原始动机是什么？
Biz:我们应该从 Odeo 开始。我们在 Odeo 工作，我们对播客服务没有应有的热情。我们没有使用它，这是一个问题。Twitter 的出现是因为 Ev 给了我们一些思考不同问题的自由。

Biz:我们(杰克·多西和斯通)做了一件事，我们有两个星期的时间来做东西和演示。建造它，在周末试用它。如果坚持，我们可以继续努力。我在热浪中扯下地毯，然后我的手机在我的口袋里震动，它是 Ev。上面说他在喝黑皮诺。我意识到我完全被这个产品吸引住了。所以我们决定继续努力。一开始是“好吧，这看起来很有说服力”。早先有人说“twitter 很有趣，但它没什么用处”。曾经说过“冰淇淋也不是”。只是好玩又怎么样？SXSW 2007 对我们来说是一个巨大的分水岭，我们第一次看到了该工具的真正潜力。看到人们在推特上谈论一个很好的会议。

问:第一个版本花了多长时间构建？
比兹:这看起来很像杰克画的一幅简单的素描。一个更新的盒子，当时连个问题都没有。我想它在说“你的情况如何。”它也非常专注于手机短信。整个想法是关于短信，这就是为什么它是 140 个字符。但是有一个概念是完全不同的——因为它是状态，它不是基于时间重复它们，而是像好友列表一样显示它们。第一个版本的商标是有史以来最丑的。
Biz:第一个版本有点滑稽，但很可爱，但后来我们变聪明了。我们学到了，“不要自作聪明”。除非是 t 恤，那就 ok。
问:当你第一次推出这个的时候，你意识到它会有多大吗？是的，我们知道巴拉克·奥巴马会用它。我知道两年后我会上奥普拉脱口秀。

问:伸缩性怎么样，这是个问题吗？我们告诉创业公司，如果你有这个问题，你是幸运的……【to】Ev:这是一个严重的问题，但我不会改变这个建议。我们的问题不是它爆炸了，无法扩展，而是做出了一些错误的选择。我们一次又一次学到的最重要的一课就是专注。少做事。当核心的东西不起作用时，我们做了各种各样的事情(Twitter 通过 IM，a IM)。到了我们开始拉东西进来的时候，这更痛苦，因为有一部分用户真的喜欢它。我们的工程师内部组织得不是很好。许多聪明的家伙完成了惊人的壮举，但他们是一个小团队，并在我们前进的过程中不断完善。
Biz:即使只有十个人，我们也认为我们都知道每个人在做什么，但这是错误的。你仍然需要努力沟通。

问:你是如何估计市场规模的？
伊娃:我们没有。接近初创公司有不同的方式。我欣赏谷歌的一点是，我们说“这个东西很大，我们要打败它”。这是企业的 CS 方法。另一种方法是“如果我们成功了，这可能会成为一件事”。我和 Blogger 在谷歌的时候遇到了这个问题。人们问博客的市场有多大？我说“我不知道”，但是如果我们把它做得很棒，很多人会去做。Twitter 没有市场，除了我们知道它很酷。
Biz: **如果我们认为这是浪费时间的话，我们的一些工程师认为这是浪费时间，“如果你给它加上视频，也许我会继续工作”。**

问:你早期得到的建议中有没有你希望得到却没有得到的？
*沉默*。我记得有一次周末回家，带着这个关于 Odeo 的天才计划回来，让我看一看。这是我们如何使它成为一个成功的企业的整个计划。我想，哇，这真是天才，但是播客也是如此。然后睡在上面。我们想成为播客之王吗？我们一直在仔细检查。

问:你对任何在 Twitter 上开发项目的人有什么建议？
Biz:继续前进当你需要帮助时请告诉我们
Ev:我们希望继续扩大平台，帮助生态系统。twitter 的很多价值来自第三方开发者。我们正在加强我们的平台团队(工程和开发支持方面)。

问:我知道当你开始的时候，人们质疑你在做什么。人们误解了什么？
Biz:有趣，琐碎，有人称之为互联网的‘宋飞正传’。
Ev:我们俩在博客方面都有相似的经历。博客也有类似的模式。博客不仅仅是一家公司，更是一场运动。我花了很多年为博客辩护。人们问你为什么在网上充斥这些垃圾？

你认为自己会上市吗？Ev:有这个可能。我们的目标是改变世界，我们将尽我们所能建立最好的公司来实现这个目标，发挥 Twitter 的潜力。在某种程度上，上市可能是最有意义的。我们远没有想到这一点。以前我可能会说“这太荒谬了”，但现在我不会这么说了。我们可以比 IPO 或被收购更有创意。

问:API 对你的成功有多重要？
商业:巨大。真的很早，这个承包商在德国工作，不想从德国支付短信。说我想写点东西让我在即时通讯上写。所以我们想，为什么不做一个 API 呢(在 Rails 中是一行代码)。发生了一些事情。Twitterific 的家伙做了很酷的 Mac 应用程序。Twitter 的桌面应用很酷。以前，如果你在网上发帖，它会说“通过”网络或“通过短信”。我们确实“通过 twitterific 发布了”。这只是开始。现在成千上万不同的应用程序。

问:地理定位。
Ev:我们正在将地理定位功能植入 Twitter。我们在侧面得到了初步的支持。您可以通过我们模糊确定的位置进行搜索。位置很大。我们对地理定位感到兴奋的原因是，我们不知道它将如何改变我们的业务。我们在 API 中发布位置，最终在我们的界面中。
