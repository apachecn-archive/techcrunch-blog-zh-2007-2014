# 可读性刚刚成为 Instapaper 的出版商付费层

就我个人而言，我认为可读性 T2 背后的概念非常棒。它给你一种方法，让你可以在网上阅读所有你喜欢的文章，而没有网络上所有的混乱——主要是广告。最近推出的支付组件更有趣，因为它给你一种方式来偿还(以非常小的方式)你经常在网上阅读的你最喜欢的作家/出版物。但是作为一个为这样的出版物工作的作家，显然我是有偏见的。我意识到这个想法并不适合每个人。对一些人来说，这将是一个相当艰难的过程。但卖家可能变得更容易了。

当 Readability [上周重新推出](https://web.archive.org/web/20230203001517/http://blog.readability.com/2011/01/the-new-readability/)时，他们提到了与流行的书签服务 [Instapaper](https://web.archive.org/web/20230203001517/http://instapaper.com/) 的合作。这种整合的主要组成部分是一个可读性移动应用程序，由 Instapaper 的 Marco Arment 在 Instapaper 的 image 中制作(并将很快发布)。这对可读性非常好，因为 Instapaper 已经是 iOS 平台上非常受欢迎的应用程序。但是今天的一点消息更好。正如 Arment [今天在 Instapaper 博客上宣布的](https://web.archive.org/web/20230203001517/http://blog.instapaper.com/post/3127304418)，你现在可以发送你的 Instapaper 日志了。

这意味着您可以使用可读性的支付元素，而根本不需要真正使用可读性(除了最初的支付设置之外)。因此，如果你喜欢可读性，但仍然更喜欢使用 Instapaper，这很好——这给了你两个世界的最佳选择。

当您链接您的帐户时，有一个选项可以发送您在 Instapaper 上保存的所有内容以提高可读性，或者只发送您存档的项目以提高可读性。那也很有意思。它为你提供了一种有选择地向某些出版商/作者付费的方式。

阿门特写道:

> 现在，您可以从 Instapaper 自动支持出版商:只需在 Instapaper 的帐户屏幕中链接您的可读性帐户。每当您保存文章时，Instapaper 都会通知可读性，它们将包含在您的可读性帐户的出版商计算中。
> 
> 你将直接支持你自动保存到 Instapaper 的网站。

同样，这并不适合所有人。有些人会想，当你可以免费使用 Instapaper 时，你为什么要这样做？你当然不需要。但是如果你曾经想要回馈一些东西，这是一个简单的方法，使用一个已经非常流行的服务。

可读性允许你每月贡献少至 5 美元，其中 70%直接给作家/出版商(显然，在你那个月读过的各种各样的书中分开)。这当然是一个可能会失败的模式——但是如果它成功了，对于传统的基于广告的模式之外的网络作者来说，这可能是一个不错的小奖励。

Instapaper 也在测试微支付订阅模式。
