# 澄清困惑:这是“新的 Windows 8 用户界面”，而不是“Metro 用户界面”

自从我们听到关于微软[放弃](https://web.archive.org/web/20221007134934/https://beta.techcrunch.com/2012/08/03/as-windows-8-nears-public-debut-microsoft-ditches-metro-brand/)其新的触摸界面和应用程序的“[地铁](https://web.archive.org/web/20221007134934/http://en.wikipedia.org/wiki/Metro_(design_language))”名称的[第一个谣言](https://web.archive.org/web/20221007134934/https://beta.techcrunch.com/2012/08/09/metro-ms-windows8/)以来，关于新界面应该叫什么名字有很多困惑。是新的 Windows 用户界面吗？Windows 8 风格的 UI？现代用户界面？这些是 Windows 8 应用还是 Windows Store 应用？还是地铁和所有的谣言都是假的？即使就在 Windows 8 发布的前一天，大多数人仍然不知道该叫它什么(我只是在发布前花了大部分时间与微软合作伙伴进行简报)。为了澄清事实，我今天向微软询问了它自己的官方指导(因为在微软内部，人们也在使用不同的名字)。

根据公司发言人的说法，这是微软的官方指导:当你提到用户界面时，它是“新的 Windows 用户界面(UI)”或只是“Windows 用户界面”。当谈到应用程序时，它是“Windows 应用程序”或“Windows 8 应用程序”

然而，这真的消除了困惑吗？我不确定。毕竟，除了面向 ARM 平板电脑的 Windows RT 之外，所有传统的 Windows 应用程序仍然可以在面向 x86 的 Windows 8 上运行。毕竟，即使在 Windows 商店里，也有很多老派的桌面应用。这些是不能从商店直接买到的，但它们仍然列在那里，并附有截图和其开发者网站的链接。

微软为什么改名仍然是一个谜，甚至到现在，微软内部的许多团队仍然使用旧名字(今天的 [Skype 公告](https://web.archive.org/web/20221007134934/https://beta.techcrunch.com/2012/10/22/microsoft-debuts-skype-for-windows-8-offers-modern-design-live-tiles-focus-on-tight-windows-8-integration/)刚刚谈到了“现代设计”)。

旧的“Metro”名称的一大优势是，它强调了这样一个事实，即这些是用于触摸界面的应用程序，而不是桌面应用程序。就像消费者可能会对 Windows RT 和 Windows 8 之间的差异感到困惑一样——毕竟，这两种操作系统都运行在非常相似的机器上——围绕 Metro 的整个困惑并没有帮助处于最关键过渡期之一的微软。
