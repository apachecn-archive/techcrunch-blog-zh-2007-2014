# 零售是店内产品库存的场所

对于零售商来说，使用网络应用来增加实体店的客流量似乎是一种获得回报的策略。今年 3 月， [Forrester 报告](https://web.archive.org/web/20230202212857/https://techcrunch.com/2010/03/08/forrester-forecast-online-retail-sales-will-grow-to-250-billion-by-2014/)称，“在线研究，线下购买”消费者市场代表了 9170 亿美元的消费者支出，占当时美国零售总额的 30%。随着越来越多的开发人员为这个市场构建应用程序，对产品库存和商店位置数据有了需求。进入[retaligence，](https://web.archive.org/web/20230202212857/http://www.retailigence.com/)，一家在 Founders Institute 孵化的秘密创业公司，将[Localeze](https://web.archive.org/web/20230202212857/http://www.crunchbase.com/company/localeze)-模式应用于店内产品库存和列表。

在这家初创公司公开上市的同时，Retailigence 还宣布，它已经从 [Draper Fisher Jurvetson](https://web.archive.org/web/20230202212857/http://www.crunchbase.com/financial-organization/draper-fisher-jurvetson) 、 [Quest Venture Partners](https://web.archive.org/web/20230202212857/http://www.crunchbase.com/financial-organization/quest-venture-partners) 、 [Dave McClure](https://web.archive.org/web/20230202212857/http://www.crunchbase.com/person/dave-mcclure) 、 [ZIG Capital](https://web.archive.org/web/20230202212857/http://www.crunchbase.com/financial-organization/zig-capital) 、Global Brain Corporation 和其他天使投资者那里筹集了 150 万美元的种子资金。

本质上，Retailigence 是一个开放的 API，面向那些希望使用实体商店位置、产品列表和可用性的开发人员。这家初创公司拥有与零售商后端系统、销售点平台和库存的连接器，并将这些数据拉入云中的数据库。目前，Retailigence 能够从 Target、BestBuy、RadioShack、ToysRUs、Home Depot 和许多其他零售商那里获取数据。总的来说，它的数据库包括来自 50，000 多家商店的 300 多万种产品，

目前，有 30 到 40 名开发者在使用 Retailigence 的 API。这家初创公司表示，世界上最大的报纸出版商之一(他们拒绝透露出版商的名称)正在使用 API 来标记产品的图像，并为用户提供点击图像的能力，以找到该产品在商店最近的位置的可用性。API 的另一个用例是 GPS 设备，它可以调用附近商店的产品列表和可用性。
