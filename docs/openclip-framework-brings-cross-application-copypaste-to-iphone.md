# OpenClip 为 iPhone 带来了真正的复制和粘贴功能，但它不是解决方案

![](img/5448a907f6de97d22643fba9876bf006.png)

真的很遗憾。一个开发团队合作，最终为 iPhone 带来了一个合法的复制和粘贴解决方案——一个可以在不同应用程序之间移动数据的解决方案(以前的解决方案只能在同一个应用程序内移动数据)。但是，尽管看到一个开发团队共同努力克服 iPhone 的一个缺点是很棒的，但它可能不会有太大的用处。

几周前， [Proximi](https://web.archive.org/web/20221006021803/http://magicpad.proximi.com/) [发布了](https://web.archive.org/web/20221006021803/http://www.beta.techcrunch.com/2008/07/28/copy-and-paste-finally-coming-to-the-iphone-sort-of/)一款名为 MagicPad 的新文本客户端，包括复制和粘贴功能——这是 iPhone 一直缺乏的功能，也是许多用户一年多来一直要求的功能。复制和粘贴是通过一个相当直观的界面实现的，虽然它可能没有苹果设计的解决方案那么完美，但它完成了任务。不幸的是，所有的复制和粘贴都被限制在 MagicPad 中。

现在 [Zac White](https://web.archive.org/web/20221006021803/http://zacwhite.com/blog/) 已经为这个问题开发了一个名为 [OpenClip](https://web.archive.org/web/20221006021803/http://www.openclip.org/) 的开源解决方案，这个平台将允许开发者包括跨应用程序的复制和粘贴。OpenClip 利用了 iPhone 上的一个共享空间，开发者可以访问该空间，并且不会违反 SDK 协议，尽管苹果可以想象会拒绝包含它的应用程序。来自扎克的博客:

> 基本上，您所要做的就是包含一些类，并使用非常简单的 API 来复制数据或粘贴数据。特殊的部分是交叉应用。在鸡尾酒中复制一杯鸡尾酒，并将其粘贴到 MagicPad 中(这是实际操作的视频)。
> 有一定的局限性。这在技术上符合所有苹果协议。使用这个的应用程序完全有可能上不了 App Store。没有任何真正的原因，除了它最终会踩到苹果的脚趾。也可想而知，这个建立在其上的技术将来会崩溃。希望打破这一点的更新也带来复制和粘贴支持。

但即使这些应用程序获得批准，苹果的邮件和 Safari 应用程序仍将缺少复制和粘贴功能。虽然第三方应用程序是 iPhone 的重要补充，但许多用户将大部分时间花在手机上，要么是搜索网页，要么是使用电子邮件，这些功能仍然会被严重错过。

也就是说，新举措很可能最终会促使苹果发布自己的解决方案。看到这些开发者一起工作令人振奋，但这让苹果看起来像一个守口如瓶的坏脾气——显然手机可以复制和粘贴，所以为什么不给我们呢？
