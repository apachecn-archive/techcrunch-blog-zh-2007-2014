# openSUSE 12.1 发布 

> 原文：<https://web.archive.org/web/http://techcrunch.com/2011/11/16/opensuse-12-1-released/>

由 SUSE 发起的社区开发 Linux 发行版 openSUSE ，今天发布了 12.1 版本。乍一看，openSUSE 12.1 非常符合 Fedora 和 Ubuntu 的最新版本:GNOME 3.2、 [systemd](https://web.archive.org/web/20230204004140/http://en.wikipedia.org/wiki/Systemd) 等。但是仔细观察就会发现今天各种 Linux 产品之间有一些真正的差异。例如，Fedora 16 允许您为文件系统选择 [btrfs](https://web.archive.org/web/20230204004140/http://en.wikipedia.org/wiki/Btrfs) ，而 openSUSE 12.1 默认使用 btrfs。这个版本中还包含了许多其他有趣的改进。

btrfs 提供的文件系统特性在 openSUSE 12.1 上通过 [Snapper](https://web.archive.org/web/20230204004140/http://en.opensuse.org/Portal:Snapper) 得到了增强，这是一个用户空间实用程序，用于拍摄、比较和恢复文件的快照。这有巨大的潜力让用户的生活更美好。这当然不会消除备份需求，但这是朝着正确方向迈出的坚实一步。类似地，openSUSE 12.1 包括对 [ownCloud](https://web.archive.org/web/20230204004140/http://en.opensuse.org/SDB:OwnCloud) “纤细而强大的私有云软件”的支持。再说一次，这不会消除 Dropbox 或类似的服务，但这是朝着自由软件领域普通终端用户自给自足的正确方向迈出的一步。

对于希望跟上自由软件最新发展的超级用户来说，openSUSE 12.1 包括 [Tumbleweed](https://web.archive.org/web/20230204004140/http://en.opensuse.org/Portal:Tumbleweed) ，这是一个经过测试的稳定软件的滚动更新。这取代了一个完整、复杂系统的严格的定期发布。Tumbleweed 不会每六个月或十二个月升级一次整个系统中的所有软件，而是会在有新的稳定版本时升级单个组件。

正如 Zonker [在 G+](https://web.archive.org/web/20230204004140/https://plus.google.com/u/0/109381089725745119133/posts/6Pwekvsf6uv) 上所说，现在有很多东西可以区分不同的 Linux 发行版。它过去主要是类似 Red Hat 和类似 Debian 的，但是现在每个发行版都有自己稳定的引人注目的特性。
