# 脸书向开发者开放其 Stream API。让对话大战开始吧！TechCrunch

> 原文：<https://web.archive.org/web/https://techcrunch.com/2009/04/27/facebook-opens-up-its-stream-api-to-developers/amp/>

正如我们本周末所推测的，脸书已经通过一个新的 API 为开发者开放了它的活动流。现在，任何开发人员都可以创建包含实时流的新应用程序。第一批利用这种新 API 的应用程序之一是 [Seesmic Desktop](https://web.archive.org/web/20230131232811/http://techcrunch.com/2009/04/07/seesmic-unveils-a-formidable-new-twitter-client-to-rival-tweetdeck-seesmic-desktop/) ，这是一个 Twitter 客户端，它现在通过这种 API 添加你的脸书 feed(Tweetdeck 过去已经通过其他更具限制性的方式这样做了)。脸书还创建了自己的桌面通知客户端来演示可以用 API 构建什么。

我刚刚和脸书的平台营销总监伊森·比尔德通了电话，他告诉我整个 feed 将通过一个 API 调用提供。如果开发者愿意，他可以重新创建整个脸书主页，或者从提要中截取一部分，重新组合，使之变得更有趣。首先，我预计大多数 Twitter 客户会把脸书流作为一个额外的选项。例如，在 Tweetdeck 上，你可以阅读你的活动流，但不能在线回复。新的[脸书开源 API](https://web.archive.org/web/20230131232811/http://www.techmeme.com/090427/p37#a090427p37) 是双向的，因此它将允许开发者构建允许应用内部双向通信的应用。

这是一件[大事](https://web.archive.org/web/20230131232811/http://www.techcrunchit.com/2009/04/26/facebook-drops-other-shoe-tomorrow/)。它有可能将脸书与 Twitter 并列在所有这些桌面和移动客户端应用程序中，在这些应用程序中，大量的实时对话正在发生，并让它与 Twitter 进行正面竞争。哪个对话流更有趣，哪个就占优势。

但除了与 Twitter 的军备竞赛，stream API 将为脸书内外的许多新应用开辟可能性。一个显而易见的方法是为你的活动流提供更好的过滤选项。举例来说，创建一个显示你的信息流中最受欢迎或评论最多的项目的应用程序是很简单的。或者，现在这种信息流可以插入各种社交搜索引擎，为你提供社交化的实时结果。见鬼，如果我能搜索我自己的活动流，我会认为这是一个巨大的飞跃。但是脸书仍然只允许开发者缓存数据 24 小时，所以你不能建立一个非常强大的搜索引擎或者返回超过一天前的结果。
