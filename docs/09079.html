谷歌放弃“Flash地图应用编程接口”

原文：https://web.archive.org/web/http://techcrunch.com/2011/09/02/google-abandons-maps-api-for-flash/

关于Flash即将衰落的更多消息是，谷歌宣布将为Flash“放弃”谷歌地图API(应用编程接口)。这个API以前允许开发者在他们基于Flash的应用程序中添加谷歌地图功能。

然而，谷歌表示，截至目前，该API的使用是有限的，只有少数应用程序利用了谷歌地图Flash API独有的功能。

最初于2008年5月发布的API，类似于的JavaScript版本，但被设计用于Flash应用中。开发人员可以使用API通过各种服务操作地图并向地图添加内容，这使他们能够在网站上嵌入交互式地图应用程序。

Flash API的地图实际上并没有被完全取消——而是被废弃了。这意味着它将继续根据 Maps API服务条款运行，但不会开发新功能，只会修复“关键错误、退化和安全问题”。

换句话说，API基本上被抛弃了。

相反，Flash开发者被鼓励通过Maps API v3迁移应用程序，谷歌表示，这提供了其他好处，如街景、融合表格集成、位置搜索和对移动浏览器的完全支持。谷歌地图JavaScript API v3论坛提供了帮助。

如上所述，鉴于人们对API的独特特性兴趣有限，这一举动并不令人惊讶。然而，在某种程度上，这一决定似乎与谷歌近几个月来支持Flash的其他举措相矛盾。例如，去年夏天， Adobe和谷歌联合宣布Flash Player将很快内置到Chrome网络浏览器中，从而消除用户单独下载、安装和更新插件的需要。

此外，谷歌的移动操作系统因其对Adobe Flash和基于Flash的应用程序的支持而闻名。

在今天的博客文章中，谷歌还补充说，它也继续支持Flash作为Chrome的开发平台，但更多细节的链接会转到一个死URL 。
