# UJAM 进行了一次用户界面改造，看起来它将成为网络的 GarageBand TechCrunch

> 原文：<https://web.archive.org/web/http://techcrunch.com/2011/07/15/ujam-gets-a-ui-overhaul-as-it-looks-to-become-the-garageband-of-the-web/>

我们已经有一段时间没有听到音乐初创公司 UJAM T1 在 TechCrunch Disrupt NYC 2010 上的消息了。这项服务于去年五月在一家私人公司推出，并于去年十二月悄悄地在 T2 向公众开放。从那以后，什么都没有。但是现在他们回来了。

UJAM 的联合创始人 Peter Gorges 说，安静的原因是该公司在过去的六个月里一直在低头，彻底检查，调整和完善用户界面，使其更容易访问。Gorges 解释说，当它第一次推出时，UJAM 真的有很多漂亮的功能，有一个功能性的，但不是特别容易使用的用户界面。他说，部分原因是因为 UJAM 的创始人与音乐专业人士合作了这么长时间，他们可能对用户期望过高。这次改革应该有助于改变这种状况。

UJAM 的承诺是，理论上它允许任何人只需哼一首曲子就能创作一首歌——如果你想要的话，还可以配上背景音乐。因此，UJAM 最大的变化是引入了 UJAM 向导，这是有道理的。这是一系列新的步骤，帮助您完成歌曲创作过程，从最初的旋律(您可以哼或唱)，到选择风格，到编辑每个音符和制作歌曲结构。

其他改进包括:风格选择器现在更容易使用，它可以让你选择是想让你的歌曲听起来像舒缓的爵士乐还是糟糕的摇滚。整个用户界面看起来更好了。

好用吗？我试着即兴创作了一首歌，遇到了一些困难——例如，歌曲的曲式步骤实际上如何影响我的录音并不明显(很明显，你在选择将引子、合唱和独唱放在哪里，但这并不明显，因为这实际上会对你的录音产生什么影响)。所以，是的，还有一点学习曲线。但这是任何一种音乐创作软件都固有的问题:如果太容易，你会得到高度通用的音乐，如果太难，没人会使用它。即使 UJAM 还没有达到完美的平衡，它也越来越接近了。他们会发布视频来帮忙。

UJAM 目前仍然基于 Flash——Gorges 表示还没有足够的音频 API 支持转换到 HTML5 但该公司打算进行转换。另一件要注意的事情是:随着网站变得更加强大和用户友好，它可能会很好地适应将先进的、传统的基于桌面的应用程序转移到浏览器的趋势(这项任务部分是由谷歌带头的，谷歌希望为其 Chromebook 项目提供强大的网络应用程序)。换句话说，UJAM 最终可能成为网页版 GarageBand 的替代品。
