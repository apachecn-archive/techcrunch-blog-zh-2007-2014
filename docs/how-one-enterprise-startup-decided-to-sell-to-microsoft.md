# 一家云企业初创公司如何决定出售给微软 

> 原文：<https://web.archive.org/web/https://techcrunch.com/2014/03/29/how-one-enterprise-startup-decided-to-sell-to-microsoft/>

More posts by this contributor

**编者按:** *Tomio Geron 是 startup**exit rund**的内容负责人。这是科技并购市场系列文章的一部分。*

当 Ursheet Parikh 在 2012 年将他的公司 StorSimple 出售给微软时，这是迄今为止云基础设施领域最大的交易之一。但这并不是唯一不寻常的地方。

这也是一笔相当大的交易，他扮演了银行家的角色，参与了交易谈判。这并非闻所未闻，但这是一个有趣的例子，说明这种过程如何在初创企业收购中发挥作用。在解释他为什么以及如何在没有银行家的情况下处理收购之前，了解一下该公司是如何发展的是有帮助的。

![Ursheet Parikh](img/e5515fedbf307cd80a5823fb590d9015.png)

乌尔谢特·帕里克

StorSimple 成立于 2009 年，将多个存储系统的功能(主存储、备份、归档和灾难恢复)结合到一个存储设备中，该设备集成了亚马逊、Rackspace、微软 Azure 和谷歌等公司的云存储服务。Parikh 曾在思科工作，有过创业经验，与 Brocade 的 Guru Pangal 一起创办了这家公司。这两个人非常适合在 StorSimple 上工作，因为 Parikh 在 WAN 加速方面有着良好的记录，而 Pangal 在存储方面有专业知识。

在早期，该团队非常重视对大约 40 个客户的产品进行测试。帕里克认为，一些专注于企业的初创公司过早地推出产品。相反，他建议创业企业在向公众开放产品之前，要确保产品是正确的。“不要过早选定产品。这些产品需要时间来烘烤。过早宣布全面上市可能是一个错误，尤其是在产品的市场适应性得到验证之前。”

过早发布产品的挑战之一是很难改变产品方向或焦点。如果创业公司在可能是利基业务的早期获得了大客户，这可能是一个大问题。“一旦有人买了产品，给你一张 5 万或 10 万美元的支票，他们就有了很大的影响力，”帕里克说。

## **改变方向？**

曾经，一些企业客户希望像 Mozy 或 Carbonite 一样，将 StorSimple 作为云备份服务的一部分。这可能会让公司朝着一个完全不同的方向发展。但帕里克认为，企业云备份不是一个好生意。

Parikh 解释了他们的理由:“我们最初受到了诱惑，但我认为传统的企业云备份效率太低。数据量太大了，而且没有投资回报。我们认为最好的方法是将主存储和备份存储结合在一起。这最终成为企业的最佳决策之一。”

StorSimple 于 2011 年向公众开放了该产品，并开始起飞，突破了预期。“对于企业基础设施初创公司来说，测试一个产品可能需要两到四个季度的时间来完成，另外两到四个季度才能达到可扩展的销售指标。在全面上市后的六个季度内，StorSimple 的季度收入在几名销售代表的帮助下已经突破了 1000 万美元的运行速度。这证明了我们拥有适合建立业务的产品市场。”

## **战略投资**

截至 2012 年年中，StorSimple 已拥有 100 多家企业客户，成为云集成存储领域的领导者，并被零售、石油和天然气、制造、服务、消费品、医疗保健、政府和银行等行业的主流企业客户所使用。

StorSimple 决定发起一轮由现有投资者牵头的新的风险投资，以扩大销售、营销和运营。生态系统中有兴趣将 StorSimple 产品推向市场的大型云和存储公司被邀请加入这一轮投资。

“StorSimple 设备必须由拥有云服务的企业客户部署。大多数 StorSimple 客户发现亚马逊和微软是他们最有活力的供应商。StorSimple 的大部分交易都是与作为云提供商的微软合作完成的，”Parikh 说。“我们与这两家公司都有渠道，但由于它们的销售队伍和渠道存在，我们会更快地与微软达成交易。这是一个伟大的方式来建立一个公司，同样的方式，Citrix 利用微软。”

然而，StorSimple 最初接触的一家投资公司在投资回合结束前提出收购该公司。当时，微软也提出收购该公司。

## **卖还是不卖**

StorSimple 不得不决定是出售还是尝试建立一个独立的大企业。“我们的产品符合市场需求，发现自己正处于战略十字路口，”帕里克说。“我们需要决定是否要加倍努力。我们不想走资本密集型道路，与现有的存储供应商展开正面较量。”

StorSimple 的战略是与大型云提供商合作，并且已经与亚马逊进行了“合作竞争”，亚马逊最近发布了一款与 StorSimple 竞争的存储产品。在这一点上，StorSimple 不得不决定是否也与微软竞争。

“我们决定探索合并方案，因为我们认为我们的产品会对配套产品和大型云供应商的销售队伍产生更大的影响，”帕里克说。“现在的问题是，我们是应该聘请一位银行家，还是应该自己管理这个过程。”

## **去银行还是不去银行**

帕里克决定在董事会的帮助下自己完成收购过程，而不是雇佣一名银行家，原因有三:

1.  他觉得他和他的团队有专业知识。“在创始人和董事会之间，我们有超过 100 起并购的经验，每个人都愿意卷起袖子与我合作。”
2.  在这个早期阶段，有一个银行家增加了他不需要的另一层。“当它是一家早期公司时，个性和与收购公司的战略契合度至关重要，创始人必须密切参与。”
3.  与现有客户和合作伙伴的谈判非常微妙。“如果 StorSimple 选择不出售，我们仍然必须确保这些公司不会感到不安，因为它们对我们作为一家独立公司的成功至关重要。”

Parikh 处理报价，并在交易过程中充当银行家。“策略是努力谈判，但要友好。即使你不卖，你也不想惹恼他们，因为他们是重要的上市合作伙伴。”

最终，微软对这笔交易感到满意，史蒂夫·鲍尔默(Steve Ballmer)称 StorSimple " [是我最喜欢的收购之一](https://web.archive.org/web/20221123160904/http://view.officeapps.live.com/op/view.aspx?src=http://www.microsoft.com/investor/downloads/events/Ballmer_FAM_2013.docx)，“吹捧它是一种可用于本地 Windows 或 Linux 服务器的云服务。鲍尔默说:“对我们来说，这种东西真的爆炸了，它帮助我们的客户理解了为什么要加倍投资我们这个混合云供应商。”。

## **收购后**

大多数收购公司会定期检查被收购的团队，以确保一切顺利进行。在微软的案例中，该公司表示 StorSimple 的部署在收购后的六个月内增长了 700%。据微软公司副总裁[布拉特·安德森](https://web.archive.org/web/20221123160904/http://blogs.technet.com/b/in_the_cloud/archive/2013/11/20/the-storsimple-acquisition-one-year-later.aspx)称，它也已经成为 Windows Azure 存储产品的“核心部分”。

“收购方将在收购一年后进行更大规模的事后评估，”帕里克说。“然而，在 StorSimple 交易中，我们同意每六到八周进行一次审查，这样你就知道你在正轨上。”

微软首席执行官塞特亚·纳德拉帮助确保任何问题都得到迅速解决，并确保公司专注于 StorSimple 业务计划。微软还确保其现有团队与 StorSimple 密切合作，以实现收购。“当微软收购 StorSimple 时，微软有一个庞大的团队在开发类似的产品，”Parikh 说。“收购后，微软将这个团队与 StorSimple 产品结合起来，这种结合是交易成功的关键因素之一。”

这种一致并不总是发生在收购中。但这是帕里克在交易过程中确定的关键因素之一——他可以与微软团队合作，他们会为相同的目标而努力。
