# 比如“iPad 版推特”？看看它在脸书的同行，face pad TechCrunch

> 原文：<https://web.archive.org/web/https://techcrunch.com/2011/01/02/facebook-ipad-facepad/>

[![](img/bed6d303daeb5d8380ed0d7b8c736b88.png)](https://web.archive.org/web/20230202225456/http://itunes.apple.com/us/app/facepad-facebook-for-ipad/id412133981?mt=8)

尽管每天都有成千上万——也许甚至数百万——的人在搜索，但在 iPad 首次亮相九个月后，脸书仍未推出一款原生的应用程序，这已经成为一个不那么神秘的技术问题。

当然，这并没有阻止一些有进取心的开发者推出他们自己的脸书应用程序——由第三方制作的“[友好](https://web.archive.org/web/20230202225456/http://itunes.apple.com/us/app/friendly-for-facebook/id400169658?mt=8)脸书”，已经成为 App Store 最成功的应用程序之一。但是这个应用程序并不完美，现在它有了一个新的挑战者: [Facepad](https://web.archive.org/web/20230202225456/http://itunes.apple.com/us/app/facepad-facebook-for-ipad/id412133981?mt=8) ，这是一个脸书应用程序，显然从 Twitter 的创新 iPad 应用程序中吸取了许多设计线索，该应用程序于 9 月[发布](https://web.archive.org/web/20230202225456/https://techcrunch.com/2010/09/01/twitter-for-ipad/)。

相似之处显而易见，但这并不是一件坏事。最大的问题涉及新窗口的处理方式:就像 iPad 上的 Twitter 一样，当你点击一个内容的链接时，Facepad 会在屏幕右侧滑动一个新窗口。这让你可以快速跳转到一篇链接的文章，然后回到脸书，而不会丢失你的位置，或者在多个朋友档案之间切换——使用普通的网络浏览器不太容易。该应用程序允许你一次打开几十个这样的标签，所以你可以在几秒钟内快速浏览一堆个人资料。

它相当光滑，人们似乎喜欢它。自上周发布以来，Facepad 已经实现了相当大的有机增长。在最初的 12 小时内，该应用程序提供了超过 100 万次广告展示，24 小时后翻了一番，达到 200 万次。现在这款应用的页面浏览量高达数千万，是 App Store 上第 15 大最受欢迎的应用。

不过，就其核心而言，Facepad 仍然是 touch.facebook.com 的翻版，这是一个基于网络的社交网络版本，脸书已经针对触摸屏进行了优化(前面提到的另一个第三方应用 Friendly 也使用 touch.facebook.com 作为其核心)。这意味着，除了左手边方便的导航条和滑动窗口，这个应用程序的大部分仍然像一个普通的网站，这并不理想。

联合创始人 Cole Ratias 表示，随着 Facepad V2 的推出，这种情况将会改变——该团队正在努力使整个应用程序更像一个原生应用程序，这样你就可以像在 iPad 的默认照片应用程序中一样浏览相册。Ratias 还表示，该团队不想实现任何感觉半生不熟的东西——例如，他指出了 Friendly 中的聊天功能，它占据了整个屏幕，感觉有点笨重。Facepad 也将提供聊天功能，但要等到它能以更小的窗口覆盖的形式呈现之后。

Facepad 是一家名为 [Loytr](https://web.archive.org/web/20230202225456/http://loytr.com/) 的新创公司的第一款产品，该公司有更大的野心，不仅仅是成为浏览脸书的更好方式(尽管他们也会接受这一点)。未来，该公司计划将一个游戏平台集成到 Facepad 中，该平台以支持脸书连接的休闲游戏为特色，这些游戏使用 Loytr 的 API 进行货币化和其他功能(这样 Loytr 就可以获得一部分收入)。该公司在脸书之外还有其他计划，不过 Ratias 拒绝透露更多细节。

当然，毫无疑问，脸书将发布更适合 iPad 大显示屏的*——但根据首席执行官马克·扎克伯格的评论，当它到来时，它可能会基于 HTML5 。扎克伯格说，脸书目前没有足够的资源为每个平台开发单独的原生应用程序，因此制定基于网络的解决方案是有意义的。*

需要注意的一件大事是:当前版本的 Facepad 有一个 bug，会导致它在运行 iOS 3.2 的 iPads 上崩溃。这是一个较旧的版本，缺少 4.2 中引入的关键功能，如多任务处理，但仍有大量用户在运行它。Facepad 已经提交了修复该漏洞的更新，正在等待苹果公司的批准。**更新**:新版本已经通过审核，可以[在这里](https://web.archive.org/web/20230202225456/http://itunes.apple.com/us/app/facepad-facebook-for-ipad/id412133981?mt=8)抢。
