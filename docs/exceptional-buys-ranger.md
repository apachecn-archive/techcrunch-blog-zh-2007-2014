# Exceptional 收购 Ranger，为其“DevOps”平台注入了一剂监控强心剂

在错误跟踪/监控领域有一些整合。为云开发者提供错误跟踪服务的 exceptual 宣布收购 [Ranger](https://web.archive.org/web/20221208174542/https://rangerapp.com/) 监控服务，以扩展这家美国公司的“DevOps”平台。交易的条款仍未披露，尽管 excellent 首席执行官乔纳森·西格尔告诉我，这是一笔“六位数的购买”，只涵盖 Ranger 产品(刚刚更新到 2.0 版本)和现有客户群。

在此之前，[早些时候](https://web.archive.org/web/20221208174542/https://beta.techcrunch.com/2012/02/07/exceptional-acquires-error-tracking-application-airbrake/)在今年 2 月被 Exceptional 收购，当时这家初创公司通过从网络设计和开发公司 thoughtbot 收购竞争对手 [Airbrake](https://web.archive.org/web/20221208174542/http://airbrakeapp.com/) 来补充其错误跟踪产品。再次，财务条款没有公布。

Ranger 于 2010 年推出，Heroku 的客户使用它来监控他们基于云的应用程序。据说它与 Exceptional 和 Airbrake 非常吻合——客户现在可以访问“任务关键型 DevOps 平台”，该平台会告诉他们*何时*和*为什么*他们的应用程序宕机。大概是这样的。

Exceptional 声称有超过 120，000 名开发人员正在使用其错误跟踪平台，这对一家没有接受任何外部资金的公司来说并不坏。收购 Ranger 旨在巩固这家初创公司在该领域的领导者地位。

Ranger 的创始人 Shay Frendt 在一份声明中说:“将 Ranger 加入他们的家庭意义重大，监控和错误跟踪齐头并进，因此它使整个工具集对开发人员更加有用。”

与此同时，Siegel 补充道:“对于包含许多移动部分的复杂应用程序设计，您需要一个智能监控平台来准确地告诉您哪个服务停止了。”
