# 谷歌文档发现更多安全漏洞

安全顾问[阿德·巴卡](https://web.archive.org/web/20230403114103/http://www.barkah.org/)与我们联系，提醒我们与谷歌文档相关的几个严重的安全问题，谷歌文档是世界上最著名的搜索引擎公司的基于网络的办公软件，赋予了它的[使命](https://web.archive.org/web/20230403114103/http://www.google.com/corporate/)一个全新的意义，使世界上的信息可以普遍访问。在他关于软件、基础设施和安全的博客上，Barkah 概述了他在调查一些潜在安全漏洞时发现的至少三个问题。

因为他做了正确的事情，就他的发现联系了谷歌(但在五个工作日之后没有收到任何回应)，我们希望这篇文章将有助于触发该公司的工程团队尽快堵塞漏洞。如果你错过了，本月早些时候我们[发现了一些谷歌文档的重大隐私错误](https://web.archive.org/web/20230403114103/https://techcrunch.com/2009/03/07/huge-google-privacy-blunder-shares-your-docs-without-permission/)，该公司后来证实并修复了这些错误(我们也为此对它们进行了 ping 操作)。

**更新:**谷歌已经在官方文档博客上发布了其对这些问题的立场[(他们不认为有重大安全风险)](https://web.archive.org/web/20230403114103/http://googledocs.blogspot.com/2009/03/just-to-clarify.html)

怎么了？

首先，很明显**当你在一个受保护的文档中嵌入一张图片时，它会被上传到一个谷歌服务器上，在那里你没有授权访问该文件的人仍然可以看到并下载它**，甚至在你删除了有问题的文档之后*。我已经上传了一张图片到我的账户中一个受保护的文件中进行测试，然后马上删除了这个文件。如果你看到嵌入在这篇文章顶部的图片，或者点击[这个链接](https://web.archive.org/web/20230403114103/http://docs.google.com/File?id=dczjts34_227g3234gdg_b)发现你仍然可以看到图片，这意味着上面的检查通过了。*

我同意 Barkah 的观点，他写道:

> 如果您将图像嵌入到受保护的文档中，您会希望图像也受到保护。如果你删除一个文档，你会期望任何嵌入的资源也被删除。最终结果是潜在的隐私泄露。

图像可能包含个人和专业的机密信息，基本上只需要找到图像的专用 URL，任何人都可以自由访问它，这是一个巨大的隐私错误。

第二，看起来如果你与任何人分享一个带有图表的文档——这是 Google [昨天推出的一个功能—](https://web.archive.org/web/20230403114103/http://googledocs.blogspot.com/2009/03/drawing-on-your-creativity-in-docs.html),**这个人将能够查看嵌入到文档中的任何图表的任何版本。这基本上意味着，如果您创建了一个包含敏感信息的图表，然后决定在以只读模式共享文档之前删除其中的一些信息，那么与您共享该图表的人只需稍微调整一下 URL，就可以恢复到以前保存的版本，从而发现您认为自己仍对他/她隐藏的内容。**

Barkah 提出的第三个问题是一个如此严重的错误，以至于他还没有进入它背后的机制的细节，等待 Google 的进一步研究和反馈。安全专家声称**如果你取消了其他人访问你的文件的许可，在某些情况下，他们仍然可以在你不知情的情况下获得这些文件。**

如果最后一个说法被证明是正确的，我将离开谷歌文档，永远不再回来。

更新:谷歌发言人回应:

> 我们非常重视用户信息的安全性，并且正在调查研究人员提出的问题。根据我们收到的信息，我们不认为谷歌文档存在重大安全问题。一旦有更多信息，我们将与您分享。
