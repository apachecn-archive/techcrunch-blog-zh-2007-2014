脸书的开放图:这取决于“开放”这个词是什么意思

原文：https://web.archive.org/web/https://techcrunch.com/2010/04/23/facebook-open-graph/

抓起爆米花。一场严肃的书呆子之战正在酝酿。

继脸书几天前在f8上宣布了大的开放图形之后，许多所谓的“开放网络”的领导者反对脸书用“开放”这个词来描述它的宏伟计划。虽然开放的图形可能是很多东西，但它不是开放的，这是许多人都有的感觉，就像Erick早先展示的。

具体来说，他们中的大多数都瞄准了网络上随处可见的新赞按钮(包括TechCrunch)。这是一个明显的目标，因为它是开放图协议中最明显的部分(至少到目前为止)。投资者/Hunch联合创始人克里斯·迪克森(Chris Dixon)正在领导一个新的 OpenLike 按钮的工作(尽管他希望其他人来负责)。谷歌的开放网络倡导者克里斯·梅西纳已经在博客文章中抨击了脸书的Like按钮。

任何人都可以在脸书宣布后立即预测到这些反应——事实上，我也预测到了。事实是，脸书是网络上最强大的力量之一，他们现在正在利用这一地位引入一个新的平台，这将是的，帮助他们。令人震惊，真的。一个想做一些对自己有利的事情的公司。但我相信，脸书，至少部分相信，这也将使更大的网络变得更好。但这对一些人来说还不够好，因为它还没有完全开放。不要介意，很多这些完全开放的解决方案总是被提倡，但由于这样或那样的原因，从来没有成功过。

但是对我来说最有趣的是脸书实际上有一个开放网络的人在那里工作。去年8月，脸书聘请戴夫·雷科登担任“高级开放项目经理”Recordon长期以来一直是开放网络讨论的中心人物。例如，在2005年，他帮助创建了OpenID。所以我问了Recordon他对最近这场混战的看法，特别是梅西纳的帖子。

作为回应，Recordon向我介绍了这个谷歌群组线索。在这里，Recordon对Messina提出的一个问题有一个很长的直接回答。我会把这两个都贴在下面。

两天前，墨西拿写道:

非常严肃地说——不是想挑衅——只是想理解开放图协议中“开放”的含义？谢谢！祝贺发射成功。

克莉丝

昨天，Recordon回答说:

我认为在这种情况下，开放意味着几件事:

首先，它旨在增加人与人之间的开放性，这是基于
能够与网络上的所有事物联系起来。在脸书，这意味着人们可以喜欢任何地方的任何网页，而不仅仅是facebook.com的网页。

第二，开放图协议增加了网络上的语义数据量，这种方式并不局限于脸书或任何一个社交网络。虽然我们都不同意引号和尖括号应该放在哪里，但最终我想我们都同意这种元数据对网络有益。

第三，它是由不止一家公司创建和实施的。我们
现在正在扩大这一人群(就在这里),并对
以精英的方式发展这一规范感兴趣。

最后，它从第一天起就在开放网络基金会
协议(http://bit.ly/6zaAYh)下获得许可。正如杰西·斯泰所写的(http://bit.ly/
c0ROCw)，这意味着它“处于完全开放的许可协议之下，其他平台创建者可以采用、使用和自由分发。”

有帮助吗？

大卫

对此，墨西拿回应道:

还有哪些公司参与了OGP的创建？

签署的协议在哪里？虽然我认为你在这里使用
OWF协议是很棒的，但似乎实际上需要一个签署的
协议，由受该协议
条款影响的各方在网上存储某人。

谢了。急着看签好的协议！

在这一点上，Recordon指出，他们可能应该将讨论转移到Open Web Foundation邮件列表。我假设任何人都可以注册(如果不注册，就不会很“开放”)，所以在那里寻找更多有趣的开放网络内斗。

Recordon有一些好的观点，但他没有指出的一点(当然)是，脸书有权做对脸书有利的事情。你可以反驳这一点的最佳论据是，对脸书有利的东西可能对我们所有人都不是最有利的。在读写网对Open Graph的彻底分解中，亚历克斯·伊斯科尔德(他也是OpenLike社区的活跃分子)对此提出了一个很好的理由:

从技术上讲，脸书的所作所为是优雅而正确的。从标记，到插件，再到API，所有这些都很现代，很棒。缺少的一点是，脸书似乎是这个等式中唯一的数据仓库——这使得整个供应严重封闭。发布者和用户没有存储数据的选择。它将单独去脸书和脸书。也许有一种方法可以修复这个系统。

当然，然后出版商没有有使用脸书的喜欢按钮。但是他们会的——我能想到近5亿个原因。喜欢也好，讨厌也好，事情就是这样。这不是善对恶。这不是黑与白的问题。一如既往，这是无数的灰色阴影。
