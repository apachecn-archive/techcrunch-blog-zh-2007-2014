# Spotify 新的嵌入式播放按钮可以让你在任何网站上收听法律歌曲

> 原文：<https://web.archive.org/web/https://techcrunch.com/2012/04/11/spotify-play-button/amp/>

Spotify 希望让任何人都更容易合法地给自己的网站配乐，哦，并在互联网上发布自己的下载页面链接。所以[今天它为新闻网站推出了](https://web.archive.org/web/20230404104946/http://www.spotify.com/se/about/play/)[嵌入式 Spotify 播放按钮](https://web.archive.org/web/20230404104946/https://embed.spotify.com/)，点击后开始播放你的 Spotify 桌面应用中指定的歌曲、专辑或播放列表。*滚石*、*赫芬顿邮报*和*卫报*都是今天将开始展示该按钮的合作伙伴，Tumblr 的博客可以直接从音频仪表板上立即将 Spotify 音乐添加到他们的帖子中。

虽然 Spotify 告诉我这一切都是为了改善音乐发现和收听体验，但它的真正好处是当没有桌面应用程序的人点击按钮时——他们会被提示下载 Spotify(如下所示)。该公司的用户获取成本应该非常高，因此通过播放按钮进行免费推广可能是使其商业模式活跃的关键。

Spotify 的播放按钮为希望从网站上播放音乐的出版商解决了一个大问题。通常情况下，他们不得不嵌入一个音频质量通常很差的 YouTube 视频，托管一个可疑的合法 MP3，热链接到另一个可能被删除的博客文件，或者在 SoundCloud 等网站上搜索合法的流媒体版本。Spotify 的播放按钮为他们提供了一种快速、稳定、可靠、高质量和合法的方式来为他们的网站配乐，并确保艺术家的工作得到报酬。

为了配置播放按钮，出版商可以右键单击 Spotify 中的任何歌曲、专辑或播放列表，复制其特殊的 Spotify URI，并将其粘贴到[https://embed.spotify.com/](https://web.archive.org/web/20230404104946/https://embed.spotify.com/)。然后，他们复制紧凑按钮或显示专辑插图的按钮的嵌入代码，并将其粘贴到他们网站的代码中。音乐甚至可以像病毒一样传播，因为其他发行商可以通过悬停在播放按钮上来获取小部件的嵌入代码。

其他网站包括 ShareMyPlaylists.com、FanRx、Popdust、The Independent、Time Out Group(纽约、巴黎、伦敦)、NME、Mashable、FanBridge、Wonderwall、The Fader、Chegg、ELLE、Noisey.com、娱乐周刊、People.com 和 SPIN.com。按钮的颜色和大小可以通过修改代码来定制，Spotify 表示，最终会在配置器中添加更多选项。

Tumblr 用户甚至可以更轻松地使用它。post composer 的音频仪表板现在允许您搜索 Spotify 内容，只需点击一下，一个优化的播放器就会插入到您的帖子中，无需嵌入代码。播放按钮也让重度 Spotify 用户更容易使用。在点击网站上的其他音乐流以避免播放两首歌曲之前，不必暂停当前的 Spotify jam，Spotify 会自动覆盖应用程序上已经播放的内容。

现在的问题是，严肃的音乐博主是否愿意显示 Spotify 播放按钮，而不是他们的旧流媒体源，即使他们可能会排除一些不想下载该应用程序的读者。否则，不熟悉的播放按钮可能会成为 YouTube 和 SoundCloud 流的副手。
