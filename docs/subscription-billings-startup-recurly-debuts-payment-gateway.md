# 订阅计费启动递归首次推出支付网关

Recurly ，一家让其他公司更容易[管理他们的订阅账单](https://web.archive.org/web/20221007161610/https://beta.techcrunch.com/2010/08/03/recurly-raises-1-6-million-to-help-companies-manage-subscription-billing/)的初创公司，发布了一个专门设计的支付网关，以处理循环账单的独特要求。

Recurly 的服务允许企业快速实现订阅计费系统，处理信用卡号码存储等任务(它还支持与 QuickBooks 等金融软件集成)。

正如总裁 Dan Burkhart 解释的那样，订阅支付领域存在动荡，因为每月有 5%到 15%的信用卡在订阅账单中失败。在许多情况下，信用卡错误会直接导致不必要的客户流失，这对于具有经常性收入流的订阅业务来说成本极高。

通过递归地运行自己的支付网关，该初创公司可以直接与支付处理者的后端集成，并更有效地管理经常性交易。

此外，Recurly 为每个 Recurly 支付网关客户提供优质客户保持报告。与大多数旨在处理一次性交易的支付网关不同，Recurly 的支付网关可以帮助企业优化客户保留率，并最大限度地减少客户流失。

Recurly 也正在转向一种不同的定价模式，收取可计费发票金额的 1.25%和每笔交易 0.10 美元，其中包括支付网关的使用。伯克哈特说，该公司正在摆脱基于交易量的分层定价结构。此外，该网关仅支持美国商家，但 Recurly 将很快增加对国际公司的支持。
