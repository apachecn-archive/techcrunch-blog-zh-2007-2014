# YC 大学毕业生和埃丝特·戴森支持的合格人员希望成为医疗保健交易的一员 

> 原文：<https://web.archive.org/web/https://techcrunch.com/2012/12/22/yc-grad-esther-dyson-backed-eligible-wants-to-be-the-stripe-for-healthcare-transactions/>

最近涌现出许多令人惊叹的初创公司，它们采用现代技术，希望改善医疗和教育。这一点很重要，不仅因为这两个行业触及到我们每一个人，还因为它们都同样被遗留的基础设施和陈旧的标准所消耗。然而，尽管创业公司继续进入这一领域——这是我们应该继续庆祝的事情——但许多人仍然回避一些最具挑战性的技术问题，这些问题的影响可能会比又一个性感的消费者移动应用程序更大。

诚然，这并不令人惊讶，因为这两个行业都面临着一些非常棘手的问题，大部分是由长期存在的遗留基础设施造成的。基础设施本身的现代化以及医疗公司集成和交换数据的方式是一项艰巨的任务，但也是一项重要的任务。

[符合条件的](https://web.archive.org/web/20221208171944/https://eligibleapi.com/)，一家总部位于旧金山的创业公司，Y Combinator 的最新一批成员，已经决定[咬紧牙关](https://web.archive.org/web/20221208171944/http://en.wikipedia.org/wiki/Bite_the_bullet)，正面迎接这个挑战。大数据将对医疗保健(以及教育和实际上每个行业)产生变革性影响，但关键是用一种智能、通用和动态的语言来表达这些数据。

为此，Eligible 着手建立一套现代化的标准化协议，以使医疗保健数据和信息更易于访问，尤其是医疗保健资格。[作为与 Clever](https://web.archive.org/web/20221208171944/https://beta.techcrunch.com/2012/06/26/clever-launches-twilio-for-ed-data/) 同属 Y Combinator 类的成员，Eligible 正在将同样的理念应用于医疗保健数据。换句话说，正如 Clever(和 LearnSprout)创建了一套现代的 REST APIs 来从学生信息系统的笨拙、封闭的数据集中释放学生数据一样，Eligible 正在为医疗保健资格做同样的事情。

目标是消除限制不同(但关键)数据集互操作性的障碍，使它们更容易管理和集成。当然，“医疗保健资格”是医疗保健基础设施的一个方面，一般人看不太多，因此它可能看起来有点无聊或晦涩(因为它是)，但它也是接受和管理医疗保健过程的一个重要部分，是使整个事情运转的一部分。

例如:每次你去医生的办公室，寻求治疗或进行手术，医院和医生都会询问保险公司，看看你有什么样的保险范围，以及在你当前的健康计划下你有资格接受哪些治疗。传统上，这些请求是手动进行的，整个过程通过电话、电子邮件甚至马车进行。

自然，在技术的帮助下，资格和信息请求过程可以自动化。当人们考虑到医院和医生每年进行大约两万亿次资格查询时(一个程序可能涉及五次查询)，人们开始理解通过自动化这些机制可以节省多少时间、金钱和带宽。事实上，现在人们认识到这是一个非常迫切的需求，奥巴马政府(根据其新的医疗保健政策)已经要求保险公司必须开始自动响应这些查询。

当然，自动化和简化传统医疗保健领域的数据交换说起来容易做起来难。合格的联合创始人 Katelyn Gleason 和 Patrice Krakow 已经接受了医疗保健 IT 特有的旧技术(以及随之而来的翻译负担),以解决他们认为对开发商和保险公司来说都是非常现实(和巨大)的问题。当然，这一过程中涉及的所有人(医生、医院、管理人员、开发商、保险公司，当然还有患者)。

[![bd692376738f57abcd0691e9d1ef547b-original](img/ffc31bcdea103f719d54652a8ea1d4df.png)](https://web.archive.org/web/20221208171944/https://beta.techcrunch.com/wp-content/uploads/2012/12/bd692376738f57abcd0691e9d1ef547b-original.png) 每年有两万亿次查询发生在许多不同的数据标准上，包括实验室和测试、保险信息、医生就诊信息、制药信息等。这些信息是至关重要的，但往往被锁在各自的筒仓里，没有多少互通。当你加上一个事实，即有大量的保险公司在一个更大的(是的，我说了)计划阵列下提供保险，汇总这些信息变得更加复杂。

为了让医疗保健提供商、保险公司以及二者之间的所有人都更方便，Eligible 聚合了来自各个保险提供商的数据，将来自 REST APIs 堆栈中每个竖井的覆盖范围数据和患者信息标准化。

对于开发人员来说，这意味着花费更少的时间进行解析，对于开发人员*和*提供商来说，与合格的 API 集成意味着能够确认有效的保险覆盖范围和人口统计信息，如账单邮政编码、性别和雇主，以及能够访问计划细节，包括诊断实验室覆盖范围、医疗支出账户共付额等。它还让医院(和开发商)的生活变得更加轻松，允许他们以电子方式请求入院、家庭医疗保健和手术的预授权码，简化了从医院前台签到到结账的整个护理链。

同样， [Stripe](https://web.archive.org/web/20221208171944/https://stripe.com/) 提供开发者友好的技术来简化将支付处理集成到应用程序和网站的复杂而模糊的过程，Eligible 希望建立第一个现代 API，允许开发者轻松交换健康数据，并在这样做的同时赚钱。开发者可以通过这些资格查询、患者信息查询等来整合 Eligible 以增加吸引力和收入。

[![aec6875b6dbfc9f6faf6be3cb346d7b0-original](img/faf11bb2bd545e7ad3f8f43a5a9aa25a.png)](https://web.archive.org/web/20221208171944/https://beta.techcrunch.com/wp-content/uploads/2012/12/aec6875b6dbfc9f6faf6be3cb346d7b0-original.png) 事实上，格里森告诉我们，Eligible 每天已经处理 8000 笔合格交易，每笔交易 5 美分，这家初创公司就已经产生了收入。该公司已经与 100 多家不同的供应商和医疗保健公司以及 60%的医院合作，这位联合创始人表示，这个数字正在快速增长。[Passport Health communication s](https://web.archive.org/web/20221208171944/http://www.passporthealth.com/index.aspx)只是一个例子，因为该公司已经通过 Eligible 每天处理数千笔交易。

随着奥巴马医改强制要求医疗保健资格申请实时自动化，医疗保健组织和初创公司正在寻找一种更简单、更准确的方式来为自己的应用程序访问这些信息，Eligible 有一个真正的机会将这种早期增长转化为长期增长。

Eligible 已经完成了艰难的跑腿工作并编写了代码，这样开发者和医疗保健公司就不必这么做了，这本身就是银行里的钱。

迄今为止，这家初创公司已经从 Y Combinator、Start Fund 和其他一些天使投资者那里筹集了 50 万美元的种子资金，其中包括 SV Angel 的李尚义、埃丝特·戴森、Anvil Capital 的 Michael Liou、埃森哲董事总经理 Anand Swaminathan 和 Andreessen Horowitz。

从 Y Combinator 毕业后，随着医疗保健信息交换平台的建立，该团队于今年冬天加入了 Rock Health 的加速器，该团队目前正在那里努力工作。
