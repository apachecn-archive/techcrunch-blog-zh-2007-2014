# 随着 XP 支持枯竭，金融公司将目光投向 Linux 和 Windows 7 

> 原文：<https://web.archive.org/web/https://techcrunch.com/2014/03/23/financial-firms-looking-to-linux-windows-7-as-xp-support-dries-up/>

# 金融公司期待 Linux、Windows 7 作为 XP 支持枯竭

不出所料，ATM 操作员和其他金融机构开始将 Linux 作为过时的 Windows XP 系统的替代品。这些组织现在才决定放弃 XP，这证明了操作系统的持久力和金融界的某种保守主义。

《计算机世界》记者 Jaikumar Vijayan 在[的文章中写道:“Windows XP 目前驱动着全球近 95%的自动取款机。”。微软将于 4 月 8 日停止对该操作系统的支持，在过去十年的大部分时间里，该操作系统是嵌入式系统和自动取款机的热门选择，正如无数次](https://web.archive.org/web/20221006073926/http://www.computerworld.com/s/article/9247096/ATM_operators_eye_Linux_as_alternative_to_Windows_XP)[提到《奇异的地方》中的蓝屏死亡](https://web.archive.org/web/20221006073926/https://www.google.com/search?q=blue+screen+of+death+atm)所暗示的那样。

新的 ATM 操作系统必须符合支付卡行业安全标准委员会(PCI SSC)的标准，并且可能有必要进行硬件更新，以将当前的 ATM(其使用寿命为 5 至 10 年)升级到 Window 7 等下一代操作系统。显然，Linux 是一个可靠的选择，因为它是开放的，甚至可以在遗留硬件上进行强化。然而，随着“PIN 和芯片”式卡开始在美国出现，更新几乎每一个 POS 终端和 ATM 的需求可能更加迫切。

据推测，金融公司可以让他们的旧机器再运行几个月甚至几年，但最终 XP 死了，尽管请求保持对嵌入式操作系统的支持，微软还是决心拔掉插头。当然，这意味着 BSOD 可能很快就会成为过去。
