# MySpace 流架构师 Monica Keller 跳到了脸书

> 原文：<https://web.archive.org/web/https://techcrunch.com/2010/02/12/myspace-stream-architect-monica-keller-jumps-to-facebook/amp/>

<amp-img src="https://web.archive.org/web/20230305231426im_/https://techcrunch.com/wp-content/uploads/2010/02/monicakellershot.png" class="shot2 amp-wp-enforced-sizes i-amphtml-layout-intrinsic i-amphtml-layout-size-defined" layout="intrinsic" i-amphtml-layout="intrinsic"><i-amphtml-sizer class="i-amphtml-sizer"></i-amphtml-sizer></amp-img>[莫妮卡·凯勒](https://web.archive.org/web/20230305231426/http://www.crunchbase.com/person/monica-keller)，MySpace 集团架构师，在推动 MySpace 的活动流和开放性方面发挥了关键作用，即将离开公司加入脸书。凯勒今晚在博客中宣布了这一消息。她将以开源和网络标准项目经理的身份加入脸书，在那里她将加入一个包括 [David Recordon](https://web.archive.org/web/20230305231426/http://www.crunchbase.com/person/david-recordon) 和 [Luke Shepard](https://web.archive.org/web/20230305231426/http://www.crunchbase.com/person/luke-shepard) 的团队。MySpace 证实凯勒已经离开公司，但拒绝进一步置评。

凯勒在推出 MySpace 的[实时流 API](https://web.archive.org/web/20230305231426/https://techcrunch.com/2009/12/09/myspace-realtime-api-google-oneriot-groovy/) 中发挥了关键作用，帮助设计了使用推送的实时流，并设计了网络的 Twitter 同步摄取。凯勒参与了流的技术方面，也参与了 MySpace 开发平台的设计。她还代表 MySpace 参加了许多会议。

虽然凯勒在她的帖子中对这家陷入困境的公司说了一些好话，但她显然对 MySpace 处理一些事情的方式不满意:

> 但是我选择了离开。虽然我能够有一些暂时的创作自由，但这不是其他工程师所享受的标准或部分，我觉得不再有一种凝聚力来提供我们所能提供的最好的东西。
> 
> 给我在 MySpace 的朋友和同事一些临别建议:
> 
> MySpace 必须有强有力的技术领导，能够吸引优秀的技术人才并做出明智的决策。重要的是，他们与世界其他地方保持联系，并致力于互操作标准和有益于最终用户的可靠产品。我的许多工程师同事都有很棒的想法和分阶段交付的计划。

这对 MySpace 来说是一个损失，但这肯定不是他们实时和开放举措的终结(这些举措比脸书的更进步)。我们听说，这些仍然由最近[晋升的](https://web.archive.org/web/20230305231426/https://techcrunch.com/2010/02/10/myspace-ceo-owen-van-natta-steps-down/) MySpace 联合总裁[迈克·琼斯](https://web.archive.org/web/20230305231426/http://www.crunchbase.com/person/mike-jones)领导，而最近[在 LinkedIn 运营活动流产品后加入](https://web.archive.org/web/20230305231426/https://techcrunch.com/2009/11/12/myspace-hires-some-social-smarts-from-linkedin/)公司的 [Christina Wodtke](https://web.archive.org/web/20230305231426/http://www.crunchbase.com/person/christina-wodtke) 参与管理团队的日常运营。
