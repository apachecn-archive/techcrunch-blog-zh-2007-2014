# SoundCloud、Deezer 和 Rhapsody 成为脸书音乐发布合作伙伴？

我喜欢伊沃·沙普，尽管我从未见过他本人。

想知道为什么吗？因为这个人只是通过阅读代码来不断挖掘有趣的东西，然后提醒我们。

在公开发布之前，Schaap 发现了[嵌入式+1 按钮](https://web.archive.org/web/20230204220304/https://techcrunch.com/2011/03/31/omg-someone-just-found-an-embeddable-google-1-button-%E2%80%93-and-it-works/)和一个[到谷歌游戏标志](https://web.archive.org/web/20230204220304/https://techcrunch.com/2011/07/15/google-games-logo-source-code/)的链接，现在他[可能已经确定了](https://web.archive.org/web/20230204220304/http://www.yvoschaap.com/weblog/more_hints_of_facebook_music_code_leaks)几家公司将与脸书合作[即将推出的](https://web.archive.org/web/20230204220304/https://techcrunch.com/2011/09/09/f8-2011/)后者的音乐服务(可能被称为 [Vibes](https://web.archive.org/web/20230204220304/http://siliconfilter.com/vibes-facebooks-upcoming-music-feature/) )。

Schaap 仔细查看了世界上主要的音乐流媒体服务，并在其中一些服务的 HTML 代码中找到了有趣的参考。几家流行的音乐流媒体服务提供商，即 [Spotify](https://web.archive.org/web/20230204220304/http://www.crunchbase.com/company/spotify) 、 [Rdio](https://web.archive.org/web/20230204220304/http://www.crunchbase.com/company/rdio) 、 [MOG](https://web.archive.org/web/20230204220304/http://www.crunchbase.com/company/mog) 、 [SoundCloud](https://web.archive.org/web/20230204220304/http://www.crunchbase.com/company/soundcloud) 、 [Deezer](https://web.archive.org/web/20230204220304/http://www.crunchbase.com/company/deezer) 和 [Rhapsody](https://web.archive.org/web/20230204220304/http://www.crunchbase.com/company/rhapsody) ，在他们的曲目、专辑和艺术家页面上以迄今为止未记录的格式显示[元数据](https://web.archive.org/web/20230204220304/http://developers.facebook.com/tools/debug/og/object?q=http://mog.com/m/track/57587005)。

![](img/812d13f1254612ae7811e2a548fbd445.png)

音频桥？马上给我报名！

当然，MOG、Spotify 和 Rdio 已经被广泛报道为“脸书音乐”的合作伙伴，但值得注意的是，Deezer、SoundCloud 和 Rhapsody 是新名字。

Schaap 还看了看 Turntable.fm、Grooveshark、Last.fm、Pandora 和 Napster 之类的网站，但在它们各自的网站上没有挖掘到任何有趣的东西(至少现在还没有)。
