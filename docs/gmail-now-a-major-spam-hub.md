# Gmail 现在是一个主要的垃圾邮件中心

[![](img/a1f471c1928b7e04a932055f6a2cc846.png)](https://web.archive.org/web/20230216140656/http://www.crunchbase.com/product/gmail)

当 [Gmail](https://web.archive.org/web/20230216140656/http://gmail.com/) 首次推出时，它因其垃圾邮件过滤和安全性而受到称赞——没有垃圾邮件离开系统，也几乎没有垃圾邮件进来。但是这一切都改变了，因为咆哮企鹅的一项研究[发现](https://web.archive.org/web/20230216140656/http://www.allspammedup.com/2008/07/spammers-choose-gmail/)来自 Gmail 的垃圾邮件仅在上个月就从 6.8%上升到了 27%。大跳跃的动力很可能是谷歌账户注册验证码已经被彻底击败，几乎甚至不再碍事了。

几周前，我们宣布[验证码已经失效](https://web.archive.org/web/20230216140656/http://www.techcrunchit.com/2008/07/05/the-end-of-captchas-as-a-security-mechanism/)，因为现在有成千上万的 Gmail、Hotmail 和雅虎的现成账户可供在线购买。随着 Gmail 现在对 27%的垃圾邮件负责，信任机制已经完全崩溃，谷歌必须采取一些激烈的行动来过滤掉发送大量电子邮件的自动用户。

他们可以通过限制出站电子邮件的数量和一些常识性的用户跟踪来做到这一点。我不认为任何真正的用户会像大多数垃圾邮件脚本那样每小时发送数千封电子邮件——所以检测和关闭应该不会太难。谷歌有一个大问题，它将影响到其他 Gmail 用户，因为从该系统发出的电子邮件现在正受到其他邮件处理者越来越多的怀疑。
