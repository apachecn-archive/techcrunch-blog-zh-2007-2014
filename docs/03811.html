谷歌将空间搜索添加到地图应用编程接口

一场争夺地理开发者的心和思想的战争正在打响。上个月， Twitter开启了它的geo API ，像 SimpleGeo 和 GeoAPI 这样的服务正在提供来为那些想要创建很酷的地理应用的初创公司做很多繁重的工作。谷歌没有袖手旁观，它只是调整了谷歌地图应用程序接口，现在支持空间搜索和搜索源。

空间搜索允许开发人员在地图的某个区域搜索特定的要素。他们的应用程序将通过搜索提要调用这些搜索，搜索提要可以搜索一个框内区域或某个纬度/经度点的半径范围内。地理搜索结果可以按不同的属性排序，如距离。将这种空间搜索输入到谷歌不断增长的本地企业目录中，开发者也许可以使用API来构建一个应用程序，在地图上显示边界内的餐馆或鞋店。

能够询问10个街区半径内或一个县内的所有有趣的地方打开了各种可能性。随着所有这些地理平台的涌现，创建地理应用程序变得越来越容易。当然，没有什么可以阻止开发人员混合和匹配来自不同API的不同地理数据层。
