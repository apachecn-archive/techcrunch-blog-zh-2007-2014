# 微软的云短暂蒸发，让多达 3.65 亿用户四小时无法访问

第一封邮件在美国东部时间 11:32 到达了 TechCrunch 的举报热线。其他人很快也说了同样的话:主要的微软云服务在全球范围内都被关闭了。Hotmail、Skydrive 和微软最近推出的 Google Docs 竞争对手 Office 365 都无法访问。用户在 twitter 和我们的热线上表达了他们的愤怒，但在离线大约四个小时后，服务恢复了，微软对他们的宕机几乎没有解释。

微软的官方博客和 Twitter 账户正忙着扑灭小的公共关系突发事件，但这些事情还是发生了，发言人除了告诉用户耐心等待之外也无能为力。服务器可能会停机。数据库可以被清除。电源线可以切断。

美国东部时间今天凌晨 3 点左右，服务重新开始工作。微软通过[的 Windows Live 博客](https://web.archive.org/web/20230204115514/http://windowsteamblog.com/windows_live/b/windowslive/)宣布，DNS 服务器已经恢复，大多数客户应该在 30 分钟内看到服务恢复。

该公司在另一个博客上简要解释说，停电是由南加州停电引起的，如果你看看停机时间，这相当吻合。然而，该帖子后来被撤下，微软尚未证实该报道。(截图由 [ITnews AU](https://web.archive.org/web/20230204115514/http://www.itnews.com.au/News/271521,microsoft-office365-hotmail-blacks-out.aspx) 提供)

这恰好是不到一个月的时间里，Office 365 第二次宕机。最后一次中断持续了 3 个小时，促使微软为客户提供 25%的折扣——你必须为 Office 365 付费。这次没有打折的消息。

就这样，在欧洲大部分地区，一天开始时，微软的大部分云服务离线，几个小时后就恢复了。每一项在线服务都会出现宕机，甚至谷歌文档也不例外，但微软服务遭遇的最新打击是又一次挫折，主要是因为它对谷歌文档的回答非常重要。正常运行时间对云服务至关重要，微软到目前为止还没有完全证明它能够提供可靠的服务。希望今年剩下的时间里 Office 365 的表现会更好。
