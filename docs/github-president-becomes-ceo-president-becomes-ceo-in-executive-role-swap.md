# GitHub 总裁变成 CEO，CEO 变成总裁，执行角色互换

因此，现在显然是中级技术公司的高管们互相交换头衔的时候了: [GitHub 刚刚宣布](https://web.archive.org/web/20230115180036/https://github.com/blog/1761-new-year-new-ceo-for-github)联合创始人汤姆·普雷斯顿-沃纳将接替联合创始人克里斯·万斯特拉斯的总裁职位，后者将接替普雷斯顿-沃纳空缺的首席执行官职位。这听起来很熟悉，因为 WordPress 的创造者马特·莫楞威格刚刚[对 Automattic 的托尼施耐德做了本质上相同的事情。](https://web.archive.org/web/20230115180036/https://techcrunch.com/2014/01/13/matt-mullenweg-becomes-automattic-ceo-as-toni-schneider-shifts-focus-to-new-products/)

对 GitHub 来说，这一转变似乎旨在改变一些事情，给 Preston-Werner 时间，以更实际的方式对待 R&D 和“公司内部的增长机会”，根据他发布消息的博客帖子。他说，普雷斯顿-沃纳仍将是公司的主要代表。就万斯特拉斯而言，他将处理更多的顶层战略和总体公司愿景。

与 Mullenweg 和 Schneider 一样，Preston-Werner 淡化了这种转变的意义，指出在 GitHub 的企业结构中，角色和职责有很大的流动性。他甚至说他们已经在实践中进行了转换，现在只是在名义上进行转换。

我们在 9 月份的 TechCrunch Disrupt 上听到了 Preston-Werner 的发言，当时他讨论了该公司的 400 万用户里程碑，以及传统的以代码为中心的平台向更广泛的协作平台的转变。当时，普雷斯顿-沃纳还否认该公司最近的 1 亿美元融资是 IPO 计划的直接前兆。

有可能 Wansrath 正在帮助公司为这一转变做准备，但看起来更像是那些与年轻高管进行高度合作的技术(如 GitHub 和 WordPress)交易的公司越来越适应高管角色的转变。高层不那么僵硬可以让整个组织有更大的适应性，因此观察接受这些举措的公司在 2014 年的表现将会很有趣。
