# YouTube 在 Flash 与 HTML5 视频上的权衡

从今年早些时候 iPad 的[封杀](https://web.archive.org/web/20221206220216/https://beta.techcrunch.com/2010/01/27/apple-ipad-flash/)Flash，到一波又一波的[广告活动](https://web.archive.org/web/20221206220216/https://beta.techcrunch.com/2010/05/13/adobe-ad-apple/)、[公开信](https://web.archive.org/web/20221206220216/https://beta.techcrunch.com/2010/04/29/steve-jobs-tells-the-world-just-what-he-thinks-of-flash/)，以及随后的辩论，似乎每个人都对 Flash 在现代网络上的优点(或缺点)有自己的看法。今天，YouTube 软件工程师约翰·哈丁在该网站的官方博客[上发表了关于 HTML5 视频支持现状的评论。要点:虽然 HTML5 很棒，但它不能做 YouTube(或大多数主流视频网站)需要的一切。](https://web.archive.org/web/20221206220216/http://apiblog.youtube.com/2010/06/flash-and-html5-tag.html)

Harding 逐点解释了他的论点，从 HTML5 对标准视频格式的需求开始。政治、编解码器质量和专利纠纷使得 HTML5 无法对视频内容采用单一标准(行业标准 H.264 存在许可问题)。谷歌希望通过上个月发布的 WebM 项目和 VP8 来弥补这一缺陷。但是 VP8 距离成为行业标准还有很长的路要走。

接下来，Harding 阐述了视频流的问题——也就是说，在动态调整质量控制和缓冲方面，HTML5 并没有真正与 Flash 相匹配的东西。还有一个事实是 HTML5 不能保护版权内容不被复制(Flash 可以通过 RTMPE)。也没有办法把一个 HTML5 视频嵌入到另一个网站，至少不能像 YouTube 那样处理注释和其他相关数据。其他问题包括无法全屏播放 HTML5，没有从浏览器访问摄像头和麦克风的标准。

当然，这并不令人惊讶。今年早些时候，YouTube 开始提供 HTML5 视频播放器，但默认播放器仍然是 Flash。谷歌已经明确表示，尽管它普遍倡导开放标准，但它认为 Flash 仍有相当大的生命力。事实上，它甚至将[烘焙](https://web.archive.org/web/20221206220216/https://beta.techcrunch.com/2010/03/30/flash-player-to-come-bundled-with-google-chrome-new-browser-plugin-api-coming/)到其 Chrome 浏览器中。
