# SalesVu 为类似 Square 的移动支付服务筹集了 60 万美元 

> 原文：<https://web.archive.org/web/http://techcrunch.com/2011/11/14/salesvu-raises-600k-for-square-like-mobile-payments-service/>

该公司今天宣布，为分布式销售团队设计的新移动支付应用和 SaaS 解决方案 SalesVu 已获得 60 万美元的天使投资。与 Square 一样，SalesVu 的服务包括一个连接到 iPhone 或 iPad 的加密狗，允许其用户在旅途中使用移动应用程序处理移动支付。但与 Square 不同的是，这款设备没有插入耳机插孔，而是使用 iPhone 或 iPad 的主端口。加密狗还提供硬件加密，这是 T2 广场的竞争对手批评该公司没有包括的。

此外，SalesVu 在定价方面与 Square 针锋相对:所有交易(刷卡或键入)的价格仅为 2.7%，而 Square 的价格为 2.75%(仅针对刷卡交易)。

为了实现这些利率，该公司的做法与 Square 略有不同。首先，它不会免费分发所有的加密狗。每个企业客户都可以免费获得第一个加密狗，但之后每增加一个设备就要支付 99 美元。当然，如果公司愿意，可以通过为每个需要设备的员工注册一个单独的电子邮件地址来绕过这一要求。但这样做会妨碍企业利用 SalesVu 的另一个关键功能:基于云的集中价格管理、报告和营销基础设施。

使用 [SalesVu 的 SaaS 解决方案](https://web.archive.org/web/20230205014221/http://www.salesvu.com/)，企业可以监控移动交易，按照产品、时间和员工观察趋势，然后根据这些发现对定价和库存数据进行动态调整。所有这些都是实时发生的，并以易于阅读的格式呈现，包括图表、条形图、表格和其他报告。随着价格的变化，企业主也可以直接从 SalesVu 到脸书分享这些折扣，提供一个优惠券代码来激活交易。

由于其丰富的实时报告功能，SalesVu 正试图解决一个与 [Square](https://web.archive.org/web/20230205014221/https://squareup.com/) 甚至 Intuit 的 [GoPayment](https://web.archive.org/web/20230205014221/http://gopayment.com/) 略有不同的问题。也就是说，它不是在追逐那些以前从未负担得起信用卡支付的小企业或个人，而是在追逐那些依赖于高度移动的销售队伍的企业。理想的客户是这样一类人，他们的潜在利润会受到实时销售数据的影响，并能够根据对这些数据的分析快速调整产品库存。

至于价格呢？最低的 2.7%？根据 SalesVu 首席执行官 Pascal Nicolas 的说法，该公司必须与其合作伙伴 [Mercury Payment Systems](https://web.archive.org/web/20230205014221/http://www.mercurypay.com/) 进行相当多的谈判才能实现这一目标。但是 Mercury 认为移动支付是一个增长机会，所以它愿意冒这个险。

总部位于奥斯汀的 SalesVu 最初于 8 月 18 日向公众开放，现在在全球拥有大约 1000 名客户。它的 iPhone 和 iPad 移动应用可以作为独立的订单处理/业务管理应用，但信用卡处理部分只在美国和加拿大有效。
