# HireVue 收购了 CodeEval，这是一款面向招聘人员的程序员评估工具

[HireVue](https://web.archive.org/web/20221006112134/http://hirevue.com/) 今天宣布收购平台技术评估公司 [CodeEval](https://web.archive.org/web/20221006112134/http://codeeval.com/) 。交易的条款没有披露，但 HireVue 的创始人 Mark Newman 证实，CodeEval 团队将加入 HireVue，继续开发 CodeEval 产品。这里不收租金。

CodeEval 是由一个叫吉米·约翰的人创建的，他是 i/o Ventures 孵化项目的一部分，我敢肯定他不是做三明治的 T4。它提供了一种提供各种编程挑战的服务，招聘人员可以用它来筛选申请人。这个想法是将白板问题从面试移到流程的最开始，这样那些没有通过的人就不会进入面试阶段。

当我们去年报道这家公司时，评论者有不同的看法。根据我的经验，开发人员从来不喜欢在面试中写代码，所以这并不奇怪。这项服务给招聘人员带来的好处比给程序员带来的要多，这在就业市场上可能是个问题。一位评论者写道:“开发者从这个平台上什么也得不到:他们不是绝望的人。作为一名开发人员，我为什么要为每份工作申请做几十次测试？”

为了吸引开发人员，CodeEval 需要让开发人员通过回答同样的几个问题来申请更多的工作。目前使用这项服务的公司包括 BitTorrent、RapLeaf 和华纳兄弟。

“筛选工作候选人是将人与工作匹配的关键部分，像 CodeEval 这样的工具可以使技术筛选更加简单和快速，同时使候选人的体验更加愉快，”Puppet Labs 的首席执行官 Luke Kanies 说，他是开源 Puppet 应用程序的最初开发者。卡尼兹说，他对这个想法很感兴趣，但自己还没有使用过这种产品。

这似乎很适合 HireVue，它出售一个平台，供招聘经理管理工作面试和远程进行工作面试。它的产品，数字面试平台，被像耐克和星巴克这样的公司用来时移面试。候选人在规定的时间内在网络摄像头上提问，这样他们就不能“重做”和润色答案，然后经理可以在自己的时间观看视频，减少了进行初次面试所需的时间安排。

我认为这种东西对招聘人员和招聘经理的吸引力是不言而喻的。但是求职者呢——你怎么看？
