# 多了 Flash，谷歌是不是要割掉 HTML5 的鼻子来刁难苹果的脸？

现在外面流传着一个非常有趣的谣言。显然，谷歌即将宣布其 Chrome 浏览器和 Adobe 的 Flash 平台的某种新的合作关系， [CNET 今天早些时候报道](https://web.archive.org/web/20221227061215/http://news.cnet.com/8301-17939_109-20001370-2.html?part=rss&tag=feed&subj=Webware&utm_source=feedburner&utm_medium=feed&utm_campaign=Feed:+webware+(Webware.com))为谣言。

谷歌没有说话，但我们听到的是，这可能与谷歌[去年年底为](https://web.archive.org/web/20221227061215/http://googleblog.blogspot.com/2009/10/teaming-up-with-adobe-and-open-screen.html)签约的开放屏幕项目有关。Adobe 在一年前启动的项目旨在为网络开发者提供一个跨越各种设备的统一内容平台。谷歌关心这个问题的原因是它的 Android 移动操作系统，以及毫无疑问它将在今年晚些时候进军装有 Chrome 操作系统的上网本。

CNET 的报告除了谈到谷歌和 Adobe 之间的“更深层次的合作伙伴关系”,可能会看到 Flash 更多地集成到 Chrome 浏览器中——比现在更多——之外，没有更多细节。这意味着什么还不清楚，因为 Flash 是通过一个几乎所有浏览器都有的插件被带到所有浏览器上的。当然，插件如何在 Chrome 操作系统中工作还不完全清楚，所以这种合作关系可能意味着一些事情。

但比实际新闻更有趣的可能是谷歌和 Adobe 之间更紧密合作的象征性姿态。毕竟，谷歌似乎正处于与前死党苹果的[大战](https://web.archive.org/web/20221227061215/https://techcrunch.com/2009/08/24/500-days-of-apple-and-google/)[的](https://web.archive.org/web/20221227061215/https://techcrunch.com/2010/02/02/apple-google-multi-touch-android/)[早期](https://web.archive.org/web/20221227061215/https://techcrunch.com/2010/01/07/nexus-one-quattro-knives-apple-google/) [阶段](https://web.archive.org/web/20221227061215/https://techcrunch.com/2010/03/02/the-complaint-apples-patent-lawsuit-against-htc-is-all-about-android/)。拒绝在 iPhone 上安装 Flash 的苹果。同样是苹果，它将在周六发布不包含 Flash 的 iPad。同样的苹果公司，其首席执行官据说最近在试图说服合作伙伴放弃这项技术的同时，也对这项技术进行了抨击——至少在某种程度上，这项技术似乎正在发挥作用。

其他有趣的事情:谷歌提供更多的 Flash 支持似乎与其公开支持 HTML5 标准背道而驰。谷歌最近推出的每个新项目，从 Wave 到 Buzz，都在某种程度上推动了 HTML5。HTML5 的很大一部分是视频组件，它将允许用户播放网络视频，而不需要像 Flash 这样的插件。谷歌甚至为 YouTube 视频提供了 HTML5，而这之前只能在 Flash 上使用。

因此，一方面，谷歌似乎在说 HTML5 是未来，而另一方面，或许暗示网络仍然需要 Flash。毫无疑问，Flash 不会很快消亡，因此谷歌当然需要支持它(就像苹果的 Safari 网络浏览器一样)。但暗示 Flash 可能是网络的统一力量(正如 Open Screen 项目所指出的那样)，似乎违背了他们的论点，即 HTML5 是同样的统一力量。我不禁想知道这是否只是为了再次与苹果竞争。我想我们明天会学到更多。
