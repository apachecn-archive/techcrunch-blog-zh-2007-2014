# 脸书再次举起锤子:切断 MessageMe 对其社交图谱的访问

[MessageMe](https://web.archive.org/web/20221226083805/http://www.messageme.com/) ，[上周推出的一款应用在美国社交网络排行榜上一路飙升至第二名](https://web.archive.org/web/20221226083805/https://techcrunch.com/2013/03/08/messageme/)，该应用正面临脸书对其社交图谱访问权的敏感。

据知情人士透露，该应用与脸书的整合在今天早些时候停止了(见左)，这是该公司决定将 MessageMe 从其“查找朋友”功能中移除的结果。MessageMe 首席执行官 Arjun Sethi 拒绝对此事发表评论，脸书也没有立即回应置评请求。

此举类似于脸书上个月关闭 Voxer 访问该图的决定，尽管 Voxer 与脸书的连接已超过一年。Voxer 是另一款支持通话和语音聊天的通讯应用。脸书在[推出与其他用户免费语音通话功能竞争的同时，关闭了该应用。](https://web.archive.org/web/20221226083805/https://techcrunch.com/2013/01/16/facebook-rolls-out-voip-calling-to-u-s-ios-messenger-users/)

在该决定中，脸书引用了其平台政策的第 10 部分(这与 MessageMe 案中使用的是同一部分):

> 互惠和复制核心功能:(a)互惠:Facebook 平台使开发者能够通过 Graph API 和相关 API 构建个性化的社交体验。如果您使用任何脸书 API 来构建个性化或社交体验，您还必须使人们能够轻松地与脸书上的人分享他们的体验。(b) **复制核心功能:未经我们的许可，您不得使用 Facebook 平台推广或导出用户数据到复制脸书核心产品或服务的产品或服务。**

MessageMe 显然复制了太多 Facebook Messenger 的功能，让该公司感到不适。长期以来，脸书一直对向谷歌和 Twitter 等其最大的战略竞争对手提供访问权颇为敏感。早在 2010 年， [Twitter 当时的首席执行官 Ev Williams 抱怨公司不愿意让 Twitter 用户](https://web.archive.org/web/20221226083805/http://bits.blogs.nytimes.com/2010/11/17/twitter-still-seeks-cooperation-from-facebook/)在 Twitter 上查找他们在脸书的朋友，或者向 Twitter 发送脸书的更新。同年，谷歌和脸书[就脸书访问 Gmail 的联系人导入器](https://web.archive.org/web/20221226083805/https://techcrunch.com/2010/11/04/facebook-google-contacts/)发生了激烈的争论，因为这家社交网络不会以相反的方式发送数据。

但只是在去年，该公司才真正加强了对其他创业公司的执法力度。在上个月切断 Voxer 后，脸书澄清了其政策，称应用程序需要将内容分享回脸书，不能复制太多脸书的核心功能。它引用了相同的政策，在 Twitter 发布几小时后切断了它的 Vine，以及俄罗斯搜索引擎 Yandex 的应用 Wonder，因为它复制了太多的图形搜索。

在 MessageMe 的案例中，考虑到人们不想透露他们与谁发消息或他们私下说了什么，要求该公司分享数据有点愚蠢。

但对公司的影响可能不会太坏。Vine 在过去的两个月里蓬勃发展，在 iOS 应用商店上仍然占据着美国社交网络的第一名。[在脸书切断 Voxer 后的一个月里，Voxer 的活跃使用量似乎保持在水平](https://web.archive.org/web/20221226083805/http://insights.onavo.com/apps/ios/social-networking/89dbfdf7069135203/voxer-walkie-talkie-ptt)。

此外，MessageMe 的大部分增长实际上并不依赖于脸书。相反，它使用通讯录，这与 WhatsApp 和 Line 等其他大型通讯应用程序使用的方法相同。
