# Parse 通过分析、后台作业和 Web 奇偶校验充实了移动后端服务 TechCrunch

> 原文：<https://web.archive.org/web/https://techcrunch.com/2013/09/05/parse-developer-day/>

今天在 Parse 的第一个[开发者日](https://web.archive.org/web/20221210064622/https://beta.techcrunch.com/2013/07/16/parse-developer-conference/)大会上， [Parse](https://web.archive.org/web/20221210064622/https://www.parse.com/) 宣布了[的新功能](https://web.archive.org/web/20221210064622/http://blog.parse.com/2013/09/04/parse-developer-day-kicks-off-with-five-new-products/):运行 JavaScript 任务的后台作业；与游戏引擎 Unity 的大合作关系；管理图像和用户会话的更好方法，以及改进的网络支持，使其与移动应用程序产品不相上下。2004 年，马克·扎克伯格在台上谈到推出脸书时说道:“伙计，我真希望当时就有 Parse。”。

Parse 是脸书在 4 月份收购的移动后端即服务平台，以支持其平台服务产品。它帮助开发人员将他们的应用程序数据存储在云中，处理身份登录，发送推送通知，并从云中推出定制代码。Parse 成为脸书提供的第一个付费 B2B 服务。

当脸书收购这家初创公司时，它的平台上运行着 6 万个应用程序。尽管少数直言不讳的人抱怨脸书会偷看他们的作品，但 Parse 经历了一次大的增长，在收购后注册人数激增了 9.4 倍。到 5 月份，它的应用数量达到了 8 万个，6 月份达到了 10 万个。与此同时，Parse 还推出了虚拟主机服务，这样开发者就可以在与应用相同的系统上运行他们的网络展示或登陆页面，并轻松共享用户数据。

### 解析开发者日让它变得简单

扎克伯格用一个脸书起源的故事开始了会议。“我所有的钱是我父母为我上大学攒的 8000 美元。我们需要一些可以在服务器和路由器周围移动的东西。有人想出了在易贝购买二手路由器的主意，幸运的是，这个主意奏效了，”扎克伯格说。“我们最终管理了我们自己的所有服务器”，但是所有的后台工作分散了团队在社交网络方面的创新。

他希望 Parse 能够减轻初创公司的这种摩擦，“这样你就可以专注于打造出色的体验。”他说，在 Parse 和脸书，“我们正试图打造工具来帮助你做两件事:构建和发展你的应用。

Parse 首席执行官 Ilya Sukhar 随后出场，谈到 Parse 如何发展到处理“数十亿次 API 请求、数十亿次推送通知和数亿台与 Parse 对话的设备。”

然后，他进入了主题演讲的核心部分:为 Parse 开发人员提供的一系列新工具和集成。更多内容请见下文。

*   [**分析**](https://web.archive.org/web/20221210064622/http://blog.parse.com/2013/09/05/parse-analytics-same-great-taste-now-just-the-way-you-like-it/)–开发人员现在可以通过 Parse 跟踪他们应用的增长，推送通知的有效性、稳定性等，而不必将数据转储到 Excel 中。Parse 将自动获取有关登录、数据使用等指标，以便开发人员知道他们需要修复或加倍努力。
*   [**后台作业**](https://web.archive.org/web/20221210064622/http://blog.parse.com/2013/09/04/introducing-background-jobs/)–Parse now 允许开发人员编写小的 JavaScript 片段来调度和运行重复性任务，而无需他们自己的服务器。这些包括更快地发送电子邮件和做其他工作，因为 Parse 可以动态地将其巨大的服务器能力分配给这些任务。
*   [**Unity Partnership**](https://web.archive.org/web/20221210064622/http://blog.parse.com/2013/09/04/create-amazing-cross-platform-games-with-the-new-parse-unity-sdk/)–Unity 是一款游戏引擎，超过 200 万开发者使用它来处理图形渲染，更多用于 iOS、Android 和 Windows 游戏。合作伙伴已经创建了一个 Parse Unity SDK，它将使 Parse 开发人员在 Unity 上构建变得更加容易。
*   [**用户会话模块**](https://web.archive.org/web/20221210064622/http://blog.parse.com/2013/09/04/new-cloud-modules-for-images-and-users/)–新的用户会话模块简化了用户在移动设备和网络上的登录和注销管理，使之前的二级 web 开发体验与 Parse 的 MBaaS 不相上下。
*   [**图像模块**](https://web.archive.org/web/20221210064622/http://blog.parse.com/2013/09/04/new-cloud-modules-for-images-and-users/)–通过在解析云中运行最少的代码，开发人员现在可以在他们的应用程序中操作图像。Parse 不仅可以从用户的相机胶卷中提取图像，还可以对图像进行调整大小、裁剪、着色等操作。

这些都不一定是“令人惊叹”的特性，而是 Parse 扩展其功能的逻辑进展。会议的目标不是让开发者震惊或害怕，也不是给他们一堆东西让他们去思考。它只是想让他们的生活更轻松，这些工具应该会有所帮助。当您可以在 Parse 中完成分析、后台工作或图像处理时，为什么还要在其他地方费心呢？

当天晚些时候，由脸书项目经理道格·波弟领导的一个客户小组在一个客户小组上解释说，当 Parse 在 App Store 中出现巨大的流量高峰时，Parse 拯救了他的应用程序，使其免于崩溃。

更新:在会议期间，我和一些开发人员聊了聊他们为什么不使用 Parse。大多数担忧是关于稳定性。仍有一些人担心，如果脸书没有实现目标，它可能会缩减移动后端服务，这可能会让开发者陷入困境。脸书过去并不羞于改变其开发者平台，尽管我认为 Parse 是其平台战略中足够重要的一部分。

让开发人员远离 Parse 的最有根据的批评是关于正常运行时间。当你在 Parse 上构建时，你必须以某种方式开发你的应用，它不能在其他地方运行。令人担忧的是，如果 Parse 暂时关闭，开发者就不能快速地将他们的应用移植到其他地方，不像亚马逊这样的网络托管服务。现在，如果 Parse 出现停机，开发人员会承担风险。

一位开发人员告诉我，他希望脸书提供一个 Parse 溢价期权，附带保险，如果 Parse 倒闭，保险将补偿他失去的业务，从而将风险从他身上转移到脸书身上。这样的选择可以增强开发者的信心，让他们相信 Parse 是值得投资的。

### 为什么解析脸书的问题

总的来说，Parse 的想法是，如果它的工具可以更好地帮助开发者完成后端，他们可能会到脸书[寻求帮助，将社交整合到他们的应用程序中，并获得更多用户](https://web.archive.org/web/20221210064622/https://beta.techcrunch.com/2013/04/25/everything-but-a-facebook-os/)。通过这种方式，Parse 可以很好地与脸书整合社交登录和分享的软件开发套件以及其安装广告的应用相媲美。通过 Parse，脸书可以让更多的应用程序将内容分享回它的新闻源，这样它就可以通过广告赚钱，也可以让开发者为更多的广告付费。

脸书可能没有像 iOS 或 Android 这样蓬勃发展的移动应用平台，可以对应用内购买征税。但是，通过结合后端托管、分析、社交整合和广告来促进增长，脸书希望 Parse 能够像开发人员所依赖的操作系统一样重要。
