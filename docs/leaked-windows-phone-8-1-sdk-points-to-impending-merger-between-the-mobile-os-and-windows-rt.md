# 泄露的 Windows Phone 8.1 SDK 指出移动操作系统和 Windows RT 即将合并

今天早些时候，一名泄密者[在 Reddit 上列出了微软即将推出的 Windows Phone 8.1 的细节，这些细节是通过提前获得更新的软件开发工具包(SDK)而浮出水面的。](https://web.archive.org/web/20221207094824/http://www.reddit.com/r/windowsphone/comments/1xloor/wp81_is_going_to_have_javascript_support/)

除了一系列功能升级，这些变化还包括微软将 Windows Phone 8.1 和 Windows RT 更紧密地结合在一起。这与微软高管朱莉·拉森-格林的[声明](https://web.archive.org/web/20221207094824/https://beta.techcrunch.com/2013/11/22/microsofts-julie-larson-green-hints-at-further-windows-harmonization-across-device-classes/)一致:“我们有手机操作系统，我们有 Windows RT，我们有完整的 Windows。我们不会有三个。”很明显，微软打算——鉴于它不能也不会终结 Windows 的桌面友好风格——在某种程度上合并 Windows RT 和 Windows 8.1。

如上所述，今天的泄漏符合这种模式。微软拒绝置评。

[泄露的 Redditor 详细描述了](https://web.archive.org/web/20221207094824/http://www.geekwire.com/2014/leak-shows-microsoft-unifying-windows-phone-windows-rt-app-development/?utm_source=feedburner&utm_medium=twitter&utm_campaign=Feed%3A+geekwire+%28GeekWire%29)“通用应用”将会是一个“模板，用 HTML 和 JavaScript 更容易地从同一个共享代码的解决方案构建 WinStore 和 WinPhone 应用”。Windows RT 是微软的操作系统，几乎完全依赖于 Windows 应用商店。

统一 Windows Phone 和 Windows RT 应用程序体验将允许每个平台的现有和未来应用程序目录的交叉授粉，也许会提高开发人员对更大的 Windows 平台的兴趣。  

正如 The Verge [的 Tom Warren 指出的](https://web.archive.org/web/20221207094824/http://www.theverge.com/2014/2/11/5400660/windows-phone-8-1-features-leaked)，新 SDK 的开发者元素指向微软在未来允许 Windows Phone 和 Windows Store 应用程序在任何设备上并行运行。

Windows Phone 8.1 中还有通知中心、Internet Explorer 11、改进的 YouTube 体验、独立的视频和音乐应用程序、改革的多任务处理、帐户处理方式的改变(似乎不再有脸书，并增加了 iCloud)等等。完整的名单还不清楚，因为我们还没有完整的，公开的代码。

然而，鉴于微软似乎正在向开发者分发代码，预计将会有更多的人被释放。对微软来说是坏消息？不完全是。事实上，他们肯定已经预料到了这一点。以上都不是对公司的损害，只是他们未来的风头被部分衰减了。

在假期前令人失望的小版本 Windows Phone 8 Update 3 之后，微软即将发布的智能手机操作系统更新似乎即将完成，功能完善。希望在其 Build 大会上，微软将更好地详细说明现在基本公开的 Windows RT 和 Windows Phone 即将实现的统一。
