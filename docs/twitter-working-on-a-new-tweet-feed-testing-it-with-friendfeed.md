# Twitter 正在开发一个新的 Tweet Feed，用 FriendFeed 进行测试

如果你在过去的几个月里一直是 [FriendFeed](https://web.archive.org/web/20221006002435/http://friendfeed.com/) 的用户，你可能已经注意到推文导入的速度有多慢。但是几天前发生了一些变化——在推特上发布后，推特几乎立即出现在提要中。这是因为 FriendFeed 正在测试 Twitter 正在开发的一种新型后端 Feed。

这个新的 feed 很有趣，因为到目前为止，FriendFeed 是少数几个可以访问 Twitter XMPP feed 的服务之一。有些人将 XMPP 提要称为“消防水管”，因为它可以快速发送大量数据，这与通过 RSS 导入提要的方式不同，后者既有限又缓慢。但是 XMPP 在过去的几个月里一点也不快，所以需要一个新的解决方案。

这个新的提要仍然是实验性的，但是我们知道:它是基于 HTTP 的，并且，至少对于 FriendFeed 来说，它将取代 XMPP 提要。Twitter 正在使用 FriendFeed 作为它的试验场，但是最终，如果它成功了，它应该向其他服务开放。“希望我们能够确定哪些有效，哪些无效。Twitter 的詹森·高德曼告诉我:“无论我们从这次测试中得出什么结论，都将告诉我们如何前进。”。

到目前为止有点小故障。当 FriendFeed 在周三首次实现它时，它几乎是即时的——FriendFeed 的联合创始人保罗·布克海特[发了一条推文](https://web.archive.org/web/20221006002435/https://friendfeed.com/e/dddba9e6-0c8f-4156-9c55-1da907038125/I-just-activated-the-new-Twitter-feed-so-public/)，一秒钟后它就出现在 FriendFeed 上了。但在随后的几天里，订阅速度慢了很多，有时需要 20 分钟才能发布一条推文。据 Buchheit 称，Twitter 认为它已经修复了一个漏洞。

Twitter 是 FriendFeed 的重要组成部分，因为 tweets 在网站内容中占很大比例。当然，Twitter 正在成为许多服务的重要组成部分，所以希望这种基于 HTTP 的 feed 有朝一日将意味着所有人都可以近乎即时地访问 Twitter。
