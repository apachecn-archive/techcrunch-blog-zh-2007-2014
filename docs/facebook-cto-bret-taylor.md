# 脸书的交友化仍在继续:布雷特·泰勒晋升为首席技术官

![](img/21afeec691ac78d546a3bbae51801ecd.png)

脸书任命了新的首席技术官 Bret Taylor。FriendFeed 的联合创始人和谷歌地图的初始产品经理去年带着对 FriendFeed 的 5000 万美元收购来到了脸书。他担任了脸书平台总监的角色，并领导了最近推出的[脸书的开放图](https://web.archive.org/web/20230403184929/https://techcrunch.com/2010/04/21/zuckerbergs-buildin-web-default-social/)和开放图 API，试图让网络上的社交联系[和超链接](https://web.archive.org/web/20230403184929/https://techcrunch.com/2010/04/21/facebook/)一样重要。他在简化脸书平台的构建过程中发挥了关键作用。

越来越多，脸书看起来越来越像 FriendFeed，到处都有类似的按钮，并且更加强调中央流。泰勒从 FriendFeed 带来了很多工程和设计方面的东西，并开始将它们灌输给脸书。现在，他被提升为空缺的首席技术官，负责监督脸书平台以外的其他项目，包括搜索和新闻订阅/主页。自 2008 年 Adam D'Angelo 离开脸书创办社交问答服务 Quora 以来，该公司的首席技术官职位一直空缺。

以下是马克·扎克伯格不久前发给所有脸书员工的电子邮件，宣布泰勒升职，脸书向 TechCrunch 提供了这封邮件:

> =======================
> 马克·扎克伯格的内部邮件
> ====================
> 
> 嘿大家好，
> 
> 我有一些好消息要和大家分享。我创建了一个新角色，并邀请 Bret Taylor 担任我们的首席技术官。
> 
> Bret 大约一年前加入我们，担任平台产品总监。从那时起，他在构建我们新平台的许多部分中发挥了关键作用，包括社交插件、我们的新 graph API 和 Open Graph。自 f8 以来，已经有超过 100，000 个网站使用社交插件，我们的新 API 因其优雅和简单而受到了很多好评。此外，Bret 在许多方面帮助我形成了对产品、工程和战略的想法。
> 
> 如今，Bret 只有几个直接下属，通过提供有用的建议和积极影响许多产品的决策来完成工作。我最近一直在和他讨论他如何扮演类似的角色，与其他一些领域合作，帮助塑造我们的方向。由于 Bret 同时参与技术和产品问题，我决定在工程和产品之外创建一个新的 CTO 职位是正式确定这一新角色的最佳方式。
> 
> 在这个职位上，Bret 将向我汇报，不管理其他任何人。CTO 角色不是管理角色。在这里，建立和运营产品、工程和运营组织的角色一点也没有改变。如果你在过去会去找施瑞普、克里斯·考克斯或海利格，你现在仍然应该去找他们。(尽管，说实话，Schrep，Cox，Bret 和我都坐在同一个舱里，所以你几乎可以同时抓住我们中的任何一个。)
> 
> Bret 将继续专注于平台，但这个新角色也让他能够在其他领域提供更多帮助。Bret 一直从事的平台产品管理工作将继续向 Cox 和产品组织汇报。我们能够做出这一改变的原因之一是因为 Mike Vernal 领导工程团队所做的出色工作。我对他继续建设这个组织非常有信心。
> 
> 当我环顾产品和工程时，我们现在正在用非常有杠杆作用的小团队构建许多独特的东西。平台是整个行业的基础，我们的团队有大约 30 名工程师。News Feed 是每天超过 2.5 亿人的主页，而我们的团队只有不到 15 名工程师。我们的搜索类型与谷歌的查询数量级相同，我们的团队只有不到 15 名工程师。这些是我们将在未来几年打造的变革性产品的例子，我专注于确保我们以正确的方式打造它们。
> 
> 如果您有时间，请和我一起祝贺 Bret 的新角色。如果你有任何关于这个或其他的问题，请随时给我留言，或者在我们下一次的公开问答中提问。
