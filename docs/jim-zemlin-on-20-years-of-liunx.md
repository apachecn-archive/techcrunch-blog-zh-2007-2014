# Jim Zemlin 谈 Linux TechCrunch 的 20 年

> 原文：<https://web.archive.org/web/http://techcrunch.com/2011/08/15/jim-zemlin-on-20-years-of-liunx/>

非营利组织 Linux 基金会的执行董事吉姆·泽姆林使用 Linux 的时间和我一样长，大约是 Linux 出现时间的一半。我最近和吉姆谈论了 Linux 基金会即将到来的 [LinuxCon](https://web.archive.org/web/20230204223357/https://techcrunch.com/tag/linuxcon) ，Linux 的历史，以及未来二十年可能会发生的事情。

如果你回顾一下计算的历史，我们会看到大型老牌企业在各自的领域占据主导地位，然后慢慢衰落，在某些情况下甚至彻底消亡。40 年前，计算全是大型机和 UNIX。然后，个人电脑时代开始了，像微软 Windows 这样的桌面操作系统占据了主导地位——UNIX 和大型机仍然存在，但未能适应计算基本特性的巨大变化。在过去的十年中，我们看到了移动计算的绝对爆炸——微软仍然是一个竞争者，但不可否认的是，他们对人们使用计算设备的方式的变化反应缓慢。

另一方面，Linux 已经在所有这些平台上蓬勃发展。这有很多原因，但 Linux 长寿的根本原因无疑是它的开源根源。Linus Torvalds 在 GNU Public License 下发布了 Linux，允许人们在需要时使用和扩展它，前提是他们与世界其他地方共享他们的工作。

Zemlin 对 Linux 开源本质的长期价值进行了更深入的挖掘。根据他的说法，它允许自我形成的社区出现，以搔他们自己的痒处。这些自我形成的社区的工作以及它们之间的交叉授粉带来了意想不到的好处，其中 X 上的工作明显使 y 受益。例如，嵌入式设备的 Linux 内核中的电源管理工作明显改善了大型机 Linux，而功耗是主要的成本考虑因素。

这样，感兴趣的各方已经能够为每一个可以想到的计算平台构建 Linux:从大型机到台式计算机到电话，再到电视和汽车内的嵌入式设备。大型机 Linux 人员可能没有在嵌入式 Linux 上工作的动机，但是 Linux 或其开发模型中也没有任何固有的东西排除跨多个硬件平台的同时开发。

Zemlin 有许多 Linux 适应性的现成例子，所有这些都有助于确保它在今天和未来的相关性。我故意唱反调，向泽姆林建议说，Linux 在很大程度上是一种反应式的努力，对需求做出反应，而不是预测需求并打造新的解决方案。泽姆林很快反驳说，Linux 的开发绝对是创新的。人们只需要看看高性能计算市场中的非凡工作就能找到例子。Linux 主导了 90%的 HPC 市场，研究人员正在不断寻找新的方法来使 Linux 在该领域脱颖而出。嵌入式设备也是如此，Linux 是事实上的选择。

[![](img/b5154ede85b076a180212b2b3231fe8f.png "linux-world-domination") ](https://web.archive.org/web/20230204223357/https://techcrunch.com/wp-content/uploads/2011/08/linux-world-domination.jpg) Linux 用户习惯开“Linux 一统天下”的玩笑。有各种各样巧妙而又讽刺的海报图片，一只巨大的无尾半正式企鹅隐约出现在微软公司总部的上空。如今，莱纳斯·托沃兹不再开这些玩笑了。根据 Zemlin 的说法，Linux 是“一股不可阻挡的力量”:每天生活的很多方面都直接或间接地受到 Linux 的影响。正如泽姆林所说，“除非你是微软、Appe 或 RIM，否则你就是在用 Linux。”的确是统治世界。

有人认为 Linux 内核可以很好地预测未来的技术。Linux 内核本身获得新特性的速度要比各种 Linux 发行版快得多。因此，现在核心内核中的东西将在几个月后成为主流发行版，因为供应商们会用他们栈中的其他组件来测试这些新特性。因此，如果您观察内核提交，您会很好地了解 Linux 计算的发展方向。

泽姆林开玩笑说，他预计在今年的 LinuxCon 上，会出现许多明显的中年危机。毕竟，开源社区中的许多知名人士都是伴随着 Linux 成长起来的。Linux 基金会一直在努力工作，以确保下一代创新者准备好接过衣钵。今年的 LinuxCon 以一个[学生项目](https://web.archive.org/web/20230204223357/http://events.linuxfoundation.org/events/linuxcon/student-program)为特色，包括为五名幸运获奖者提供的[培训奖学金](https://web.archive.org/web/20230204223357/http://www.linuxfoundation.org/news-media/announcements/2011/08/linux-foundation-announces-linux-training-scholarship-recipients)，让学生有机会直接从开源世界的主要杰出人物那里学习开发方法和技术。

LinuxCon 不仅仅是一个庆祝彼此成就的大团体拥抱，而是一个不断推进 Linux 开发的机会。可能需要六个月的邮件列表来来回回才能解决的问题，可以通过五分钟的面对面聊天来解决。人们可以遇到志同道合的开发人员处理类似的问题，以加快问题的解决，分享经验，并改善 Linux 社区的整体状态，这与该社区所做的工作一样重要。

今年我将参加 LinuxCon，与开发者和高管们谈论 Linux 的过去、现在和未来。我还将参加[媒体小组](https://web.archive.org/web/20230204223357/http://events.linuxfoundation.org/events/linuxcon/media-panel)，在那里，其他记者和博客作者将分享他们对这些年来媒体对 Linux 报道性质变化的看法。
