采访:我们采访了Mac OS X Plex媒体中心的首席开发人员:在Boxee变得很酷之前，它就在做类似Boxee的东西

原文：https://web.archive.org/web/https://techcrunch.com/2010/01/15/interview-we-talk-to-the-lead-developer-of-plex-media-center-for-mac-os-x-it-was-doing-boxee-like-stuff-before-boxee-was-cool/amp/


Plex的主菜单，显示有可选的永旺皮肤

就XBMC福克斯而言，Boxee显然是热度最高的。它有风险投资资金涌入，与内容提供商达成浮华的交易，你很快就可以购买一个专用的D-Link box，以便更方便地在你的电视上使用。但是Boxee并不是唯一值得你花费时间的XBMC媒体中心。这甚至不是第一个XBMC福克走出去，并为自己的名字。Mac OS X专属的Plex ，在Boxee酷炫之前是Boxee。我最近与首席开发人员Elan Feingold进行了交谈，以更好地了解Plex是什么、它做什么以及它的发展方向。不用说，如果你在经营麦克OS X，你应该试一试。挺好的。

Plex是一个基于XBMC的媒体中心，它的前身是Xbox Media Player，一个在被入侵的微软Xbox上运行的媒体中心。Xbox Media Player于2002年首次发布，是当时功能最丰富的媒体中心。随着项目的成熟——这个项目根本没有得到微软的认可或支持——它变成了XBMC，Plex(以及Boxee、MediaPortal和Voddler，仅举几个例子)就基于这个项目。

换句话说，每当你称赞Boxee(或Plex，或任何其他XBMC衍生的媒体中心)时，请记住这一切是如何开始的:一个旨在播放和组织被黑客入侵的Xboxes上的视频、音乐和照片的业余爱好者项目。(你甚至可能想更进一步，把功劳归于mplayer和FFmpeg的开发者，这两个核心的开源“引擎”从第一天起就为Xbox Media Player提供了动力。)让我们在该表扬的地方给予表扬，就这样。

就这样，下面是我和Elan的对话。希望能有所启发。

什么是Plex？你会如何向一个完全的计算机新手描述它？你在Xbox1上使用过XBMC/XBMP吗？

Elan Feingold: Plex是一个媒体中心，也就是说它是一个为您的媒体提供“十英尺界面”的程序。你坐在沙发上，喝啤酒，看视频，看照片，听音乐。它来自于让XBMC代码在更强大的机器上运行的愿望，这种机器可以处理更高分辨率的媒体，而完美的平台似乎是Mac(尤其是Mini的出色外形)。我在一台旧的XBox上用了几年XBMC(它现在正放在我的车库里收集霉菌),我很喜欢它。

CG:开发是从什么时候开始的？开发团队有多少人？你和XBMC的人是什么关系？

elan:2007年12月，首次下载XBMC代码，试图在Mac上构建它。我妻子出城了，我基本上很无聊。目前，我们有大约五名开发人员从事Plex的不同方面的工作。当我发布了一份进度报告，上面有关于Mac端口进度的截图时，XBMC团队联系了我，并让我加入了他们的团队。在接下来的几个月里，我发布了这个端口的一些早期版本，我称之为“OSXBMC”。然而，很早就很清楚，我对这个项目有着与XBMC团队不同的目标和愿景，所以我们结束了分叉代码，不久之后就变成了Plex。

CG:哦伙计，我记得OSXBMC！我也记得它变成Plex的那一天。

依兰:你还记得我得到的关于这个名字的所有狗屎吗？

嗯，人们都是仇恨者。

Elan:你不可能取悦所有人。

CG:对。

Elan:那段(OSXBMC时期)对我来说是一段相当艰难的感情时期，仅仅是因为我在代码上所做的所有工作以及所有的争论和尖酸刻薄。

CG:哇，好的。现在我不得不问你(或团队)从Plex上赚到钱了吗？(你卖了多少个马克杯？)或者这是一种爱的劳动？

Elan:我们有一个捐赠链接(右上角),人们对它非常慷慨，但它并不比覆盖服务器、服务、设备等所需的多多少。我想我们已经卖了一打左右的杯子了！所以现在这对我们所有人来说都是一项充满爱的工作。

明白了。现在，Plex仅仅是一项爱的工作，还是你们希望有一天能为自己赚几美元？

Elan:不要误会我的意思，我们都想全职做这件事，并获得报酬。

CG:你们似乎比Boxee更有“独立”的感觉。

Elan:众所周知，我喜欢留时髦的胡子。

CG:多么惊人的反应。

依兰:但这正是我们喜欢用来形容自己的词，“独立”

CG:好的，Boxee的一大特色是你可以在上面安装各种应用。如果我错了，请纠正我，但是Plex已经有一段时间了。事实上，Xbox上的XBMC几乎没有你可以安装的应用程序，Shoutcast radio，X-Link Kai。基本上，Boxee没有发明媒体中心应用程序的想法，我有点觉得这是一种看法…

Elan:是的，我们一年多前就有相同的功能了。我们有超过130个“应用程序”它们比Boxee或XBMC的同类产品更容易编写。我们实际上有自己的“应用商店”，叫做Plex媒体服务器，在Boxee推出他们的“应用盒子”或其他什么东西之前运行。

CG:那么开发一个Plex应用程序需要多少编码知识呢？有哪些比较成功的？

Elan:基本上，你需要了解一点Python，这是非常容易学习的。你可以用10-15行代码编写一个Plex应用程序，因为我们有一个非常好的框架来编写它们，它是由我们的主要开发人员之一James Clarke开发的。我们最受欢迎的是苹果电影预告片，You Tube，Hulu，网飞，MTV音乐视频，BBC iPlayer，Vimeo。到目前为止，我们的应用程序下载量已经超过了一百万次。

现在你们和那些人(Hulu，网飞等)有什么问题吗？)?我好像回想起Boxee和Hulu有一段时间的来往。

Elan:是的，所以网飞和Hulu是截然相反的两个阵营。我们和网飞进行了很多友好的交谈，我们和他们的关系很好。我们在他们的应用库中被列为网飞官方认可的应用。有了Hulu，我们的关系就没那么融洽了。他们已经采取了相当多的直接行动，试图打破我们对他们网站的支持。我真的很感激，因为它帮助我们使我们对这类网站的支持更加强大。我受宠若惊，他们曾经在HTML中命名一个div为“PL”，我想它代表“Plex”(因为它试图破坏我们)。

CG:太棒了。嗯，不太好，你知道我的意思。你和其他内容提供商有类似的问题吗？

Elan:大多数其他内容提供商(如Spotify，许多其他瑞典提供商)真的喜欢在我们的平台上。许多人与我们接触，甚至提供支持。有趣的是，这似乎取决于国家。出于某种原因，瑞典内容提供商似乎真的很喜欢Plex，有趣的是，瑞典拥有世界上最高的Plex人均用户。

CG:美国排名如何？你有类似“我们从德克萨斯下载了100次，从巴黎下载了400次”的统计数据吗？

Elan:我必须找到这份报告，但肯定在前10名中，不要感谢内容提供商！

你和Boxee的人聊过吗？他们有没有试图用丰厚的风投资金雇佣你的一个开发人员(或你)？

Elan:有趣的是，当Avner Ronen来毛伊岛看我时，我实际上为他做了晚餐。除此之外，我不认为他接触过我们队里的任何人。我给他做了一道非常辣的绿色泰国咖喱，我们友好地聊了聊。

让我们谈一会儿拳击。比方说，Netgear就制作一个类似Boxee盒子的盒子与你们接洽，你们会考虑吗？

Elan:我们不相信人们还想要另一个盒子。也就是说，我们确实意识到600美元的Mac Mini太贵了，无法获得我们想要的眼球数量。

CG:苹果电视仍然动力不足，无法正常运行，对吗？

Elan:所以我们致力于以更低的价格向人们提供Plex，只是不是用一个硬件。是的，我认为苹果电视不是一个可行的平台。动力不足，封闭。

与苹果的其他产品相比，苹果电视的表现也不太好。

Elan:最近XBMC支持一种可以进行高清解码的Broadcom芯片，这引起了一些热议，但你必须拆开你的Apple TV并安装芯片，我们认为许多不住在父母地下室的人不会这么做。

CG:没错，我就是那个孩子，改造Xboxes，彻夜不眠试图弄明白这些事情。

依兰:是的，我一开始是一名电气工程师，所以我也是那个孩子。

CG:那么你认为现在最理想的Plex设置是什么？我曾经用一根长长的HDMI电缆从iMac连接到电视，然后将iMac插入一个便宜的5.1接口。显然，不是每个人都会这么做…

Elan:如果你买得起，新款Mac Minis非常棒:安静，小巧，完美播放高清内容。再加上一个环绕接收器，你就万事俱备了。但是，是的，人们也使用他们的笔记本电脑和iMacs。我们的用户中有50%使用Mac Mini，20%使用MacBook Pro，12%使用MacBook，11%使用iMac，3%使用Hackintosh。很明显他们中有五个人使用XServe。

CG:太棒了，某个在服务器农场的家伙在夜班时看电影。现在Plex是面向“普通人”还是更高级的用户？

Elan: Plex面向的是普通媒体用户，我们意识到，要让它更容易使用，我们还有很多工作要做。我们不想把它弄得像前排一样沉闷，但是现在有两个锋利的边缘，你可能会伤到自己。

你是说像一个新的用户界面？还是说这个综合体太像瑞士军刀了？

是的，就像一个进化的用户界面。软件项目必须记住的一件事是，添加一个特性会增加非零的支持成本。我们并不认为拥有更多功能就一定更好或更有吸引力，除了对一小部分人。

CG:是的，我是那种宁愿看到软件做好一件事，也不愿看到100件事做得不好的人。

Elan:准确地说，开源项目在这方面是出了名的糟糕。

是的，他们是。那么你现在正在开发一个新的UI吗，或者这是你将来要做的事情吗？

Elan:我们正在改进当前的界面，我们已经花了相当多的时间来模拟事物，甚至构建Flash界面来测试它们的可用性。

哦，在我忘记之前，你有官方的推特账号吗？我听说推特风靡一时。

Elan: @Plexapp

CG:太棒了。那么，Plex的下一步是什么，只是继续开发吗？

依兰:嗯，我们一直在努力开发Plex/Nine系列。

CG:还会有像Plex媒体服务器这样的东西出现吗？有点莫名其妙。

Elan:是的，在接下来的几个月里，你会看到一个非常有趣的新功能。

CG:太棒了。有什么提示吗？

elan:Plex/Nine系列还将推出新的图书馆系统，用于管理您的本地内容。它从头开始建造就是为了踢屁股。

我只是想问，是你们建造的吗？

是的，新的图书馆系统是100%由我们建造的。因此，我认为，对于Plex来说，这应该是令人兴奋的一年，因为新的库、改进的用户界面和这一令人惊讶的新功能。

CG:非常酷。除了Plex你还做什么？

Elan:白天我是自由软件顾问。只有到了晚上，我才换上前面有巨大“P”的衬衫。

我们什么时候能看到《丛/九》？

Elan:这似乎是每个人都在问的问题！

CG:就像一个粗略的估计，或者你可以说“当它准备好了！”

Elan:我想说的是，在接下来的几个月里，你会看到一些新的东西出现。随着时间的推移，我们将发布不同的片段。我们将有稳定的Plex/seven系列，然后我们将开始在Plex/Nine版本中引入一些新功能，供人们使用。我们的主要关注点之一一直是稳定性，这就是为什么到目前为止稳定的Plex/Eight系列已经发布了五个版本，重点是修复错误。

CG:苹果曾经联系过你们吗？也许只是想说，“嘿，很酷的应用程序，伙计们。”

Elan:我们从未听说过他们，尽管一些在苹果零售店工作的人评论说，“我向顾客推荐Plex！”但是苹果公司甚至没有承认我们的存在。这对我们来说很好，我们一直试图保持良好的一面，不为被黑的苹果电视等发布Plex。

CG:你有没有考虑过一个iPhone应用程序，比如一个花哨的遥控器之类的？

Elan:嗯，现在已经有一些支持Plex的iPhone遥控器了。他们似乎工作得很好。

哦，我没有iPhone，所以我不确定。

没有，先生！

Elan:再见，谢谢！
