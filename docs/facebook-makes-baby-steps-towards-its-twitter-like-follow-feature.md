# 脸书向其类似 Twitter 的“关注”功能迈出了一小步

在过去的几天里，我们收到了许多关于脸书一个看似新现象的提示:人们收到了一些他们试图添加为好友但尚未接受其好友请求的人的状态更新。实际上，他们可以看到这些用户的状态更新，却看不到他们的个人资料——这在脸书上是前所未有的，至少直到最近才出现。那么，脸书终于开启了它几个月前向 T2 承诺的类似 Twitter 的“关注”功能了吗？该功能将允许用户开始接收来自他们不是朋友的人的状态更新。

不完全是。事实证明，这一功能是第一次[在六月下旬](https://web.archive.org/web/20230327132128/http://blog.facebook.com/blog.php?post=98499677130)宣布的，位于脸书关于其发布者升级的测试版发布的博客帖子中，它让你选择你想让谁看到给定的更新。现在，只要你添加了一个状态更新为“所有人”的脸书好友，你就可以在你的新闻订阅源中收到这些更新，即使他们从未接受你的好友请求。

从某种意义上说，这类似于将来某一天的“follow”功能，但它不是一个特别好的替代品。首先，没有一种简单的方法来判断某人是否将状态更新设置为“所有人”，所以每次你将这些人添加为好友时，你都是在瞎猜。在这个过程中，你还会在收件箱里收到大量的好友请求。尽管如此，这个专题，尤其是我们的读者对它的反应，显示了当它是单向和双向联系的混合体时，脸书会变得多么令人困惑。

当前的设置显然很奇怪。自 3 月份以来，你可以将个人资料和状态更新的某些部分设置为 [everyone](https://web.archive.org/web/20230327132128/http://blog.facebook.com/blog.php?post=60186587130) ，这使得通过脸书搜索进入你的个人资料或在其他人的墙上看到你的名字后点击你的名字的任何人都可以看到它们。但仍然没有一个选项可以让用户“关注”某人，并在他们的新闻订阅中接收他们的状态更新，而无需成为脸书的朋友。该功能仍然为允许单向连接的*页面*保留。用户可以自由创建他们自己的脸书页面，尽管这些页面不允许你有一个单独的朋友群来访问更多的个人数据，比如你的照片。但是个人资料和页面之间的区别是[认为](https://web.archive.org/web/20230327132128/https://techcrunch.com/2009/03/04/liveblogging-facebooks-open-door-press-conference/)正在消失，所以也许他们很快就会有那个功能。一切都很混乱。

希望一旦脸书部署其备受期待的[新隐私功能](https://web.archive.org/web/20230327132128/https://techcrunch.com/2009/07/01/live-blog-the-facebook-privacy-conference-call/)，这一切都将澄清，这将为他们带来更多的控制，并有望带来更直观的体验。然而，这次更新也将更加强调与每个人分享——我认为这可能会导致更多的问题。
