# Linux 的现状——连苹果都在开源

大企业在打电话——他们想谈谈 Linux。

现在，云已经出现，并且大部分都是建立在开源基础上的，Linux 也在不断发展。大企业明白这一点，他们想搭上这趟列车。就连苹果也在适应。

Linux 基金会执行董事吉姆·泽姆林在圣地亚哥为第一个[云开放](https://web.archive.org/web/20221006231152/http://events.linuxfoundation.org/events/cloudopen)做了主题演讲，他指出一切都在变成一种服务。这些服务正走向企业的最前沿，它们是基于开源构建的。

所以如果你想掌握软件，你必须掌握开源。否则你做不到。没有它，就无法在服务领域取得成功。

谷歌、IBM 和英特尔都表明他们正在投资开源软件。甚至你想不到的公司也在采用开源技术。

泽姆林说，苹果是世界上最封闭的公司之一，它实际上使用了很多开源软件。苹果 iPhone 中的授权信息证明了这一点。在 iPhone 的法律部分，它引用了 Linux 内核开发者 [Ted Ts'o](https://web.archive.org/web/20221006231152/http://en.wikipedia.org/wiki/Theodore_Ts'o) 的代码。Linux Suse 也在那里。

Zemlin 指出，苹果拥有数千亿美元的现金，足以购买惠普、英特尔和戴尔的总和。相反，苹果公司购买了通用 Unix 打印系统(CUPS)的版权，该系统现在安装在所有的 Linux 和苹果系统上。

使用 Linux 的公司不止苹果一家。曾经将开源等同于共产主义的微软，现在是 Linux 内核项目的顶级贡献者。VMware 正在赶上这股潮流。该公司收购了 Nicira，它是 OpenStack 的顶级贡献者，open cloud 致力于构建自己的基础设施。本周，VMware 提交了加入 OpenStack 社区的申请。在 VMworld，高管们表示，此次收购是向开源靠拢的战略的一部分，而不是远离开源。

IT 的未来是软件。它会比人们更勇敢。这种创新将来自构建开放云环境的工程师。事实是，开放对不同的人来说有很多含义。在多个云上运行并不容易。这不仅仅是代码的问题，也是系统的问题。开源云运动面临的挑战是，它如何帮助将各个部分结合在一起，使云成为真正的联邦。
