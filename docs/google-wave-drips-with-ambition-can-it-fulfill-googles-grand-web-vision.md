# 谷歌浪潮雄心勃勃。新网络的新交流平台。

> 原文：<https://web.archive.org/web/https://techcrunch.com/2009/05/28/google-wave-drips-with-ambition-can-it-fulfill-googles-grand-web-vision/>

![google_wave_logo](img/722ebaabdcdd9e0ff60a4676241010eb.png "google_wave_logo")昨天，在[谷歌 I/O 主题演讲](https://web.archive.org/web/20230405021901/https://techcrunch.com/2009/05/27/live-from-google-io-2009/)期间，谷歌工程副总裁[维克·冈多特拉](https://web.archive.org/web/20230405021901/http://www.crunchbase.com/person/vic-gundotra)展示了一个宏伟的愿景，谷歌认为网络将朝着 HTML 5 标准前进。虽然我们还没有做到这一点，但除了微软之外，所有主要的浏览器玩家都已经准备好进入下一阶段，这将包括在浏览器中运行 3D 游戏和电影的能力，而无需额外的插件。但是谷歌想更进一步，为这个新时代提供一种全新的交流方式。叫做 [Google Wave](https://web.archive.org/web/20230405021901/http://wave.google.com/) 。

现在，每个人都在网上使用电子邮件和即时消息，但是想象一下，如果你能把这两种形式的交流联系在一起，并在其上添加许多功能。从最基本的形式来看，这就是 Wave 的本质。Wave 是由谷歌澳大利亚悉尼办公室的兄弟 [Lars](https://web.archive.org/web/20230405021901/http://www.crunchbase.com/person/lars-rasmussen) 和 [Jens Rasmussen](https://web.archive.org/web/20230405021901/http://www.crunchbase.com/person/jens-rasmussen) 和 [Stephanie Hannon](https://web.archive.org/web/20230405021901/http://www.crunchbase.com/person/stephanie-hannon) 开发的，Wave 诞生于这样一个想法，即尽管电子邮件和即时消息仍然很成功，但它们都是在很久以前创建的。我们现在有了一个更加强大的网络，充满了内容，充满了分享的欲望。或者正如 Lars Rasumussen 所说的，“如果电子邮件是今天发明的，它就会是波。”

看了一个冗长的演示，虽然听起来很荒谬，但我不得不同意。Wave 提供了一种非常时尚和简单的方式来导航和参与网络交流，这使得电子邮件和即时消息看起来很陈旧。更好的比较巧合的是，这家公司是由另一群(前)谷歌人创办的。但是 Wave 是一个不同的产品，原因有很多，而且看起来有更高的目标——所有这些我都会在下面谈到。

[![google_wave_snapshots_inbox](img/09b5cd6eb1f3a7b76c3c318ba582352b.png "google_wave_snapshots_inbox")](https://web.archive.org/web/20230405021901/https://techcrunch.com/wp-content/uploads/2009/05/google_wave_snapshots_inbox.png)

**特色**

在我们的演示过程中，Wave 团队不断重申，该产品基本上仍处于初级阶段。虽然它诞生于 2004 年延斯·拉斯穆森的一个想法，但它被搁置了(当时他和兄弟拉斯正忙于构建最终成为谷歌地图的东西)，直到 2007 年才被复活，并最终在几个月前进行内部测试。谷歌不断重申这一点的原因是，目前存在的功能集只是他们最终目标的一小部分。也就是说，已经有很多事情要做了。

Wave 有一个左侧边栏“导航”,下面有一个来自谷歌联系人的联系人列表。但屏幕的主要部分是你的 Wave 收件箱。这看起来和你的 Gmail 收件箱很相似，除了它的特色是每个主题中你朋友的脸。也有数字指示器表示该线程中是否有新内容。这是与 Gmail 的一个重要区别——它不仅仅是关于新邮件，在这些浪潮中可以有任何种类的新内容。

点击任何一个 wave 线程都会在收件箱右边打开另一个窗格，显示完整的 wave。假设一波是朋友发来的消息，你想回复它。如果他们目前不在线，你可以像在 Gmail 中一样，在他们的邮件下方操作。除了没有庞大的新消息创建器弹出打开，你只是开始在你朋友的消息下面输入。但是也许你想回复他们信息的某个特定部分——你也可以简单地在你要回复的部分下面输入。

可能你想再加一个好友到浪里。你可以走到你的联系人框前，把他们的照片拖到 wave 中。这是事情真正开始变得有趣的地方。如果这位朋友想了解 wave 中其他人已经在谈论的内容，他们可以通过使用“回放”功能来实现。这有点像倒带，看看过去发生了什么，你可以通过它的变化来观察它的进展。

[![google_wave_concurrent_edit](img/ef397872dc4c3dae0009244e65a72ea2.png "google_wave_concurrent_edit")](https://web.archive.org/web/20230405021901/https://techcrunch.com/wp-content/uploads/2009/05/google_wave_concurrent_edit.png)

但是如果浪潮中的两个人同时在线，你们可以在同一个浪潮中实时交流。只需开始键入，您的朋友就会在您输入单词时看到它们，反之亦然。很明显，这是一个类似即时消息的元素，但关键是，它只是潜在的波浪对话的一小部分。如果你不喜欢实时交流的想法，在这种交流中，别人可以在你打字的时候看到你在打什么，你可以进入“草稿”模式来隐藏你的单词，直到你准备好发送它们。

假设在多人的帖子里有一个人你想私下传达信息。你可以很容易地在波浪中中断私人谈话。显然，只有你和另一个接收者才能看到这个信息，但是对你们两个来说，它会留在波本身的流动中，保持它在上下文中。

但是 Wave 不仅仅是传统的信息和回复方式，我们已经习惯了电子邮件和即时消息。您也可以通过并发的小组协作来编辑维基风格的东西。任何试图在谷歌文档上对文档进行分组编辑的人都知道，这很快就会变得棘手。但是 wave 提供了一个很好的用户界面和实时的编辑更新，以确保即使是几个人在编辑 Wave 中的东西时也不会互相踩踏。当某人正在编辑某个东西时，您会看到他们的名字被一个颜色鲜艳的方框勾勒出来，旁边是他们正在实时进行的编辑。如果你感到困惑，你可以使用我在上面描述的回放功能跳来跳去，看看编辑的内容。

从这里我们走得更远。假设你想在 Wave 上分享一些最近的照片，如果你有一个安装了 Gears 的浏览器，你所要做的就是将照片拖放到 Wave 窗口中。值得注意的是，这是让 Wave 在没有任何附加功能的情况下无法与“现代”网络浏览器(火狐、Chrome、Safari 和 Opera)完全兼容的一个致命弱点。(“现代”是谷歌对微软互联网浏览器的被动攻击方式。)谷歌表示，它希望看到这种功能被添加到 HTML 5 标准中，因为它真的简化了这种类型的共享。

[![google_wave_yes_no_maybe_inbox](img/29835a420297486219b446607fba520e.png "google_wave_yes_no_maybe_inbox")](https://web.archive.org/web/20230405021901/https://techcrunch.com/wp-content/uploads/2009/05/google_wave_yes_no_maybe_inbox.png)

这真他妈的酷。如果你与其他几个人分享 wave 线程中的照片，从你将照片拖到你这边的 wave 中的那一刻起，你的朋友就可以在他们的屏幕上看到照片的缩略图。wave 中的每个人都可以合作修改图片的标题，你也可以观看图片的幻灯片。

但是图片只是开始。其他可以在 Wave 中分享内容包括谷歌地图(可以编辑)、游戏、活动邀请等等。而这些只是 Wave 团队自己想到的例子。这就把我们带到了波的下一个点。

**波任网络交流**

谷歌不仅仅把 Wave 看作是另一个你可以在一个网站上使用的网络应用，它希望你可以在网络上的所有网站上使用它。比方说，你有一个博客。作为一个帖子，你可以与公众分享一个 wave，并允许其他人看到你和你的 wave 中的其他人在做什么。这些访问你博客的人甚至可以直接从你的博客加入进来，所有的信息都会直接放入原始的 wave 中。

这可能有几种方式。你可以在你的网站上启用 wave 上的匿名协作，让用户能够用你网站上已经有的某种方法登录，比如评论用户名，或者他们可以用他们的 Wave/Google 名称登录并互动。我们询问使用类似脸书连接的东西作为编辑 wave 的方法，虽然拉斯·拉斯穆森说他们还没有专门为这种功能做任何事情，但他们正在考虑这一点——尽管你可以肯定谷歌会更喜欢朋友连接。

Waves 也可以作为它们自己的实体发布在网络上。这将使他们和他们的内容被谷歌的机器人索引。但是 wave 团队很小心地注意到，如果有东西在网上向公众发布，你可能会在主 Wave 窗格中看到 Wave 中有一个很大的指示器。

[![google_wave_inbox_chess](img/e03370b5bb3d41b1ca56d9d3defdfec0.png "google_wave_inbox_chess")](https://web.archive.org/web/20230405021901/https://techcrunch.com/wp-content/uploads/2009/05/google_wave_inbox_chess.png)

谷歌希望 Wave 登陆的不仅仅是博客，而是你能想象到的几乎任何类型的网站。有很多不同的用途。例如，许多公司使用某种类型的管理系统进行电子邮件或即时消息之外的交流。我们用的是 T4 的 Yammer，当我在 VentureBeat 的时候，我们用的是 FriendFeed 的私人房间。Yammer 不错，但它基本上只是你自己系统上的 Twitter。FriendFeed room 更有活力，但是 FriendFeed 托管了它。Wave 可以提供两个世界的最好的东西，它们都是为在自己的服务器上托管 wave 的公司甚至个人服务的。

这是整个想法的关键之一。谷歌不希望 Wave 成为它的另一个应用，它希望 Wave 成为一个交流平台，它可能已经开始，但在网络上的许多不同地方蓬勃发展。这就引出了下一个问题。

**开发者，开发者，开发者**

曾在微软工作过的 Gundotra 认为，比尔·盖茨早期坚持拥有一个强大的开发者社区是 Windows 成功的关键之一。(我们都知道现任 CEO 史蒂夫·鲍尔默对开发者的看法。)对开发者的这种重视也有助于 Android 和 iPhone 等新平台的发展。如果 Wave 要成功，整个团队都知道这将再次归功于开发者。

虽然 Rasmussen 兄弟和 Hannon，以及大约 50 名目前在谷歌从事该项目的开发人员已经构建了一个非常有趣的框架，但如果开发社区接受它，这只是 Wave 潜力的一个缩影。周五，谷歌将向开发者开放 Wave 的 API，让他们体验一下。希望在短期内，将会有大量的小工具、扩展、混搭和有趣的网站围绕 Wave 概念建立起来。

[![google_wave_map_yes_no_maybe](img/1d335dec7af7cd2b192266ff794e6d88.png "google_wave_map_yes_no_maybe")](https://web.archive.org/web/20230405021901/https://techcrunch.com/wp-content/uploads/2009/05/google_wave_map_yes_no_maybe.png)

这个想法是让这个系统尽可能的开放。该团队希望看到某个人创造的波与另一个人创造的波进行交流。“我们希望它成为像电子邮件一样的开放系统。我们希望其他服务也能建立 Wave 服务，即使是在与谷歌竞争的情况下，”拉斯·拉斯穆森告诉我们。

考虑到这一点，谷歌计划开源 Wave。这将是 Google Wave 的第三阶段。第一个是 Google Wave，这是一款由 Google 创建、开发并最终以网络应用的形式发布给公众的产品。第二个是 Google Wave，即[平台](https://web.archive.org/web/20230405021901/http://code.google.com/apis/wave)，我们在上文中将其概述为一个可供开发者参与和创造的系统。但是第三个方面是 Google Wave，协议，这是它作为 web 通信平台的存在。在 Waveprotocol.org 的[网站了解更多信息。](https://web.archive.org/web/20230405021901/http://www.waveprotocol.org/)

因为它将是开源的(因为谷歌已经准备好公开发布)，它将不会只是谷歌在负责它会变成什么。正如它对 Android 所做的那样，很大程度上将由开发社区来决定它的发展方向。或者，至少，这是希望。

**新网**

所以，如果你读到这里，你可能会想 Wave 要么听起来很棒，要么你对它到底是什么感到困惑。它确实是那种你必须亲眼看到才能理解的产品。不幸的是，除非你是开发人员，否则你不可能马上看到它。正如我们在开始时提到的，Wave 仍处于早期阶段，但谷歌从中看到了足够的前景，它希望让开发者社区尽早参与进来——这就是为什么我们看到它在谷歌 I/O 上推出。

同样重要的是要注意，Wave 在很大程度上是围绕谷歌专注于 HTML 5 的关键基础:画布元素、视频元素、地理位置、应用程序缓存以及数据库和网络工作者。你可以在奥赖利雷达或昨天的[我们的现场报道](https://web.archive.org/web/20230405021901/https://techcrunch.com/2009/05/27/live-from-google-io-2009/)中读到更多关于这些[的内容，但 Wave 的关键之一将是网络工作者。这种功能允许你在浏览器之外运行后台进程，这样就不会像运行非常丰富的应用程序那样缓慢 Wave 就是这样。](https://web.archive.org/web/20230405021901/http://radar.oreilly.com/2009/05/google-bets-big-on-html-5.html)

Web Workers 有助于将浏览器转变为下一代网络应用的更加成熟的发射台。这是昨天主题演讲的要点，今天的主题演讲提供了迄今为止 Wave 中这种新型应用的最佳范例。

这是一个非常有趣的概念，你真的需要看到它的实际应用。它雄心勃勃——我们很喜欢——但这也给它带来了失败的可能性。但伟大的产品就是这样诞生的。如果谷歌能够成为向我们的网络数字生活彻底转变的领导者，潜在的回报将是巨大的。

[![google_wave_inbox_add](img/2ebfa5715cf9f8bd55a8f5d715966ee7.png "google_wave_inbox_add")](https://web.archive.org/web/20230405021901/https://techcrunch.com/wp-content/uploads/2009/05/google_wave_inbox_add.png)

**更新**:我们参加了第二天的 Google I/O 主题演讲，团队正在展示 Google Wave。有许多令人印象深刻的特性我们甚至没有深入研究(是的，有这么多)。我最喜欢的一个不是 只在 Wave 中搜索实时，而是真的直播。例如，如果您从“Here”中删除一个“e ”,它将从该单词的结果中消失。同样，如果你重新加上“e ”,它会立刻再次弹出。

另一个很棒的功能是高级拼写检查器，它不仅检查字典中的拼写，还查找句子中的上下文。认为这可行是疯狂的，但是当他们炫耀它的时候，它确实可行。

是的，Wave 将与 Twitter 一起工作。该团队自己为它创建了一个小工具，他们称之为“Twave ”,它从你的流中引入推文，并带有你的联系人的 Twitter 图标。你可以在 Wave 中回复这些推文，它们会回到你的 Twitter 流中。但最好的功能是 Twave 的搜索功能，它可以实时扫描 Twitter，并在新结果出现时实时更新。你可以用它来实时追踪任何你想要的东西。

现在团队正在展示一种可以实时翻译成其他语言的 wave。再次令人印象深刻。

谷歌 I/O 上的演示在巨大的起立鼓掌声中刚刚结束。如果今天有任何迹象，这将是一个大事件。
