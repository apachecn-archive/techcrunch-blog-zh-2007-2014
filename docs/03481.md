# Twitter 缩短的 URL 扩展对 Bit.ly 链接不起作用

![](img/44cb6a3bbbed8b83d26a4f348aefe8b2.png)通常，当你使用 [Twitter 的搜索服务](https://web.archive.org/web/20230306173925/http://search.twitter.com/)时，你可以避免盲目点击缩短的网址——这是一个[安全风险](https://web.archive.org/web/20230306173925/https://techcrunch.com/2009/09/24/video-symantec-shows-the-danger-of-shortened-twitter-links/)——通过扩展它们，看看它们会把你带到哪里。我一直在使用它，我甚至已经习惯了在网络上使用 [Brizzly](https://web.archive.org/web/20230306173925/https://techcrunch.com/2009/09/21/brizzly-faster-better-more-open/) 作为 Twitter，部分原因是它会自动扩展任何缩短的 URL。

我建议任何人不要在不知道链接到哪里的情况下点击短网址，即使它来自你认识和信任的人，因为这也不能保证安全链接。在这方面，Twitter 搜索现在似乎不再扩展[bit . ly](https://web.archive.org/web/20230306173925/http://bit.ly/)URL，这是 Twitter 使用的[默认网址缩写](https://web.archive.org/web/20230306173925/https://techcrunch.com/2009/05/06/url-shortening-wars-twitter-ditches-tinyurl-for-bitly/)，这并没有帮助。

**更新:**噗，又起作用了。

我们不确定这个问题是什么时候开始出现的，但 Twitter 应用开发者 [Mallikarjun Reddy 今天早些时候注意到了这个问题](https://web.archive.org/web/20230306173925/http://www.malliks.in/2009/09/twitter-breaks-its-tinyurl-expansion.html)，并补充说这不是 Bit.ly 方面的问题，因为它的 URL 扩展 API 似乎工作得很好。幸运的是，大多数其他网址缩写的扩展仍然有效，但由于 bit.ly 是社交网络上使用最广泛的工具，这不是一件简单的事情。即使 bit.ly 尽了自己的职责来警告用户恶意链接，Twitter 也有责任尽可能地保护用户。

Twitter 工程师的待办事项清单上又多了一项。

**更新:**是的，当我们向[我们的 Twitter 流](https://web.archive.org/web/20230306173925/http://twitter.com/techcrunch)推送内容时，我们使用 bit.ly。不，那和我上面写的完全没有关系。
