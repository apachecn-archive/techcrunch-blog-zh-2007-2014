# Trsst 是后斯诺登时代互联网的安全推特

在一个推文成为新短信的世界里，有些人想要传播他们的想法——但要有可接受的安全水平。这就是 [Trsst 进入](https://web.archive.org/web/20230129222525/http://www.kickstarter.com/projects/1904431672/trsst-a-distributed-secure-blog-platform-for-the-o)的原因。简而言之，它是一个加密的信息平台，可以将你的短信息转换成 p2p 风格的协作系统。

我意识到这有点难以理解(我仍然有些难以理解)，但本质上它将是一个签名的、加密的、去中心化的系统，模仿 Twitter 的功能。这意味着你的个人对个人的信息将是完全安全的，更重要的是，你广播的信息将被数字签名。数据库将是分布式的，比特币式的，所以没有一家公司控制它。

“这个项目的起源是不久前阅读了戴夫·维纳的一篇名为“[用 RSS](https://web.archive.org/web/20230129222525/http://threads2.scripting.com/2013/march/makeATwitterOutOfRss) 制作 Twitter”的帖子，我认为这很有启发性，然后有一些布伦特·西蒙斯的帖子，然后我偶然发现了一些 JavaScript 客户端加密库，并融入了比特币项目的一些想法，”创始人迈克尔·鲍尔斯说。鲍尔斯是一位老企业家，他认为这将是一个很酷而且有用的副业项目。

“真正的与众不同之处在于 Winer，因为我们试图让一切都变得非常简单。用户无法知道幕后发生的所有事情。他们去一个网站，注册，然后他们就离开了，”鲍尔斯说。“该服务可以与现有的 http 服务器一起工作，它们仍然提供 rss 服务，因此易于采用和大量互操作，但也有空间像真正的 p2p 网络一样真正优化联合。也就是说，那些其他项目都是由真正聪明的人组成的，我们将合作并重用我们能得到的任何开源代码。”

他要求 48，000 美元来支付制作费用。12 美元就可以进入测试版——比 App.net 在全盛时期花在 T4 身上的钱还要少——而且因为这个项目表面上是开放的，由用户控制，而不是中央服务器。

为什么我们需要 Trsst？鲍尔斯写道:

在任何时候，这些公司的董事和股东可能会撤销他们对你的隐私承诺，他们甚至可能在没有通知你的情况下这样做。这些公司运营所在的政府可能会随时颁布法律，将他们所拥有的数据(包括您的个人身份信息和存储的通信内容)据为己有或收归国有。
这可能已经发生了。没有不受政府管辖的公司。没有地方是安全的。

我认为这有可能实现吗？我表示怀疑。然而，这是一个由精明的家伙创造的崇高目标。我认为像 Twitter 这样的广播技术已经无处不在，普通用户不会转移到其他地方——App.net 就是明证。然而，我们肯定都需要社交媒体的安全性，这是实现它的一种方式。
