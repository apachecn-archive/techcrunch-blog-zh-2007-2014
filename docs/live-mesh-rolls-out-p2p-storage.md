# Live Mesh 推出 P2P 存储技术 

> 原文：<https://web.archive.org/web/https://techcrunch.com/2008/07/21/live-mesh-rolls-out-p2p-storage/>

# 实时网格推出 P2P 存储

[![](img/1eeee13a37393dabf18eb46926583455.png)](https://web.archive.org/web/20230221013821/http://www.crunchbase.com/product/windows-live-mesh)

微软[的](https://web.archive.org/web/20230221013821/http://blogs.msdn.com/livemesh/archive/2008/07/11/service-update-new-build-new-features-coming-today-0-9-3103-2.aspx) [Live Mesh](https://web.archive.org/web/20230221013821/http://www.mesh.com/) 团队今天宣布，在最新的客户端更新中，Mesh 节点之间的 P2P 存储现已可用。以前，网格上的存储通过利用分配给微软中央托管平台上每个用户的 5GB 来工作。用户现在能够在连接的节点之间自动同步他们的文件。

这项新技术的工作原理是在计算机之间同步文件，但元数据仍然集中存储(尽管它不计入用户配额使用)。除了 Live Desktop 之外，所有网格对象和文件夹都可以在设备之间直接复制。下面的视频来自 Live Mesh 团队，他们在谈论 P2P 技术，以及他们如何解决冲突和其他问题，这些问题在过去一直困扰着其他 P2P 存储尝试。

[http://channel9.msdn.com/posts/Dan/414732/player/](https://web.archive.org/web/20230221013821/http://channel9.msdn.com/posts/Dan/414732/player/)
[大卫·斯蒂尔和特雷弗·罗宾逊:实时网状 P2P 同步如何工作](https://web.archive.org/web/20230221013821/http://channel9.msdn.com/posts/Dan/David-Steere-and-Trevor-Robinson-How-Live-Mesh-P2P-Syncing-Works/)

正如我们之前写的那样，目前 Mesh.com 上的应用程序是基于开放格式和标准构建和开发的更广泛的开发平台的实现。Live Mesh 已经向任何持有微软护照的人开放了几个星期，Vista 和 XP 客户端也可用于 Live Mesh，Mac OS X 客户端正在开发中。
