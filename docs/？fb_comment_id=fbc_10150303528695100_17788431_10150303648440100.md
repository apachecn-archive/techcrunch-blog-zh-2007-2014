# 为什么马克·扎克伯格对 Google+的首次公开回应是正确的

![](img/7cafea80623a090e9e317dff2feb015b.png)

***编者按**:这篇客座博文由 MySpace 前总裁、创始人、第一个朋友[汤姆·安德森](https://web.archive.org/web/20230203142946/http://www.crunchbase.com/person/tom-anderson)撰写。它改编自他在 Google+ 上发表的一篇[评论，经允许转载。你现在可以在](https://web.archive.org/web/20230203142946/https://plus.google.com/112063946124358686266/posts?tab=mX)[脸书](https://web.archive.org/web/20230203142946/http://www.facebook.com/myspacetom)、[推特](https://web.archive.org/web/20230203142946/http://twitter.com/#!/myspacetom)和[谷歌+](https://web.archive.org/web/20230203142946/https://plus.google.com/112063946124358686266) 、上找到汤姆*

今天在脸书新闻发布会上，一名记者问马克·扎克伯格对 Google+有什么看法。扎克伯格回应说，许多公司都将建立像视频聊天这样的东西，但脸书的竞争对手也必须先建立他们的社交网络。脸书的工作就是不断创新。这是一个完全合理的回答，当然，他是完全正确的——挑战是获得用户群，并使他们容易使用你的产品。为脸书做了一切。集成看起来很棒。

一些专家抱怨说这项技术并不新，但这不是重点。举个例子:在 MySpace，我们推出了扎克伯格在 2007 年宣布的东西(试着谷歌一下[“MySpace Skype 合作伙伴关系”](https://web.archive.org/web/20230203142946/http://www.google.com/search?sourceid=chrome&ie=UTF-8&q=myspace+skype+partnership)), MySpace 在 2004 年也有一对一的视频聊天。关键是，当时人们还没有做好准备——现在是时候了，脸书已经有了用户基础。庞大的用户群([7 . 5 亿](https://web.archive.org/web/20230203142946/https://techcrunch.com/2011/07/06/zuck-confirms-that-facebook-now-has-750-million-users/))与可以说是最好的语音/视频技术(Skype)的简单整合造就了这一新闻。

扎克伯格还在对 [Hangout 问题](https://web.archive.org/web/20230203142946/https://techcrunch.com/2011/07/06/facebook-video-chat-google-hangouts/)的回应中指出，一对一的视频聊天将是更常见的用例(Google+有“Hangout”，允许 10 个用户同时进行视频聊天)。同样，完全合理，而且可能是正确的。许多网站都有群组视频聊天，Google+不是第一个，Hangout 也不是游戏改变者。这里你需要的是用户群，目前只有脸书有，人们更有可能一对一地交谈(就像我们在电话里做的那样，废话)。

我认为，他的声明中更有趣的部分是他在介绍 Skype 整合时对 Google+的含蓄回应。扎克伯格说的是，脸书上的群组被 7.5 亿用户群中的一半积极使用。“Groups”实际上是脸书对“朋友列表”的第二次尝试，扎克伯格去年承认，[没有获得牵引力](https://web.archive.org/web/20230203142946/https://techcrunch.com/2010/08/26/facebook-friend-lists/)(人们不想做把人放入列表的工作)。

脸书集团的功能是在某种程度上设计，使用户谁愿意做的工作，可以。有人邀请你，你就加入了这个团体，而你不必采取任何行动。(其实你得做点工作才能出群！)扎克伯格指出，这也是好友请求的工作方式——总是有少数人做所有的好友，而我们其他人只是跟着做，更容易获得“批准”脸书的群组是为了克服好友列表问题而设计的。他们发展很快，尽管 95%的用户都懒得建立自己的群组。

如果你仔细想想，这是脸书处理许多事情的聪明方法:建立一个应用程序平台，让开发者社区来做繁重的工作。创建一个翻译平台，让用户用人类已知的每一种语言翻译脸书。创建一个群体特征，让那 5%的人为其他 95%的人创建群体。就像土耳其机器人，但是我们没有报酬。(除非你是 Zynga！)

还有待观察的是，从长远来看，用户会更喜欢哪种模式——脸书“群组”——其功能更像一个内置论坛的老派雅虎群组。或者 Google+“Circles”——这更像是一个电子邮件分发列表与 Twitter 的结合，提供了更好的评论。这两者实际上非常相似，但是每一个都可能在某些方面比另一个做得更好。思考每个模型做得更好可能是解开“模型”将“赢”的关键。
