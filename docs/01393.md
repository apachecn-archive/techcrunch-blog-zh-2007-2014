# 雅虎拥抱语义网——期待互联网迅速组织起来

> 原文：<https://web.archive.org/web/https://techcrunch.com/2008/03/13/yahoo-embraces-the-semantic-web-expect-the-web-to-organize-itself-in-a-hurry/amp/>

雅虎对所有开放事物的拥抱今天仍在继续——预计一个小时左右，他们将宣布扩展他们的开放搜索平台，这是我们上个月写的。

在之前的公告中，雅虎谈到了他们的计划，即允许第三方利用可能对用户有用的结构化数据来改变和增强搜索结果。今天，他们将提供更多关于开发者平台的细节，并将宣布对一些[语义网](https://web.archive.org/web/20230126105804/http://en.wikipedia.org/wiki/Semantic_Web)标准的支持。

这一切意味着什么？这意味着我们可以期待网络很快地自我组织起来。这关系到雅虎搜索的大量流量，以及[任何人](https://web.archive.org/web/20230126105804/http://techcrunch.com/2007/03/09/this-is-cool-unless-it-achieves-consciousness-and-kills-us-all/) [或](https://web.archive.org/web/20230126105804/http://www.google.com/)可能选择在这些数据基础上构建应用。

雅虎对语义网标准的支持，如 RDF 和微格式，正是网站采用这些标准的动机。不再是分散在网络上的语义孤岛(想想[缠绕](https://web.archive.org/web/20230126105804/http://techcrunch.com/2007/10/19/twine-launches-a-smarter-way-to-organize-your-online-life/))，雅虎将把所有可用的语义信息整合在一起，就像搜索引擎应该做的那样。到目前为止，很少有应用程序需要来自第三方的适当结构化的数据。今天这种情况发生了变化。

雅虎产品管理总监艾米特·库马尔和其他人在昨天的简报会上给我举了一个例子，那就是 LinkedIn——如果他们用微格式标记用户资料页面，雅虎搜索就能理解内容以及各部分内容之间的关系。然后，雅虎可以在雅虎搜索中以智能的方式展示这些数据。*“随着对 LinkedIn 的结构化数据(包括在我们的索引中)的更深入了解，我们将能够为用户的网站提供更有吸引力和更有用的搜索结果，”*雅虎表示。这是它在搜索结果中的样子(见我们在雅虎开放搜索上的[前一篇文章](https://web.archive.org/web/20230126105804/http://techcrunch.com/2008/02/25/yahoo-announces-open-search-platform/)了解这是如何实现的):

任何第三方都可以利用他们的语义数据为雅虎搜索创建 mod(雅虎将在几周内推出一个测试程序，以及一个开发者发布会)。一些幸运儿会被默认添加到所有搜索中。

一些细节正在被披露，雅虎承诺在几周内提供更多信息。他们说他们将在一开始支持一些微格式:hCard、hCalendar、hReview、hAtom 和 XFN。它们将支持来自 Dublin Core、Creative Commons、FOAF、GeoRSS、MediaRSS 等的词汇表组件。他们将支持 RDFa 和 eRDF 标记，以便将它们嵌入到现有的 HTML 页面中。最后，雅虎将支持 Amazon A9 [OpenSearch](https://web.archive.org/web/20230126105804/http://www.opensearch.org/) 规范，并对深度网络数据的结构化查询进行扩展。

Erick Schonfeld 在二月份写了一篇文章，敦促雅虎完全开放搜索以与谷歌竞争。雅虎还没有朝这个方向前进。但他们看起来肯定最终会到达那里。
