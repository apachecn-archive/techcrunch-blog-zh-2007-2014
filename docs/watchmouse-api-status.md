# WatchMouse 监控 26 个流行的 API，所以你不必

网站监控初创公司 [WatchMouse](https://web.archive.org/web/20230304024034/http://www.watchmouse.com/en/) 正在推出一项名为[API-status.com](https://web.archive.org/web/20230304024034/http://api-status.com/)的新服务，这是一个显示流行的公共 API 的实时可用性和性能的网站。

受监控的 API 列表包含 26 个流量最大的网络服务，包括谷歌搜索、谷歌地图、必应、脸书、Twitter、SalesForce、YouTube、亚马逊、易贝等。

API-status.com 每五分钟从全球 42 个位置调用并检查每个 API 的有效结果。如果系统在其中任何一个上检测到异常或超过四秒的延迟，结果将被记录为错误，API 的状态将在网站上显示为不可用。这些错误用于计算每个站点的可用性或正常运行时间的百分比。

关于 API-status.com 的详细信息包括 7 天的历史，以及 24 小时的概览和每个 API 的国家/地区的性能指示。

WatchMouse 在 2009 年 12 月 16 日至 2010 年 1 月 16 日期间密切监控所有 26 项网络服务的可用性。为了便于调查，初创公司只选择了每个 API 的一个函数；最简单的一种，需要认证。这意味着他们的调查不包括对全部 API 的深入测试，而是提供对每个 API 的可用性和工作功能的估计。

如上所述，测试站点的方法包括一个简单的 API 调用和检查有效结果。这通常意味着一个认证动作，包括登录，然后是搜索或列表动作，再加上对预期结果的检查动作。使用 WatchMouse 公共状态页面工具实时检查每个站点。

WatchMouse 发现 Yammer 的 API 可用性最低(正常运行时间为 96.09%)，而亚马逊、谷歌地图、谷歌搜索、Last.fm 和雅虎地图的可用性最高，正常运行时间为 100%。更多结果[此处](https://web.archive.org/web/20230304024034/http://www.watchmouse.com/en/SPI/2010/API_availability.php)。
