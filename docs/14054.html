更多B2D: TappingStone希望通过其用户行为预测即服务TechCrunch来挖掘对数据科学家的需求

原文：https://web.archive.org/web/https://techcrunch.com/2012/10/18/more-b2d-tappingstone-wants-to-tap-the-need-for-data-scientists-with-its-user-behavior-predictions-as-a-service/

为开发者服务的创业公司，即所谓的B2D，无疑正在崛起。我开始记不清有多少次我被推销类似于“像Twilio但不像X”这样的东西。这是有道理的。随着资金在硅谷和其他地方流动，反过来，可用的工程人才短缺，为初创公司提供外包部分技术的能力似乎是一个不容错过的商机(Semil Shah在最近的TC客座博文中很好地提出了这一点)。

总部位于伦敦的 TappingStone 希望抓住这个机会，它已经在私人测试版中推出了用户行为预测即服务。通过利用该公司的 API 和后端算法，应用程序开发者有望构建更智能的应用程序。根据TappingStone返回的数据，开发人员可以为每个最终用户个性化他们的应用程序，这样做不仅可以改善用户体验，还可以更有针对性地进行追加销售，或者通过预测哪些用户在永久离开之前不太可能留下来，来做其他有趣的事情，如课程变动。

这只是两个相当原始的例子。但是，从本质上讲，开发者将他们的用户数据插入TappingStone，然后查询其API以获得用户行为预测。他们根据这些预测采取什么行动是他们自己的事情，但是TappingStone做了很多繁重的工作，这就打开了各种可能性。

联合创始人兼首席执行官Simon Chan表示:“我们相信，出于多种原因，对TappingStone这样的服务的需求将会非常巨大。”。“首先，像脸书、谷歌、LinkedIn、Zynga、Flipboard等大公司。在数据工程上投入了大量资金。如今，应用程序必须是智能的。消费者只是期望如此。其次，随着智能手机和平板电脑的增长，人们消费更多的应用程序，更多的人希望为这个不断增长的市场开发应用程序。第三，尽管有需求，但数据科学家和工程师严重短缺。创业公司几乎招不到他们需要的人，而许多公司却感受到了留住人才的痛苦。”

这是一个不错的提议，尽管不承认存在竞争对手是不负责任的。灵光乍现，陈让我注意到了预言师。我敢说还有其他的，以及开发者可以利用的开源推荐引擎和算法。

然而，Chan说TappingStone的不同之处在于，它没有采取一刀切的方法。相反，他声称它能够基于每个特定的数据集和用户行为自适应地构建一个独特的算法。

“为各类app提供这样的自助式行为预测服务，在技术上很有挑战性。例如，新闻应用程序和餐馆发现应用程序对行为预测的需求完全不同:一个对时间非常敏感，而另一个则更依赖于位置，”Chan解释道，他的背景包括在硅谷、香港和中国建立各种社交初创公司，所有这些都需要开发“智能”功能，这是TappingStone的部分灵感。

我请Chan提供一些可以利用TappingStone添加智能功能的应用类型的实例。

他举的一个例子是一款基于免费增值模式的手机游戏，该游戏可以使用TappingStone来预测哪些玩家更有可能成为付费用户。他们还可以通过向那些最终最有可能转化的客户提供个性化的应用内购买折扣来增加收入。

或者，一个旅游应用程序或酒店预订应用程序可以做一些类似于亚马逊的事情——“去过巴黎的人也喜欢慕尼黑”——以及推荐要做和要看的事情、用户可能喜欢的下一次旅行，甚至预测用户何时想去另一次旅行。

诸如此类。

就TappingStone计划如何赚钱而言，它将遵循与Twilio 类似的模式，并向开发者收取每次使用费用，按数据大小和每次预测呼叫定价。我确信，对于较小的初创公司来说，它会很便宜，甚至可能是免费的，但对于像Pinterest这样的东西来说，它的规模会很有竞争力。

“我们还没有决定确切的价格范围。但它会非常便宜，尤其是与内部数据科学家相比。
