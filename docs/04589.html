开放政府倡议:让技术人员解决政府问题TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2010/05/22/the-open-government-initiative-enabling-techies-to-solve-problems/

当奶奶在她时髦的iPad上翻阅相册时，政府机构(和大多数公司)在笨重的基于网络的前端处理关键任务交易，这些前端通过欺骗主机以为它们连接到CRT终端来工作。这些系统是用汇编和COBOL这样的计算机语言编写的，维护起来要花一大笔钱。我写过关于加州的遗留系统和浪费在维护这些系统上的数十亿美元。考虑到政府官员的短暂任期、根深蒂固的政府承包商的游说以及企业计算世界的缓慢变革步伐，我对大变革并不乐观——即使是在未来十年。但是在另一方面还有希望:开放政府倡议。这为企业家提供了他们自己解决问题所需的数据和API。他们不需要等待政府更新其遗留系统；他们可以简单地构建自己的应用程序。

联邦政府的开放数据计划data.gov于一年前由联邦首席信息官维韦克·孔德劳发布，包含47个政府信息数据集。这已经增长到超过250，000个数据集。已经构建了数百个应用程序来利用这些信息。一些州和地方也紧随其后，其中最引人注目的是旧金山市。

2009年6月，旧金山首席信息官克里斯·维因(Chris Vein)推出了一款应用，允许市民通过Twitter访问该市的311呼叫中心。除了打电话，公众还可以发推文提醒城市注意坑洞，或者了解城市的绿色倡议。这导致了 Open311 API的出现，它通过第三方应用程序提供对政府数据的访问。创业者已经用这个搭建了一些有用的app，比如 CitySourced 、 MyCityWay 、 SeeClickFix 和 TweetMy311 。在这一成功的鼓舞下，该市更进一步——开放所有非私人数据。该市的创新主管杰伊·纳特正在建设DataSF.org，他称之为“该市政府数据的一站式网站”。

这让一些著名的旧金山居民如craiglist创始人克雷格·纽马克非常兴奋。克雷格说，旧金山似乎非常重视提供更好的客户服务，同时通过让市民访问理应属于他们的数据来省钱。他称赞这座城市勇敢地面对“精英影响力兜售者”。上周，在吃早餐时，他告诉我，他愿意花自己的钱为那些创造出解决现实生活问题的最具创新性应用的企业家提供奖励:比如修补一个坑洞，询问现在是否是去车管所的好时机，或者也许是了解一个人的纳税情况。

Web 2.0的传奇出版商和大师Tim O'Reilly也是开放政府的忠实粉丝。奥莱利以其通过检测来自阿尔法极客的“微弱信号”来预测未来的能力而闻名，他说，几年前，他开始注意到开发者正在从笨重的政府网站上抓取政府数据(如当地犯罪统计数据)，并以对公民更有用的方式显示这些数据。他注意到“一场新兴的开放政府运动的信号”。因此，在2008年，他决定重点鼓励奥巴马政府利用这些优势。在一个名为Gov 2.0的新系列会议的支持下(其中之一是下周在华盛顿特区开始的 Gov 2.0博览会)，奥赖利开始与主要政府官员会面；起初倾听他们的挑战和观点；然后把机会还给他们。社交媒体当时在华盛顿像野火一样蔓延，最初与Gov 2.0这个术语紧密联系在一起。奥莱利传达的信息是，采用社交媒体不足以实现真正的改变；政府需要从科技行业吸取教训，开始搭建一个平台。

O'Reilly的目标是让美国所有地区都像旧金山一样使用Open311。但是挑战之一是跨地区标准化API，并创建标准。如果这一努力成功了，开发人员将能够编写通用的应用程序，将请求发送到公民当时所在司法辖区的正确部门。毕竟，你手机上的应用程序不应该只在一个城市的范围内运行。

还有其他的布道者，比如美国代码组织的詹妮弗·帕尔卡，她正致力于“帮助Web 2.0时代最聪明的头脑改造城市政府”。Pahlka说，城市面临着前所未有的压力，与预算削减和过时的技术作斗争。“如果城市可以利用网络的力量变得更加高效、透明和参与性，而不是削减服务或增加税收，那会怎么样？”她问道。“美国研究员项目代码”是以“为美国而教”项目(T2)为蓝本的。它与城市官员和领先的网络开发人才合作，确定并开发网络解决方案，然后分享并推广到全美各城市。

底线是，企业家有新的机会为他们的社区和国家做好事，并在这样做的同时积累财富。我曾感叹在硅谷，我们有世界上最伟大的头脑和最深的投资资金池专注于开发脸书和推特应用，而不是建立做好事的企业。这是一个仍然构建这些应用程序并做好事的机会。
