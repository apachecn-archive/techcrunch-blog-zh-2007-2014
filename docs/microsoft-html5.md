# 微软同意苹果和谷歌的观点:“网络的未来是 HTML5”

![](img/f36a1db7817955ad530a2a3ffe87aebe.png)

我们提到过 2010 年将是 HTML5 的[大年吗？苹果和谷歌正在大力推动这项技术，现在微软也是如此。当](https://web.archive.org/web/20221208223313/https://beta.techcrunch.com/2010/01/01/ten-technologies-2010/) [Internet Explorer 9](https://web.archive.org/web/20221208223313/http://ie.microsoft.com/testdrive/) 问世时，它将支持 HTML5，并有助于使其在网络上更加普遍。

“网络的未来是 HTML5，”微软 IE 总经理 Dean Hachamovitch 在一篇关于网络视频的博客文章[中写道。微软仍然支持 Flash，但是 HTML5 和 Flash 相持不下。通过支持 HTML5，微软给了网站设计者一个放弃 Flash 的理由。](https://web.archive.org/web/20221208223313/http://blogs.msdn.com/ie/archive/2010/04/29/html5-video.aspx)

这篇文章特别谈到了微软计划只支持 HTML5 视频的 H.264 编解码器。同样，Flash 播放器现在也支持 H.264。但 H.264 视频越多，对 Flash 播放器的需求就越少，因为这些视频可以直接在 HTML5 浏览器中播放，如 IE9、Safari 或 Chrome。

而且，正如苹果公司首席执行官史蒂夫·乔布斯昨天在他的“我们不需要没有臭味的东西”的演讲中所说，H.264 对移动设备更加友好:

> 为了在播放视频时实现长电池寿命，移动设备必须在硬件中解码视频；用软件解码太费电了。现代移动设备中使用的许多芯片都包含一种称为 H.264 的解码器，这是一种行业标准，用于所有蓝光 DVD 播放器，并被苹果、谷歌(Google)、Vimeo、网飞和其他许多公司采用。
> 
> 虽然 Flash 最近增加了对 H.264 的支持，但目前几乎所有 Flash 网站上的视频都需要一个老一代的解码器，该解码器不是在移动芯片中实现的，必须在软件中运行。区别是惊人的:例如，在 iPhone 上，H.264 视频可以播放长达 10 个小时，而在软件中解码的视频在电池完全耗尽之前只能播放不到 5 个小时。

哈查莫维奇更有外交手腕。他还指出，“闪存确实存在一些问题，尤其是在可靠性、安全性和性能方面。”尽管如此，他补充说，太多的消费者依赖 Flash，所以微软将继续与 Adobe 合作，使它变得更好。
