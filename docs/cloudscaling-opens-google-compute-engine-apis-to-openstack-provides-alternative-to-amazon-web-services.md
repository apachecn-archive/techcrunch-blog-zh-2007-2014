# Cloudscaling 将 Google 计算引擎 API 开放给 OpenStack，提供亚马逊网络服务的替代方案

[Cloudscaling](https://web.archive.org/web/20221207012749/http://www.cloudscaling.com/) 已经向 OpenStack 开放了[谷歌计算引擎](https://web.archive.org/web/20221207012749/https://cloud.google.com/products/compute-engine)(GCE)API，允许使用开放联合云的客户访问谷歌庞大的基础设施。

旧金山初创公司 Cloudscaling 是以亚马逊网络服务(AWS)风格构建开放分布式云的先驱之一。该公司将提供工程和代码开发，以将 GCE APIs 实现到 OpenStack 项目中，open stack 项目是两年前开始的开放云项目。它现在是世界上最大的开源项目，拥有代表 88 个国家 850 多个组织的 5，600 多名个人成员。OpenStack 基金会已经从会员和 21 家白金和黄金企业赞助商那里获得了 1000 万美元的资金。

Cloudscaling 创始人兼首席技术官 Randy Bias 表示，OpenStack 社区中的任何人都可以下载代码，并为他们的 OpenStack 版本添加补丁。然后他们将能够使用 GCE API。他说，代码是在测试的基础上提供的，已经过良好的测试，但在野外需要更多的时间来学习它需要如何进化。

Cloudscaling 将支持 GCE API 代码，要么作为 OpenStack 的一部分，要么作为一个单独的补丁。Bias 表示，他们将接受 OpenStack 社区提供的对代码的更改和更新。

这一举动是 OpenStack 和 GCE 的一个转折点——提供了一些证据，证明 OpenStack 可能会成为 AWS 的一个选项，AWS 是市场上无可争议的领导者。Bias 在一次电子邮件采访中对我说:

> 我们相信这在市场上创造了选择。重要的是，客户能够创建云之间的联合解决方案，这意味着支持主要公共云系统的 API，并在它们之间提供最大程度的兼容性。兼容性始于 API，止于架构和行为的保真度。
> 
> 这里的问题是关于可移植性，这是 OpenStack 吸引人的核心。谷歌是云计算领域的新生力量，但却拥有世界上最深厚的基础设施。它将成为 AWS 的重要竞争对手。它还表明，随着客户意识到 OpenStack、Google 和 AWS 都提供负担得起、易于访问的服务，可以运行轻量级应用程序或任务关键型遗留软件套件，这些软件套件是老牌企业巨头声称最好在自己的硬件上运行的，私有云潮流将不得不放缓。

联合云是这里的目标，但即使有谷歌在，在我们看到客户有能力在不同的云之间移动之前，仍有许多发展要走。
