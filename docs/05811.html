Windows Home Server已死，Windows Home Server万岁？TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2010/12/01/windows-home-server-is-dead-long-live-windows-home-server/

Windows Home Server无疑是微软有史以来最好的产品之一。它带来了成熟服务器的大部分功能，没有任何麻烦。几年前，惠普(HP)和宏碁(Acer)等原始设备制造商迅速加入该平台，并推出了以WHS为中心的出色产品线，几乎任何消费者都可以在一台比传统电脑能耗更低的小型电脑上设置全面的备份解决方案、媒体仓库和网络连接存储单元。对于微软、硬件制造商甚至美国消费者来说，这似乎是一款成功的产品。但这种看法显然是错误的。微软现在正在准备名为Vail的WHS的下一个版本，该版本不包括以前定义该平台的特性和功能。

当然，这些举动已经导致WHS服务器的粉丝群在一个由困惑和遗弃问题引发的互联网聊天室爆发。当微软证实了惠普将不会生产基于下一代操作系统的服务器的报道时，人们认为Windows Home Server现在已经死了。死了，取消了，或者其他一些同义无生命的名字可能不是正确的描述，因为微软显然正在开发这个平台，但下一个版本的WHS在其核心理念和目标人口方面似乎足够不同，足以证明名称的改变甚至死亡法令。

从2007年开始，我家里就有一台Windows Home服务器在运行。在WHS之前，我的服务器解决方案包括一个旧的Windows XP系统，其中有一大堆不同的网络共享，分布在各种各样的PATA硬盘上。我想我有七个硬盘，大小从80GB到250GB不等，装在一个完整的塔里。Windows Home Server的驱动器扩展器功能完美地创建了一个虚拟文件系统，允许网络共享跨越不同的物理驱动器。设置实际上只是两次点击添加更多的硬盘驱动器。当然，这只是一个JBOD系统，但是在我从2007年开始运行这些服务器的这些年里，我还没有丢失一点数据。即使有问题，Windows Home Server也允许多磁盘冗余和驱动器复制。

我发现驱动器扩展器在惠普、宏基和其他公司的现成型号中同样有用。想要更多储物空间吗？拉出一个滑动硬盘托盘，放入2TB硬盘，滑入，点击几个按钮，几分钟后，您的总存储容量将增加2TB。该解决方案完全依赖于软件，几乎可以在任何硬件上工作。

但是唉。这一关键特征已不复存在。微软几天前宣布，Vail将转而依赖硬件RAID解决方案，而不是像Drive Extender这样的内置软件解决方案。一方面，原生使用RAID可能会提高数据传输速度，老实说，在当前版本的Windows Home Server中，这一点从来都不出色。在驱动器之间或甚至在千兆位网络上传输通常会遇到facepalm类型的速度，但勒德分子简单的设置和管理仍然使WHS值得。

微软声称此举是为了提高网络负载效率，我对此表示欢迎。但Windows Home Server并不是删除以消费者为中心的功能的合适平台。如果有什么不同的话，Drive Extender尽管有缺陷，但仍然是购买该产品的最好理由。硬盘的价格不断下降，体积不断增大。用Windows Home Server制作一个大型的集中式家庭媒体保险库是一件无关紧要的事情。任何有能力打开装有新硬盘的纸箱的人都可以在大约7分钟内完成这项工作。

互联网目前猜测，微软正在将Windows Home Server转变为更像小型企业的解决方案。很公平，但是为什么不建立一个独立的建筑来摆脱DE而支持RAID而不完全杀死支援呢？这是大多数粉丝在过去一周想知道的，以及这个平台是否只是在变得无关紧要。

然后是今天关于惠普取消MediaSmart产品线的公告。这些是随该平台推出的型号，突然，就在传言Drive Extender不会出现在下一个版本中的几天后，惠普完全终止了该系列。这就像看着科比在菲尔杰克逊被解雇几天后以自由球员的身份离开湖人。(作为一名凯尔特人的球迷，这将是我的梦想，但这一点很清楚)

这使得宏碁和宁静PC成为两个值得注意的硬件合作伙伴，尽管惠普提供的whs比其余两家公司的总和还要多。突然，就像那样，市场上潜在的Windows Home Server的数量减少了一半，这似乎表明Windows Home Server正在快速走向…某个糟糕的地方。总有一个机会，互联网上的抗议将导致微软的权力以太重新插入驱动器扩展器回到平台或想出一个快乐的媒体。如果不是，Windows Home Server也可能会死，因为这将是它目前的大多数粉丝，甚至可能是硬件合作伙伴的看法。
