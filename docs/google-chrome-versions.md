# 谷歌 Chrome 新版本每六周发布一次

> 原文：<https://web.archive.org/web/https://techcrunch.com/2010/07/22/google-chrome-versions/>

# 谷歌浏览器的新版本每六周发布一次

凭借 Chrome 网络浏览器，谷歌一直痴迷于速度。现在他们正在加速它的另一个方面:发布稳定版本的频率。谷歌表示，现在的目标是每六周发布一个新的稳定版本——大约是目前速度的两倍。换句话说，为即将到来的 Chrome 6、7、8 和 9 做好准备。

那么谷歌为什么要这么做呢？因为他们创建新功能的速度非常快，他们希望确保所有用户都能尽快获得这些功能。这些快速的迭代也将让谷歌为 Chrome 制定更坚定的时间表。因此，项目经理将能够为在特定时间内可以完成的工作量设定现实的目标。

但谷歌强调的最重要的事情可能是减轻了他们工程团队的压力。在旧的、更长的发布模式下，工程师们会迫于压力，试图在截止日期前完成新功能，或者冒着被删减、几个月不出现的风险。有了新的发布时间表，即使某些东西在一个版本中还没有准备好，离下一个版本的发布也只有六周的时间了。

谷歌还试图淡化即将到来的版本快速迭代:

> 由于我们将继续在每个新版本中增加我们的主要版本(即 6.0、7.0、8.0、9.0)，这些数字将开始比以前移动得快一些。请不要对版本号变化的速度做过多解读——它们仅仅意味着我们正在通过发布周期，我们已经准备好将更新的版本发布到你的手中！

谷歌并不认为这六周的周期是“在理想条件下运行的”，所以它可能会不时地下滑。尽管如此，他们现在迭代的速度还是很令人兴奋的。即使版本号没有多大意义，其他浏览器也必须确保他们的用户知道这一点，否则它可能看起来像是 Chrome 在绕着他们跑。
