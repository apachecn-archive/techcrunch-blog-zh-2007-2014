# Q Factor 筹集了 650 万美元的 A 轮融资，通过消除等待来解决移动设备上的富媒体交付问题

美国初创公司 Q Factor 的目标是解决无线设备上的富媒体内容交付问题，让人们更少——甚至没有——等待观看视频或访问大文件，该公司已经完成了来自[适马 Prime Ventures](https://web.archive.org/web/20230326060907/http://www.crunchbase.com/financial-organization/sigma-prime-ventures) 和 [Venrock](https://web.archive.org/web/20230326060907/http://www.crunchbase.com/financial-organization/venrock) 的 650 万美元 A 轮融资。

这家总部位于马萨诸塞州沃尔瑟姆的初创公司成立于 2012 年初，但一直保持低调。它今天拉开帷幕，宣布其 A 系列，并详细介绍其技术。

来自适马的投资者约翰·西蒙和 Venrock 的迈克·泰勒加入了 Q Factor 的董事会，加入种子投资者泽纳斯·哈奇森和 Q Factor 创始人兼首席执行官苏巴什·罗伊的行列。

那么 Q 因子构建了什么？一个纯软件系统进行无线分组恢复——它将这种技术称为 DRP(又名:动态分组恢复)——并声称它“显著”改善了无线设备上“富媒体内容”的传输和观看体验。

系统通过定位丢失的数据包(因为它知道正在发送什么)并替换它们以保持内容的到来来做到这一点。罗伊说，这样做的目的是“消除”你在移动设备上播放视频、下载文件或访问大文件时的等待。

以下是 Q Factor 在其网站上详细介绍该软件的方式:

> DPR 做三件重要的事情:
> 
> *   恢复丢失的数据包
> *   保持高传输速率
> *   最大限度减少网络拥塞
> 
> DPR 可以动态查找所有丢失的数据包，无需重新传输，确保用户交互所需的所有数据都在需要时存在，并帮助提高网络吞吐量。当您的应用程序已经被*“q factored”时，您就不用再等着看那个可怕的纺车了。*

DRP 是通过 SDK 部署的——因此 Q Factor 需要吸引拥有大量媒体应用的开发者注册。它认为其他可能受益的应用包括媒体播放器、虚拟专用网、云备份和多人游戏。该公司的软件要到 2014 年才能上市。

“通过无线方式访问网络内容正日益成为标准，但如今的用户体验非常不稳定。Q Factor 的技术提供了一个强大的解决方案，可以大幅提高性能，”适马的普通合伙人西蒙在一份声明中对这轮融资进行了评论。

“Subhash 和他的创始团队对这个市场有着无与伦比的经验和知识，”Venrock 的 Tyrrell 在另一份声明中补充道。“这种人才和他们的颠覆性技术的结合将被证明是通过无线传输内容的游戏规则改变者。”
