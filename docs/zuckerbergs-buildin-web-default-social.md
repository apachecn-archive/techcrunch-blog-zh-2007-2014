# 扎克伯格:“我们正在建立一个默认为社交的网络”

![](img/f1334c8c3d7192b8dddbfbf6066cecc1.png)

今天在脸书的 [F8 会议](https://web.archive.org/web/20230325060517/https://techcrunch.com/2010/04/21/hacking-the-graph-live-from-facebooks-f8-conference/)上，马克·扎克伯格展示了他将网络变成“即时社交体验”的计划这个超级社交网络的构建模块是脸书新的 Open Graph 和社交插件，其中包括 Facebook.com 以外网站上随处可见的新按钮、[无需点击脸书连接即可自动登录的功能](https://web.archive.org/web/20230325060517/https://techcrunch.com/2010/03/26/facebooks-plan-to-automatically-share-your-data-with-sites-you-never-signed-up-for/)，甚至还有一个[脸书社交吧](https://web.archive.org/web/20230325060517/https://techcrunch.com/2010/03/27/facebook-meebo-bar/)，其中包括几个插件和脸书聊天(再见，Meebo)。

我们以前报道过所有这些新功能，但今天扎克伯格把它们放在上下文中:“我们正在建立一个默认为社交的网络。”脸书是如何做到这一点的？首先，脸书为开发人员重新设计了图形 API，这样他们不仅可以看到人们之间的社交联系，还可以看到和创建人们与他们的兴趣之间的[联系——事物、地点、品牌和其他网站。扎克伯格称之为开放图(相对于社交图)。这真是一个有趣的图表。](https://web.archive.org/web/20230325060517/https://techcrunch.com/2010/04/19/facebook-twitter-interests/)

脸书希望开发者围绕兴趣和事物创建开放图的子集。Yelp 可能围绕餐馆创建一个，Pandora 可能围绕音乐创建一个，网飞围绕电影创建一个。添加一些“喜欢”按钮，任何时候有人在网上任何地方用一个脸书喜欢按钮喜欢一家餐馆、一首歌或一部电影，这些信息就会流回到开放的图表中。这样 Yelp 就会知道你和你的朋友在其他地方喜欢什么餐馆，并在给你推荐时考虑到这一点，或者 Pandora with music，等等。(Yelp 和 Pandora 是真实的例子，网飞不是)。

脸书正在从人们的实时信息流中获取一些信息，并将其放到网络上。扎克伯格说:“这条小溪是短暂的。“它会在那里呆几个小时，然后大部分时间都会漂走。服务不理解你和那家餐厅之间的语义联系。”但是现在脸书可以了。脸书希望网络不再仅仅由超链接来定义(这有利于像谷歌这样的搜索引擎)，而是由社会联系、喜欢和不喜欢、兴趣来定义，这些都是编码的和机器可读的。“我们的目标是使用开放的图表，这样人们无论走到哪里都可以获得即时的社交体验，”他说。

将此开放图表添加到插件中，如网站可以放在网站上的活动流和推荐流小部件。为了进一步消除与这个开放图形交互的摩擦，甚至不需要登录。即时社会化和即时个性化是网站的目标。想象一下，来到 TechCrunch，看到你的脸书朋友“喜欢”的每一个故事，或者去 Amazon.com，看到产品的同样情况。(再说一遍，这些都是暂时假设的例子，但为什么不呢？).这个开放的图表雄心勃勃。只要等到脸书插入目标广告。
