# Spotify 不再接受新的应用提交，计划扩展 Web API 并专注于移动 SDK

> 原文：<https://web.archive.org/web/https://techcrunch.com/2014/03/24/spotify-shutters-app-submissions/>

# Spotify 缩减应用目录，专注于外部生态系统

音乐流媒体服务 Spotify】宣布不再接受 App Finder 的新应用提交，App Finder 是其自己的应用目录，用于在其桌面客户端内驻留和操作*的第三方应用。这一举动实质上意味着它关闭了我们所知的 Spotify 应用平台。然而，与 API 1.x 兼容的现有应用将继续在 Spotify 桌面应用中可用——因为它将重点转移到第三方 Spotify 集成的移动 SDK，并计划扩展其当前相当有限的 Web API。*

后者虽然没有细节，但可能意味着位于 Spotify 之外*的 Spotify 支持的网络应用程序将最终获得一个公平的竞争环境，与那些以前在桌面客户端内运行的被批准的应用程序竞争。对于希望整合 Spotify 的应用程序来说，这是一个更加“开放”的网络的胜利吗？我们将不得不看看结果如何，但这一举动肯定很有趣。*

我的理解是，Spotify 曾打算将第三方 Spotify 应用集成到他们的网络播放器和移动应用中，但由于涉及的复杂性，他们在一年多前取消了这一集成。

尽管主要品牌和初创公司都在开发官方的 Spotify 应用程序，列在其应用程序查找器中，包括 Last.fm、musiXmatch、Rolling Stone、Soundrop 和 Tunewiki 等，但其平台播放整体上有多成功也是一个问号。Spotify 应用程序可以在音乐流媒体服务自己的客户端中访问，这似乎是一个伟大的想法，但如果有足够多的人知道并使用它们，这是值得怀疑的。

我去年采访过的一家初创公司承认，其 Spotify 应用最初给了注册用户一个不错的提升，但在平台周围的公关活动减弱、应用目录变得更满之后，注册用户很快就减少了。

但也许真正的故事不是关于一个失败的平台，而是向移动的转移，让人想起在线游戏。Spotify 在其官方开发者更新中对此事说得差不多了，指出其移动 SDK 仍然是该公司的主要关注点。

Spotify 写道:“合作伙伴一直在要求我们提供一种更便宜、更简单的方式，在 Spotify 中拥有一席之地，而开发者希望有更多方式将 Spotify 与他们自己的应用程序集成在一起。”。“再加上移动设备日益增长的重要性，我们意识到我们需要适应。”

它已经适应了。

“未来几个月，我们将为 Spotify 的合作伙伴增加新功能，扩展我们的 Web API 功能，并发布新的移动 SDK，”该公司补充道。
