# 实时分析:Salesforce 的贝尼奥夫谈论社会企业战略、Chatter 等

> 原文：<https://web.archive.org/web/https://techcrunch.com/2009/11/20/realtime-crunchup-salesforces-benioff-talks-social-enterprise-strategy-chatter-and-more/>

![](img/c0bc3c0529daa2838b7769976a19b764.png)

Salesforce.com 联合创始人兼首席执行官马克·贝尼奥夫(Marc Benioff)正与 TechCrunch IT 编辑史蒂夫·吉尔默(Steve Gillmor)和 TechCrunch 联合编辑埃里克·舍恩菲尔德(Erick Schonfeld)坐下来讨论企业的社会化。贝尼奥夫最近为 Salesforce 发布了自己的社交策略:Chatter。Chatter 在该公司的 Dreamforce 活动上首次亮相，它允许任何公司通过安全的私人社交网络为其业务进行实时协作。内容、应用程序和人员现在将在平台内拥有个人资料、订阅源和群组，使他们能够通过统一的流连接起来。此外，开发人员现在将能够利用 Chatter 的 API 来构建平台外的社交企业应用程序。虽然 Chatter 看起来和感觉上都像是企业的社交网络，但 Benioff 很快就去掉了这个绰号，更愿意称这个平台为协作工具。

*下面找我的笔记(转述):*

爱德华斯诺登:我们和 Salesforce.com 的首席执行官马克·贝尼奥夫在一起，他在过去的几天里宣布了一些事情。

MB:我们有 19，000 人参加我们的梦想力量大会。我们成功发布了一款名为 Chatter 的新产品。它把史蒂夫·吉尔默称之为实时的东西与企业结合起来。它带来了真正的企业计算能力。我请来了 Salesforce 的 Chatter 产品主管斯蒂夫·费舍，我想对 Chatter 做一个概述。

史蒂夫和我一起创办了一家软件公司，他在斯坦福大学教授计算机科学，在苹果公司工作。

Chatter 背后的核心思想是为企业带来实时性，这是企业迫切需要的。这为流中的信息提供了上下文。它不仅提供了一个社交应用，也为人们提供了一个创新平台。Chatter 获取一个数据库对象，并将其转换为人员和简档。

MB: Chatter 是 Force.com 平台的社交延伸。

MB:我在脸书上有 5000 个朋友。但为什么我对这些陌生人的了解比我自己的员工还多。对于我不了解的人，我比我最重要的员工更聪明。

ES:企业社交网络一直致力于为用户获取实时社交数据。

MB:我们还没有深入其中。我们正在寻求反馈。我们有一个总体战略。我们认为这是一个独特的策略。除了微软 Sharepoint，我们还需要一些新的东西。这是我们的第四朵云。

ES:你认为这个应用会有更多的用户互动吗？

MB:我们正在向消费者世界寻求已经普及的东西，如 feeds、个人资料，然后加入计算机技术，为企业创造这种东西。我们试图在我们的平台上实现实时。我们对 SaaS 充满热情，但合作是下一件事。为什么这只能发生在消费者身上。为什么 a 不能跟上我的 PowerPoint 演示？
