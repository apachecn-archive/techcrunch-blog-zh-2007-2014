# StumbleUpon 将推出 su.pr ShortUrl 服务 TechCrunch

> 原文：<https://web.archive.org/web/https://techcrunch.com/2009/03/12/stumbleupon-to-launch-supr-shorturl-service/amp/>

 <amp-img src="https://web.archive.org/web/20230207093751im_/https://techcrunch.com/wp-content/uploads/2009/03/supr.jpg" class="border amp-wp-enforced-sizes i-amphtml-layout-intrinsic i-amphtml-layout-size-defined" alt="" layout="intrinsic" i-amphtml-layout="intrinsic"><i-amphtml-sizer class="i-amphtml-sizer"></i-amphtml-sizer></amp-img> 

StumbleUpon 正准备在未来几周内推出一项名为 [su.pr](https://web.archive.org/web/20230207093751/http://su.pr/) 的 shortURL 服务(一项为长 URL 重定向提供短别名的 web 服务，如 [TinyURL](https://web.archive.org/web/20230207093751/http://tinyurl.com/) 或 [Bit.ly](https://web.archive.org/web/20230207093751/http://bit.ly/) )。

创始人[加勒特·坎普](https://web.archive.org/web/20230207093751/http://www.crunchbase.com/person/garrett-camp) [在推特上宣布了](https://web.archive.org/web/20230207093751/http://twitter.com/gmc/status/1273057527)这项新服务，但没有任何关于 3 月 3 日可能会有什么的描述。在一次电子邮件交流中，他说这将是一个在 Twitter 和脸书等网站上分享 StumbleUpon 链接的短 URL 服务，并将在一两周内推出。该网站目前重定向到 StumbleUpon。

Digg 正在计划它自己的类似服务，我们上个月末报道过。

Digg 的服务将在一个 Digg“工具栏”包装器中显示附加信息，如链接的总流量，用户只需在任何 URL 前添加 digg.com/就可以轻松创建链接(短 URL 将被创建并显示)。我不知道 StumbleUpon 到底在计划什么，但可以肯定的是它会有类似的功能，而且可能会在 Digg 之前发布。

StumbleUpon 和 Digg 对这些服务的好处是当人们点击短 URL 时的流量。人们在 Twitter 帖子上添加了很多链接，但空间限制(每篇帖子 140 个字符)要求短 URL。
