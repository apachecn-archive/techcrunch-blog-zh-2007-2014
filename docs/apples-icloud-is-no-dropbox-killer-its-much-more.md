# 苹果的 iCloud 不是 Dropbox 黑仔(它远不止如此)

> 原文：<https://web.archive.org/web/https://techcrunch.com/2012/02/16/apples-icloud-is-no-dropbox-killer-its-much-more/>

随着今天 OS X 下一个版本的发布，也就是 OS X 10.8，也就是山狮，苹果更加深入地将其 iCloud 服务整合到操作系统中。将文档存储在云中不再是一种额外的增值功能，而是操作系统本身的一部分。苹果似乎在说，云只是另一个驱动器，保存到云应该看起来和感觉上与保存到你的文档文件夹或桌面没有什么不同。

当然，这个想法并不新奇。这就是像 [Dropbox](https://web.archive.org/web/20230127141218/http://www.dropbox.com/) 这样的初创公司今天正在做的事情:制作一个看起来与其他任何驱动器一样，但可以从任何机器访问的驱动器。虽然从表面上看，很容易将 iCloud 称为“苹果版的 Dropbox”，但事实实际上更复杂:它是关于建立一种新的计算范式。

在 Mountain Lion 中测试新的 iCloud 集成时，一个文件可以在多个位置打开——比如你的 Mac、iPad 和 iPhone——当发生变化时，它几乎会立即实时出现在所有三个设备上。您不必等待通知或重新加载文件。它就这么出现了。虽然人们的第一反应是 iCloud 正在迅速变成苹果自己的 Dropbox 的改良版，但它也是谷歌文档(Google Docs)和传闻已久的谷歌硬盘(Google Drive)的激烈竞争对手。

然而，谷歌的理念是文件创建本身可以迁移到云中。一个在线办公套件“足够好”，如果没有本地套件好的话。“足够好”将因易用性而胜出。在几乎完全相反的观点下，苹果的 iCloud 正在做相反的事情:将云的能力带给更丰富、更强大的原生应用。这不仅包括 iWork 中的 office 应用程序，而且通过使用开发者 API，它将扩展到任何需要启用 iCloud 的应用程序。虽然现在，iCloud 对第三方的支持更加有限，但 API 会及时改进。最终，在苹果平台上运行的任何应用程序(桌面或移动)都将拥有在不同安装之间移动数据的工具。

为了实现向云的无缝过渡，苹果将云深深嵌入到新版 OS X 中，一直到“打开”和“保存”对话框。Mac Store 应用将能够立即保存到本地文件系统或 iCloud。iCloud 也被嵌入到 Finder 中，显示按应用程序排序的实时文件列表。管理这些文件有一种类似 iOS 的天赋:例如，你可以把它们一个一个拖放到另一个上面来创建一个文件夹。甚至这里的背景看起来像 iOS 跳板。

但苹果的 iCloud 不仅仅是为了建立一个更好的 Dropbox，而是为了保持一切同步:邮件、联系人、日历、提醒、书签、笔记、照片、账户等等。

目前，终端用户将 iCloud 视为这个互联网位置，这表现为一个新的选择:“存储到 iCloud？”好像 iCloud 仅仅是一个在线存储箱！但这几乎感觉像是我们习惯的世界(物理硬盘)和云未来之间的过渡步骤。有趣的是，这个中间步骤在某种程度上歪曲了云的简单性。云实际上并不是“天上的硬盘”(你好，[微软](https://web.archive.org/web/20230127141218/http://live.skydrive.com/))。无论我们身在何处或使用何种设备，这种结构都允许我们维护单一的计算环境。
