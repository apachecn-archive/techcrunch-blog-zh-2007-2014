# Windows Phone 应用可以在 Windows 8 上运行吗？

> 原文：<https://web.archive.org/web/http://techcrunch.com/2011/09/05/windows-phone-apps-windows-8/>

当你看到安卓和苹果在美国合计 70%的智能手机市场份额时，相比之下，微软的份额只有区区 6%，而且还在不断缩小，这似乎表明，在 Windows Phone 真正开始之前，游戏就已经结束了。Windows Phone 是一个不错的移动操作系统，甚至是[承诺的](https://web.archive.org/web/20230205033313/https://techcrunch.com/2010/10/20/review-windows-phone-7/)，但迄今为止，它未能抓住开发者或消费者的心。微软能做些什么来改变这种情况吗？它会将 Windows Phone 与下一代桌面操作系统 Windows 8 更紧密地捆绑在一起吗？

有一些线索表明，这正是微软计划要做的。当我们第一次看到 Windows 8 的[预览视频](https://web.archive.org/web/20230205033313/https://techcrunch.com/2011/06/02/windows-8-gorgeous-shell-video/)及其[触摸和基于磁贴的界面](https://web.archive.org/web/20230205033313/https://techcrunch.com/2011/06/01/windows-8-brings-entirely-new-look-and-feel-unifies-touch-and-pc-interfaces/)时，人们立即想到了 Windows Phone，它有着非常相似的界面。两者都没有使用应用程序图标，而是使用磁贴来显示底层应用程序的数据和图像，而无需打开它们。瓷砖本身成为一个仪表板，显示每个应用程序背后的实时数据。

[![](img/2f0c19797cbc0e2db9591f0d74e3f84e.png "windows-phone-7")](https://web.archive.org/web/20230205033313/https://techcrunch.com/wp-content/uploads/2011/09/windows-phone-7.jpg)

Windows Phone 和 Windows 8 是两个独立的操作系统。但是，如果微软让 Windows Phone 应用程序在 Windows 8 电脑上运行变得非常容易，会怎么样呢？目前，Android 或苹果 iOS 设备上的大多数移动应用程序都生活在自己与桌面分离的世界中(将应用程序从 iOS 移植到 OS X 是可能的，但似乎不是一种非常受欢迎的做法)。如今，连接到桌面的链接通常是通过网络完成的。如果 Windows mobile 应用程序在桌面上有同步的副本，并以 Windows 8 应用程序磁贴的形式出现，这可能会给 Windows phone 带来竞争对手目前缺乏的优势。这也符合微软经典的拥抱和扩展战略，即利用其在桌面领域的主导地位扩展到其他领域。这种策略在后 PC 时代可能不再奏效，但仍值得一试。

我们可能会看到所有这些都与 Windows Phone 8 和微软即将推出的 Windows 8 开发框架“Jupiter”结合在一起。就像阿波罗神是木星的太阳一样，Windows Phone 8(代号阿波罗)也是与 Windows 8 相关的。正如我们的莎拉·佩雷斯[所写的木星](https://web.archive.org/web/20230205033313/https://techcrunch.com/2011/08/20/is-jupiter-the-future-of-windows-pc-phone-tablet-tv/):

> 木星可能最终成为统治它们的“一个框架”。这意味着，只需重用现有代码并做一些小的调整，就有可能将已经用 Silverlight 编写的数千个 Windows Phone 应用程序移植到 Windows 8。或者甚至没有调整。(那部分还不清楚)。如果是这样，这将是为 Windows Phone 8(顺便说一下，代号为“阿波罗”，是“木星”的儿子)或 Windows 8 构建的开发者的技术优势。

正如我上面提到的，即使这种策略成功地创造了大量跨平台的移动 PC 应用，也不清楚这是否足以让 Windows Phone 有所不同。但是它明确指出了移动和桌面世界在不久的将来会融合，而不仅仅是在 Windows 上。你所有的移动应用程序也应该以某种形式出现在你的桌面上。当然，不是完全相同的应用程序，因为移动应用程序是为触摸界面、定位和利用手机硬件(如摄像头和加速度计)而构建的。相比之下，桌面应用程序仍然需要为鼠标和键盘而设计。但是底层数据可以支持两个平台上的本地应用。

在过去的十年里，桌面应用已经让位于网络。如果它们的受欢迎程度因移动应用程序而卷土重来，岂不是很讽刺？
