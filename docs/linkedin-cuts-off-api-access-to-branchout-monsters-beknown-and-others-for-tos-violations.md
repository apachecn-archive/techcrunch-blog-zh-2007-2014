# LinkedIn 因违反 TOS 切断了对 BranchOut、Monster's BeKnown 和其他 API 的访问 

> 原文：<https://web.archive.org/web/https://techcrunch.com/2011/07/01/linkedin-cuts-off-api-access-to-branchout-monsters-beknown-and-others-for-tos-violations/>

![](img/c0ed45769d4a95a13111454403294b17.png) **【独家:**职业社交网络 LinkedIn 称，因违反服务条款，该公司已关闭了许多开发者的 API 访问。接入 LinkedIn API 的六个开发者包括专注于脸书的职业网络 BranchOut、Monster 的社交招聘应用 [Beknown](https://web.archive.org/web/20230203072756/http://apps.facebook.com/beknown/welcome) 、品牌管理应用 [Visible.me](https://web.archive.org/web/20230203072756/http://visible.me/) 、简历服务 [Daxtra](https://web.archive.org/web/20230203072756/http://www.daxtra.com/) 、职业声誉管理器 [Mixtent](https://web.archive.org/web/20230203072756/http://www.mixtent.com/) 和 [CRM-Gadget](https://web.archive.org/web/20230203072756/https://crm-gadget.appspot.com/) 。

BranchOut 和 Monster 的类似(和[最近推出的](https://web.archive.org/web/20230203072756/http://www.insidefacebook.com/2011/06/27/monster-beknown/))应用 BeKnown 的访问关闭尤其令人惊讶。根据 LinkedIn 的说法，BranchOut 被比作脸书的 LinkedIn，它违反了该网络的 API TOS，因为它的[计划](https://web.archive.org/web/20230203072756/https://techcrunch.com/2011/05/11/exclusive-branchout-raises-18m-for-facebook-focused-professional-network/)是一个高级企业招聘搜索工具。该网络表示，对 LinkedIn 内容的访问收费是不允许的。

LinkedIn 表示，它切断了对 BeKnown API 的访问，因为该应用程序使用 LinkedIn APIs 发送消息来推广 BeKnown(从而从 API 中获利)。LinkedIn 还担心 BeKnown 将对与 API 相关的企业服务收费，类似于 BranchOut。Mixtent 和 Visible.me 也因为同样的原因被关闭。CRM-Gadget 和 Daxtra 都因存储 LinkedIn 会员数据而被关闭。

在 BranchOut 和 BeKnown 的案例中，很难不想到整个 Twitter-UberMedia 的崩溃，其中 Twitter 因违反 TOS 而关闭了对 UberMedia [的 API 访问，包括商标、隐私和货币化违规。UberMedia 是 Twitter 的直接竞争对手，拥有大量第三方客户。](https://web.archive.org/web/20230203072756/https://techcrunch.com/2011/02/18/twitter-suspends-ubermedia-clients-ubertwitter-and-twidroyd-for-violating-policies/)

同样，BranchOut(现在是已知的)在某些方面是 LinkedIn 的竞争对手。由 Accel、Norwest、Floodgate 和 Redpoint 支持的 BranchOut 可以让你通过脸书的朋友建立关系网并找到工作。该公司还允许你从 LinkedIn 导入技能、教育和工作经历。该公司允许品牌和组织向用户发布职位。这家初创公司一直在 LinkedIn 尚未涉足的领域——脸书——成长。

LinkedIn 有 20，000 名开发者使用它的 API，除了一些失误，它与开发者的关系一直相当好。今年 1 月，LinkedIn [关闭了对 CubeDuel 的访问](https://web.archive.org/web/20230203072756/https://techcrunch.com/2011/01/14/cubeduel-goes-viral-too-quickly-stumbles-over-linkedin-api-limits/)，这是一项将热门与否的最佳(或最差)与专业社交网络相结合的服务。显然，CubeDuel 超出了 LinkedIn 的 API 限制，但这实际上是这家初创公司[的错。](https://web.archive.org/web/20230203072756/https://techcrunch.com/2011/01/14/cubeduel-goes-viral-too-quickly-stumbles-over-linkedin-api-limits/)

LinkedIn 表示，如果这些开发者和初创公司遵守网络的服务条款，它愿意恢复其 API。LinkedIn 与一些开发者有合作协议，初创公司为 API 付费(他们可以从中获利)。

但 LinkedIn 或许可以从 Twitter 与其开发者之间的微妙关系中学到一些东西，并且肯定应该非常小心地在这些水域中航行。

**更新:**

BranchOut 针对 LinkedIn 的举动发表了以下声明:

> 在 BranchOut，我们认为专业网络的下一代平台是脸书。LinkedIn API 的变化对 BranchOut 体验几乎没有影响，因为只有一小部分用户在使用它。也就是说，我们认为用户数据应该归用户所有，应该允许人们与提供最大效用的新服务和上下文共享他们的数据。
> 
> 我们分析了我们的统计数据，这给我们带来了一个非常令人兴奋的发现——即我们正在一个比上一代职业服务所解决的更大的受众群体中引起一场浪潮。BranchOut 用户不仅包括专业社交网络用户，还包括全球 7 亿脸书用户，他们希望利用自己的社交网络在商业、招聘、销售和求职方面为自己提供帮助。例如，除了白领专业人士之外，我们的用户还包括大学生、零售业、制造业、酒店业、军队、政府以及其他尚未在社交网络中找到专业声音的人。我们很高兴成为第一家为全球观众提供相关专业网络解决方案的公司。

这是怪物的回应:

> 我们对 LinkedIn 的决定感到惊讶和失望，我们认为这不仅违背了 LinkedIn 用户的利益，也违背了 LinkedIn 声称代表的东西——开放和连接。职业社交网络本质上是社会性的，LinkedIn 只是限制了他们随时随地联系的能力。他们剥夺了用户控制如何以及何时分享个人资料和个人联系人的权利。我们还注意到，就在 Monster 发布 BeKnown 的几天后，LinkedIn 决定屏蔽该 API，而几个月前就有其他面向网络的应用程序在使用该 API。虽然 LinkedIn 的这一举措给用户带来了不便，但 BeKnown 的成员将继续从所有最大的在线网站建立自己的网络，包括脸书、雅虎、谷歌和 Monster。
