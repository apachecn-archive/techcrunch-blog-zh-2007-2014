# FanBridge 的新脸书品牌页面 Omni-App 结合了来自 12 个合作伙伴的功能

脸书页面可以托管任意多的应用程序，但不幸的是，大多数访问者从未离开过单一的默认登录选项卡。为了让其品牌和乐队客户在默认标签下提供电子商务、众筹、音乐、视频和票务，[营销平台 FanBridge](https://web.archive.org/web/20230204161426/http://www.fanbridge.com/) 今天与 12 家开发商建立了合作关系。客户可以同时将 Topspin、IndieGoGo、SoundCloud、Stickam 和 Songkick 以及其他合作伙伴的功能集成到免费的 FanBridge Fan Page Creator 应用程序中。通过创建一个[合作伙伴生态系统](https://web.archive.org/web/20230204161426/http://www.damntheradio.com/)，FanBridge 可以成为聚合其他应用程序的中心，而不必自己构建它们。omni-app 的灵活性和知名品牌合作伙伴可以帮助 FanBridge 从脸书页面管理巨头那里吸引客户，如 Buddy Media 和 Vitrue，这些公司的品牌客户通常会在他们的页面上显示多个独立的应用。

FanBridge Fan Page Creator 的核心是减少脸书页面标签应用系统中固有的摩擦。页面访问者登陆一个应用程序，但必须使用屏幕一侧的小导航菜单来访问其他应用程序。他们很少遇到麻烦，因此品牌通常要么用一个不能说明全部情况的应用程序来定义自己的身份，要么麻烦地频繁更换默认应用程序。 [Fanzila](https://web.archive.org/web/20230204161426/https://techcrunch.com/2011/09/15/fanzila-upgrades-facebook-pages-with-apps-for-blogs-forums-contest-and-more/) 试图通过创建一个多功能应用程序来解决这个问题，但它自己构建所有的模块，而不是选择通常要求的功能的最佳提供商。

FanBridge 为用户提供了更简单的跨应用程序体验，这可以转化为更多的参与、转化、捐赠和更多有利于品牌的行动。FanBridge 的产品副总裁 AJ Magnuson 解释说，其合作伙伴都开发了高质量的独立应用程序，但“有些人很难获得牵引力，或者只想专注于核心功能”，而不是建立其他功能或完整的内容管理系统。与 FanBridge 的集成使他们无需太多额外的开发工作就能增加发行版。

FanBridge 的客户可以添加他们的内容，并确定通过 CMS 仪表板显示哪些合作伙伴功能。很快，FanBridge 将从其授权的页面中收集关于合作伙伴功能使用情况的数据，以向客户推荐哪些功能最适合他们。它希望通过免费应用吸引客户，然后向他们追加销售其社交营销套件和其他服务。

对于用户和页面管理员来说，脸书在应用发现方面做得非常糟糕。通过其生态系统，FanBridge 可以充当策展人，让品牌只访问最高质量的应用程序，并让他们不必选择访客是首先看到他们的电子商务店面、视频频道还是另一个应用程序。
