# 出色的新 API 初创公司动态生成 1，000 个视频

[![](img/5e6219227e44d40d6f616df8f4eb93c5.png) ](https://web.archive.org/web/20221007224605/http://www.crunchbase.com/company/stupeflix) [Stupeflix](https://web.archive.org/web/20221007224605/http://stupeflix.com/) 是一家法国初创公司，他们提出了一种全新的创建、处理和编辑在线视频的方式。从表面上看，Stupeflix 自动从图片、音乐和视频中生成专业的视频。如果这听起来像是 [Animoto](https://web.archive.org/web/20221007224605/http://animoto.com/) ，那么你是对的，但是在引擎盖下有一些关键的不同，这使得 Stupeflix 完全不同，并且潜在地具有更大的价值。我不会轻易这么说。

Animoto 和 Stupeflix 完全不同的地方在于他们的方法和商业模式。Stupeflix 有效地提出了一个 API，它基于 XML 描述描述 flash 视频中的视频、文本、使用和图片。所以你编辑的不是视频，而是 XML。这意味着你可以通过改变标签来编辑视频，或者告诉他们的引擎为你想要生成的每个视频运行不同的效果。iMovie 将只创建一个视频，并需要一个丰富的包来编辑它的呈现方式。使用 Stupeflix，你只需编辑 XML，加上“rotate”或“fade left”这样的标签。今天，Stupeflix 为其视频编辑网络应用程序推出了网络界面，作为演示，他们在不到 60 分钟的时间内，直接从维基百科内容中自动生成了 1000 多个视频。
