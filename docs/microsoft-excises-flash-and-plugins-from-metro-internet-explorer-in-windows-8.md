# 微软在 Windows 8 中删除 Metro Internet Explorer 的 Flash 和插件 TechCrunch

> 原文：<https://web.archive.org/web/http://techcrunch.com/2011/09/15/microsoft-excises-flash-and-plugins-from-metro-internet-explorer-in-windows-8/>

微软已经确认了其对 Metro 中独立和单独优化的生态系统的承诺，通过一直有趣的[构建 Windows 8 博客](https://web.archive.org/web/20230203053417/https://blogs.msdn.com/b/b8/archive/2011/09/14/metro-style-browsing-and-plug-in-free-html5.aspx)宣布 Metro 中的 IE10 将不支持插件。当然，这也包括 Flash，它给这份声明带来了同样的味道，如果不是同样的物质的话，就像苹果著名的拒绝那项特殊技术一样。这要温和得多，尽管它仍然意义重大。

微软一直是插件的粉丝，但是 IE 的最新版本更加面向标准。IE6 只是一个网络插件的容器，但是 IE9 更加集成和精简。IE10，至少在 Metro 中，将把它带到下一个合乎逻辑的步骤，那就是完全消除插件。

事实上，这是一个务实的决定。Windows 8 的平板电脑形式将专注于基本的网络消费、定制的全屏应用以及平板电脑般的长寿命和易用性。对于过去可能需要 Flash 或其他插件的许多用例(视频和音频占大多数)，新的标准和容器正在使本机实现更快、更简单、更安全。由于桌面形式的 8 将随时可用，所以切换到功能更全的 IE10 很容易。只需在需要插件的页面上点击一个按钮，你就会进入全桌面模式。

这不是最优雅的解决方案，但这是*的一个解决方案*，而苹果只提供了冰冷的安慰。一些互联网是 iOS 用户无法访问的，简单明了，Windows 8 平板电脑不会出现这种情况。这对许多人来说是一种优势，尽管可能也有同样多的人接受了苹果的前提，即如果它在 iPad 上不工作，它可能就不值得。

不过，在 Metro 版本的应用程序中减少这一功能让我有点担心，因为这似乎使它不太像一个“真正的”应用程序，进而使 Metro 模式下的 Windows 8 平板电脑不太像一台“真正的”PC。这是一个单一的设计决策，还是众多应用中地铁/平板电脑应用稍逊一筹的一个？

仔细想想，这并不像有些人想的那样是一种妥协。简化应用程序的决定有实际的好处。iPad 上的 Safari 是最简单的，它牺牲了大量的功能。为什么要对微软有双重标准？在这里，他们提供了一个“哑”版本，可以容忍或切换到一个完整的版本。它们提供了苹果没有的选择。

此外，几乎可以肯定的是，插件将得到像 Firefox 这样的第三方浏览器的支持。微软只是在选择它想在自己的平板电脑上定义的市场。成为瑞士军刀是一个太大的承诺，尤其是在 Windows 上，它本身已经是一把大得不切实际的瑞士军刀。

然而，我仍然觉得微软可能无法兑现其早期的承诺，即 Metro 界面将与我们一些人熟悉和喜爱的桌面一样强大和功能齐全。我希望这是真的，但是如果没有魔法，我不认为一开始会是那样。

Adobe 已经就这个话题[在这里](https://web.archive.org/web/20230203053417/http://blogs.adobe.com/flashplatform/2011/09/flash-support-on-windows-8-and-metro.html)说了些什么，在这里它提到它将通过 Air 和可能的 app store 把 Flash 内容带到 Metro。这是否会是一次愉快的经历还有待观察(很大程度上取决于内容本身；许多 Flash 项目都没有针对触摸进行优化)，但值得注意的是，这远远不是苹果一直在推动的与 Flash 的完全分离。

你可以在这里阅读我们的[Windows 8 早期印象](https://web.archive.org/web/20230203053417/https://blogs.msdn.com/b/b8/archive/2011/09/14/metro-style-browsing-and-plug-in-free-html5.aspx)，但不幸的是，我们设备中的 wi-fi 认证有问题，所以我们无法测试浏览器。不要担心:在操作系统发布之前，我们将有足够的时间在接下来的一年左右时间里测试它。
