# HTML5 开发者 Joe Hewitt:“应用商店长期存在”

在 TechCrunch[Mobile First CrunchUp](https://web.archive.org/web/20230203084758/https://techcrunch.com/2011/07/29/watch-the-mobile-first-crunchup-live-here/)上，著名的移动开发者[乔·休伊特](https://web.archive.org/web/20230203084758/http://www.crunchbase.com/person/joe-hewitt)与 TC 作家 MG·西格勒谈论了本地和移动网络应用的未来。

休伊特是最著名的脸书移动开发者，在他厌倦了苹果公司坚定的应用商店规则后，他停止了脸书的 iOS 工作。收购 Parakey 五年后，Hewitt 最终离开了脸书，现在他正致力于自己的项目，主要是基于 HTML5 的项目。"

在今天的演讲中，休伊特对苹果的立场似乎更加温和，甚至告诉西格勒，他是一个 iPhone 迷，“我从未说过我不喜欢这个产品。”

休伊特还表示，他听说苹果在通常严格的规则方面已经有所改善，“他们对开发者的反应更快，等待时间也更短。”休伊特也承认，在过去的两年里，他个人没有向苹果应用商店提交任何东西。

休伊特认为，尽管许多人现在编写原生应用程序是因为它时尚和令人兴奋，但有些内容绝对不能在平台上工作，他提到了*金融时报*作为一个成功决定用 HTML5 网络应用程序[代替 iOS 参与的机构的例子。他希望随着移动网络变得越来越功能化，更多的开发者会跟随这一趋势，但本地应用不会很快走向任何地方。](https://web.archive.org/web/20230203084758/http://www.macrumors.com/2011/06/07/financial-times-wont-give-apple-a-cut-drops-ios-for-web-app/)

“应用商店长期存在..(例如)游戏对应用商店来说很棒，”他说，他指的是这种格式的支付能力，以及游戏将最大程度上受益于 iOS 保留的数亿张信用卡的事实。休伊特认为，其他格式，如新闻和杂志应用程序，并不理想，“用户体验确实受到影响，”他说。
