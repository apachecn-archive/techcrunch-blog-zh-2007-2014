# 苹果得到一个地图制作工具。这给谷歌带来了什么？

如果你今晚还没有看够[基于地理位置的新闻](https://web.archive.org/web/20230103225838/http://techcrunch.com/2009/09/30/twitters-geolocation-api-appears-to-be-live-but-most-of-you-are-lost/)，这里还有一个非常有趣的消息。《计算机世界》的塞思·温特劳布今晚报道说，看起来苹果已经悄悄地收购了一家在线地图公司。

苹果对 Placebase 的收购实际上发生在今年 7 月，一家使用 Placebase 地图的合作公司的创始人[在推特上发布了相关消息](https://web.archive.org/web/20230103225838/http://twitter.com/fredlalonde/status/2514358118)。但是它被大多数人忽略了，因为这是唯一的新闻。但温特劳布今晚翻出了 Placebase 首席执行官贾隆·瓦尔德曼(Jaron Waldman)的 LinkedIn 个人资料，果然，他现在是苹果“地理团队”的一员。

这是为什么这非常有趣:这很可能标志着谷歌和苹果之间的关系再次出现裂痕。至少，苹果收购了谷歌地图的竞争对手，这可能是埃里克·施密特不得不从苹果董事会辞职的另一个原因(他在八月辞职)。但更重要的是，这样的收购可能会让苹果摆脱对谷歌地图的依赖，苹果在 iPhone 和 iPhoto 电脑软件上使用谷歌地图。

显然，人们对苹果拒绝谷歌语音应用程序一事议论纷纷，但也要记住，他们也拒绝(或不批准)谷歌的 Latitude 应用程序，迫使谷歌开发基于浏览器的版本。苹果给出的不批准的理由是，这会让用户对 iPhone 上内置的地图应用程序感到困惑——这个应用程序运行着，是的，谷歌地图。

谷歌实际上帮助苹果建立了整个地图应用，除了让他们使用他们的地图数据。至少可以说，这整件事很奇怪。但随着 Placebase 的收购，这一切可能很快就会结束。

也可能不是。当然有可能苹果只是意识到了地理定位的重要性，尤其是在移动领域，并希望获得该领域的人才。虽然 Placebase 是谷歌地图的竞争对手，但它也略有不同，专注于不同的图层和定制。
