# 创业试验:Quipol 试图在拥挤的市场中扰乱在线投票 TechCrunch

> 原文：<https://web.archive.org/web/https://techcrunch.com/2011/12/13/startup-trials-quipol-wants-to-disrupt-online-polling-but-can-it-withstand-the-competition/>

作为一家初创企业，这是一个脆弱的局面，不是吗？这里你有一个你认为很棒的产品，也许它甚至有点颠覆性，但你不会天真到认为你是唯一一个想到这个想法/产品的人。在任何时候，一些大型或资金更充足的初创公司可能会把你的想法/小工具/功能/工具/产品作为他们的优先事项，只要花一点营销费用，你就完蛋了。Bootstrapping，你依靠用户的反馈来告诉你什么可行，什么不可行，迭代，并希望你的用户群有机增长，直到你有一个可持续的业务。

这就是希望，对吗？来自印第安纳州的 23 岁设计师马克斯·约德(Max Yoder)也是如此——只是在约德的例子中，他在印第安纳波利斯的卧室里创办了自己的创业公司。Yoder 最近从大学毕业，那时他去了 Compendium 工作，了解创业公司、媒体软件，并帮助偿还他的学生贷款。为了帮助他的事业，他去年被选为奥尔企业家研究员，在过去的七个月里，他一直在创建自己的创业公司。

最近，Yoder 开始全职工作于 [Quipol](https://web.archive.org/web/20221007114913/http://quipol.com/) 。Quipol(发音为“kwi-poll”)是一个测试版的 web 应用程序和嵌入式小部件，使用 HTML5、CSS3、Ruby on Rails 和[响应式设计](https://web.archive.org/web/20221007114913/http://blog.quipol.com/post/9840678306/responsive-web-design)设计，让内容制作者创建简单的一个问题的投票，可以嵌入到博客帖子和网页中，以鼓励观众的反馈，并帮助他们吸引更多的读者。这部分是投票系统，部分是评论工具，但事实是竞争很激烈——这种情况以前也发生过。

还有像 [GoPollGo](https://web.archive.org/web/20221007114913/http://gopollgo.com/) 这样令人敬畏的网站(完全披露:创始人 Ben Schaechter 曾经是 TechCrunch 的开发人员，也是他的朋友)，这些网站允许用户创建与大量 Twitter 集成的实时投票。有[民意调查机构](https://web.archive.org/web/20221007114913/http://www.crunchbase.com/company/poll-authority)、[无处不在的民意调查](https://web.archive.org/web/20221007114913/http://www.crunchbase.com/company/poll-everywhere)、[精确民意调查](https://web.archive.org/web/20221007114913/http://www.crunchbase.com/company/precision-polling)(由调查猴子抢先获得)，像 WordPress 这样的 CMSes 也有自己的民意调查功能，这要感谢像 [PollDaddy](https://web.archive.org/web/20221007114913/http://polldaddy.com/) 这样的初创公司。

这就是 Quipol 真正的竞争对手:内容管理平台拥有或提供的已经到位的投票窗口小部件。但 Yoder 认为 Quipol 可以很好地补充其中的一些内容，以及 Disqus 和 Livefyre 等评论平台。因此，Yoder 希望通过 Quipol 设计的简单性(它看起来很棒)和它的“普遍性”，这项服务能够与已经占据该空间的其他服务有所区别。

在这一点上，Quipol embed 代码可以在 WordPress.org、Tumblr、Squarespace 和 Blogger 上运行，除了社交评论之外，在 Posterous 和脸书上运行良好。那是什么意思？用户可以像往常一样在脸书和 Posterous 上投票和发表评论，但如果他们选择将评论分享到他们的社交网络，社交网站将链接回 Quipol.com 的投票——而不是其内嵌的来源。不幸的是，在这种情况下，Posterous 和脸书不能很好地使用 Quipol 的 javascript，但 Yoder 说他目前正在努力寻找一些解决方法。

有趣的是，Quipol 目前没有定制选项，它只有 400 乘 600 的形式，这是 Yoder 目前正在努力解决的另一件事——这就是用户反馈的来源。但是，Quipol 仍然做得足够好，它是一个伟大的工具。评论使用 Twitter 和脸书认证来阻止垃圾邮件，脸书和 Twitter 的链接提供了一个简单的“拇指向上”或“拇指向下”投票机制，如果你的意见不属于这两个类别之一，就可以发表评论。

投票和评论可以被网站管理层的 Quipol 创建者监控。然而，读者除非投票，否则无法看到 Quipol 的结果，而管理层提供基本的分析，直到用户提供足够的数据来显示他们想要看到的内容，这将使 Yoder 能够扩展分析产品。他还计划在不久的将来将 YouTube/Vimeo 视频嵌入到 Quipols 的媒体部分，[看起来就像这个](https://web.archive.org/web/20221007114913/http://media.tumblr.com/tumblr_lqfxwpN4RJ1qgsbtb.jpg)。

这是一个非常简单的工具，Yoder 设计了一个伟大的产品——问题是这里是否有一个真正的业务，或者它是否最适合作为一个更大实体的功能。当 Quipol 嵌入到上下文相关的博客帖子中时，它的效果最好，可以帮助博客作者吸引和增加他们的受众(增长部分是通过 Quipol 的社交评论功能实现的，Yoder 说)。
