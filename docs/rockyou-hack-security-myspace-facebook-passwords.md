![](img/36f85565311e70063c84fc35967442a8.png "ry") [今天早些时候有消息传出](https://web.archive.org/web/20230403194353/https://techcrunch.com/2009/12/14/rockyou-hacked/)社交应用网站 [RockYou](https://web.archive.org/web/20230403194353/http://www.rockyou.com/) 遭遇数据泄露，导致超过 3200 万用户账户被曝光。更严重的是，发现 RockYou 在其数据库中以纯文本形式存储所有用户帐户数据，将所有这些信息暴露给攻击者。rock 你还没有通知用户这个漏洞，他们的博客出奇地沉默——但是安全漏洞的细节越来越糟糕。

第一个问题是 RockYou 试图淡化整个事件，首先通过不通知用户来掩盖它，然后在官方声明中淡化它，称它是一个只影响“旧”应用程序的问题。对最初的入侵负有责任的黑客公布了他检索到的一小部分数据集，并能够表明他不仅可以访问他们的整个数据库，而且密码也明文存储。这件事现在看起来比最初怀疑的还要糟糕。数据集还包含一个表，其中 RockYou 存储了社交网络和其他合作伙伴网站的用户凭据。

该数据库由一个包含合作伙伴数据的表和另一个存储了用户输入的那些合作伙伴站点的凭据的表组成。这包括社交网络，如 MySpace 和 webmail 账户。

最初的利用利用了一个微不足道的 SQL 注入漏洞，这种技术已经被很好地记录了十多年。这种利用漏洞的方法在执行过程中极其简单，但在影响上却是灾难性的 RockYou 和网站用户现在正在艰难地学习这一点。更令人惊讶的是，这种情况没有更早发生——因为 RockYou 平台是安全漏洞和不良做法的瑞士奶酪。

### 你错在哪里

**糟糕的密码策略**

RockYou 帐户创建只强制使用最小长度为 5 个字符的密码，没有大小写、数字或标点符号的要求。该平台实际上鼓励简单的密码，不允许任何标点符号。

![](img/fd7bd7a41755ce8e2eb465eb358af8da.png)

**密码中的明码**

RockYou 您仍然明文存储密码，并通过电子邮件明文传输用户密码。尽管攻击发生在 10 多天前，并且 RockYou 知道这次攻击，但今天注册 RockYou 帐户的用户仍然会将他们的密码以纯文本形式存储，并以明文形式通过电子邮件发送给他们。

![](img/83f6cebb9c840850fc94b561c91b763a.png "ru2")

**密码反模式**

RockYou 在共享数据或应用程序时，提示用户将其第三方站点凭据直接输入到 RockYou 站点。脸书的整合需要适当的脸书认证，今天的 MySpace 整合也应用了类似的技术，但是对于大多数其他网站来说，同样的旧的疯狂的密码请求表单仍然存在。告诉您的用户您不会存储他们的密码不是一个解决方案。

![](img/8f1c8a2a1c0f1e13377ed9faa29597f4.png "ru3")

**可怕的反应**

RockYou 几天前就知道了这个漏洞，黑客的嘲讽让这个问题变得众所周知，RockYou 也做出了回应(尽管他们的用户仍然没有意识到这个问题，除非他们在网上阅读新闻)。

下次你注册网络服务时，花点时间看看他们在告知用户数据泄露方面的立场，看看他们有多尊重用户的隐私。

你对一件非常严肃的事情沾沾自喜。他们没有采取措施来纠正导致违规的问题，也没有以适当或充分的方式解决用户的问题。适当的反应应该是关闭网站几个小时，并强制用户输入新密码，新密码将以哈希或加密的形式存储。可悲的是，公司能够如此自满地逍遥法外，因为大多数用户不会发现这一点，大多数用户永远不会受到它的影响，而且服务提供商对用户的私人数据不负任何责任。
