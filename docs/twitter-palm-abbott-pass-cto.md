# 随着阿博特从 Palm 跳槽到 Twitter，帕斯成为了第一位 Twitter 首席技术官

上周五，有消息称，Palm 的软件和服务主管迈克尔·阿博特(Michael Abbott)将离开公司，他负责 Palm 的 webOS 平台。这个由[通过一份监管文件披露给](https://web.archive.org/web/20230304035341/http://www.businessweek.com/idg/2010-04-16/palm-s-software-chief-quits.html)的消息引起了轩然大波，因为它进一步凸显了 Palm 的脆弱地位——该公司的股票价格在最近几个月一直在暴跌，现在它正试图找到一个买家。同一份文件披露了 Palm 让高级员工留下来的计划:在出售的情况下，提供利润丰厚的保留方案。很明显，艾伯特不想和这件事扯上任何关系。他一定有充分的理由。现在我们知道原因了。

我们已经从公司获悉并证实，雅培将加入 Twitter，担任该公司新的工程副总裁。为了腾出空间，Twitter 现任工程副总裁 [Greg Pass](https://web.archive.org/web/20230304035341/http://www.crunchbase.com/person/greg-pass) 将被提升为首席技术官——成为这家[快速增长的](https://web.archive.org/web/20230304035341/http://twitter.com/JoinTheFlock/status/12469813406)公司的第一位首席技术官。

大约两年前，Pass 作为 Summize 收购案的一部分来到了 Twitter。他之前是 Summize 的 CTO(和联合创始人)，在此之前是 AOL 的系统架构师和 ToFish 的联合创始人。显然，Summize 的收购对 Twitter 来说很好，因为搜索现在是该服务的一个关键部分，并在该公司开始赚钱的努力中起着主导作用——推广的推文[将首先完全依赖 Twitter 搜索。](https://web.archive.org/web/20230304035341/https://techcrunch.com/2010/04/12/full-details-on-twitters-long-awaited-ad-platform/)

与此同时，雅培之前是。微软的. NET 在线服务。在此之前，他共同创立了客运公司和复合软件。但显然 Palm 的演出是他的重头戏，因为是他把 webOS 介绍给了世界。虽然它可能没有拯救 Palm，但几乎每个人都认为该软件是创新的，是 Pre 的最好部分。

Twitter 继续打造一个地狱般的团队。
