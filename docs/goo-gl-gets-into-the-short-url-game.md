# Bit.ly 刚拿到 Fu.kd:脸书和谷歌进入短网址游戏

![](img/d6837e5d28d002d3d620c349c18836e2.png)

在我们听到[报道](https://web.archive.org/web/20230324211525/http://www.insidefacebook.com/2009/12/14/facebook-testing-new-url-shortener-fb-me/)脸书新的网址缩短器后不久，谷歌[推出了](https://web.archive.org/web/20230324211525/http://googleblog.blogspot.com/2009/12/making-urls-shorter-for-google-toolbar.html)自己的服务，恰当地称为 [goo.gl.](https://web.archive.org/web/20230324211525/http://goo.gl/)

目前，它只用于谷歌工具栏和 Feedburner。谷歌刚刚宣布这项新服务是工具栏的共享功能，可以让你直接从工具栏共享网页。shortener 不是一项独立的服务，不能用于“更广泛的消费者使用”也就是说，至少目前是[。](https://web.archive.org/web/20230324211525/http://twitter.com/mattcutts/status/6672989465)谷歌保证其 shortener 将是稳定和安全的，以帮助保护用户免受恶意网站的点击。不出所料，谷歌承诺提供快速链接服务。

谷歌进军网址缩短领域并不令人惊讶；许多平台正在内部开发自己的 shorteners，比如 Digg 和脸书。脸书的新网址缩短器正被用于其手机界面的网址。目前还不清楚 Fb.me 是否会推广到整个平台。

当然，脸书和谷歌在这一领域的冒险都将威胁到 bit.ly，这是 Twitter 和许多 Twitter 客户端上使用最广泛的网址缩写和默认服务。到目前为止，bit.ly 已经迅速成为标准的 shortener。但是脸书和谷歌能产生的大量的短链接可能会很快淹没比特链接的数量。然而，链接背后的数据才是有价值的。谁能收集所有缩短链接的最统一的视图，谁将最终获胜。

你不得不认为谷歌最终会更广泛地推出这项服务，以扩大其覆盖范围，但目前还不清楚这是否会发生。(尽管谷歌的马特·卡茨[在推特](https://web.archive.org/web/20230324211525/http://twitter.com/mattcutts/status/6672989465)上表示，对谷歌自己产品的限制只是“暂时的”，这项服务有可能在未来“开放”)。
