Kii云为拥有2500万最终用户的移动开发者平台打开了大门

一个名为 Kii Cloud 的移动后端即服务(MBaaS)今天正式面向开发者开放，号称拥有2500万终端用户和一群分发合作伙伴，包括 NTT docomo 。在过去两年的开发中，Kii Cloud将API和SDK与专有后端结合起来，抽象出构建服务器的复杂性。

MBaaS更多时候是一种迎合开发者的方式。这里的想法是，开发人员编写应用程序，并将其推送到Kii Cloud，而不必构建自己的服务器和后端。关键在于，Kii通过API和SDK来推销简单性和可伸缩性，这些API和SDK使用了最初由db4objects作为开源项目开发的对象数据库技术。其遍布日本、韩国和中国的广告网络和分销渠道使其影响力巨大。

Kii如何工作

开发人员为其应用程序编写代码，并部署到Kii。SDK提供身份服务作为跨设备用户身份验证的核心功能。Kii的用户概要文件包含JSON编码的对象，允许用户相对容易地构建他们的概要文件。

Kii也在推销其数据和分析能力。再说一次，他们说的优势是易用性。SDK可用于iOS和Android设备。我们将看到“在没有任何后端编程的情况下向它扔任何类型的数据”是多么容易，但是现在我们将假定他们是无辜的。

Matsumura在一次采访中对我说，Kii的价值来自于它能够利用数百万用户来访问它的应用程序。Kii拥有自己的广告网络，让开发者能够直接销售和管理自己的广告。

Kii Cloud是圣克拉拉和东京Synclore公司的伺服软件的故事。Servo最初是为网络运营商、设备制造商和开发者提供API和SDK。Synclore成立于2008年，是对日本诺基亚公司的管理层收购。它基于其专有的手机备份系统，为Docomo和软银等运营商提供软件。

这两家公司于2010年7月合并，并从那时起建立了自己的服务。Miko Matsumura是Kii管理团队的成员。他负责新的Kii云的开发者关系。他说，2500万终端用户使用的应用和服务来自与京瓷、软银和夏普等分销合作伙伴和赞助商合作的开发者。这项服务目前处于公开测试阶段，至少在年底前是免费的。

Servo的创始人Christof Wittig是Kii的首席执行官和Kii Capital的董事总经理。在Kii之前，他在db4objects担任创始人兼首席执行官，该公司于2008年12月被Versant Corp .收购。db4objects开发、商业许可并支持db4o，这是一个开源对象数据库。Kii中的面向对象技术来源于Wittig的技术背景。它是Kii服务不可或缺的一部分，允许复杂的交互，这种交互在开发人员向平台添加功能和不同数据集的过程中发挥作用。

Kii是第二代云公司。这是一家工程公司，由经验丰富的技术专家开发，他们带来了同步和数据库技能集，以创建一种面向移动市场的服务。但这家公司的国际风味将是公司成功的一个因素。像NTT·多科莫这样的公司的发行方式会有所不同。这些公司将计费嵌入到他们提供的服务中，这使得Kii对于希望接触全球受众的开发人员来说是一个有吸引力的产品。

总之，我认为开发人员需要仔细考虑他们使用的移动后端。有很多选择。 Kin Lane 涵盖了所有的提供商，包括 Stackmob 、 Kinvey 和 Parse 。Kii声称，它一开始就是最大的供应商，拥有庞大的最终用户群。但这是一个非常拥挤的市场，也是一个年轻的市场。Kii面临的挑战将是从人群中脱颖而出，并展示其后端能够扩展到全球多个地区的数百万用户。
