# Google Buzz 增加了 Rebuzz 功能——但不要这么叫它，那太逊了

![](img/456b4d8a4280ca03c44dfc8ce14d7d16.png "b")

不管是什么样的社会服务，一个共同的关键特征就是分享的能力。脸书有“分享”，Tumblr 有“转发”，当然，Twitter 有“转发”等等。该功能为用户提供了一种简单的方式来做一些社交活动，而不必自己做太多事情。今天， [Google Buzz](https://web.archive.org/web/20221209033825/http://www.google.com/buzz) 正在获得自己的这种功能——但不要称之为“Rebuzz”，而是简单地称之为“Reshare”

当添加到 Buzz 的[当前“评论”和“喜欢”(以及电子邮件)的](https://web.archive.org/web/20221209033825/https://beta.techcrunch.com/2010/02/09/if-google-wave-is-the-future-google-buzz-is-the-present/)武库中时，Reshare 完成了其大多数竞争对手已经建立的社交圈。它的工作方式和你预期的完全一样:如果你看到一个你喜欢的 Buzz 帖子，并想与你的粉丝分享，只需进入帖子底部，然后点击“重新分享”按钮。一个输入区将会下拉，你可以在你分享的任何评论上添加你自己的评论。这将被注入到你的追随者的信息流中。

这个的关键可能是在你分享的任何东西上面留下你自己的评论的能力。这使得该功能更像是 Tumblr 的转发，而不是 Twitter 的转发。不能在推特转发中留下自己的评论，这在推特用户中引起了很大争议。以前，转发是社区做的一件有机的事情，你可以简单地通过复制和粘贴“RT @username”之前的推文来分享一些东西 Twitter 希望使这一过程更加简化，将其融入服务中，但没有加入你自己的评论。因此，有些人仍然用旧的手动方式来做(或者使用用旧方式做的客户端)。

![](img/1a25f5b6cdbafbf3b25a5eec353fbb16.png "a")

关于 Buzz Reshares 的几点说明:

*   正如你所料，你只能分享公共的嗡嗡声
*   这是一个双击的过程(“重新分享”然后“发布”)。谷歌曾考虑过让它一键完成，但最终决定(在下拉列表中)让有限的人群分享一些东西是很重要的。
*   当你分享内容时，每个分享文章的人都会出现在你评论的下方(以一个小图标的形式)。
*   多次重新共享的相同内容将会折叠在您的订阅源中最新内容的下方。
*   reshare 函数将对话分支到一个新线程。谷歌对此想了很多，但最终决定这是目前最好的方法。不过，他们也在考虑一个非分叉选项。
*   喜欢一个分享的帖子只是“喜欢”重新分享的版本，而不是最初的分享。

所以，关于名字。我问谷歌为什么选择 Reshare，而不是明显的“拒绝”他们让我看下面的图片。显然，Rebuzz 曾被考虑过，但最终被否决了，因为它“听起来有点蹩脚”这在某种程度上是正确的，但出于品牌的目的，我认为这可能是正确的选择。我怀疑人们可能会这样称呼它。他们也考虑过“转帖”，但希望避免引入新的动词。所以最终他们选择了最字面的意思。

Buzz 团队表示，重新分享的能力是最受欢迎的功能之一。它在 15 周内竞争了 16 个新功能(他们一直在争取每周一个新功能)——当然，这些功能中有许多是用户要求的，以使 Buzz 不那么烦人。现在是[到达](https://web.archive.org/web/20221209033825/https://beta.techcrunch.com/2010/02/22/google-buzz-review/)。

这一重新分享功能将在一天内向所有 Buzz 用户推出。
