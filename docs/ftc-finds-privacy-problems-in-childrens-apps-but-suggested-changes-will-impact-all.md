# FTC 发现儿童应用程序中存在隐私问题，但建议的更改将影响所有 TechCrunch

> 原文：<https://web.archive.org/web/https://techcrunch.com/2012/02/16/ftc-finds-privacy-problems-in-childrens-apps-but-suggested-changes-will-impact-all/>

我相信孩子们是未来。(什么，太快了？)但是在新的联邦贸易委员会关于儿童移动应用的报告中，参考了当前移动开发者使用的数据处理实践，儿童*就是*未来。它们是在当今移动应用生态系统中如何处理我们个人信息的未来指标。

尽管新的报告专门为儿童应用程序提出了建议，但现在显然有一股愤怒和违规的暗流在涌动(这主要归功于 [addressgate](https://web.archive.org/web/20221006192645/https://beta.techcrunch.com/2012/02/08/addressgate-path-fallout/) )。人们，不仅仅是父母，需要控制和理解他们的数据是如何和为什么被收集、使用和共享的，以及这到底意味着什么。问题是，*这是怎么做到的*？

在 FTC 题为“[面向儿童的移动应用:当前的隐私披露令人失望](https://web.archive.org/web/20221006192645/http://ftc.gov/os/2012/02/120216mobile_apps_kids.pdf)，[的报告中，该机构发现](https://web.archive.org/web/20221006192645/http://bits.blogs.nytimes.com/2012/02/16/f-t-c-issues-report-on-apps-and-data/)苹果应用商店和安卓市场上的应用几乎没有提供关于他们数据收集做法的信息。大多数时候，父母没有办法阅读应用程序的描述，以做出任何明智的决定，即它是否适合孩子。

应用程序能够访问各种个人信息，包括一个人的姓名、联系人列表、通话记录、唯一标识符、精确位置等。应用程序还能够连接到社交媒体，通常包含广告，这为数据收集开辟了其他渠道。

正如[亚历克西亚已经指出的](https://web.archive.org/web/20221006192645/https://beta.techcrunch.com/2012/02/14/leave-path-alooooooooone/)，当前设置的大部分问题及其引起的隐私问题都与移动应用商店本身(即苹果和谷歌)普遍缺乏执行力度有关。联邦贸易委员会支持这一点，基本上是说，如果没有商店自己的指导方针的适当执行，开发商就没有什么动力去遵守。当然，他们没有动力向父母提供更多的特别披露。

此外，联邦贸易委员会表示，为儿童设计的移动应用程序甚至可能违反 COPPA，也就是儿童在线隐私保护法案。这项规定要求在线服务的运营商，包括交互式移动应用，在收集 13 岁以下儿童的信息之前，提供通知并获得父母的同意。在接下来的 6 个月里，联邦贸易委员会将审查应用程序，以确定是否有一些违规。很可能，很多人是。

但这份报告中详细描述的问题的根源与这样一个事实有关，即没有简单、通用、标准的方法来理解一个应用程序在幕后做什么或为什么做。联邦贸易委员会计划在 2012 年通过一个公开研讨会来解决这个问题，在研讨会上，它试图定义“移动隐私披露”应该如何工作，包括它们如何能够“简短，有效，并在小屏幕上为消费者所用。”

苹果和谷歌已经有了自己的方式来提醒用户应用程序的访问请求。例如，当一个应用程序想要使用你的位置时，iOS 应用程序会显示弹出窗口，而 Android 会询问你是否同意允许应用程序访问它请求的数据，这些数据会在下面列出。这两种方法都不足以更深入地了解应用程序可能需要什么数据(如地址簿，[和路径](https://web.archive.org/web/20221006192645/https://beta.techcrunch.com/2012/02/08/addressgate-path-fallout/)的情况)，以及这些数据是否共享、与谁共享以及用于什么目的。

鉴于最近发生的事件，FTC 描述的这种“移动隐私披露”的创建显然不是一个仅限于孩子的冒险。如果需要额外的家长控制来阻止与儿童应用程序相关的潜在侵入性数据共享行为，显然下一步是让所有人都可以使用这些控制。

*这可能不是一件坏事。*毕竟，为什么开发人员不应该首先实施防止 addressgate 之类问题发生的技术解决方案呢？为什么应用程序不应该对它们请求的数据更加小心呢？当然，答案是他们*应该*，而且如果抢在应用淘金热之前上市，可能更多的人*会*。

在许多情况下，违规的应用程序并不“邪恶”,而是懒惰的编码、年轻的开发人员和被忽视的错误的结果。把政府的要求放在他们面前，指数级的应用程序增长就会放缓。代码必须反复检查。app 审核进度变长。也许谷歌真的要对安卓应用实施一个应用审查程序。在清理代码的过程中，应用程序会被一遍又一遍地提交、拒绝和重新提交。一切都以隐私为名。再说一次，这可能不是一件坏事，但它肯定会改变游戏。
