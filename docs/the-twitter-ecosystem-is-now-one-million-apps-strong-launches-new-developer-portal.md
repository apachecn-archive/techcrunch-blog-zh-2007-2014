# 100 万个强大的应用程序，Twitter 推出新的开发者门户

Twitter 刚刚发布了一篇博文，称该平台目前支持 100 万个注册的第三方应用，由全球超过 75 万开发者开发。并且每 1.5 秒注册一个新的应用。此外，自 2010 年 12 月以来，已有超过 5 亿美元投资于 Twitter 生态系统公司，超过 10 亿美元用于收购。

对于 Twitter 的开发者生态系统来说，这些是令人印象深刻的统计数据，考虑到该平台一年前仅支持 15 万个应用。今天，该公司还在[推出了](https://web.archive.org/web/20230204200220/https://dev.twitter.com/blog/new-twitter-developer-site)新的开发者门户，旨在为第三方开发者创建一个更加集中的平台。

使用 Drupal 启动的门户网站包括一个新的[论坛](https://web.archive.org/web/20230204200220/https://dev.twitter.com/discussions)，摆脱了 twitter 之前使用的邮件列表。讨论可以通过“热门话题”进行过滤，以获得最受欢迎的对话。并且可以订阅自己最感兴趣的类别和线程。“开发者茶点”部分旨在为开发者提供更多的社交功能。

新的[开发者博客](https://web.archive.org/web/20230204200220/https://dev.twitter.com/blog)将包括 API 公告、事件、技巧和操作方法、应用案例研究、产品见解等等。[文档](https://web.archive.org/web/20230204200220/https://dev.twitter.com/docs)在结构和可搜索性方面得到了改进。

Twitter 还推出了一个新的、简化的应用管理器，并改进了 dev.twitter.com 的搜索，增加了开发者团体邮件列表中的过滤器和文档。

Twitter 与其开发者的关系一直很脆弱，新的门户网站是改善沟通的一种方式。三月份，Twitter [基本上告诉开发者](https://web.archive.org/web/20230204200220/https://techcrunch.com/2011/03/11/twitter-ecosystem-guidelines/)避免在原生客户端上与他们竞争。这并不是说 Twitter 不想让开发者构建他们的平台，他们只是不想让开发者构建模仿 Twitter 自己服务的客户端。正如我们过去所写的，这是[的事情，不是个人的。](https://web.archive.org/web/20230204200220/https://techcrunch.com/2011/02/19/this-is-business-not-personal/)但是据报道，联邦贸易委员会[正在调查该公司的开发者政策是否有反竞争的倾向。](https://web.archive.org/web/20230204200220/http://www.bloomberg.com/news/2011-07-01/ftc-said-to-have-begun-antitrust-inquiry-into-twitter-s-developer-policies.html)
