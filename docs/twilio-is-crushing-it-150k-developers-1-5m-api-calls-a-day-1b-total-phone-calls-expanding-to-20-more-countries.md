# Twilio 正在摧毁它:15 万个开发人员，每天 150 万个电话，5 亿个总电话，20 个新国家 

> 原文：<https://web.archive.org/web/https://techcrunch.com/2012/10/17/twilio-is-crushing-it-150k-developers-1-5m-api-calls-a-day-1b-total-phone-calls-expanding-to-20-more-countries/>

今天，Twilio 的首席执行官杰夫·劳森在他的[云电话初创公司大会](https://web.archive.org/web/20221205112058/http://www.twilio.com/conference)上发表了热情洋溢的主题演讲，其中穿插了大量统计数据和引语，如“通信现在属于软件领域。我们专注于用户，我们会赢。”

Twilio 现在为优步、易贝和 Hulu 等科技巨头提供支持，拥有 15 万名开发者，并正在向国际扩张，以颠覆传统的电信行业。

这一消息今天在 Twilio 的开发者大会 [TwilioCon](https://web.archive.org/web/20221205112058/http://www.twilio.com/conference) 上公布，联合创始人兼首席执行官杰夫·劳森(Jeff Lawson)走上舞台，宣布 Twilio 成功打破了运营商霸权。“在电信行业，竞争环境已经变得公平，”他说，因为从昂贵的硬件转向基于云的软件，比如 Twilio 提供的那种。

Twilio 今天宣布，现在有 150，000 名开发人员使用其 API，每天有近 150 万次电话呼叫，有 5 亿次电话呼叫是在 Twilio 平台上完成的。[更正:Twilio 每天总共处理了 5 亿个电话呼叫和 150 万个电话呼叫，而不是我们最初发布的 10 亿个电话呼叫和 150 万个 API 呼叫]。

Twilio 正在加速向另外 20 个国家引入本地号码，这将使其覆盖的国家总数达到 40 个。这是一个重要的增加，考虑到过去 Twilio 已经像这个一样以增量[增加。您可以在此](https://web.archive.org/web/20221205112058/https://beta.techcrunch.com/2012/08/29/cloud-based-telco-disruptor-twilio-adds-5-more-markets-plus-local-numbers-in-4-as-it-preps-a-move-into-asia/)查看受支持国家的完整列表[。](https://web.archive.org/web/20221205112058/http://www.twilio.com/international)

这一扩张意味着 Twilio 现在可以为开发者提供所有有人居住的内容的本地电话号码(南极洲除外)，进入它以前没有涉足的市场，如澳大利亚、亚洲部分地区和拉丁美洲。(Twilio 的产品副经理 Lisa Weitekamp 指出，在 Twilio 没有提供数字的所有国家中，澳大利亚是最受欢迎的国家。)

尽管云电信势头强劲，但这并不意味着它是一场一马当先的比赛。老牌电话 API 提供商 Voxeo Labs 在其 Tropo 开发社区中拥有 200，000 名开发人员，比现在的 Twilio 还要多。还有来自竞争对手的[消息称，Twilio 在开发者关系方面失去了一些关键人才](https://web.archive.org/web/20221205112058/http://www.diggz.org/index.php/2012/10/16/twilio-is-like-the-worst-girlfriend/)，这可能会削弱对该公司的乐观情绪。

该公司正在通过其创新的基于 API 的服务与世界各地的运营商进行竞争，这种服务允许开发者在网络和移动应用程序中创建基于云的电话号码，比运营商提供的类似服务成本更低，并且更容易在多个地理位置部署——首先，用户不需要与各国建立关系。

但是运营商们现在开始接受这个项目:例如，Telefonica 正在努力开发 Blue Via API，它最近增加了第一个外部运营商 Telenor T1，并计划在不久的将来扩展到更多。(有趣的补充说明，Telefonica 前 Blue Via 开发者关系主管詹姆斯·帕顿(James Parton)现在为欧洲的 Twilio 工作。上个月，Twilio 宣布了其与 AT & T 的[最大的合作伙伴关系，尽管由于一些关于交易的材料被撤回，整合的部署似乎被搁置。](https://web.archive.org/web/20221205112058/https://beta.techcrunch.com/2012/09/26/twilio-att-advanced-communications-suite/)

这些合作关系使得 Twilio 不仅开始关注更普遍的服务——例如语音服务在 20 个国家的扩张，而且更加关注服务质量。为此，今天它还宣布了一项新的全球低延迟(GLL)和优质服务，该服务由“采用地理战略数据中心和智能路由”创建。通俗地说，这确保了在它的网络上打电话不仅划算，而且听起来不错。

它还可能引入一个想法，即 Twilio 可能在某个时候开始为用户提供不同的服务质量水平，类似于运营商的做法。

韦特坎普指出，Twilio 从语音开始，但也在寻找将其他服务扩展到更多市场的方法。她指出，短信在许多国家需要单独的移动协议，这需要一些时间来解决。她不愿就 Twilio 是否以及何时推出视频通话或计费等全新服务发表评论。

Twilio 产品管理总监托马斯·斯齐亚沃尼(Thomas Schiavone)表示:“在全球范围内达成运营商交易和部署基础设施是一项重大任务。“但我们知道，我们将通过通话质量来衡量，这是确保我们为全球客户提供最佳体验的唯一方式。”

Twilio 继续致力于提供一个覆盖全球的 API。7 月，该公司宣布向 200 多个国家发送短信，并支持多种语言，如阿拉伯语、中文、日语、希腊语、俄语等。

Twiliocon 传达的重要信息是优先考虑客户体验的理念。劳森称赞了 Twilio 的开发者，称“你关注的是用户而不是脚手架。其他人被大硬件和大软件束缚住了。你会赢的。”
