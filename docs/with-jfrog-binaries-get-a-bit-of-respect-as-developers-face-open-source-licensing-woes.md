# 有了 JFrog，二进制文件得到了一点尊重，因为开发人员面临开源许可的困境

二进制文件，即应用程序中包含所有性感代码的部分，作为软件和服务如何构建的文档，正变得越来越重要。

根据以色列一家为开发人员管理二进制文件的公司对财富 500 强公司的 150 名开发人员进行的调查，开源软件经常忽略这种文档。调查结果有点自私，但它们确实说明了应用程序开发市场的状况。以下是一些亮点:

*   接受调查的 65%的开发者表示，这种糟糕的文档和许可是开源软件的最大问题。一些首要问题是未知的所有者、法律问题和关于软件的低效说明。JFrog 最近要求对发布在 [Bintray](https://web.archive.org/web/20221006120130/http://www.jfrog.com/home/v_bintray) 服务上的开源二进制文件附加许可证，这是其基于社区 SaaS 的社交平台，用于分发和提供二进制文件。
*   90%的开发人员使用源代码控制和二进制存储库作为他们与其他开发人员交换库和代码的主要手段。这意味着他们不再像以前那样使用内部文件共享、磁盘或 USB。现在的世界是协作的——开发人员需要像共享源代码一样共享他们的二进制文件。
*   大约 42%的开发人员使用二进制存储库管理器。这反映出市场发生了多大的变化。JFrog 的代表指出，三年前，90%的人甚至不知道拥有一个在线知识库意味着什么。

JFrog 提供诸如 Bintray 和 [Artifactory](https://web.archive.org/web/20221006120130/http://www.jfrog.com/home/v_artifactorycloud_overview) 这样的服务，允许组织建立他们自己的在线私有库。通过这些协作服务，JFrog 试图将自己确立为二进制文件的主导平台。自从[谷歌改变其二进制托管政策](https://web.archive.org/web/20221006120130/http://google-opensource.blogspot.com/2013/05/a-change-to-google-code-download-service.html)以来，他们的努力得到了支持，GitHub 也做出了自己的改变，它否决了一项上传功能，该功能允许用户存储独立于 GitHub 上托管的源代码的任意文件。一位发言人表示，就其存在而言，该功能不如 GitHub 体验的其他部分质量高，也不太适合大多数用户的工作流程。

为了应对这种转变，JFrog 在 4 月份宣布了将 GitHub 二进制文件自动迁移到其服务中的能力。从那时起，JFrog 见证了 140%的新注册增长。当 Google Code 宣布关闭二进制支持时，注册量也大幅上升。

事实是，谷歌擅长提供开发者资源，帮助开发者通过丰富的服务和 API 构建应用。GitHub 最擅长的是帮助开发人员在他们的代码上合作。

JFrog 希望成为一个同样重要的二进制协作平台，并帮助缓解 OSS 许可的困境。
