# 微软在 Release 中收购了 InCycle 的发布管理服务，以改进其 Team Foundation Server

微软[已经收购了](https://web.archive.org/web/20230404063303/http://www.microsoft.com/en-us/news/Press/2013/Jun13/06-03InReleasePR.aspx) [InCycle 的](https://web.archive.org/web/20230404063303/http://www.incyclesoftware.com/) [发布管理解决方案](https://web.archive.org/web/20230404063303/http://www.incyclesoftware.com/inrelease/) InRelease，这是一个在不同环境下自动将应用组件部署到目标服务的工具。InCycle 是一家加拿大公司，专注于应用生命周期管理(ALM)和微软的发布管理解决方案。NET 平台。

由于微软只是收购 InRelease 业务部门，公司的其余部分将继续独立运营，专注于 ALM 咨询。InCycle 成立于 2002 年，去年，该公司与另一家专注于 ALM 的微软合作伙伴 Sirk Technologies 合并，该公司在旧金山和西雅图设有办事处。

InRelease 支持敏捷。NET 团队来自动化他们的应用程序的发布周期。它直接从微软的 Team Foundation Server(微软自己的 ALM 和 DevOps 解决方案)自动部署到所有环境，包括生产环境。这确保了所有部署都以相同的方式完成。它还自动化了批准工作流和审计支持。

微软开发部门的公司副总裁 S. Somasegar 在今天的一份声明中表示:“随着企业被迫以越来越快的速度开发和部署高质量的应用程序，DevOps 是 ALM 中越来越重要的一部分，也是首席信息官越来越感兴趣的一个领域。”。InRelease 连续交付解决方案将使 Visual Studio Team Foundation Server 从开发到生产的发布过程自动化，有助于更快、更简单地交付应用程序。

InRelease 已经与 [Team Foundation Server](https://web.archive.org/web/20230404063303/http://tfs.visualstudio.com/en-us) 直接集成，InCycle 希望这次收购能够帮助它向 Team Foundation Server 添加发布管理功能。微软没有具体说明除了[之外它具体计划如何这样做，指出](https://web.archive.org/web/20230404063303/http://blogs.msdn.com/b/somasegar/archive/2013/06/03/teched-2013.aspx)“这次收购将扩展微软在 ALM 和 DevOps 领域的产品”，并且它计划集成“从 TFS 到生产，都在一个解决方案中，并且都与 TFS 集成”
