# Softcover 是一个新的面向技术作者的自助出版平台

Softcover 是一家初创公司，为自助出版电子书提供所谓的“无摩擦”平台。

联合创始人 Michael Hartl 自己也是一名作家，他写了 [Ruby on Rails 教程](https://web.archive.org/web/20221209070132/http://ruby.railstutorial.org/)。(他也是一个重复创业者，创建了由 T4 Y Combinator 支持的社交网络初创公司 Insoshi 。)Hartl 通过电子邮件告诉我，从与 Addison-Wesley 的出版交易到教程网站的直接销售，这本书已经赚了 75 万美元 Softcover 就是基于这样的经历:

> 最初，Softcover 来自于挠自己的痒痒:我想制作 Ruby on Rails 教程的后续产品，这是一个书籍和截屏系列，已经成为 web 开发的领先入门之一。我从经验中知道制作一个新网站，安装一个销售系统，并上传文件以供使用是多么痛苦，我不想为每一个新产品都这样做。我还想使用我定制的 Rails 教程制作系统(或者功能相当强大的系统)从通用源文件输出多格式电子书。最后，我希望对其他产品拥有与 Rails 教程相同的控制级别，包括对我的客户列表的完全访问和对定制域的支持。我找不到这样的服务，所以我和几个朋友建立了 Softcover。

这基本上就是 Softcover 通过电子书自助出版系统提供的东西，该系统最近刚刚退出了私人测试版。它允许作者在写作时查看其图书的 HTML 版本，在 Softcover 网站上发布 HTML、EPUB、MOBI 和 PDF 电子书，并将这些电子书与截屏和其他媒体捆绑在一起，以创建优质包。(哈特尔说，电子书可以包括“语法突出的代码列表、编号的表格和图形、数学方程式和链接的交叉引用。”)作为交换，创业公司拿走销售收入的 10%。

哈特尔强调，初创公司仍将给予作者很大的控制权。首先，如果他们愿意，他们可以提供免费版本(他将 Ruby on Rails 教程的成功归功于此)。此外，该系统是开源的，作者将可以访问他们的销售列表，他们可以使用自定义的网络域名，所以没有什么可以阻止他们离开 Softcover，如果他们选择的话。

凭借代码和截屏支持等功能，Softcover 似乎特别适合技术作者，但 Hartl 表示，它也可以制作“格式不太复杂的文件，如非小说书籍和小说。”他补充说，他希望降低使用 Softcover 制作系统的技术壁垒(目前它“最适合那些习惯使用文本编辑器和终端窗口的作者”)，并引入营销指南来帮助新作者建立受众。

Softcover 是由 Ruby on Rails 教程的利润资助的。哈特尔说，他对“在正确的条件下”接受外部投资持开放态度。
