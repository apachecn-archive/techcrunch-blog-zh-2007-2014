# 重新设计:复制粘贴的帽子电子邮件模板

> 原文：<https://web.archive.org/web/http://techcrunch.com/2011/07/11/the-techcrunch-redesign-a-copy-and-paste-hatemail-template/>

这是一条绝对的、不可改变的定律:每当一个备受瞩目的网站进行重大重新设计时，普通读者的反应将是愤怒。

不是困惑，不是有分寸的批评，而是完全的愤怒——就好像他们醒来后发现有人溜了进来，重新安排了他们妻子的面部特征。

因此，几分钟前，我怀着忐忑不安的心情访问了 TechCrunch，发现我们的[大肆吹嘘的](https://web.archive.org/web/20230203142218/https://techcrunch.com/2011/07/08/heads-up-techcrunch-will-look-a-little-different-next-week/)重新设计已经“上线”，包括巨大的标题字体、怪异的底层图像以及——哦，天哪——一个新的标志。一个新的标志，在我没有受过艺术训练的眼里，看起来像是用一只忍者神龟的尖尖画的。

首席设计师戴夫·费尔德曼(Dave Feldman)在这里为重新设计的发表了一篇激烈的、先发制人的辩护，解释它如何满足“大胆、原始、快节奏”的美国在线科技博客 TechCrunch 的需求。我发现戴夫的帖子很吸引人，尤其是因为我不知道有个叫戴夫·费尔德曼的人在 TechCrunch 工作。(你好戴夫！)

至于重新设计——坦率地说，我不确定我对它的感觉如何。我不确定我是否喜欢顶部标题对我大喊大叫。我不确定顶部的动画标志是粗体、原始还是快节奏的。对于右下角的“也在 AOL TECH”框，我真的犹豫不决。

但这就是问题的关键:对我或任何人来说，判断新设计是否比旧设计更好还为时过早。就像学习使用一部新手机一样，我们都需要花几天时间来适应新事物，并弄清楚它是否真的更容易使用。我们需要更长的时间来进行客观的衡量——页面浏览量、加载时间；诸如此类的废话——决定新造型是否“成功”。

然而，这些都不应该——也不会——阻止你，我们尊贵的读者，告诉我们你有多讨厌新设计的每一个细节。你们中的一些人已经在 Dave 的帖子上加入了[有用的反馈](https://web.archive.org/web/20230203142218/https://techcrunch.com/2011/07/11/redesigning-techcrunch-we-picked-this-logo-just-to-piss-you-off/)，不同地描述这个设计为“可怕的”、“可怕的”、“有史以来最糟糕的重新设计”和“甚至不能与 Gizmodo 相提并论”(哎哟)。但是你们其他人呢？如果说我对戴夫有所了解的话(其实我并不了解)，那就是，在这个新东西上工作了几个月之后，他简直迫不及待地想听到你有多讨厌他(和他的团队)的劳动成果。
