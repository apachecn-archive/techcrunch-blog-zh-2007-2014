# IBM 的蓝云是网络计算的另一个名字

IBM 想要一些 Web 2.0 的魔力。这就是它今天宣布 Blue Cloud 的原因，Blue Cloud 是一套 T2 的云计算产品，将于 2008 年第一季度提供给公司客户。当然，云计算只是 Web 计算的另一个名称。这意味着大规模的服务器群、大规模的存储，以及支持互联网规模的应用和使用模式的能力。

亚马逊、谷歌、雅虎和 Salesforce.com.com 都是当今消费者和企业已经可以使用的云计算的例子。这些大多是特定应用程序的形式，但亚马逊提供了其 Web 服务套件，这是通用云计算产品的集合——计算周期、存储、通信。甚至 Salesforce.com 也通过 AppExchange 向其他公司提供云计算基础设施。IBM 在竞争中应该不会有任何问题。

与大多数企业数据中心相比，Blue Cloud 更像是一种分布式计算架构。它基于一个名为 [Hadoop](https://web.archive.org/web/20221203090710/http://lucene.apache.org/hadoop/) 的开源项目，该项目管理大型计算机集群的计算资源。Hadoop 包括一个开源版本的 MapReduce，谷歌用这个软件在世界各地的服务器上有效地分配计算任务。

所以 IBM 基本上是把这种大规模计算架构带给它的企业客户。这对企业应用程序来说是好事，因为这种分布式架构适合于已经侵入企业的 Web 2.0 应用程序。问题仍然是 IBM 将如何实现蓝云。它会在自己位于世界各地的托管服务器农场上提供这种服务，还是会教大公司的首席信息官们如何在自己的数据中心构建自己的迷你谷歌？

我猜可能两者都有一点。
