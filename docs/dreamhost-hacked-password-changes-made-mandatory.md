# DreamHost 被黑，密码修改成为强制性的

又是一天，又是一次黑客攻击。这次数据被泄露的是哪家公司？梦幻主机。

根据 DreamHost 的状态博客，该公司检测到“在他们的一个数据库中有未经授权的活动”。换句话说:有人在不该窥探的地方窥探，DreamHost 注意到了脚印。

唉，该公司没有透露太多关于黑客攻击性质的信息，除此之外，他们“没有证据表明客户密码在此时被窃取”。尽管如此，他们仍然要求所有 Shell/FTP 帐户(不是 DreamHost 客户用来登录计费/后端系统的帐户，而是他们用来访问和维护其实际网站的用户帐户)重置密码。)面向似乎是所有**dream host 客户。如果你今天发现自己无法登录你的 DreamHost FTP 帐户，那是因为你的密码已经被禁用了。**

也许这是不言而喻的，但是:如果你是 DreamHost 的用户，并且你在互联网上的其他地方使用了类似的密码，那么现在是时候把它们都换掉了。
