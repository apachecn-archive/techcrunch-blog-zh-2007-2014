# 后端服务提供商 StackMob 来到 Android TechCrunch

> 原文：<https://web.archive.org/web/http://techcrunch.com/2011/10/12/backend-service-provider-stackmob-comes-to-android/>

面向移动开发者的后端服务提供商 StackMob 今天发布了重大消息:它现在可以在 Android 上使用了。随着今天的发布(仍处于私人测试阶段——见下文邀请函), StackMob 将向 Android 开发者提供对其服务的全面访问，包括一个[开源 Android SDK](https://web.archive.org/web/20230204184440/https://github.com/stackmob/StackMob_Android) 和对 Android 推送通知的支持。

这个平台，[名爵西格勒曾称之为](https://web.archive.org/web/20230204184440/https://techcrunch.com/2011/05/16/stackmob-raises-7-5-million-to-be-the-heroku-for-mobile/)“移动的 Heroku”(在 [StackMob 与 Heroku](https://web.archive.org/web/20230204184440/https://techcrunch.com/2011/08/24/stackmob-heroku/) 合作之前)！)，是一个基于云的系统，旨在解决移动开发者的所有后端需求，包括社交整合、存储、消息传递、API 创建、分析、货币化等。这是一个灵活、可扩展的解决方案，如果需要，它甚至允许开发人员使用自定义代码向他们的应用程序添加复杂的逻辑。

联合创始人 Ty Amell 解释说，新的 Android SDK 提供了与 iOS SDK 相同的功能，这意味着开发人员可以通过用户名向给定用户发送推送通知，而不必担心他们当时使用的是什么设备。StackMob 会处理那部分。

推送通知功能适用于运行 2.2 及更高版本的 Android 设备，Amell 表示，Android 平台上约 78%的设备都使用该功能。

有兴趣注册 Android stack mob 私有测试版的开发者可以从这里开始:[http://www.stackmob.com/tc-android](https://web.archive.org/web/20230204184440/http://www.stackmob.com/tc-android)。有 300 个空位。

我们被告知，未来，StackMob 计划通过更详细的应用分析和 API 分析来进一步构建其分析系统。它最近对其后端进行了彻底检查，将架构从专有系统改为开源 Flume，主要是因为该公司喜欢支持开源，并且因为…嗯，为什么要在有可用系统的地方构建自己的系统呢？该公司仍未公开宣布退出私有测试版时的定价情况，但私有测试版的开发者可以免费测试该平台。

StackMob 此前在 5 月份的一轮融资中筹集了 750 万美元，由 Trinity Ventures 牵头，现有投资者 Harrison Metal 和 Baseline Ventures 参与。它现在在旧金山有 15 名员工，但正在寻求招聘更多员工。
