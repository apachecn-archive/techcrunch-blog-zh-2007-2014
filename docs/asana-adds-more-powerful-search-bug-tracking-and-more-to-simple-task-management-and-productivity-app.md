# Asana 为简单的任务管理和生产力应用程序| TechCrunch 添加了更强大的搜索、错误跟踪等功能

> 原文：<https://web.archive.org/web/https://techcrunch.com/2013/04/09/asana-adds-more-powerful-search-bug-tracking-and-more-to-simple-task-management-and-productivity-app/>

[Asana](https://web.archive.org/web/20230406162018/http://asana.com/) 是一家试图围绕任务(而非电子邮件)重新设计工作场所的知名生产力初创公司，今天宣布了一项重大更新，增加了更强大的搜索功能、错误跟踪功能和经理报告。

搜索是此次发布中最大的更新，增加了全文搜索、结构化搜索和可自定义保存的搜索视图。联合创始人贾斯汀·罗森斯坦解释说，人们花大量时间在组织内部寻找信息。但是有了新的搜索功能，浅野用户应该再也不会面临这个问题了。

本质上，到目前为止，Asana 的默认视图是“项目视图”。现在，随着高级搜索功能的推出，浅野提供了一个工作“搜索视图”，让您可以在浅野中心窗格中看到任何搜索的结果，从简单的关键词搜索到丰富的结构化搜索。这些视图可以按任务到期日、创建时间或修改时间排序。您也可以在任务之间导航以查看其详细信息，或选择多个任务以一次性更改它们。您可以保存这些视图，以创建自定义报告，并在每次切换到这些视图时进行更新。

是这样的。对于简单的全文搜索，您可以在搜索框中键入并选择“搜索任务”对于结构化搜索，单击搜索框右侧的箭头。然后，您可以指定受让人、项目、标签、附件、完成状态、截止日期等。然后，您可以将范围缩小到未完成的任务、带有附件的任务或未分配给您的任务。一旦创建了要再次使用的搜索，您可以单击搜索标题旁边的星号将其保存。

此外，Asana 正在推出新的经理报告功能，以帮助经理跟踪他们的团队和项目。您只需将您的队友添加到“分配给”字段，然后按项目或标签过滤，以深入了解您需要的信息。例如，罗森斯坦说，经理可以全面了解他们的团队目前正在进行的所有任务、他们的状态以及与这些任务相关的通信。

最后，由于 Asana 在开发者社区中相当受欢迎，这家初创公司在错误跟踪方面加倍努力。Rosenstein 说 Asana 与许多开发人员交谈过，以确定他们对 bug 跟踪的需求，并且一遍又一遍地听到当前的 bug 跟踪工具达不到标准。

因此，搜索已经更新，以便更好地跟踪错误。例如，您可以使用保存的搜索功能来允许 QA 团队观察尚未被 QA 的已完成的 bug，或者让客户服务团队观察他们在被分配和完成时打开的 bug。

罗森斯坦说，这次更新是 Asana 作为生产力应用程序的一大进步。“这只是我们计划的开始，”他说。“搜索是我们产品的核心支柱之一，我们是一家查询集体记忆的公司。”
