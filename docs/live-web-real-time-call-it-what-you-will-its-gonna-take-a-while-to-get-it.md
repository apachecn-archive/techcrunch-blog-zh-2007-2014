# 实时网络直播。。。随你怎么称呼，需要一段时间才能理解 TechCrunch

> 原文：<https://web.archive.org/web/https://techcrunch.com/2009/06/30/live-web-real-time-call-it-what-you-will-its-gonna-take-a-while-to-get-it/>

*这篇客座博文的作者是[玛丽·霍德](https://web.archive.org/web/20221006195424/http://www.crunchbase.com/person/Mary-Hodder)，涉足的创始人[。在涉足之前，霍德为许多初创公司提供咨询，在 Technorati 做研究，并在伯克利写了她的硕士论文，重点是查看博客数据的实时网络搜索。](https://web.archive.org/web/20221006195424/http://dabble.com/)*

实时搜索并不是什么新鲜事。这是一个我们已经研究了至少十年的问题，十年后我们可能仍会试图解决它。这是一个非常困难的问题，我们过去称之为" [live web](https://web.archive.org/web/20221006195424/http://blogs.law.harvard.edu/doc/2008/09/26/the-live-web/) search "，这个词是由 [Allen Searls](https://web.archive.org/web/20221006195424/http://globealive.blogspot.com/) ( [Doc 的儿子](https://web.archive.org/web/20221006195424/http://doc-weblogs.com/2003/04/21#theWorldLiveWeb))创造的，指的是在包括搜索在内的所有因素中，以时间为元素的活的 web。

将名称改为“实时搜索”似乎是将注意力重新集中到时间问题上的一种方式，时间是过滤器的一个重要元素。至少在过去十年里，我们仍然面临着同样的问题。Erick Schonfeld 前几天提到的公司中没有一家看起来与之前的在线搜索/发现公司有什么不同。新的似乎都是在问题开始时摸索，事实上似乎是在做“最近的搜索”，而不是真正的实时搜索。虽然我确信他们已经在他们的系统上下了很大的功夫，但是他们并没有比老的实时网络搜索系统更有问题。所有新的都给出了一个相反的编年史观点，其中大多数将 Twitter 与一些东西混合在一起:博客数据、其他微博数据、照片，创建了某种最近趋势的顶级列表。有些是有背景的，比如一段时间内的活动计数，或者一个趋势持续了多长时间，或者 Technorati 和 Sphere 早年都尝试过的直方图(Crowdeye)。或者显示有多少链接指向某个东西，或者有多少推文。所有这些似乎都容易受到垃圾邮件和其他降低用户体验的活动的影响，并且似乎都没有真正提供人们希望看到的上下文和质量过滤器，如果这真的有效的话。所有人似乎都苦于需要学习我们在博客搜索和主题发现中已经学到的经验。

从 1999 年开始，公众可用的出版系统将时间的价值融入到正在出版的内容中(想想 Pyra，它现在是 Blogger、Moveable Type、WordPress 和 Flickr，还有许多其他的)，以及搜索和发现系统，用于那些已经出版的内容，如 Technorati、Sphere、罗约、Blogpulse、Feedster、Pubsub 和其他内容，沿着记忆的道路走下去。。。(顺便说一句，出于披露的目的，我应该声明我在 2004 年为 Technorati 工作了 10 个月，并以这样或那样的形式为几乎所有其他人提供咨询或建议)。

我于 1999 年在加州大学伯克利分校开始研究这个问题，并最终在 SIMS(或现在所知的 iSchool)完成了关于实时网络数据搜索和主题发现的硕士论文。从 2000 年到 2004 年，SIMS 的人会对我说，“你在博客和数据上做什么，这太奇怪了。这有什么关系？”但是时间元素是我在常规搜索中丢失的迷人的部分。它是让事情成为新闻的元素，也是可以在短时间内将项目组合在一起的元素，以显示传统新闻媒体经常忽略的关注点和活动(直到最近他们认为实时网络活动很有趣)。

在 2005 年的火人节上，一个炎热安静的下午，在一个树荫下，我记得和 [Barney Pell](https://web.archive.org/web/20221006195424/http://www.crunchbase.com/person/barney-pell) (他后来创建了 [Powerset](https://web.archive.org/web/20221006195424/http://www.crunchbase.com/company/powerset) )进行了四五个小时的谈话，内容是关于实时网络和实时网络搜索，如何做，这意味着什么，如何理解并向用户展示时间，多少是发现，多少是搜索，你可以获得的数据是如何结构化的，你对数据的依赖程度如何，你可以从这些数据中获得什么意义，等等。谢尔盖·布林(Sergey Brin)坐在那里听着，几个小时后，他问我，“什么是*实时网络*什么是*实时网络搜索*？”因为巴尼和我已经进行了一次深度潜水，我以为谢尔盖知道我们在谈论什么，所以这让我很惊讶，但我解释了为什么我认为时间是常规搜索中一个巨大的缺失元素，这就是我从事的搜索类型。巴尼和我又继续了几个小时。天气变凉了，是时候去欣赏艺术了，就这样结束了。但这些年来，我一直在想，谷歌在实时网络方面处于什么位置，以及随着时间的推移，他们可能会做些什么。Twitter 似乎在刺激他们。

2006 年，史蒂文·利维和布拉德·斯通的《生活网络》新闻周刊封面故事第一次在一个全国性论坛上探讨了这个问题。

当我看到最新一批搜索创业公司时，我想:为什么我们又在以同样的方式做这件事？重新发明轮子？有人在数据或界面方面有什么创新吗？有人在我们之前学过的后端或 UI 的基础上构建吗？

坦白说，我们的过滤器很烂..我想，如果名字的改变让我们重新思考更好的过滤器，那么，我应该感到高兴。我对我们迄今为止糟糕的过滤器负有部分责任，因为在致力于解决这个问题的过程中，我为各种各样的实时网络或实时或任何当下的词语做出了贡献，试图解决这个问题。我们非常擅长发布我们的想法和愿景，带有时间戳，但不太擅长事物的过滤方面。信息搜索和发现的旧方法是打开报纸或杂志，用编辑过滤和放置的信息翻页，当你完成时，你说，“好的，我知道了”(不管你是否真的知道)。但媒体变得自满，错过了故事，随着博客发布和 Flickr 等网站照片发布的便利，我们可以用整个网络取代纸张并补充我们的信息需求。唯一的问题是，这是整个网络。雪崩。我们在网上感到焦虑，因为缺少一两个印刷新闻来源曾经给我们的过滤和编辑的优雅。

我在 2002 年做了一项研究，在 2004 年和 2008 年又重复了一次。我让用户追踪他们一周的在线信息摄入。每项研究中只有 30 人，从 Craiglist 广告中随机选择，但我发现每 30 人一组的平均在线新闻和信息网站时间在 2002 年为 1.25 小时，2004 年为 1.85 小时，2008 年为 2.45 小时。这些人不在硅谷，但他们都有宽带，住在美国。他们中的每一个人都报告了一定程度的焦虑，他们觉得为了获得信息，他们需要接受大量的数据。他们经常通过增加他们保持消息灵通的时间来处理这个问题。他们不知道更好的过滤器实际上可能会减少他们的焦虑。

正如 Erick 指出的，解决这个问题的紧张关系存在于记忆和意识之间；或者像鲍勃·怀曼和[萨利姆·伊斯梅尔在发布会上所说的那样:回顾与前瞻。这是问题的一部分。但是还有更多。](https://web.archive.org/web/20221006195424/http://www.angstro.com/node/58)

发现确实意味着你必须引入时间作为一个元素。不能期望用户知道正在冒泡的是什么，或者将命名最新事物的特定短语。

有些人会说“迈克尔·杰克逊”，有些人会说“MJ”，有些人会说“流行之王”。作为一个话题，迈克尔·杰克逊其实很简单。我记得有一次在 2003 年为一个实时网络搜索和发现系统做可用性测试，我们要求用户在谷歌新闻和各种实时网络系统上搜索在澳大利亚发现“巨型海洋生物”的事件。但由于所有报道它的媒体都来自澳大利亚，他们都称它为“巨大的鱿鱼”，所有后续的美国来源包括博客都抄袭了澳大利亚语，所以最近没有“巨大的海洋生物”的点击量测试人员必须创造性地思考如何获得他们知道的信息，然而这是一个语义上的飞跃。一个搜索测试者因为拒绝放弃而哭了，她如此坚定地要在我们测试的任何一个在线系统中找到结果。我们恳求她停下来；很痛苦。好的发现可能会有所帮助。

发现和实时网络搜索的另一个关键因素是获取结构化数据，因为谷歌用来从网络上获取数据以进行定期追溯网络搜索的蜘蛛搜索使理解已发表作品的时间变得更加困难。如果你只知道你什么时候抓取了页面，那就很难和时间一起工作。另一方面，Twitter 有结构化的数据，因为所有的东西都在他们的筒仓中发布，所以他们的网站提供完整的数据流，以结构化的格式获取数据。他们知道每条推文的时间。更不用说数据可以通过 API 获得。这是引出搜索意义的最有效的方法，因为对于搜索和发现来说，你肯定知道每段数据的上下文，时间是支点之一。

您还需要为后端搜索数据库获得正确的数据模型，以便获得意义和链接指标。你需要理解不同的数据，知道这些东西对用户(而不是工程师)意味着什么，并找出垃圾邮件和不良行为者的问题。有数据的原始上下文，也有用户界面，当试图让时间对许多用户来说是可理解的时候，这是非常困难的。事实上，有些人认为与普通用户交流时间元素是如此困难，以至于进行时间聚焦搜索实际上是一个“高级搜索”问题。

如果设计不当，系统会导致用户不自然地产生扭曲的数据。如果该系统涉及某种权威或受欢迎程度的过滤器，他们就会受到幂律效应的影响(Technorati 称他们的指标为“权威”，但博客的入站链接数不是权威，它们只是受欢迎程度的衡量标准)。什么是[幂律](https://web.archive.org/web/20221006195424/http://en.wikipedia.org/wiki/Power_law)效应？当一个系统驱动活动去非自然地强化最初导致某些东西出现的行为。例如，如果过滤器的一个指标计算点击顶部搜索的人数，那么点击越多，该项目在搜索列表顶部停留的时间就越长，即使它本来会更早从列表中掉出。相反，如果过滤器的指标涉及由想象力驱动的自发行为，如写推文，那么在过滤器顶部显示这些项目可能不太可能推动活动。然而，如果你向用户展示结果，当看到一个热门话题时，他们可能会在看到热门话题之前没有想到就开始发关于这个话题的微博。换句话说，通过揭示你关注的指标，你可以推动用户改变他们的行为。通过驱动行为，幂律分布将一些权力保持在顶端，因为它们在顶端或者可以驱动它们更高。它变成了一个循环。因为没有区分一个单元的质量或强度，也没有区分这个单元对一个主题领域的一群用户意味着什么，所以直接计算数字并不明智。

例如，如果我们制作一个系统来统计 Om Malik 的入站链接，并将其称为权威，不管是什么主题，我想 Om 都会同意，即使是他也不会在例如现代舞或金属加工等主题上有很大的权威和洞察力，如果他碰巧在博客帖子中提到这些词的话。但是在宽带问题上，他绝对是权威。但 Technorati、OneRiot 和其他服务采用指标计数，并将其应用于所有主题、所有环境、所有搜索结果匹配，没有上下文，使用户看到的信息质量随机化。它们可能会在整个网络中提供一个过滤器，但是在判断什么是有用的或无用的方面，它们不会给我们任何真正的帮助。这就是为什么主题社区是有帮助的，一旦你找到一个好的编辑过滤器，在人情味的驱动下，你会非常珍惜它，因为它是这样一个时间和能量的节省者。

我并不幻想我们很快就能解决实时网络、实时搜索甚至最近搜索的问题。我们没有。我们也没有接近解决发现。但我希望我们会。宜早不宜迟。因为我现在就需要。机会是巨大的。这意味着真正用算法构建我们今天以人的形式拥有的编辑过滤器，同时平衡暴民的活动。解决这个问题，奖金会很大。
