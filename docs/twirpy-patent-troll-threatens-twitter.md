# Twirpy 专利巨魔威胁 Twitter

![](img/53175b10e20443b9412346871141e3b3.png)

一个名为 [Kootol 软件](https://web.archive.org/web/20221207003636/http://www.kootol.com/)的专利巨魔已经让[Twitter](https://web.archive.org/web/20221207003636/http://www.crunchbase.com/company/twitter)进入警戒状态。这家“公司”的公司标志(和名字)令人怀疑地联想到谷歌，今天早上[说](https://web.archive.org/web/20221207003636/http://www.businesswire.com/news/home/20110603005342/en/Kootol-Software-Limited-Sends-Caution-Notice-Twitter)它已经向比兹·斯通、杰克·多西和公司发出警告通知，表达对可能侵犯知识产权的“关注”。

正在讨论的专利申请(尚未分配专利号)的标题是“一种用于通信、广告、搜索、共享和动态提供期刊订阅源的方法和系统”

据称，该专利涵盖了“提供动态和实时通信的创新技术”。根据 Kootol 发布的新闻稿，这项“发明”允许人们使用单向或双向消息以及订阅网络其他用户的帖子来发布和发送消息，并为这些用户提供实时搜索。

Kootol 的网站菜单中提到了“产品”和“解决方案”，但需要注意的是 Kootol 目前并没有提供任何产品或解决方案。

他们似乎在寻找投资者来开发类似 Twitter 的竞争对手。只不过不是真正的*，因为这不是那种会建造任何东西的公司。*

他们所谓的发明专利申请也已经在印度、加拿大和欧洲提交，但值得注意的是，Kootol 没有基于其中任何一项对 Twitter 提起诉讼，至少目前没有。

总部位于印度的 Kootol 董事 Yogesh Rathod 声称，他是该专利申请的发明人和所有者，并在一份新闻声明中表示，Kootol 目前正在“审查其地位”。他们表示，向 Twitter 发送警告通知只是为了让初创公司有机会在“最早的阶段”审查专利申请。

基本上，他们试图让 Twitter 许可他们的技术。

Yogesh Rathod 说他总共拥有大约 60 项专利。
