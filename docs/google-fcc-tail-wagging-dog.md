# 谷歌对联邦通信委员会:停止让语音网络尾巴摇互联网狗 TechCrunch

> 原文：<https://web.archive.org/web/https://techcrunch.com/2011/06/20/google-fcc-tail-wagging-dog/>

![](img/a5373e8fe797b6c7dff767d5d15342db.png)

在北美，互联网承载的流量是语音网络的近 160 倍，但许多法规和运营商间的流量费用都是基于语音网络统治的快速倒退时代。谷歌在给 FCC 的一封信中称之为“摇尾巴”，敦促他们不要对 IP 网络征收过时的每分钟语音流量费。随着 IP 语音流量接近传统电路交换语音流量，这正在成为一个问题。谷歌的律师在信中写道:

> 相对于其他形式的通信流量，独立语音流量已经在显著减少；事实上，正如附件中所描述的，在短短几年内，大部分语音流量将是基于 IP 的。因此，FCC 不应该允许相当于遗留语音有线服务的非常小的尾巴摇摆所有通信业务交换的非常大的狗。具体来说，每分钟语音流量的始发和终接费用是电路交换电话时代遗留下来的一个持久但不受欢迎的问题，并不最适合现代 IP 流量和网络。

谷歌用一系列生动的幻灯片展示了网络不断变化的本质。早在 1997 年，美国的互联网流量只有每月 3，300 兆兆字节，而语音网络的流量为每月 54，000 兆字节。三年后的 2000 年，语音流量达到每月 66，000 兆字节的峰值，而互联网流量增长了八倍多，达到每月 28，000 兆字节。

![](img/b8fb378222fc475e546993b0a5210ae7.png)

到 2005 年，消费者 IP 流量已经达到每月 669，000 TB(其中 2 TB 是 IP 语音流量)，而语音流量已经缩减到每月 48，000 TB。

2010 年，北美的消费者 IP 流量完全让语音流量相形见绌，每月 5.7，而老化的语音网络为 36，000 TB。更何况 IP 语音流量(Skype，Google Voice 等。)每月传输 21，000 TB，占旧交换网络传输量的近 60%。

![](img/77ffbe6061a918024662f1ae86cd204b.png)

所有这些网络，包括数据和语音网络，都将走向 IP 网络。谷歌估计，到 2015 年，北美的消费者 IP 流量将增加两倍以上，达到每月 1940 万兆兆字节，而语音网络将进一步缩减至每月 26000 兆字节。IP 语音流量将几乎达到每月 21，000 TB(见上图)。包含所有幻灯片的整封信嵌在下面。
