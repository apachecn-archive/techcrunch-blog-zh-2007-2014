# 谷歌现在使用 ReCAPTCHA 解码街景地址

> 原文：<https://web.archive.org/web/https://techcrunch.com/2012/03/29/google-now-using-recaptcha-to-decode-street-view-addresses/?guccounter=1&guce_referrer=aHR0cHM6Ly93d3cuZ29vZ2xlLmNvbS8&guce_referrer_sig=AQAAACo3u1vXZLHiQSvfo5FMpza6cKENV6RYqvWDJEqQ0DrvTlLpAHV0tdFHzz-6p54jpNhV9OYujnECak2BshMCDxzvMHvMB-kyMTA_ifKwFOAPj4h2A0xU1mP2SJuo2XXCMz3UnN45AnTALygzik1LGTUvTUCqzbB-7MfW3omWBBsA>

你是否已经开始在网上看到那些看起来很像谷歌街景中的门牌号的照片？好吧，事实证明，他们就是这样。谷歌证实，它目前正在进行一项实验，涉及使用 reCAPTCHA 垃圾邮件打击系统，通过让用户识别街道名称和商业地址等内容来改善谷歌地图中的数据。

[对于那些不熟悉的人来说，reCAPTCHA](https://web.archive.org/web/20230327080517/http://www.google.com/recaptcha) 是卡耐基梅隆大学最初开发的系统，旨在改善验证码的使用(又名“区分计算机和人类的完全自动化公共图灵测试”)——这是一种扭曲的文本，旨在阻止机器人注册在线账户。reCAPTCHA 技术是谷歌在 2009 年收购的[，如果你使用网络，你肯定以前用过它。这就是为什么网站上的安全问题要求你识别图片中的单词和数字，以验证你是人类。](https://web.archive.org/web/20230327080517/https://techcrunch.com/2009/09/16/google-acquires-recaptcha-to-power-scanning-for-google-books-and-google-news/)

该系统旨在减少垃圾邮件和欺诈，但它也有助于将书籍和报纸等印刷材料中的文本数字化。例如，谷歌一直在使用 reCAPTCHA 为谷歌图书和谷歌新闻档案数字化内容。

然而，在过去的几天里，[一些](https://web.archive.org/web/20230327080517/https://twitter.com/#!/dcfvg/status/183563332689727488/photo/1) [用户](https://web.archive.org/web/20230327080517/https://skitch.com/mikesten/8cu66/aframe-signup-create-account-utf8-andplan-personal) [已经](https://web.archive.org/web/20230327080517/http://bitinn.net/9466/)看到另一种类型的 reCAPTCHA 出现了——照片。新的 reCAPTCHAs 展示了一幅图像，其中一面包含用户熟悉的扭曲文本，而另一面显示了一张有些模糊(好像放大了)的数字照片。这些数字显然是街道地址，这导致一些猜测，谷歌是从谷歌街景中提取这些数据的。

这种新的 reCAPTCHA 已知会出现在 Google 的 AdWords 网站上，特别是在托管关键字工具的页面上。不过，你不会总是看到这个新的 reCAPTCHA——例如，今天早上我刷新了[这个页面](https://web.archive.org/web/20230327080517/https://adwords.google.com.au/o/Targeting/Explorer?__c=1000000000&__u=1000000000&ideaRequestType=KEYWORD_IDEAS&defaultView=2)十几次，但仍然无法让它出现。您的里程可能会有所不同。

据谷歌发言人称，该系统不仅限于街道地址，还包括街道名称，甚至交通标志。不过，我们在野外还没有发现其他类型的。

谷歌说:

> 我们目前正在进行一项实验，让街景图像中的字符出现在验证码中。我们经常从街景图像中提取街道名称和交通标志等数据，以使用商业地址和位置等有用信息来改进谷歌地图。基于这些 reCaptcha 测试的数据和结果，我们将确定使用图像是否也是一种有效的方法，来进一步完善我们的工具，以打击与机器和机器人相关的在线滥用。

尽管许多用户现在才注意到新图片的出现，但谷歌表示，该实验实际上在几周前就开始了。
