# AppStoreHQ 利用 Twitter 获取 iPhone 应用推荐 TechCrunch

> 原文：<https://web.archive.org/web/https://techcrunch.com/2010/01/21/appstorehq-twitter-app-recommendations/>

已经有很多笔墨详细描述了苹果的应用程序商店是多么的成功，现在已经有超过 10 万个应用程序。然而，如此庞大的应用数量带来了一个挑战:当有如此多的应用可供选择时，很难找到真正有趣的新应用。苹果尽最大努力在商店中突出高质量的应用程序，它提供了一个天才的功能，但这往往会留下一些需要的东西。 [AppStoreHQ](https://web.archive.org/web/20221006104925/http://www.appstorehq.com/) 是一个利用新的数据来源 Twitter 来帮助推荐应用的网站。该网站根据谁在推特上发布了关于 iPhone 应用程序的建议，并可以根据你自己的推特为你提供建议。

该网站查看 Twitter 的公共流，检查推文中返回到应用商店的链接。每当有人链接到一个应用程序，他们就认为这是用户的投票。当你访问该网站的主页时，你可以浏览目前在 Twitter 或 web 上最热门的应用程序(后者是由博客文章提到最多的应用程序决定的)。或者，如果你想要个性化的推荐，你可以在推特上发布一两个你喜欢的应用。有一个问题是，你必须在你的推文中包含几个关键词中的一个，如“iPhone”或“AppStoreHQ ”,否则系统不会捕捉到它。该服务只解析带有这些关键字的链接，因为跟踪每个短 URL 和链接以查看它们是否链接到 iPhone 应用程序将需要惊人的处理能力。

一旦你在推特上发布了一些应用程序，系统就会根据其他类似用户发布的内容提出定制建议。显然，你发的应用越多，推荐就越好(这也是服务病毒式增长的一个好方法)。该系统还考虑了其他一些数据来源，包括你在 AppStoreHQ 上喜欢的任何应用程序，以及你在浏览网站时实际点击过的应用程序。

首席执行官克里斯·德沃尔表示，虽然这项服务可以开发一个情感引擎，以确保推文中链接的应用程序确实得到了积极的评论，但人们很少会在推文中链接到他们不喜欢的应用程序，所以他们不会进行这种处理。

我喜欢使用推文进行应用推荐的想法——写推文不像写博客那么复杂，所以网站可能能够获得更多数据进行分析，而且它还能够快速展示热门的新应用。也就是说，该算法还可能不得不忍受来自 Twitter 的大量垃圾邮件。我认为 AppStoreHQ 的布局还需要改进，因为它的菜单系统看起来笨拙而乏味。

AppStoreHQ 实际上已经存在了一段时间，它根据博客帖子中的应用链接提供推荐。它还有一个标有白色标签的 iPhone 应用搜索引擎，目前集成在许多其他流行的 iPhone 应用新闻和社区网站中。

我们已经介绍了许多其他的应用商店推荐服务，包括[合唱团](https://web.archive.org/web/20221006104925/http://www.beta.techcrunch.com/2009/11/03/chorus-delivers-customized-app-recommendations-to-your-iphone/)、 [Appolicious](https://web.archive.org/web/20221006104925/http://www.beta.techcrunch.com/2009/12/21/appolicious-now-delivers-personalized-app-recommendations-to-the-iphone/) 、 [AppsFire](https://web.archive.org/web/20221006104925/http://www.beta.techcrunch.com/2009/06/30/recommend-your-favorite-iphone-apps-with-appsfire/) 和 [Appsaurus](https://web.archive.org/web/20221006104925/http://www.beta.techcrunch.com/2009/12/30/appsaurus-iphone-genius/) 。
