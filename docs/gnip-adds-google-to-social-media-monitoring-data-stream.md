# Gnip 将 Google+加入社交媒体监控数据流

向企业应用程序提供社交媒体数据的 Gnip 今天宣布，它将把来自 Google+的数据整合到其流中。Gnip 将为 Google+提供一个基于[社交平台新 API 的关键词搜索流。](https://web.archive.org/web/20230203073642/https://techcrunch.com/2011/09/15/google-plus-api-2/)

作为背景，Gnip 是社交流的 API 枢纽，从 Twitter、脸书和其他社交来源收集数据，并将其推送到其他数据消费应用程序和网站。使用 Gnip 平台的应用程序可以获得 100 多个订阅源和站点的公共数据流，包括 Twitter、脸书、YouTube、WordPress、Flickr 等，而无需访问这些站点或访问它们各自的 API。Gnip 也[有额外的权限](https://web.archive.org/web/20230203073642/http://allthingsd.com/20101118/twitter-partner-gnip-raises-2m-for-social-media-monitoring-data/)访问 Twitter 的数据。

Gnip 总裁、首席运营官的克里斯·穆迪(Chris Moody)说，将 Google+数据添加到数据流中完全是基于客户的需求。“根据社交对话做出商业决策的大公司以及社交媒体监控公司一直要求将此作为对脸书和 Twitter 其他信息的补充。

使用 Google+的关键字搜索流，客户选择相关的术语和短语，Gnip 使用这些术语和短语从 Google+提供匹配的公共活动、评论、照片、URL 和视频数据。Gnip 的 Google+关键字搜索流带有该初创公司的标准元数据丰富内容，包括格式标准化、重复数据删除和 URL 展开。

穆迪解释说，关键的一点是，企业实际上想要监控 Google+流。他说，拥有 4000 万用户，“这是一个有价值的来源。”
