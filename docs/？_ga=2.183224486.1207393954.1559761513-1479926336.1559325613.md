# Looker 从第一轮和 PivotNorth 中获得$2M，为商业智能| TechCrunch 构建“SQL 续集”

> 原文：<https://web.archive.org/web/https://techcrunch.com/2013/03/06/looker-takes-2m-from-first-round-and-pivotnorth-to-build-a-sequel-to-sql-for-business-intelligence/?_ga=2.183224486.1207393954.1559761513-1479926336.1559325613>

[Looker Data Sciences](https://web.archive.org/web/20230315095315/http://www.looker.com/) 是一家商业智能初创公司，由 Netscape 和 LiveOps 的一名早期首席工程师创立，如今正摆脱隐形模式，并宣布获得首轮资本和 PivotNorth 的 200 万美元资金，以基于一种更易于使用的新 SQL 方法 LookML 建立其业务，该方法被称为“SQL 的续集”

总部位于圣克鲁斯的 Looker 已经有 20 家公司在其平台上，它是我们已经观察了一段时间的一种趋势的一部分，这种趋势是一类新的企业初创公司，它们希望用一种新的方法来打破现状，将商业服务带到一个更常见的领域。

在这种情况下，这种基础与商业智能软件和大数据有关，利用企业已经收集的信息，帮助企业在未来如何经营企业方面做出更好的决策。

用创建 Looker 的前 Netscape/LiveOpps 工程师 Lloyd Tabb 的话说，迄今为止，商业智能一直是一个笨重的系统。大多数系统都是基于 SQL 的，并且需要用户具备工程和编程方面的专业知识来制定查询来询问数据。

Looker 通过其专有的 Looker BI 平台和 LookML 语言，将过去的编程查询转变为基于自然语言的查询，从而使企业中的任何人都可以进行深度分析，包括工程师和客户经理等。这也扩大了对商业智能的访问，超越了大型企业，进入了较小的组织，这些组织在过去可能没有资源来雇佣数据分析师来管理和运行大型数据库。事实上，Tabb 表示，Looker 的一些早期用户是硅谷规模较小的初创公司，员工不到 40 人。他们包括移动广告初创公司 kiip，只是被雇佣和威胁。

Tabb 指出，当一家公司与 Looker 签约时，该公司目前会使用 LookML 语言帮助建立一个适合你自己业务的初始模型。然后，这成为公司自己如何构建未来查询的基础。

他说，Looker 的想法最初是通过他与小型初创公司合作的许多经验产生的，这些公司建立在基于数据做出商业决策的基础上，随着组织的增长，这变得越来越具有挑战性。“多年来，我一直在思考如何处理数据的问题，”他在接受 TechCrunch 采访时说。“我意识到，将数据送到业务人员手中是一家公司最大的成功因素之一。”

下面是 Looker 帮助简化的查询类型的一个例子。以一家从五六家供应商那里购买某种特定产品的企业为例。他说，为了能够查看这些供应商在销售方面的表现，每次数据库查询都需要由程序员编写，以便报告数据。塔布解释说，Looker 允许一个人——任何人——创建一个覆盖整个供应商群体的查询，一个查询“需要 10 秒钟来编写”。这也意味着更接近特定业务问题的人，例如，与这些供应商打交道的买方，应该能够更快地制定和获得问题的答案。这里的想法是，许多企业都在用 SQL 积累数据，但关键是要弄清楚如何以一种简单的方式打开它。

Tabb 表示，在其早期实施中，大约一半的客户定期使用 Looker，有些人每天使用长达 20 小时，“很多人都在问问题，”他说。
