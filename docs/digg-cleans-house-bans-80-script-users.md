# Digg 清理门户，禁止 80+脚本用户

![](img/831a661aab3004715bebaf1d06ff4007.png)

有报道称，Digg 已经永久禁止 80 多名用户运行帮助他们在网站上自动执行某些任务的脚本。

这篇冗长的报道发表在 Get Smart 博客上，标题是[死神访问了 Digg](https://web.archive.org/web/20221208232843/http://www.thegetsmartblog.com/2008/09/the-grim-reaper-has-visited-digg/) 。帖子列出了 86 个用户名，这些用户名在没有预先警告的情况下被随意删除。作者和他的评论者，所有显然是 Digg 用户(或前 Digg 用户)自己的语气是戏剧性的:“这么多勇敢和勇敢的挖掘者…看到这样浪费人才是一个无法形容的悲剧。”

被封禁的用户中有一个叫 Diggboss 的用户，[开发了](https://web.archive.org/web/20221208232843/http://shegeeks.net/was-digg-right-to-ban-diggboss-for-his-helpful-script/)一个油滑的猴子脚本，用来检查朋友，看他们是否已经偷了你提交的或者喊过的东西。该脚本使用了 Digg 自己的 API，[没有自动挖掘任何故事](https://web.archive.org/web/20221208232843/http://pulse2.com/2008/09/18/upcoming-discussion-on-massive-digg-user-ban-for-running-scripts-built-on-digg-apis-the-drill-down/)，然而 Digg 的[使用条款](https://web.archive.org/web/20221208232843/http://digg.com/tos)含糊地禁止“自动访问网站”和任何“以任何方式人为改变 Digg 服务结果的有组织的努力”

在上周的一篇官方博客文章中，来自 Digg 的 Jen Burton 认为脚本是被禁止的，因为它们“给 Digg 服务器增加了额外的负载(降低了每个人的速度)”。然而，很明显，Digg 也关心防止用户通过招募他们的朋友来玩游戏。Duggboss 脚本可能没有自动提交故事，但它确实帮助用户追求一种为主页点击量而挠背的策略(这种策略使马克·库班的亲兄弟陷入困境)。

许多用户反对 Digg 解雇长期贡献者，理由是他们运行了脚本。一些人认为 Digg 给用户跳槽的理由是搬起石头砸自己的脚:

> 剧本？对此我要说的是世卫组织在乎！Digg 在流量和使用量上做的好吗？是啊！如果你问我，如果脚本能让人们更快地使用它，这个网站可能会更容易使用一点…..就这样吧！这只会对 Digg 有利，你会认为他们会理解这一点。看看接下来几个月会发生什么将会非常有趣。Digg 可能会变成一座鬼城，而 Yahoo Buzz 可能会成为下一个大师。

但那是[没什么新鲜的](https://web.archive.org/web/20221208232843/http://www.beta.techcrunch.com/2007/11/24/digg-refugees-may-be-heading-to-mixx/)。
