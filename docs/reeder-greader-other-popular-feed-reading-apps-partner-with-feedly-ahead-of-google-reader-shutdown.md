# 在谷歌阅读器关闭之前，Reeder、gReader 和其他流行的 Feed 阅读应用程序与 Feedly 合作

据 Digg.com 公司的倒计时器称，不到一个月，谷歌阅读器将永远关闭，该公司目前正在建造替代品。对于最终用户来说，这是一个简单、快速和强大的浏览新闻的前端，但对于在 RSS 生态系统中开发应用程序的开发者来说，这意味着更糟糕的事情:对 Reader API 的访问的结束。今天， [Feedly](https://web.archive.org/web/20221231062102/http://www.feedly.com/) 提供了另一种选择。

随着[最近成为更严重的竞争者之一](https://web.archive.org/web/20221231062102/https://techcrunch.com/2013/03/22/feedly-winning-on-mobile/)来接替谷歌阅读器留下的位置，该公司已经宣布与几家 RSS 应用程序开发商合作，包括 [Reeder](https://web.archive.org/web/20221231062102/http://reederapp.com/) 、 [Press](https://web.archive.org/web/20221231062102/http://twentyfivesquares.com/press) 、 [Nextgen Reader](https://web.archive.org/web/20221231062102/http://nextmatters.com/) 、 [Newsify](https://web.archive.org/web/20221231062102/http://newsify.co/) 和 [gReader](https://web.archive.org/web/20221231062102/http://www.greader.co/) ，它们将在谷歌阅读器 API 关闭之前支持 Feedly API。对于依赖这些应用的最终用户来说，这意味着应用制造商将能够在谷歌阅读器即将关闭之前无缝地将他们的用户转移到一个新的后端。

该 API 是 Feedly 的[“诺曼底”项目](https://web.archive.org/web/20221231062102/http://blog.feedly.com/2013/03/14/google-reader/)的一部分，该项目专注于克隆阅读器 API，目前正在谷歌的应用引擎平台上运行。Normandy 后端还将为 Feedly 自己的应用生态系统提供动力，包括其 iOS、Android、Chrome、Safari 和 Firefox 的应用。当 Normandy 首次亮相时，该公司回应了 100 多个开发者的请求，但只有上面列出的那些应用程序开发者收到了最初的邀请。第二批正在进行中。

除了 API 合作伙伴的消息，Feedly 还提供了一个简短的进展更新，详细介绍了来自其支持论坛上用户的主要请求。该公司表示，它已经有了新的开发和运营人员来帮助它专注于提高 Feedly 的速度，这是一个最高要求。用户还要求更好的搜索、网络版、对 Windows 8 和 Windows Phone 的支持、改进的群组共享等等。

该公司还吹捧其面向出版商的工具和面向网站的按钮，这表明这家初创公司的意图不仅仅是替代在线阅读器，而是用必要的工具帮助重新播种 RSS 生态系统。

自谷歌宣布以来，一些替代品已经开始填补空白，包括创业公司，如 [NewsBlur](https://web.archive.org/web/20221231062102/http://www.newsblur.com/) 、 [NetNewsWire](https://web.archive.org/web/20221231062102/http://blackpixel.com/blog/2013/03/the-return-of-netnewswire.html) 、 [The Old Reader](https://web.archive.org/web/20221231062102/http://theoldreader.com/) 、 [Feedbin](https://web.archive.org/web/20221231062102/https://feedbin.me/) 、 [Digg](https://web.archive.org/web/20221231062102/http://www.digg.com/reader) 等等。然而，许多公司只有一个在线界面，而没有一系列应用程序、API 和其他工具来替代谷歌阅读器曾经提供的功能。然而，Digg 现在得到了 betaworks 的支持，并且可能有资源提供服务。在更广泛的测试之前，它已经将代码转移到生产服务器上。但 Feedly 在这一领域的时间更长，所以它有一个很大的领先优势。

几周前，Feedly 宣布，自谷歌宣布关闭阅读器以来，已有 300 万用户加入。据说现在尝试这项服务的用户中有 68%转化为每周活跃用户。
