# 脸书通过抛弃 HTML5 并像 iOS 版本一样原生重建来加速 Android 应用

脸书的 HTML5 应用噩梦结束了。[今天晚些时候](https://web.archive.org/web/20230111142505/http://newsroom.fb.com/News/554/Facebook-for-Android-is-Faster-than-Ever)它发布了用于 Android 2.0 的[脸书，它用全原生基础设施取代了混合原生/webview 代码，通过使应用程序更快来回应多年的抱怨。在此之前，](https://web.archive.org/web/20230111142505/https://play.google.com/store/apps/details?id=com.facebook.katana)[8 月份的速度加倍 iOS 更新](https://web.archive.org/web/20230111142505/https://techcrunch.com/2012/08/23/facebook-for-ios-faster/)极大地改善了用户评论。这是脸书真正转型为移动优先公司的一年的恰当结束。

HTML5 恐怖故事是这样开始的。脸书正在寻找一种方法来减轻移动团队的压力。因此，它改变了其原生的 iOS 和 Android 应用程序，部分变成了围绕其 HTML5 移动网站的 webviews。这使得它可以一次推送代码来更新 m.facebook.com，同时更新两个应用程序。这也让脸书能够快速地对部分移动用户进行小规模测试，而无需强迫他们下载新版本。

当公司规模稍小，手机还不流行时，这是个不错的主意。但随着智能手机的普及和脸书用户数量的增加，敏捷不再那么重要了。它需要稳定性和一流的用户体验。它所拥有的是极其缓慢的应用程序，在屏幕和无反应的控制之间有很长的加载时间。CEO 马克·扎克伯格后来称依赖 HTML5 是脸书迄今为止最大的错误之一。

因此，九个月前，它做出了一个艰难的决定。它会走回头路，原生重建其移动应用，并专注于速度。八月份发布的 iOS 更新获得了好评如潮……或者至少没有之前那么狂热。其[应用商店评分在 3 周内从 1.5 星飙升至 4 星](https://web.archive.org/web/20230111142505/https://techcrunch.com/2012/09/13/facebook-for-ios-review/)。脸书承诺将为安卓用户提供同样的服务。

这就把我们带到了今天。【Android 2.0 的脸书将在接下来的几个小时内登陆 Google Play 商店。通过[原生重组应用](https://web.archive.org/web/20230111142505/https://www.facebook.com/notes/facebook-engineering/under-the-hood-rebuilding-facebook-for-android/10151189598933920)，应用启动更快；通知和时间线加载速度更快；由于脸书新的定制 ListView 回收器，新闻提要的故事显示得更快，滚动更流畅。由于有一个下拉横幅，你会收到朋友新发表的故事的提醒。照片现在几乎可以立即打开，因为它们被加载到设备的本机内存中，而不是 Java 堆中。

脸书解释说，这一更新“为脸书安卓应用的发展奠定了坚实的基础。现有的基础设施将继续让我们让应用程序变得更快、更流畅、功能更丰富。”

它并没有完全放弃 HTML5。脸书的移动网站仍将在 HTML5 上运行。它还将继续推动移动浏览器供应商提供更好的 HTML5 标准，希望有一天能在该协议上做得更多。

对于已经忍受了几个月的脸书安卓用户来说，今天应该感觉像是一个节日(就像一个人可以忍受十秒钟的延迟来看你在世界各地的朋友在做什么)。更快的速度意味着更少的用户挫折感，从而导致更长的会话时间。这意味着更多的内容被分享，更多的广告被观看。

速度的提高也将有助于脸书淘汰 Twitter 等轻量级应用和 Path 等移动设计杰作。不过，最重要的是，更新会让人们开心。在咖啡馆排队或在公交车上度过的短暂自由时光现在可以被一些脸书浏览所占据。虽然很微妙，但只要知道你不会花额外的时间盯着加载屏幕，就会让你更频繁地打开应用程序。

【Android 2.0 版脸书将在 Google Play 商店发售，可能是现在，也可能是未来几个小时，这取决于它推出的速度。
