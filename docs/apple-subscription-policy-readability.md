# 可读性与苹果再次尝试——订阅政策已经改变了吗？

http://twitter.com/#!/可读性/状态/40517950431559680

前几天，[可读性](https://web.archive.org/web/20230203002643/http://readability.com/)被[惹毛了](https://web.archive.org/web/20230203002643/https://techcrunch.com/2011/02/21/readability-app-rejection/)。在一封给苹果的[公开信](https://web.archive.org/web/20230203002643/http://blog.readability.com/2011/02/an-open-letter-to-apple/)中，他们指出新的订阅政策“有点贪婪的味道”。他们威胁要放弃苹果的平台，转而支持开放网络。这一信息引起了许多人的共鸣，因为开发者社区对苹果的新政策感到非常不安。但是现在，仅仅两天后，Readability 已经[重新将他们的应用程序](https://web.archive.org/web/20230203002643/http://twitter.com/#!/readability/statuses/40517950431559680)提交给苹果进行审查。显然，一封写给该公司的“情书”即将出炉。

那么，为什么会从战争转向和平呢？苹果的政策已经变了吗？嗯，没有——至少现在还没有。Readability 的理查德·齐亚德说他的推特是一个“笑话”,并说:

> 我们确实重新向苹果提交了一份解释，说明为什么我们认为他们应该批准可读性。我们没有和苹果的任何人说过话。我们不知道他们是否会批准。我们只是在上诉过程中尽最大努力解释自己。

所以还没有好消息，但可能有希望。

可读性最初爆发的原因是因为苹果拒绝了他们的应用程序，理由是它没有使用他们的应用内购买系统——该系统要求他们将 30%的收入交给苹果。该公司认为这是不公平的，也没有太大的意义，因为他们认为自己是一家软件即服务(SaaS)公司，实际上并不出售内容，而是一种服务。

这很重要，因为在此事件后不久，一封据称是史蒂夫·乔布斯发送的电子邮件声称，订阅规则是针对“出版应用，而不是 SaaS 应用”。

这是一个灰色地带，因为可读性确实提供了一些内容，但这不是他们的内容，而是重新设计的内容。尽管如此，苹果抽取他们收入的 30%会彻底改变他们的商业模式，并可能迫使他们关闭，或者干脆不做 iPhone 应用。

苹果如何对待这款应用程序，乔布斯后的评论将很大程度上说明他们对未来政策的意图。正如齐亚德所说，“敬请期待！”
