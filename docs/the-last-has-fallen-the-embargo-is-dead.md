# 最后一个倒下了。禁运已经结束。TechCrunch

> 原文：<https://web.archive.org/web/https://techcrunch.com/2009/09/23/the-last-has-fallen-the-embargo-is-dead/>

去年年底，我在 TechCrunch 上宣布了一项新政策——我们不做禁运。好吧，它比那稍微复杂一点，并且被设计在公共关系队伍中激起混乱。我们说我们会打破所有的禁运，我们还说我们会尊重对独家和一些精选公司的禁运，特别是微软和谷歌，因为它们被证明是可靠的。总的来说，我们是想混淆视听，事实也的确如此。

禁运新闻，如果你不熟悉这个术语的话:一家公司想要宣布新闻，比如产品发布会或新的融资。他们向许多媒体简要说明新闻发布的日期和时间。出版社同意在那之前不写。但是一般来说，有人会很早就去，用一个很好的借口，比如时间戳软件问题，然后每个人都会发布消息。无论是谁首先披露了这个故事，通常都会比其他人获得更多的眼球和关注，所以有很多犯错的动机。特别是因为没有人惩罚罪犯。

很多人说我们的新政策会导致 TechCrunch 的死亡。从那以后，我们的读者数量和页面浏览量翻了一倍多，所以事后来看，我不同意。但有趣的是，自那以后，科技新闻领域的禁运文化基本上已经崩溃了。混沌规则，连曾经伟大的微软和谷歌都倒下了。

这对读者来说是好事。

今年早些时候，华尔街日报[也对](https://web.archive.org/web/20230216121756/http://racetalkblog.com/2009/04/09/wall-street-journal-follows-techcrunch-no-longer-honoring-embargoes/)实施了不禁运政策，除非他们获得独家报道，这反映了我们的立场。人们吓坏了。6 月，微软对微软 Hohm 的禁令提前解除(这是 VentureBeat，一个惊人的惯犯——去年他们曾在一周内解除所有禁令，然后声称这是夏令时问题或其他问题)。

这使得谷歌成为唯一一家有能力迫使媒体遵守禁令的公司。今天早上，随着谷歌 Sidewiki，谷歌跌跌撞撞。不幸的是，坚持禁运的 PaidContent 在产品上线前几个小时就已经出发了。昨晚我浪费了三个小时测试和[写关于](https://web.archive.org/web/20230216121756/https://techcrunch.com/2009/09/23/google-steps-where-many-have-stumbled-sidewiki/)那个产品的文章，所以你可以想象我在混乱的新闻中的快乐被我懒得等待我的帖子的沮丧所冲淡。我应该在凌晨 2:30 发表，那时我已经结束了。

随着谷歌和微软不再能够实施禁运，除了放弃整个做法，真的没有什么可做的了。我个人对此很高兴。

**更新:** VentureBeat 创始人马特·马歇尔在下面的评论中回应道:

*   “迈克，在你的报道中，关于 VentureBeat 是惯犯的评论是不公平的。正如我当时告诉你的，WordPress——我们使用的软件——不支持反映夏令时，所以我们在不知情的情况下违反了几天的禁令(我们计划在晚上 9 点发布故事，但它会在晚上 8 点发布)。一句话:VentureBeat 仍然坚持禁运。嗯，与此同时，这完全是一场误会。”
