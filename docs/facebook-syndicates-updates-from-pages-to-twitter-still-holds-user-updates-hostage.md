# 脸书辛迪加从网页到推特的更新，仍然挟持用户更新

一缕光线开始从脸书的信息黑洞中闪耀出来。该网站刚刚[发布了](https://web.archive.org/web/20221006104310/http://blog.facebook.com/blog.php?post=123006872130)一项功能，允许脸书页面所有者将他们从脸书到 Twitter 的更新整合在一起——这是用户多年来一直要求的。

这听起来可能不多，但这是一件大事。到目前为止，将 Twitter 更新导入脸书还是很容易的，因为有各种各样的脸书应用程序可以让你在不同程度上控制它。但这一直是一条单行道——没有简单的方法将脸书的更新转移到 Twitter 上。今天，这种情况发生了变化。页面管理者将能够分享来自他们页面的所有更新，或者他们可以挑选他们想在 Twitter 上分享的内容类型。

当然，新功能中有一个重大遗漏:你不能将你的脸书个人资料更新到 Twitter 上。实际上，唯一从中受益的人是名人、品牌和其他维护脸书页面的用户——这是一般脸书会员不会做的。那么为什么不对所有人开放呢？

![](img/22f863986c0e547d5a2c9c3c2d81f1b4.png)

脸书说了很多关于其“开放性”的话，但是开放和数据可移植性的倡导者长期以来一直批评该网站是一个信息黑洞。当然，你可以使用脸书连接将一些数据导入第三方网站，但这些连接是短暂且有限的。

有几个原因可以解释为什么脸书会坚持向所有用户开放这项服务。其中之一就是目前由 Twitter 主导的实时搜索竞赛。脸书也希望在这一行动中分得一杯羹——它最近[改进了](https://web.archive.org/web/20221006104310/http://www.beta.techcrunch.com/2009/08/10/facebook-flips-the-switch-on-real-time-search-goes-after-twitter-where-it-hurts/)的搜索功能，不久[将鼓励](https://web.archive.org/web/20221006104310/http://www.beta.techcrunch.com/2009/07/01/live-blog-the-facebook-privacy-conference-call/)用户与“[每个人](https://web.archive.org/web/20221006104310/http://www.beta.techcrunch.com/2009/06/24/facebook-brings-privacy-controls-to-publisher/)”分享他们的更新，这意味着更新不会仅限于你社交图谱中的用户。Twitter 可能在这场竞赛中领先，但即使脸书 2.5 亿多用户中的一小部分开始与“每个人”分享，他们也很可能超过 Twitter，成为事实上的实时搜索网站。然而，如果脸书用户也将他们的状态更新整合到 Twitter 上，这种优势将会丧失。

另一方面，如果脸书最终允许用户将他们的状态更新导出到 Twitter，这实际上会增加脸书的访客数量——每次你分享新的相册或视频，它都会被托管在脸书而不是 TwitPic 这样的网站上。更进一步，如果脸书确实允许用户整合更新，同时更容易地导入他们朋友的 Twitter 更新，就像 FriendFeed 做的那样，脸书本身可能会成为一个非常受欢迎的 Twitter 客户端——一个允许在线评论、“喜欢”和其他细节的客户端，这可能有助于它实际上从 Twitter 窃取对话。关于这个想法的更多信息，请参见 MG Siegler 的这篇文章。
