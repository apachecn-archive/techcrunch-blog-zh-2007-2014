# GoDaddy 说 Crash 不是匿名的，它不是黑客，它不是 DDoS。这是内部网络问题

GoDaddy 已经证实，昨天导致[数千个网站一天大部分时间](https://web.archive.org/web/20230308145806/https://techcrunch.com/2012/09/10/godaddy-outage-takes-down-millions-of-sites/)瘫痪的 DNS 问题现在已经解决。虽然黑客行动主义团体 Anonymous 的一名所谓成员声称对昨天的情况负责，但 GoDaddy 表示，这不是外部网络攻击造成的，而是“一系列破坏路由器数据表的内部网络事件。”

该公司临时首席执行官斯科特·瓦格纳(Scott Wagner)在一份声明中进一步指出:“一旦发现问题，我们就为我们的客户和 GoDaddy.com 采取了纠正措施。”该公司表示，它已经建立了一些系统来防止这种情况再次发生。

正如该公司昨天下午在[指出的那样，在此过程中没有客户数据被泄露。](https://web.archive.org/web/20230308145806/https://techcrunch.com/2012/09/10/godaddy-looks-like-its-back-up-company-yet-to-confirm-why-it-went-down/)

昨天，一个自称匿名所有者的人声称对这些攻击负责。他甚至在昨天说，当情况已经明朗，经济正在复苏的时候，他很快就会减少他的攻击。

GoDaddy 网络中的这种不稳定性(也许只是巧合)与组织高层的不稳定性有着不幸的关联。7 月底，Wagner 临时担任 GoDaddy 的首席执行官，此前首席执行官发生了动荡，10 个月前，“杀大象”首席执行官鲍伯·帕森斯下台，由沃伦·阿德尔曼接任，仅 8 个月。

Wagner 在这篇笔记中写道，GoDaddy 到目前为止已经对其 DNS 基础设施提供了 99.999%的更新。“我们让客户失望了，我们知道这一点。”

道歉可能不足以说服一些人。正如我们昨天报道的，Asana 的达斯汀·莫斯科维茨在 Quora 上写道，停工期是对他的公司最终从 GoDaddy 转移的刺激。虽然 GoDaddy 说这不是黑客攻击，但一些人似乎怀疑这一说法。一个例子:
