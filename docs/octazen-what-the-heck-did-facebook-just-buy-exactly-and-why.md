# 奥克塔森:脸书到底买了什么，为什么？

> 原文：<https://web.archive.org/web/https://techcrunch.com/2010/02/19/octazen-what-the-heck-did-facebook-just-buy-exactly-and-why/>

脸书收购了它的第三家公司，马来西亚创业公司 [Octazen Solutions](https://web.archive.org/web/20230404142459/http://www.crunchbase.com/company/octazen) 。据 GigaOm 报道，脸书称这在很大程度上是一种人才引进。Octazen 在其主页上有一个稍微不同的故事，称脸书收购了*“公司的大部分资产，并将这些资产用于不同的方向。”*

不管怎样，这让一些人挠头。今晚，我们采访了一家竞争公司的高级工程师，他说:“脸书刚刚买下了网络上最有才华和创造力的抓取工具，它们绕过了每个人的速率限制和检测系统。”我们今晚采访的另一位了解 Octazen 产品的人说，*“脸书在通过脸书互联保护他们自己的用户数据方面是如此的道貌岸然，但 Octazen 多年来一直在从服务条款中收集用户数据，然后转售。”两个消息来源都要求保持匿名。*

脸书方面尚未对我们的置评请求做出回应。

奥克塔森到底在忙什么？该公司主要是将联系人从一个服务导入到另一个服务——例如，从脸书登录 Gmail，将你的联系人导入到那里，并添加他们为脸书好友。大部分工作都是通过 OAuth 和 API 完成的，但是 Octazen 可以更深入地获取数据。

例如，Octazen 有时会直接收集和存储用户凭证，并登录大型社交网络和其他网站，就好像他们是用户一样，比如多个来源。然后他们会下载通讯录和社交图。你在那项服务上的一部分朋友可能是向 Octazen 付费的服务(现在的脸书)的用户，你会被要求加他们为好友。但还有一个大问题是，其余的数据会发生什么，以及 Octazen 是否违反服务条款存储了一个影子社交网络，以推荐用户连接。他们可能会更深入地查看数据，例如，查看电子邮件标题信息，以更好地了解你与谁交流最多。

但 Octazen 最令人不安的部分，我们的消息来源说，是他们非常非常擅长在不被发现的情况下大规模收集数据。例如，他们可能会使用大量不同的 IP 地址来攻击某项服务，而不被发现。他们说，Octazen 可以抓取像 Twitter 这样非常公开的网站，在那里社交图出现在每个个人资料上，以一种 Twitter 不会知道的方式。

例如，在 2007 年，人们购买并运行 [Octazen 脚本，以一种非常粗略的方式收集联系人](https://web.archive.org/web/20230404142459/http://lindner.livejournal.com/12948.html):*“所以我们使用 Octazen 的这个工具包从各个网站收集联系人列表。我们曾经热切的用户(ab)如此频繁地使用这一功能，以至于 hotmail 屏蔽了我们。”*海报找到了访问 Hotmail 的 API 的方法，而不是仅仅通过抓取来获取数据，Octazen 回应道*“确实很好”*

我们的理解是，脸书已经使用 Octazen 神秘地确定了你失散多年的朋友，并建议你重新与他们联系(导致我们的收件箱收到大量电子邮件，脸书不知何故在阅读电子邮件或以其他方式获取他们不应该得到的数据)。

最大的问题是，如果脸书所做的只是通过 OAuth 和 API 导入标准的地址簿，或者像脸书使用的专有但记录良好的协议，为什么他们需要收购一家位于世界另一端的公司。言下之意是，这些家伙在大规模数据收集方面非常专业，有时可能会违反被收集网站的服务条款。

这显然只是一个可能的故事的一面，尽管是基于 Octazen 以前不光彩行为的确凿证据和通过多种渠道。但在脸书更详细地解释这次收购之前，我们没有太多的进展。

**更新:**拉里·于在脸书报道:

作为背景，超过 30%的用户目前依赖 Hotmail 以外的电子邮件提供商，雅虎！，AOL 或 Gmail 作为他们进入脸书的认证方式，寻找网站上的朋友并邀请新朋友加入。

随着我们在全球范围内寻求增长，这一比例只会增加。所以我们请来了 Octazen 工程师，因为他们非常擅长为这些包含“长尾”的替代服务建立联系进口商。这很重要，因为新用户可以很快加入，因为他们可以找到并邀请朋友。我们打算让他们专注于建立这些“长尾”进口商，并改进我们现有的解决方案。

Octazen 不会直接收集和存储用户凭证以方便联系人导入。他们的大多数客户将软件安装在自己的服务器上，因此 Octazen 在这些情况下无法访问任何用户数据。有一种按需服务，联系人导入过程将通过 Octazen 的机器运行。但对于这种按需服务，他们只存储用户名和用户密码的单向散列。这些存储的数据不能用于代表用户登录。它仅用于审计目的，因此他们可以根据成功的导入而不是尝试对客户端进行计费。
