# 雷·奥茨的秘密初创公司 Talko 正在开发一款面向消费者的移动通信应用，而不是基于云的移动后端技术

> 原文：<https://web.archive.org/web/https://techcrunch.com/2013/03/15/ray-ozzies-stealthy-startup-talko-is-building-a-consumer-facing-mobile-communications-app-not-a-cloud-based-mobile-backend/>

去年 12 月，有消息透露，前微软高管雷·奥茨已经为一家名为 [Talko](https://web.archive.org/web/20221130165849/http://talko.com/) 的秘密初创公司筹集了 400 万美元，该公司之前被称为 Cocomo。人们对该公司的计划知之甚少，但最初的猜测是 Talko 正在建立基于云的后端服务。然而，我们现在听说新服务的一个关键组成部分实际上是一个面向消费者的移动通信应用程序。

众所周知，奥齐对自己的计划讳莫如深([网站实际上是一个空白页](https://web.archive.org/web/20221130165849/http://talko.com/))。早在 2012 年 3 月，他就曾向 CNET 谈及他对基于云的后端的兴趣，这导致了多篇报道称 Talko，当时的 Cocomo，正在致力于开发这一领域的东西。诚然，基于云的后端很可能是 Ozzie 和他的团队正在努力的一个组成部分，但重点不一定是直接进入 MBaaS(移动后端即服务)领域。

Talko 的联合创始人 Ransom Richardson 最近在亚马逊网络服务会议上发言，根据 GigaOm 的[报道，引用与会者的声明，Talko 运行在 AWS 之上。这些人还表示，该公司正在为手机打造一项通信服务。](https://web.archive.org/web/20221130165849/http://gigaom.com/2013/03/12/one-secret-about-ray-ozzies-secretive-startup-is-out-it-will-tap-amazons-cloud/)

这也与我们从多个更直接的渠道听到的一致——该公司不是在建立一个移动后端业务，而是一个面向消费者的移动应用。在某些情况下，人们甚至称之为 WhatsApp/Tango/LINE 风格的移动应用程序，尽管有些不同，因为它们的功能是可以一起做某些事情，比如在不同的设备上一起浏览照片。

在过去的一年里，我们对 Talko 的了解大多来自于[的招聘信息](https://web.archive.org/web/20221130165849/http://allthingsd.com/20120105/a-tiny-bit-more-on-ray-ozzies-new-startup-cocomo/)、[的 SEC 文件](https://web.archive.org/web/20221130165849/https://beta.techcrunch.com/2012/12/03/ray-ozzies-startup-talko-raises-4-million-to-develop-cloud-based-mobile-backend-services/)以及 LinkedIn 的搜索。例如，2012 年 1 月,[《波士顿环球报》](https://web.archive.org/web/20221130165849/http://www.boston.com/business/technology/innoeco/2012/01/former_lotus_and_microsoft_exe.html)报道称，前微软高管[马特·波普](https://web.archive.org/web/20221130165849/http://www.linkedin.com/pub/matt-pope/0/55/aba)是该公司的联合创始人，与他一起的还有理查森，他也是前微软驻波士顿的员工。

LinkedIn 今天显示，雷·奥茨的儿子尼尔·奥齐(Neil Ozzie)也在团队中，埃里克·帕蒂(Eric Patey)也在团队中，他和理查森一样，曾在 Groove Networks 与雷共事。据 LinkedIn 介绍，前微软员工阿里·戈德堡、霍华德·纳格尔、理查德·斯派尔也加入了。

Talko 团队位于西雅图和波士顿。然而，招聘广告称该公司是一家“不受地点限制”的初创公司，申请时将优先考虑这些城市以及旧金山湾区的人。

消息应用非常庞大

对于那些不熟悉的人来说，最近几个月，消息应用的采用一直在激增。今年二月，Viber 的用户突破了 1.75 亿。WhatsApp 在 T2 拥有大约 2.5 亿用户。老牌 Skype 拥有约 8 亿用户，[其中每月活跃用户超过 2.8 亿](https://web.archive.org/web/20221130165849/https://beta.techcrunch.com/2012/11/20/skype-updates-its-android-app-now-tablet-optimized-and-upgraded-its-silk-audio-codec/)。LINE，流行于日本，新到美国，[1 月破亿](https://web.archive.org/web/20221130165849/https://beta.techcrunch.com/2013/01/18/line-100m-users/)。在韩国，88%的 iPhones 使用了韩国的 KakaoTalk(尽管这份报告有点老了；现在很可能更多)。微信在中国很大。 [Tango 刚刚冲击 1 亿用户](https://web.archive.org/web/20221130165849/https://beta.techcrunch.com/2013/03/07/tango-the-video-calling-app-turned-social-network-hits-100m-users-and-finally-arrives-on-ipad/)。

在美国和西欧，Facebook Messenger [也与 WhatsApp 并列榜首](https://web.archive.org/web/20221130165849/https://beta.techcrunch.com/2013/03/07/tango-the-video-calling-app-turned-social-network-hits-100m-users-and-finally-arrives-on-ipad/)。受青少年欢迎的 SnapChat 刚刚筹集了 1350 万美元，现在每月有 6000 万次“快照”。像 MessageMe 这样的新创公司仍在进入这个领域。

换句话说，消息应用市场可能很大，[但也非常分散](https://web.archive.org/web/20221130165849/https://beta.techcrunch.com/2012/12/04/global-messaging-market/)。这当然提出了一个问题:Talko 认为它可以在这方面做些什么，这会是新的或与众不同的？什么还没做？

在这一点上，Talko 以外的人似乎都不知道，但鉴于一份招聘广告提到，这款产品将“改善人们和群体的沟通和合作方式，对使用智能移动设备的每个人都有潜在的吸引力”，Talko 可能是一款 B2C 产品，但其功能也适用于我们的工作场所需求。这实际上可能会让这款应用与 Yammer 等人的产品相抗衡——顺便提一下，Yammer 现在归 Ozzie 的前雇主微软所有。

上个月在网上发现的一份招聘启事也披露了关于该应用性质的更多细节。该应用将在 iOS 和 Android 上运行，使用 HTML5 和 JavaScript，支持同步和异步消息传递，以及 WebRTC 等实时技术，并被描述为“社交”，适用于“个人和团体”，并将同时提供“移动和网络”体验。
