# Automattic 推出群组 Twitter 风格的平台

[![prologue.jpg](img/515d0666d0a6e5849faacabebf3feb9a.png)](https://web.archive.org/web/20221209112954/http://wordpress.com/blog/2008/01/28/introducing-prologue/)automatic[发布了 Prologue](https://web.archive.org/web/20221209112954/http://wordpress.com/blog/2008/01/28/introducing-prologue/) ，这是一种 Twitter 风格的群组服务，也被定位为分布式 Twitter。

据 Automattic 的创始人马特·莫楞威格称，这项新服务是用户与公司机构分享短信的一种方式，或者是不同群体之间的私人信息。Mullenweg 说，虽然它最初的目标不是成为一个分布式 Twitter，但他们在开源的基础上提供模板，如果人们想为此目的而入侵它，“欢迎你”

分布式 Twitter 的概念在去年的大部分时间里都在某些圈子里讨论。其概念是分散短信服务，因此克服了 Twitter 在服务提供方面的持续问题，或者简单地说，许多人在许多服务器上托管服务，并且他们都互相交谈。

Centernetworks 的 Allen Stern [说](https://web.archive.org/web/20221209112954/http://www.centernetworks.com/twitter-who-says-wordpress)随着 WordPress 成为博客的主导者，这可能会改变游戏规则。那鸿这是一个非常合理的想法，但是 Twitter 成功的关键有三点。一个是其庞大的用户量，通过提供最活跃和丰富的用户群，它击败了 Jaiku 等竞争对手。其次，尽管集中式服务是一个弱点，但它也是一个优势，因为当你在 Twitter 上与他人联系时，*你会在 Twitter 上与他人联系*。不知道他们所在的服务器是否是最新的、活动的或者甚至是兼容的，它只是工作(当它没有关闭或者“暂时过载”时)。第三是通过第三方工具对 Twitter 的开放访问；问问 Pownce 的 Leah Culver(她不是我的粉丝)为什么开放获取对于构建这样的东西是至关重要的。Prologue 可能会提供一些开放的访问，但它的分布式本质将意味着它最终将是一个利基产品；可能是一个很好的利基产品，但它不会很快将 Twitter 的鸟从它的栖木上打下来。
