# iPhone 上的推送通知很棒，但是...TechCrunch

> 原文：<https://web.archive.org/web/https://techcrunch.com/2009/07/12/push-notifications-on-the-iphone-are-great-but/>

在新的 iPhone 3.0 软件推出后的头几周，带有推送通知的应用程序很少出现，现在正以健康的速度推出。这很好，因为这个功能真的很有用。在某种程度上。

我现在注意到的问题是，如果你有太多打开推送通知的应用，整个系统会变得没什么用。你看，推送通知基本上是苹果允许第三方应用程序在 iPhone 后台运行的一种方式。因此，应用程序现在可以将这些推送信息发送到你的手机，让你知道是否有某种信息或更新，你应该打开一个应用程序。但如果你有很多推送消息，我发现你要么每隔几分钟就要拿出手机，要么冒着错过你可能想看的通知的风险。

问题是推送通知消息指示器不是为大量使用而构建的。如果你的手机收到多条推送消息，屏幕上只会显示最新的一条。而且即使解锁手机，也很难分辨哪些推送消息进来了。虽然你可以在应用程序图标上设置一个徽章，让你知道有一条消息，但如果它被另一条消息覆盖，你就会被迫打开应用程序，找出它是什么。

让我们明确一点:我并没有在很多应用程序上使用 Push。我现在基本上只在三个平台上(定期)使用它:Foursquare(一个基于位置的社交网络 [ [iTunes 链接](https://web.archive.org/web/20230323053423/http://itunes.apple.com/WebObjects/MZStore.woa/wa/viewSoftware?id=306934924&mt=8))、GPush(它为 Gmail 推送[，遗憾的是它还没有在 App Store 中获得批准)和 Boxcar(它为 Twitter @replies 和 DMs](https://web.archive.org/web/20230323053423/https://techcrunch.com/2009/07/07/push-gmail-comes-to-the-iphone-%E2%80%94-through-an-app-if-its-accepted/) [ [iTunes 链接](https://web.archive.org/web/20230323053423/http://itunes.apple.com/WebObjects/MZStore.woa/wa/viewSoftware?id=321493542&mt=8))提供[)。在某一点上，我也为 AIM 使用了](https://web.archive.org/web/20230323053423/http://www.readwriteweb.com/archives/boxcar_iphone_twitter_client_with_real_push_notifi.php)，但这本身就太难处理了。

在这三者之间，我在一天中的正常时间内每隔几分钟就会被窃听一次(如果不是在高峰时间更频繁的话)。虽然这没问题，因为我可以更改音频通知等内容，但我希望能够看到自上次查看手机以来收到的所有内容的完整列表。谷歌的安卓平台以更好的方式处理这个问题，有一个顶部下拉菜单可以分解你的通知(这也有点不同，因为应用程序可以在安卓的后台运行)。当然，这只是在你解锁手机之后，但这仍然比目前的苹果手机方法理想得多。

![iphone_status_screen](img/c18398315db01d892f7dd2ca22daba82.png "iphone_status_screen")但最好是在手机还锁着的情况下，用某种方式将这些信息分开。我在想，首先也许可以将推送通知、短信和电话按颜色分开，并在屏幕上的列表中显示出来。然后，有办法进一步分解屏幕上的推送通知，可能是把应用程序图标放在每个图标旁边，然后说一些类似(4)新的 Foursquare 消息的话，就像苹果目前对短信所做的那样。

[这个家伙去年做了一个很好的模型](https://web.archive.org/web/20230323053423/http://www.robertsdonovan.com/?p=148)，但是那是在推送通知还没有出现之前。根据其专利，苹果也一直在考虑这个问题。现在更需要这样的系统(右)。

当然，这一切都应该是用户可调整的设置，但这似乎是一件很容易的事情。因为许多人很快就会发现，目前处理推送通知的方式并不简单，除非您只使用一个每隔几个小时就会收到一次消息的应用程序。随着越来越多支持推送的应用程序每天出现，问题只会变得更糟。
