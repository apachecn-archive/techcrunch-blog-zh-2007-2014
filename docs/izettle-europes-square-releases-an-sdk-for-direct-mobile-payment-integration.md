# 欧洲的 Square ize ttle 发布了一个 SDK，用于 iOS TechCrunch 上的直接移动支付集成

> 原文：<https://web.archive.org/web/https://techcrunch.com/2014/01/23/izettle-europes-square-releases-an-sdk-for-direct-mobile-payment-integration/>

总部位于瑞典的移动支付初创公司 iZettle——因其与智能手机或平板电脑相连的小硬件而被称为“欧洲广场”——今天又迈出了一步，使其服务在其运营的九个国家更加普遍:它发布了一个 [SDK](https://web.archive.org/web/20221005183620/http://developer.izettle.com/) ,将直接将 ize ttle 移动支付集成到第三方应用程序中。

它比 Square 和 PayPal 等美国同行走在了前面——尽管后者预计将在本季度的某个时候发布一款，并且已经在 T4 与优步等知名合作伙伴进行了测试。

很明显，iZettle 引用了一家汽车服务公司作为其 SDK 的参考客户。

“iZettle 的支付功能已经成为我们应用程序的一个重要方面，”Taxi 24/7 和 iView Inc .的创始人兼首席执行官泽利亚布·吉玛在一份声明中说。“客户越来越多地使用我们的应用程序来预订他们的旅程，由于 iZettle SDK，最近增加了一个支付选项，这真的有助于提高客户满意度和重复业务。”

该 SDK 将首先用于 iOS，然后在今年晚些时候扩展到 Android，它是 iZettle 去年推出的 API 的扩展。与 API 和 iZettle 的直接解决方案一样，收费将在 2.75%之间，随着交易量的增加，收费将降至 1.5%(这里是对其定价的完整解释)。

API 解决方案还将 iZettle 支付选项集成到第三方应用程序中，但当客户使用它时，点击“支付”按钮会触发 iZettle 应用程序打开。现在，体验将保留在最初的应用程序中——这将创造更好的用户体验，并对使用它的商家更具吸引力。

“在我们不断追求卡支付民主化的过程中，我们现在向已经拥有一款应用程序，或者正在开发下一款轰动一时的 iOS 应用程序，并且需要支付解决方案的开发者开放我们的平台，”iZettle 联合创始人兼首席执行官 Jacob de Geer 在一份声明中说。

虽然 iZettle 一开始主要直接针对那些规模太小、接受信用卡支付在财务上不可行的商户(因为银行和信用卡公司的收费)，但它越来越多地与其他人合作，以覆盖这些小商户和那些较大的商户——这表明通过移动应用程序和加密狗处理的支付越来越被主流接受。

这包括从 Banco Santander 获得 660 万美元的融资，以拓展拉丁美洲的新市场。它目前活跃在巴西和墨西哥。iZettle 公司的 CMO·约翰·本德兹告诉我，到目前为止，这些推广进展“非常顺利”。“我们在巴西的第一个月就有 4 万名用户注册了。”

自 2010 年成立以来，iZettle 已经筹集了超过 5000 万美元，投资者还包括 Index Ventures、Greylock、万事达卡和美国运通。目前，iZettle 活跃在英国、西班牙、德国、瑞典、丹麦、挪威、芬兰、墨西哥和巴西。对于是否有计划筹集更多资金或有什么国际扩张计划的问题，该公司不愿作出回应，只是表示确实在关注更多增长。

iZettle 发展的另一部分是更多地与销售点硬件提供商合作，将其支付后端和硬件加密狗集成到更大的支付系统中，以销售给更大的商家或那些已经有支付系统的商家。SDK 将帮助 iZettle 更好地与后者集成。

“我们很高兴与 iZettle 团队密切合作，开发市场上首批紧密集成的支付设备 SDK 之一，提供我们的客户喜欢的出色用户体验，”基于 iPad 的销售点系统开发商 iKentoo 的创始人 Serge Sozonoff 说。
