# HTC 不会用 Bootloader 解锁来勾搭 One X 车主

不久前，当 HTC 为许多设备发布了它的[引导程序解锁工具](https://web.archive.org/web/20221007223801/https://beta.techcrunch.com/2011/08/15/htc-finally-offers-bootloader-unlock-u-s-users-must-wait/)时，它赢得了一些人的心。它基本上允许 Android 手机的所有者将定制的 rom 加载到他们的手机上，以换取他们可能已经享有的任何保修权利。

这是一个公平的交易，考虑到很大一部分 Android power 用户更喜欢 Android，主要是因为他们有这个选项，但没有坚实的硬件，加载不同版本的 Android 的能力变得越来越没有吸引力。

[HTC One X](https://web.archive.org/web/20221007223801/https://beta.techcrunch.com/2012/05/04/some-guys-have-all-the-luck-htc-one-x-hitting-doorsteps-ahead-of-schedule/) 也是如此。这是一款优秀的手机，可能是 HTC 有史以来最好的手机。但不幸的是，该公司不会为这种特殊的手机提供引导加载程序解锁工具。

据 [MoDaCo](https://web.archive.org/web/20221007223801/http://www.modaco.com/page/news/_/android/htc-on-the-att-one-x-bootloader-lock-r483) 报道，这是来自 HTC 的官方消息:

> HTC 致力于倾听用户的心声，让客户满意。自从宣布我们对可解锁引导加载程序的承诺以来，HTC 在过去的六个月里已经努力让我们的客户能够解锁超过 45 台设备上的引导加载程序。然而，在某些情况下，限制会阻止某些设备参与我们的引导加载程序解锁程序。请放心，HTC 致力于帮助开发者解锁 HTC 设备的引导加载程序，我们将在未来继续解锁更多的设备。

HTC 指出它提供这种功能的手机数量是正确的，它目前的旗舰设备没有得到同样的待遇，这确实有点奇怪。 [MobileBurn](https://web.archive.org/web/20221007223801/http://www.mobileburn.com/19465/news/htc-one-x-for-att-unable-to-use-bootloader-unlock-tools) 怀疑 AT & T 可能与这一限制有关，我倾向于同意。

毕竟，这不是我们第一次看到运营商出于他们自己的，有时无法解释的原因篡改手机的功能。还记得威瑞森[在 GalNex](https://web.archive.org/web/20221007223801/https://beta.techcrunch.com/2011/12/06/verizon-couldnt-be-more-full-of-it/) 上屏蔽了谷歌钱包，理由是这家运营商对竞争对手 ISIS 的 NFC 支付系统的支持吗？

我们已经联系了美国电话电报公司，想弄清楚蓝色大航母是否真的要对这一不幸事件负责。一旦我们知道更多，我们会随时通知你。
