# Automattic 收购文件共享服务 Cloudup 以构建更快的媒体库并实现共同编辑 

> 原文：<https://web.archive.org/web/https://techcrunch.com/2013/09/25/automattic-acquires-file-sharing-service-cloudup-to-build-faster-media-library-and-enable-co-editing/>

WordPress.com 的母公司 automatic 已经收购了[的文件共享服务](https://web.archive.org/web/20221205115124/https://beta.techcrunch.com/2013/06/21/cloudup-is-a-fast-dead-simple-way-to-share-and-view-files-on-any-platform-without-the-folders/) [Cloudup](https://web.archive.org/web/20221205115124/https://cloudup.com/) ，该服务于 6 月开始测试。此次收购将有助于 Automattic 改善 WordPress.com 的两个特别功能:用于上传视觉内容的媒体库和后期编辑，使多个用户能够同时编辑。

Cloudup 是 ed-tech 公司 [LearnBoost](https://web.archive.org/web/20221205115124/https://www.learnboost.com/) 的一个分支，成立于 2010 年，这意味着它包括相同的团队和现有的投资者。LearnBoost 联合创始人兼首席技术官 Guillermo Rauch 表示，教室管理服务也将继续在 Automattic 下运营。

一开始，Cloudup 团队将专注于修改帖子编辑器，并用 Cloudup 替换 WordPress.com 的媒体库。Automattic 创始人马特·莫楞威格说，这项服务比 WordPress.com 目前的媒体上传程序更先进、更优雅、更直观。“没有多少人说他们最喜欢的[WordPress.com]特色是媒体库，”他指出。

这是轻描淡写。Cloudup 的技术将使多方能够同时编辑和写作，这意味着作家可以起草文本，而摄影师可以将图像上传到同一个帖子。该服务还允许在文件上传完成前共享文件，这意味着第二方可以在文件完成前开始查看文件。

Mullenweg 说，他认为 WordPress.com 即将推出的共同编辑功能与 Google Docs 等服务不同。他说，后者从根本上将文档视为文档，而博客帖子或页面由于其视频、图像和图库功能而具有更丰富的体验。

Mullenweg 没有估计用户何时会看到集成的成果，但他说他们的意图是尽快实现。

据 Rauch 称，Cloudup beta 是一个完整的产品，因为为该系统提供动力的一些开源技术已经开发了多年。不过，Mullenweg 表示，在扩展 Cloudup 基础设施并将其与 WordPress.com 和 WordPress.com 的 Jetpack 整合方面，可能还有大量工作要做。

自 6 月份以来，Cloudup 已经积累了约 10，000 名活跃用户，将继续独立于 WordPress.com 提供服务。Automattic 将继续运行它并添加新功能，该服务将很快向公众开放。

虽然这是一个简单的收入来源，但 Mullenweg 表示，目前还没有计划利用这项服务赚钱。

“我们认为 Cloudup 是世界上本质上有用的东西，Automattic 在 WordPress.com、VIP、VaultPress 和 Akismet 的核心业务对我们公司来说已经足够好了，”他说。

Mullenweg 在 T1 之前已经说过，他的目标是让 WordPress.com 为互联网上的大多数网站提供动力——它们目前的市场份额刚刚超过 20%——而实现这一目标的第一步只是将 WordPress.com 作为一个平台进行改进。Cloudup 有助于实现这一目标。对于像 TechCrunch 这样使用该平台的网站来说，能够更快地共同编辑帖子和上传照片无疑是使用 WordPress 流程的一个改进。

Mullenweg 说，这次收购既是为了人才也是为了技术，因为他已经意识到 LearnBoost 的开源精神，并跟踪了 Rauch 的工作多年。

此次收购使 Automattic 的员工总数超过 200 人。虽然 Cloudup 的大部分员工都来自旧金山，但也有三名员工来自世界各地。其中一人将成为 Automattic 在巴西的第一名员工，这将该公司的业务扩展到 29 个国家。

Mullenweg 说，将自动化团队发展到预期的数千名员工将有 95%是招聘，只有一小部分来自这样的收购。
