# 企业移动性:设备、安全性、设计和分发 TechCrunch

**编者按:** *[塞米沙](https://web.archive.org/web/20221006232314/http://blog.semilshah.com/about)为 [Swell](https://web.archive.org/web/20221006232314/http://www.swell.am/) 做产品，是* TechCrunch *专栏作家，[投资人](https://web.archive.org/web/20221006232314/http://www.angel.co/semil)。他在 [Haywire](https://web.archive.org/web/20221006232314/http://blog.semilshah.com/) 写博客，你可以在 Twitter 上关注他，地址是[@ semi](https://web.archive.org/web/20221006232314/http://www.twitter.com/semil)。*

每个星期天，我都会为这个专栏写一些与手机相关的东西。迄今为止，它主要是关于面向消费者的应用、[设备传感器](https://web.archive.org/web/20221006232314/http://blog.semilshah.com/2013/12/09/on-mobile-the-location-arrow-may-finally-be-pointed-in-the-right-direction/)、[用户界面](https://web.archive.org/web/20221006232314/http://blog.semilshah.com/2013/12/16/the-shifting-ways-in-which-well-interact-with-mobile-apps/)、像[推送通知](https://web.archive.org/web/20221006232314/http://blog.semilshah.com/2013/10/14/the-precise-art-of-mobile-push-notifications/)这样的策略，以及一系列其他主题。然而，我还没有深入研究面向业务、面向企业的移动用户和考虑事项。部分原因是因为我不在大公司工作，也不与他人关系太密切。然而最近，在我的一些谈话中，针对企业环境的移动应用的话题再次出现。2013 年，我还在由风险投资公司[emergency Capital](https://web.archive.org/web/20221006232314/http://www.emcap.com/)组织的一次移动活动中主持了一个小组讨论，该公司专注于 SaaS，并根据不同类型的企业应用制作了这张图表。虽然已经有很多关于这个主题的文章，但我想写这篇文章，从基本原则出发，不做任何假设，并分享我关于创始人和投资者可以从这些类型的移动产品中寻找什么的思考过程。然而，最重要的是，我想向大家学习如何在这些环境中获胜，所以请发表评论或发推特，或者联系我，告诉我你的想法。

**设备:**我听过这里的每个角度。在一些大公司，员工可以自带设备上班。公司会支持员工的要求。但是，随着设备数量的快速增长、设备更新速度的加快以及上市的新设备(尤其是非 iOS 设备)打破了生态系统，这变得更加困难。一位创始人对我说，他相信我们会看到从 BYOD 到 CYOD 的转变，或者“选择你自己的设备”，雇主预先选择一组受控的设备，员工可以从中选择。这些条件在每个公司肯定是不同的，取决于他们采取什么样的政策。

**安全合规:**即使在美国国家安全局的爆料之前，企业安全显然也是一件大事。举例来说，这可能就是为什么大公司会搬到 CYOD 世界。公司担心客户端和基于云的数据安全性，随着最新一批手机允许不同形式的文件共享，不需要传统数据源来支持，这种担心可能会加剧。从员工的角度来看，我想很多人可能会选择让自己的私人手机保持活动状态，从而携带两部手机，部分原因是为了将工作和生活与工作之外的生活分开，部分原因是因为他们担心隐私。

**以企业为中心的应用与跨界应用:**这是一场关于企业是否需要特定应用解决方案的辩论，或者是以消费者为中心的应用(或从消费者层面的原则中汲取的应用)更有可能获胜。我不知道答案是什么，似乎成功的道路并不只有一条。像 Box 这样的产品是为企业级客户设计的，Yammer 这样的产品借鉴了消费者级的设计原则，而 Mailbox 这样的产品，仅举一例，可以帮助其母公司 Dropbox 在大公司的员工队伍中推广自己的应用套件。除此之外，我们还可以在企业设置中“[自带应用](https://web.archive.org/web/20221006232314/http://arielseidman.com/post/53860251234/from-byod-to-byoapps)”。

对我来说，这是最重要的一点…

**分发，病毒式的自下而上，还是自上而下的命令？当我看到新的团队在组建和开发新产品时，这就是我要面对的问题。我不知道最好的方法是什么。起初，世界的发展方式似乎是合乎逻辑的，即假设员工将开始使用消费级应用，一些人将“跨越”到他们的工作中，点燃让它在同事中发展所需的火花。这就是今天的应用程序如何变得巨大，并成为突破性的水平。然而，在公司环境中，雇主可能会要求员工使用特定的应用程序，并每天使用。这可能会迫使公司的每个人不仅下载和安装特定的应用程序，还会设置通知，允许位置跟踪，并在一天内多次使用该应用程序。**

如今，收视率最高、收入最高的“商业”应用大多来自传统提供商(如 Adobe)、基于企业巨头(如 Salesforce)构建的应用、少数新进入者(如 Square)以及一系列小型企业相关工具，如扫描仪应用等。也许我的下一个专栏将关注小型企业的应用程序，但现在，我想听听你对哪些企业级应用程序做得好，机会在哪里，安全问题是什么，以及谁将推动分发的看法。这当然是一个巨大的机会，随着移动分发对成千上万的移动开发者构成真正的挑战，大公司内部的条件可能会带来有吸引力的机会。
