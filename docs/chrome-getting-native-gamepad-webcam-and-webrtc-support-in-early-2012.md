# Chrome 将在 2012 年初获得原生游戏手柄、网络摄像头和 WebRTC 支持

判断 Chrome 何时更新并不容易，而且无论如何，这些变化并不总是那么显著，甚至不值得怀疑。但是，谷歌开发者倡导者保罗·金兰最近在利物浦的[开发者大会](https://web.archive.org/web/20221208234839/http://liverpool.develop-conference.com/)上发言称，有两个值得关注的功能将于 2012 年初面向用户推出。

会议的重点是游戏开发，所以它的特点也是以游戏为中心的也就不足为奇了。首先，Chrome 将支持即插即用的游戏手柄、网络摄像头和麦克风，其次，新生的实时通信协议 [WebRTC](https://web.archive.org/web/20221208234839/http://www.webrtc.org/) 也将开始实施。这打开了无缝视频聊天和类似游戏服务的大门。

这并不是说已经有大量的游戏和应用程序在等着利用游戏手柄和 WebRTC。例如，虽然它将扩展 Chromebooks 的功能，但它也将扩展任何运行 Chrome 的上网本或台式机的功能。尽管如此，让浏览器平台原生支持外设是使其更少地成为应用层而更多地成为主要操作系统层的重要一步。

替换驱动程序层总是会带来各种各样的复杂问题，所以这还没有被正确实现也就不足为奇了。来自操作系统驱动程序和设备管理器的直通已经成为标准，这可能需要一段时间来取代。但是如果它能工作，它就能工作，没有人会关心是微软、苹果、罗技、Adobe、谷歌还是哈利伯顿让他们的网络摄像头工作。他们只想和他们的孩子视频聊天。

金兰还表示，谷歌正在开发一种 OnLive 类型的流媒体视频/游戏服务，这不足为奇。Google +整合也几乎是必然的:“Blank 在玩现代战争 3！点击这里观看。点击此处租赁。”自然，这项服务最难的部分是许可协议，OnLive 和其他公司在这方面领先了很长时间。谷歌拥有的服务器。他们需要的是内容。

“第一季度”就像金兰得到的一样具体，无论如何做出这些决定还为时过早，所以毫无疑问，当我们接近发布日期时，我们会听到更多。
