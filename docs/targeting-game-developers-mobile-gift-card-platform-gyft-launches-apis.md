# 针对游戏开发商，移动礼品卡平台 Gyft 推出 API TechCrunch

> 原文：<https://web.archive.org/web/https://techcrunch.com/2013/03/27/targeting-game-developers-mobile-gift-card-platform-gyft-launches-apis/>

[Gyft](https://web.archive.org/web/20230126215229/http://www.gyft.com/) ，TechCrunch Disrupt SF 2012 决赛入围者，致力于[将塑料礼品卡行业带到移动领域](https://web.archive.org/web/20230126215229/https://techcrunch.com/2012/09/10/gyft-is-moving-the-plastic-gift-card-industry-to-your-iphone/)，今天宣布其[API](https://web.archive.org/web/20230126215229/http://www.gyft.com/partners/)正式可用，该 API 允许开发者将 Gyft 集成到网站、应用或其他服务中。然而，正如 Gyft 联合创始人兼首席执行官 Vinny Lingham 解释的那样，主要焦点是移动应用程序开发者——这是一个有兴趣在他们的移动游戏中提供礼品卡作为奖励的群体。

“很多开发者都有积分、奖励和未兑换的积分，他们希望为用户提供一种兑现这些积分或积分的方式，”林厄姆解释道。他建议，对于这些开发者来说，Gyft 是比使用收取交易费用的账单支付系统更好的选择。

“我们专注于移动领域，”他补充道。“现在有很多手机游戏。这是个巨大的市场。林厄姆指出:“那些已经拥有手机、正在玩手机游戏的人，将会对使用手机礼品卡持更加开放的态度。”。

[![Yappem](img/f3009a72d8291186f8c14f95f445ab13.png)](https://web.archive.org/web/20230126215229/https://techcrunch.com/2013/03/27/targeting-game-developers-mobile-gift-card-platform-gyft-launches-apis/yappem/) 虽然应用内的 [API 集成](https://web.archive.org/web/20230126215229/http://www.gyft.com/partners/)可以帮助新客户在 iOS 和 Android 上使用 Gyft 自己的应用，但兑换卡的客户不会*让* *到*在他们的手机上安装 Gyft 应用。如果他们愿意，他们可以在网上访问 Gyft，登录并打印出他们的礼品卡。

第一个使用 Gyft APIs 的合作伙伴是 Yappem，这是一个与品牌和企业分享客户体验的社交网络。林厄姆说，大约 10 个集成将在 4 月底上线，Gyft 预计在接下来的几个月里会有“数百个”集成上线。

这是 Gyft 首次推出面向开发者的功能。到目前为止，这家初创公司一直在开发面向消费者的各种移动礼品卡“钱包”，允许用户存储他们的塑料礼品卡的数字版本，购买礼品卡，或在特殊场合与朋友分享卡片，例如生日。

Gyft 拒绝提供活跃用户或下载量的细节，但表示其服务目前已有“数十万”用户，自六个月前[推出](https://web.archive.org/web/20230126215229/https://techcrunch.com/2012/09/10/gyft-is-moving-the-plastic-gift-card-industry-to-your-iphone/)以来，其存档礼品卡的储值已增长至 500 万美元。该公司还开始针对用户提供小额免费礼品卡(如 Gap 的 5 美元)，以鼓励用户与 Gyft 的零售商合作伙伴一起消费。在一个案例中，20%开卡的人去实体店购物。林厄姆说，平均消费是信用卡价值的 4 到 5 倍。

当然，这是早期的数据，但随着公司的发展，它将与零售商合作，与他们持有礼品卡的客户群建立联系，为那些潜在的购物者提供激励，以在线或离线的方式进入商店进行购买。但目前，该公司专注于产品改进和推动消费者采用该应用程序——通过移动游戏生态系统传播其服务可能有助于实现这一目标。
