# 栈溢出达到 1000 万 Uniques，大胆地去没有问答网站去过的地方

![](img/8221199cacc704c02bdd7263d6914ee2.png)

似乎问答网络[栈溢出](https://web.archive.org/web/20230202220747/http://www.stackoverflow.com/)已经很好地利用了它在五月份收到的[600 万美元的资金，截至昨天已经突破了](https://web.archive.org/web/20230202220747/http://www.readwriteweb.com/archives/stackoverflow_business_funding.php)[1000 万独立月访客](https://web.archive.org/web/20230202220747/http://www.quantcast.com/p-c1rF4kxgLUzNc#traffic)大关。虽然 Q & A 网站像 [Quora](https://web.archive.org/web/20230202220747/http://www.quora.com/) 银行一样是一个大的单体网站，但 Stack Overflow 通过仔细地将 Q & A 游戏分成不同的社区，推出 34 个不同的网站，主题从骑自行车、烹饪到 IT 安全，显示了成功。

在第一轮融资之前，该公司只有三名全职开发人员，其中包括摇滚明星天使投资人，如罗恩·康韦、克里斯·迪克森、纳瓦·拉维坎特和卡特里纳·法克。从那时起，创始人[乔尔·斯波尔斯基](https://web.archive.org/web/20230202220747/http://www.crunchbase.com/person/joel-spolsky-2)和[杰夫·阿特伍德](https://web.archive.org/web/20230202220747/http://www.crunchbase.com/person/jeff-atwood "Jeff Atwood")雇佣了 24 名新员工，其中包括 Stack Overflow power 用户马克·格拉维尔。

尽管(或因为)它的编程根基，Spolsky 告诉 TechCrunch，该公司在过去六个月中最大的成就是建立了 [Stack Exchange](https://web.archive.org/web/20230202220747/http://stackexchange.com/) ，它允许用户[为 Stack Overflow 风格的 Q & A 网站建议新的垂直领域](https://web.archive.org/web/20230202220747/http://area51.stackexchange.com/)，如象棋、航空和密码学。这是一个明智的举动，因为使用共识模型来决定建立哪些网站可以确保到达后的临界质量和高质量的答案，如[这个](https://web.archive.org/web/20230202220747/http://math.stackexchange.com/questions/11669/mathematical-difference-between-white-and-black-notes-in-a-piano)。

栈溢出流量猛增，在六个月内从 600 万增加到 1000 万(花了两年时间才达到第一个 600 万)。Spolsky 说，“*世界上几乎每个程序员都知道堆栈溢出，并定期访问，通常是因为我们有他们在谷歌中输入的问题的答案——新的堆栈交换网站正开始迅速传播到程序员的最初受众之外。”*以我的经验来看，对程序员有巨大吸引力的东西通常会成为主流和流行趋势，因为程序员花在网上的时间最多。

举个例子:新的 Stack Overflow 垂直网站才成立几个月，但根据内部指标，已经显示出 33%的月增长率。除了流量，成功还基于多种标准，其中最重要的是至少有一个投票赞成答案的网站问题的百分比。斯波尔斯基说，所有的新网站都有 90-100%的投票率。

就未来的计划而言，斯波尔斯基正专注于堆栈交换，并鼓励更多新的垂直领域的增长，在这些领域，专家们可以聚集在一起，*“最终，我们认为将有数百或数千个这样的网站为详细的问题提供极高质量的答案，所以我们正在努力的最重要的事情是真正培养利基社区。”*

该网络最近还从 Jason Cohen 和 Dharmesh Shah 那里获得了一小笔几乎是象征性的 5 万美元的资金，并刚刚发布了一系列新功能，包括类似 [Campire 的](https://web.archive.org/web/20230202220747/http://campfirenow.com/)互动聊天。

愿最佳问答网站(或多个网站)胜出。
