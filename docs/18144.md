# Wit.ai 拥有适用于任何应用的语音接口 API，它希望成为自然语言的 Twilio

去年，语音技术巨头 Nuance [悄悄收购了 VirtuOz](https://web.archive.org/web/20230328145641/https://techcrunch.com/2013/01/10/in-our-inbox-a-memo-that-says-nuance-has-bought-virtuoz-to-ramp-up-its-enterprise-services/) ，这是一家为在线销售、营销和支持提供虚拟助手的开发商——这是一款“企业的 Siri ”,将 PayPal 和 AT & T 等公司视为客户。现在，VirtuOz 的创始人兼首席执行官亚历山大·勒布伦(Alexandre Lebrun)重返创业世界，推出了 [Wit.ai](https://web.archive.org/web/20230328145641/http://wit.ai/) ，这是一个平台和 API，允许开发者将语音识别和自然语言界面集成到任何应用程序或硬件中。

用 Lebrun 的话来说，这里的想法是将“Twilio 或 Stripe 模型”有效地应用到语音界面的世界，其中 Wit 能够理解用户的意图，以及语音识别。

希望将这一点融入他们的应用程序的开发人员输入几行 Wit.ai 代码；这是第一次，开发者本身不必成为该领域的专家，也不必面对从其他地方引进技术知识的巨额费用。

为了与创业英雄 Twilio 和 Stripe 保持一致，Lebrun 和他的联合创始人 Willy Blandin 申请了 Y-Combinator 课程，现在是该课程的一部分。除此之外，Wit.ai 也取得了一些令人印象深刻的进展。

在开放测试版服务的两个月内，仅仅通过口头宣传，它就获得了 2000 名开发者的注册；它已经签下了几个大客户，包括一家大型设备制造商、一家汽车公司、一个电视频道和几家家庭自动化初创公司。(我已经同意暂时不提他们的名字，但我向你保证这是一个值得称赞的列表。)Lebrun 和他的小团队目前在智能手表制造商 Pebble 位于帕洛阿尔托的办公室工作。

正如 Lebrun 向我描述的那样，作为一家新的创业公司，推出 Wit.ai 的动力来自两个地方。

首先，基于 API 的自然语言平台仍然是一个非常新的概念。这是因为，除了专门为特定用例定制的应用程序之外，涉及自然语音的平台很难创建。这实际上意味着大多数应用程序的语音界面对大多数应用程序发行商来说成本太高、时间太紧。

当 Lebrun 还在运行 VirtuOz 时，开发者的需求和满足需求的挑战是他经常遇到的事情。

“当有人要求我们建造一个智能代理时，通常需要花费 10 万美元和三个月的时间，”他说。“这是非常技术性的。你必须考虑语法、特定术语、不同的用例。”

然而，对于新一代设备，自然语言将是最自然的交互方式。

“史蒂夫·乔布斯把 Siri 放在 iPhone 上，但这是绝对必要的吗？大多数人不喜欢它，因为它没有真正做任何你已经不能用手机上的其他功能做的事情，”他说。

你也可以说，这可能是为什么苹果没有努力使它成为比现在更强大的功能的部分原因。感觉就好像人们习惯了它就足够了。

相比之下，Lebrun 表示，Wit.ai 专注于开发人员如何能够为下一代设备进行创作，“在你没有键盘的地方。想想 Nest 或谷歌眼镜或许多其他可穿戴设备。除了语音，别无选择。”

创建 Wit.ai 的第二个原因与更广泛的创新问题有关。Lebrun 已经在一家主要的语音技术公司工作，那么为什么不在那里推出 Wit.ai 作为一个项目呢？

他告诉我，除了大公司可能非常抵制可能蚕食其现有收入的技术这一事实之外，他觉得大型、老牌组织在孵化和发展新概念方面往往非常具有挑战性。

“如果你问 Nuance 为什么不能开发这个，我会回答，为什么美国电话电报公司没有创造 Twilio？它只是必须是一个局外人，”勒布伦解释说。

可以肯定的是，Wit.ai 并不孤单:你可以想象，还有其他自然语言和语音识别专家也在研究如何为类似 Siri 的技术创建一个 API。潜在的候选人包括[罗宾实验室](https://web.archive.org/web/20230328145641/https://techcrunch.com/2013/12/18/yahoosiri-app-is-actually-a-real-prototype-built-by-robin-labs-not-yahoo/)，或者[亚马逊](https://web.archive.org/web/20230328145641/https://techcrunch.com/2013/01/24/amazon-gets-into-voice-recognition-buys-ivona-software-to-compete-against-apples-siri/)以及更多[亚马逊](https://web.archive.org/web/20230328145641/https://techcrunch.com/2013/04/17/sources-say-amazon-acquired-siri-like-evi-app-for-26m-is-a-smartphone-coming/)、 [Nuance](https://web.archive.org/web/20230328145641/https://techcrunch.com/2012/08/06/nuance-nina/) 和[英特尔](https://web.archive.org/web/20230328145641/https://techcrunch.com/2013/09/13/intel-has-acquired-natural-language-processing-startup-indisys-price-north-of-26m/)。

Wit.ai 在技术层面所做的事情在很大程度上利用了大数据架构带来的好处和进步。

Wit.ai 已经整合了几个并行运行的语言处理引擎，包括在卡耐基梅隆大学开发的开源 [CMU 斯芬克斯](https://web.archive.org/web/20230328145641/http://en.wikipedia.org/wiki/CMU_Sphinx)项目。通过使用机器学习，Wit.ai 可以智能地组合结果，并涵盖通用语言和专业词汇。Lebrun 将 Wit.ai 描述为“它们之上的虚拟层”,它将不同的服务编织成一块布。从这个意义上说，它不同于另一家公司 [Ask Ziggy](https://web.archive.org/web/20230328145641/http://www.ask-ziggy.com/) ，后者也创建了一个 API，将自然语言理解应用程序，但不提供处理引擎集成。

然后，还有一个众包数据的元素被扔进了这个混合体。Wit.ai 利用了一个命令数据库，这些命令实际上是使用其 API 的开发人员可能为他们的特定应用输入的所有不同的现有和附加短语的聚合——他们可以选择这样做，尽管这不是必要的。

“虽然自然语言很难，但人们想要的东西也有很多重叠，”他说。“当每个开发人员给出一些例子时，Wit 理解并把这个例子与其他类似的例子联系起来，这些新短语就成为更大数据集的一部分。”

这受到了开发商的欢迎。

“Wit 将自由形式的语言转化为结构化数据的神奇能力，通过帮助我们理解每个问题的含义，改变了 TalkTo 从企业获得答案的能力，”TalkTo 的联合创始人兼首席技术官赖利·克莱恩(Riley Crane)说，talk to 是一家初创公司，其免费应用程序允许用户从当地商店获得答案，而无需打电话给他们。“他们的 API 通过在我们人类生活的非结构化世界和严格的机器代码世界之间架起一座桥梁，为开发人员提供了一种新的、急需的超级能力。”

其核心是，Wit.ai 试图打破自然语言和语音识别技术多年来的孤岛式发展方式。“这是语音和语言行业的一场革命，”Lebrun 说。
