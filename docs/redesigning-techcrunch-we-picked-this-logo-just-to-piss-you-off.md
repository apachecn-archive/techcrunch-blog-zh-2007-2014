# 重新设计 TechCrunch:我们选择这个标志只是为了激怒你

> 原文：<https://web.archive.org/web/https://techcrunch.com/2011/07/11/redesigning-techcrunch-we-picked-this-logo-just-to-piss-you-off/>

制作过程历时数月。一路上经历了无数的曲折。它在一次收购中幸存了下来。协调是一种挑战，也是一种荣幸。我很荣幸向大家介绍全新的、完全重新设计的 TechCrunch。特别感谢我们的发布赞助商[戴尔](https://web.archive.org/web/20221205192524/http://www.dell.com/us/p/xps-15z/pd.aspx)。

[正如迈克尔·阿灵顿在周五](https://web.archive.org/web/20221205192524/https://beta.techcrunch.com/2011/07/08/heads-up-techcrunch-will-look-a-little-different-next-week/)发布的，我是[戴夫·费尔德曼](https://web.archive.org/web/20221205192524/http://www.crunchbase.com/person/dave-feldman)，自 2011 年初以来，我一直担任 TechCrunch 重新设计的产品经理。该项目始于去年秋天，在 AOL 收购 TechCrunch 之前。到 12 月，它需要产品管理——向设计机构提供反馈和指导(代码&理论)，定义产品需求，了解 TechCrunch 不同寻常的协作编辑流程，确定信息架构，并最终协调开发和发布。迈克向美国在线的布拉德·加林豪斯寻求产品经理的项目领导。他求助于 AOL 的消费者体验主管 Matte Scheinker(我的经理)。Matte 的团队专注于需要额外产品、设计和/或工艺专业知识的“战略项目”。他同意接手这个项目，让我负责这个案子。

TechCrunch 很大胆。它是生的。节奏很快。每天有 50 篇文章穿过主页，我们不能简单地重新设计面向读者的网站；我们也必须考虑 CMS。我们想支持你们，读者，因为你们阅读了我们发布的每一篇文章。我们想给你更好的文章和主题的背景，新的方式来切割内容，这样你就可以挖掘你感兴趣的东西。我们获得了巨大的 RSS 流量，也希望继续为这些读者提供完整的体验。

我不记得我们经历了多少轮的线框和标识。在我们第八次拒绝视觉设计后，我把自己和 Code & Theory 的创意总监锁在一个房间里，我们调整、讨论和修改，直到我们有了我们都喜欢的东西，这就是你今天在这里看到的东西的前身。

### 该网站

这是一个更简单的 TechCrunch。特色帖子的传送带不见了。取而代之的是钉在河的顶端的特色文章(寻找小图钉图标),以及没有广告时标题中的特色。我们用一个统一的分享按钮取代了混乱的社交网络图标。一个一致的、网站范围的外观和感觉意味着更少的干扰会分散你对内容的注意力。

我们已经将 CrunchGear、MobileCrunch 和 TechCrunchIT 分别滚动成[beta.techcrunch.com/gadgets](/web/20221205192524/http://techcrunch.com/gadgets)、[beta.techcrunch.com/mobile](/web/20221205192524/http://techcrunch.com/mobile)和[beta.techcrunch.com/enterprise](/web/20221205192524/http://techcrunch.com/enterprise)，这样更容易找到你想要阅读的文章。(在过渡期间，这些网站上的一些旧帖子可能会消失几天，但它们会回来。)我们创造了更丰富的方式来展示文章:当你浏览主页时，你会看到各种各样的布局，每一种都是作者选择的，以最好地代表她的作品。

今天的 TechCrunch 应该会快一点。我们从头开始构建了一个新的代码库。我们对 HTTP 请求进行了批处理，将 API 调用转移到了服务器端，并修改了加载顺序以提高性能。这只是一个开始:从下周开始，我们将优化这个东西。

我已经看到了很多与最近的 [Gawker 重新设计](https://web.archive.org/web/20221205192524/https://beta.techcrunch.com/2011/02/17/gawker-redesign/)的比较——主要是担心我们会步他们的后尘。我认为我们对 Gawker 太苛刻了:他们看到了传统博客格式的缺点，决定尝试一些新的东西，一些类似应用的东西。他们犯了很多错误。但核心思想是一个耐人寻味的想法，我为他们敢于冒险而鼓掌。也就是说，Gawker 的方法不适合 TechCrunch。如果他们背离了博客的格式，我们加倍努力，通过提炼和扩展来解决同样的缺点。

### CMS

关于 [WordPress](https://web.archive.org/web/20221205192524/http://www.crunchbase.com/company/wordpress) 我最喜欢的一点是它的扩展性。我们今天和昨天在同一个平台上，但是为作者和编辑建立了新的工具。特色文章和固定文章都有截止日期，所以编辑不必回去手动取消特色文章。选择文章布局就像点击一个按钮一样简单。自动调整图像大小意味着更快的加载时间和更少的失真照片。选择哪些文章放在主页上是一件简单的事情。

### 外观和标志

新标志是我们最有争议的变化。我喜欢它，尽管这不是偶然的:在找到我们喜欢的之前，我们在《代码与理论》中经历了很多很多选择。它大胆、简单、功能多样。它适用于任何环境——从微小的单色图标到海报上的马赛克。它完全符合 TechCrunch 的品牌。不，我们没有在《我的世界》建造它。我们使用了 AOL Paint，它在 AOL CD 上是免费的，并且具有这个可爱的 UltraLogoMatic2000 特性。

整体的外观和感觉反映了 TechCrunch 大胆，有时不敬的本质。它不会在后院举行茶话会，也不会在最时髦的艺术画廊里与黑色高领毛衣的人群一起闲逛。这是一个比它的竞争对手爆出更多新闻的设计，它喜欢代码迷们每天 22 小时工作来构建改变世界的产品。这是希瑟、迈克和我看到的第一个也是唯一的设计，我说*是的，就是它。*它尖叫着 TechCrunch。

我认为今天的发布是一个测试版。还有更多的功能即将推出:增强的社交功能；浏览文章和深入主题的方法；查看帖子的新选项；更好的导航；和改进的内容过滤。作家和编辑更好的工具。性能大幅提升。

归根结底，TechCrunch 是关于科技行业、报道科技行业的人物以及痴迷于科技的读者的。最重要的是，这种重新设计是为了提供最好的车辆。我们将不断增强、完善和修改，以使它变得正确。敬请关注。
