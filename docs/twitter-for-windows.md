# Twitter:没有 Windows 客户端的计划——但永远不要说不

> 原文：<https://web.archive.org/web/https://techcrunch.com/2011/01/06/twitter-for-windows/>

很久很久以前，Twitter 只是一个集成了一些短信的超级简单的网站。然后它变成了一个稍微强大一点的网站。然后就变成了 iPhone 应用。然后是一个安卓应用。然后是一个 iPad 应用。然后是一个非常健壮的网站。黑莓、Windows Phone 等。现在，到今天为止，有一个官方的 Mac 桌面客户端，Mac 版 Twitter。几乎所有的基础现在都被覆盖了——除了一个:窗户。

今天早上发布了 Mac 版 Twitter 之后，我问该公司是否有计划下一步开发基于 Windows 的桌面版本。"*目前，我们只有 Mac 版本。一位 Twitter 代表告诉我，“我们没有推出 PC 版的计划——尽管我们从未说过永不*。换句话说，最终，是的，可能会有一个。

正如 Twitter 今天在他们的博客文章中提到的，“*我们收购了 atebits，专注于推出我们自己的 Twitter iPhone 应用程序。从那以后，我们不断被要求为 Mac 提供新版本的 Tweetie。我们认为新版本非常符合我们的目标，即确保主流用户在流行平台*上获得最佳体验如果 OS X 是一个流行的平台，那么 Windows 是什么？一个超级超超人气平台？虽然这些用户可以选择原生体验，但许多人运行在 AIR 上，坦白地说，这是一种非常糟糕的体验。

所以，我再次怀疑我们可能会看到 Twitter 在某个时候转向 Windows 客户端。如果微软能够更好地统一 Windows 体验，就像他们昨天在 CES 上建议的那样，这将是尤其真实的。

当然，Twitter 在为 Mac 开发 Twitter 上也有很大的领先优势。当他们去年收购 Tweetie 和它的开发者 Loren Brichter 时，他已经在为 Mac 版 tweet ie 2——后来成为 Mac 版 Twitter 的软件——而努力工作了。正如博客帖子中提到的，“*Mac 版 Twitter 是由 Loren 和他的团队在 10 月 Twitter 的第一个黑客周*更新的新版本。”显然，从头开始构建 Windows 客户端需要做更多的工作。但是，如果在同一个[黑客周活动](https://web.archive.org/web/20230202231924/https://techcrunch.com/2010/10/22/twitter-hack-week/)中，有人在开发一个，我也不会感到惊讶。

更大的问题可能是这对整个 Twitter 生态系统意味着什么？趋势是第三方开发者开发一些东西，然后 Twitter 会推出他们自己的版本——通常会更好。那么 Mac 版 Twitter(也许最终会是 Windows 版)会摧毁其他桌面客户端吗？这似乎不太可能，因为桌面体验差异很大，甚至比移动体验更大。 [Erick 不喜欢](https://web.archive.org/web/20230202231924/https://techcrunch.com/2011/01/06/twitter-mac-stripped-down-meh/)Mac 版 Twitter，因为他是 TweetDeck 用户，一个完全不同的桌面客户端。我讨厌 AIR，所以到目前为止我还是喜欢 Mac 版的 Twitter。

此外，尽管所有这些客户端都很有趣，各有各的独特之处，但我确信 Twitter 的主要关注点仍然是核心产品:网站。如果说有什么东西扼杀了这些本土客户，那最终可能就是它。
