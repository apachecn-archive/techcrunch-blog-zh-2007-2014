# 微软紧急修补 IE，因为网络聚集在 IE6 周围

当微软更新软件时，它通常喜欢批量更新，通常称之为“补丁星期二”今天[在](https://web.archive.org/web/20221207193827/http://blogs.technet.com/msrc/archive/2010/01/19/security-advisory-979352-going-out-of-band.aspx)提到，在围绕其网络浏览器，特别是 IE6 的漏洞的争议中，微软已经决定“走出频带”,作为一个独立的补丁发布更新，并将尽快完成。

虽然帖子中没有具体说明，但这一举动似乎是对 IE6 因最近中国对一些大型网络公司的大规模攻击而受到指责的直接回应。你一定听说过，这次[包括谷歌](https://web.archive.org/web/20221207193827/http://www.beta.techcrunch.com/2010/01/12/google-china-attacks/)，这促使他们说他们将[停止在中国审查搜索结果](https://web.archive.org/web/20221207193827/http://www.beta.techcrunch.com/2010/01/17/the-price-of-google-in-china/)，并可能因此被踢出这个国家。微软[否认](https://web.archive.org/web/20221207193827/http://news.yahoo.com/s/afp/20100114/ts_alt_afp/uschinaitcompanyinternetgooglerightsmicrosofthp)它是这次黑客攻击的目标，但[承认 IE](https://web.archive.org/web/20221207193827/http://news.bbc.co.uk/2/hi/technology/8460819.stm) 中的一个漏洞至少是部分原因。据微软称，针对浏览器的攻击仍在继续。

在提到这一安全升级时，微软还表示，它建议所有客户升级到 Internet Explorer 8，这是最新版本的浏览器，具有更好的安全性。当然，[微软自己的调查](https://web.archive.org/web/20221207193827/http://www.microsoft.com/technet/security/advisory/979352.mspx)发现 IE8 也容易受到攻击。简直是噩梦。

袭击之后，甚至坚定的微软支持者也加入了一个已经存在多年的合唱:微软[应该杀死 IE6](https://web.archive.org/web/20221207193827/http://www.beta.techcrunch.com/2009/07/23/woah-people-really-dont-like-ie6/) 。不幸的是，有许多公司仍然不得不使用浏览器，因为他们有专门为运行浏览器而构建的系统。

当然，这也是微软自己的错，因为当其他人开始团结在网络标准周围时，他们多年来一直决定使用自己的专有网络代码。虽然该公司在这方面越来越好，但就与网络标准的[兼容性](https://web.archive.org/web/20221207193827/http://www.beta.techcrunch.com/2009/09/22/google-turns-internet-explorer-into-chrome-yes-seriously/)而言，IE 仍然是迄今为止最差的浏览器。微软现在仍在努力与 W3C——网络标准监管机构——更好地合作。这仅仅是因为 IE 继续流失市场份额。

以下是微软今天声明中的关键内容:

> 鉴于此问题引起的高度关注、客户如何保护自己的困惑以及不断升级的威胁环境，Microsoft 将针对此漏洞发布带外安全更新。
> 
> 鉴于对客户的影响，我们非常认真地对待带外更新的决定，但我们认为在这个时候发布带外更新是正确的决定。我们将在明天提供发布的具体时间。
