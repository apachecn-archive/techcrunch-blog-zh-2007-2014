# Linux 碎片化:好还是坏？

> 原文：<https://web.archive.org/web/https://techcrunch.com/2010/05/11/linux-fragmentation-good-or-bad/>

[![](img/37bd3469c87381b50331b257719c3523.png "Scattered puzzle pieces next to solved fragment, by Horia Varlan")](https://web.archive.org/web/20221006010709/http://www.flickr.com/photos/horiavarlan/4273913228/) 
每个人和他们的兄弟都可以制造一个 Android 设备，这意味着 [Android 的销量超过了 iPhone](https://web.archive.org/web/20221006010709/http://www.mobilecrunch.com/2010/05/11/the-real-reason-android-is-outselling-the-iphone-android-is-everywhere/) 。但正如约翰帖子上的一些评论所指出的，设备激增导致消费者在试图找到合适的设备进行购买时感到焦虑。同样的现象在 Linux 市场已经发生了很长时间，消费者试图弄清楚他们是否需要或想要 Fedora、Ubuntu 或 OpenSUSE。碎片化是一个问题，当你在这个背景下看待事物的时候；但是 Linux 市场真的那么分散吗？Linux 基金会的执行董事吉姆·泽姆林不这么认为。

在他的新博客文章中，“[碎片化对 Linux](https://web.archive.org/web/20221006010709/http://www.linux.com/news/featured-blogs/158-jim-zemlin/307570-fragmentation-is-good-and-bad-for-linux) 有好有坏”，Zemlin 指出，所有的 Linux 变种都使用相同的内核，由一群聪明、专注的人维护。当然，有人*可以*开始维护他们自己的 Linux 内核，但这是一个昂贵、耗时的提议，对最终产品没有太大好处。类似地，大多数 Linux 解决方案使用相同的 X.org 窗口解决方案，相同的 GNOME 或 KDE 桌面环境，以及相同的 OpenOffice.org 应用程序套件。不同的发行版可能使用这些产品的不同发布版本，但是它们都来自一个统一的上游供应商的事实意味着 Linux *发行版*可以专注于集成问题，而不是特定的应用程序问题。

> 这些不是 Linux 的随机版本；每一项努力都得到主要行业参与者的大力支持，每个参与者都创建了自己的生态系统；谷歌、惠普、英特尔、诺基亚、亚马逊等。这对 Linux 有什么坏处？Linux 是关于选择和允许公司，项目和个人竞争和繁荣。API 差异化允许公司参与竞争，并激励他们不断增强平台。参与者可以根据特定项目的网络优势和参与的难易程度，选择加入或退出 Linux 应用程序 API 工作。

事实上，在工作中我使用 Red Hat Enterprise Linux，而在家里我使用 Ubuntu。我们在工作中使用的许多研究应用程序来自专门针对 Red Hat Enterprise Linux 的更长发布周期的 ISV，这些应用程序是根据 RHEL 支持的库的特定要求构建的。虽然我们使用 RHEL，但是许多 ISV*也支持 SuSE，这使得我们可以选择使用哪个 Linux 发行版。对于想要更多“哦，闪亮的！”的桌面用户来说 Linux 应用程序开发的特性有 Fedora 和 Ubuntu，两者都提供了更快的发布周期和更有趣的特性开发，但由于更频繁的 API 和 ABI 更改，ISV 不太愿意将它们作为目标。*

> Android、Meego、Chrome 和 webOS 都是基于 Linux 的，但没有人会对他们正在构建的应用程序类型或他们正在进入的市场感到困惑。这里的现实是，除了苹果、RIM 和微软，几乎没有人在用 Linux 之外的任何东西构建客户端计算设备。在各种 Linux 系统之上将有多个应用程序生态系统，它们将在计算栈的较低层次保持统一。这为共享 R&D 和市场竞争提供了极好的平衡。现在重要的是，行业参与者要与可信的行业参与者支持的一项或多项 Linux 工作保持一致，并确保这些工作继续在上游开发他们的代码。

(当我写这篇文章时，我的阅读器收到了蒂姆·布雷的一篇新博文:[实用开源](https://web.archive.org/web/20221006010709/http://www.tbray.org/ongoing/When/201x/2010/05/08/Using-Android-Open-Source)。这是一个非常简洁的介绍，说明获取 Android 源代码并开始使用它取得进展是多么容易。不完全相关，但是足够相似以至于激起我对 Linux 碎片化的兴趣。)

在与 [Devin](https://web.archive.org/web/20221006010709/http://www.crunchgear.com/author/devin) 谈论 Linux 碎片化时，他观察到大多数人并不关心他们设备上的 Linux，因此碎片化只是发烧友群体的问题。他肯定有所发现。普通消费者可能会意识到苹果、安卓和 Palm 之间的品牌差异，但他们不知道安卓实际上是 Linux 的一个版本。消费者不会关心 Linux 碎片，因为他们不会看到太多碎片:他们会看到“Android”，它恰好建立在丰富的 Linux 内核和库组件之上。只要他们的通话没有中断，他们的应用程序还在运行，他们就会幸福地忽略任何 Linux 碎片。
