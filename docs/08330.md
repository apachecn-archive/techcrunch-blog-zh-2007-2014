网络聊天比网络更古老，但它仍然是让人们参与网站的最佳方式之一。理由很充分:人们喜欢互相交谈。或者至少，他们喜欢分享他们的观点，因为他们知道有人在某个地方正在阅读他们的观点。

现在，由 Y Combinator 支持的创业公司 [Envolve](https://web.archive.org/web/20230131013850/http://www.envolve.com/) 正在通过推出一个提供聊天即服务的 API，让将聊天框集成到你的网站变得更加容易。它可以让你用一小段 JavaScript 代码将聊天整合到几乎任何网页中。你可以试试他们在 [TagChat](https://web.archive.org/web/20230131013850/http://tagch.at/) 上的一个聊天框，它可以让你谈论 Twitter 上的热门话题。

对于 Envolve 在这里所做的一些背景，最好看一下该公司的历史。Envolve 实际上已经存在一年了，已经迭代了该产品的几个版本。首先，它提供了一个浏览器聊天栏，类似于 Meebo 和脸书聊天。该栏提示用户创建 Envolve 帐户，这将允许他们与当前正在浏览的网站上的朋友以及其他集成了 Envolve 的网站上的朋友聊天。

这并没有得到太多的接受(人们不想创建另一个帐户)，所以该网站后来允许网站将工具栏与他们自己的用户帐户系统合并——这种做法进展得更好。该工具栏现已整合到 20，000 个网站中，包括由环球音乐运营的 Destructoid 和 11 个网站。总的来说，合作伙伴网站每天发送 400，000 条消息，总发送量超过 4，500 万条。

到目前为止，Envolve 的聊天产品已经提供了侧边聊天，允许用户与他们的朋友进行交流。今天的发布是不同的，因为它允许网站以编程方式为单个页面生成聊天框。因此，举例来说，一个在线零售商店可以为它的每一个商品自动创建一个聊天框，而不必为每个商品手动插入嵌入代码。该 API 目前仅限于桌面网站，但移动版本正在开发中。

在后端，Envolve 提供分析和选项来帮助网站管理员减少垃圾邮件——你可以要求用户在开始聊天之前进行身份验证，并且还有一些默认过滤器。

Envolve 提供两种定价级别:更基本的“消费者”级别将允许多达 150 名用户同时在网站上聊天，而企业级别没有限制，并根据网站收到的访问量收费。请注意，即使用户没有真正开始聊天也算作一次访问，因为他们仍然会看到其他人在聊什么。

Envolve 的竞争对手包括 [Meebo](https://web.archive.org/web/20230131013850/http://www.meebo.com/) (拥有 Meebo 酒吧)和 [Wibiya](https://web.archive.org/web/20230131013850/http://www.wibiya.com/) 。
