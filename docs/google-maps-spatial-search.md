# 谷歌将空间搜索添加到地图应用编程接口

![](img/109b421aaffa389ae6297c17a27af0db.png)

一场争夺地理开发者的心和思想的战争正在打响。上个月， [Twitter 开启了它的 geo API](https://web.archive.org/web/20221006075048/http://www.beta.techcrunch.com/2009/11/19/twitter-location-api/) ，像 [SimpleGeo](https://web.archive.org/web/20221006075048/http://simplegeo.com/) 和 [GeoAPI](https://web.archive.org/web/20221006075048/http://geoapi.com/) 这样的服务正在[提供](https://web.archive.org/web/20221006075048/http://www.beta.techcrunch.com/2009/11/19/more-details-on-simplegeo-the-aws-for-location/)来为那些想要创建很酷的地理应用的初创公司做很多[繁重的工作](https://web.archive.org/web/20221006075048/http://www.beta.techcrunch.com/2009/12/09/geoapi-creation/)。谷歌没有袖手旁观，它只是调整了谷歌地图应用程序接口，现在支持空间搜索和搜索源。

空间搜索允许开发人员在地图的某个区域搜索特定的要素。他们的应用程序将通过搜索提要调用这些搜索，搜索提要可以搜索一个框内区域或某个纬度/经度点的半径范围内。地理搜索结果可以按不同的属性排序，如距离。将这种空间搜索输入到谷歌不断增长的本地企业目录中，开发者也许可以使用 API 来构建一个应用程序，在地图上显示边界内的餐馆或鞋店。

能够询问 10 个街区半径内或一个县内的所有有趣的地方打开了各种可能性。随着所有这些地理平台的涌现，创建地理应用程序变得越来越容易。当然，没有什么可以阻止开发人员混合和匹配来自不同 API 的不同地理数据层。

**2017 年 11 月更新**:你可以在[谷歌地理开发者博客](https://web.archive.org/web/20221006075048/http://googlegeodevelopers.blogspot.com/2009/12/maps-data-api-bringing-geospatial.html)上找到更多信息，该博客刚刚就这个主题发布了一个帖子。
