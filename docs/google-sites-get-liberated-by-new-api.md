# 新的 API 解放了谷歌网站

![](img/6de47a8befbf5f33deda8990efa4caa7.png)在过去的 18 个月里[谷歌网站](https://web.archive.org/web/20221208225834/http://sites.google.com/)为企业提供了一种不需要 HTML 知识就能快速建立自己网站的方法，这是一种帮助协调内部工作以及建立面向消费者的网站的简单方法。但是对这项服务有一个相当大的抱怨:如果你想把数据带到其他地方，没有简单的方法导出数据。如今，随着谷歌推出新的网站应用编程接口，情况发生了变化。

对于那些不熟悉它的人来说，Sites 是 Jotspot 的[翻版](https://web.archive.org/web/20221208225834/http://www.beta.techcrunch.com/2008/02/27/it-took-16-months-but-google-relaunches-jotspot/)，谷歌早在 [2006](https://web.archive.org/web/20221208225834/http://www.beta.techcrunch.com/2006/10/31/google-acquires-wiki-company-jotspot/) 就收购了 Jotspot(尽管这两款产品看起来完全不同)。该产品是谷歌易于使用的网站和 wiki builder，被企业广泛使用，但也有消费者选择。

新的 API 是谷歌最近[推出的](https://web.archive.org/web/20221208225834/http://blogoscoped.com/archive/2009-09-15-n32.html) [数据解放阵线](https://web.archive.org/web/20221208225834/http://www.dataliberation.org/)的一部分，该阵线由谷歌的一个团队组成，其“唯一的目标是让用户更容易将他们的数据移入和移出谷歌产品”。换句话说，谷歌试图确保，如果你对它的某项服务不再满意，它应该可以相当容易地拿起东西去别的地方，而不会丢失你的任何数据。它也给了企业一个创建自己的本地备份的机会——谷歌称这是网站最受欢迎的功能之一。但是除了数据导出之外，你还可以用新的 API 做很多事情。

企业现在将能够从第三方应用程序更新他们的网站页面(谷歌提供了一个例子，当一个新的线索被添加到你的 CRM 时，更新一个网站页面)。您还可以使用 API 将您的整个 Google Sites 帐户下载到您的桌面上，如果您在一个互联网连接最少的地区，这将非常有用。

你可以通过查看[开源导入/导出项目](https://web.archive.org/web/20221208225834/http://code.google.com/p/google-sites-liberation/)和[Sharepoint Move for Google Apps](https://web.archive.org/web/20221208225834/http://www.ltech.com/google-apps/products/sharepoint-move)来了解 API 的能力，这两个项目都使用了 Sites API。
