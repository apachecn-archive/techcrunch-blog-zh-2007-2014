# 要求 Twitter 用 Google+匕首自杀 

> 原文：<https://web.archive.org/web/https://techcrunch.com/2011/07/18/dont-drink-and-power-use/>

有时候，权力使用者很容易沉醉于那种权力。

我和其他人一样对此感到内疚。当我不断抱怨我的 Gmail 帐户太慢时，我忘记了我还有整整 20 GB 的电子邮件——谷歌公关总是善意地提醒我这是不正常的。(我想出了自己的解决方案——完全戒掉电子邮件。)[罗伯特·斯考伯](https://web.archive.org/web/20221007063021/http://www.crunchbase.com/person/robert-scoble)也许是完美的超级用户。每次一项新服务推出，他都会尽可能多的加好友或关注更多的人，以测试极限。然后，当达到限制时，他不可避免地抱怨，或者服务不能很好地与那么多连接一起工作。因为，你知道，大多数人没有三万好友。(斯考伯也没有——但那是另一回事。)

无论如何，Scoble 是权力之王使用一个产品到它的临界点，然后抱怨它。最近的一个例子:斯考伯[抱怨【Quora 作为一个博客平台真的没什么好的——](https://web.archive.org/web/20221007063021/http://scobleizer.com/2011/01/30/why-i-was-wrong-about-quora-as-a-blogging-service/)[一个稻草人的论点，他自己建立了](https://web.archive.org/web/20221007063021/https://beta.techcrunch.com/2011/01/31/quora-quora-quora-quora-quora-quora-quora/)。这个周末，他又回来了。这一次他愤怒的受害者是一个熟悉的人:Twitter。

在一篇名为“ [Google+让 Twitter 变得无趣，Twitter 应该对此做些什么”](https://web.archive.org/web/20221007063021/http://scobleizer.com/2011/07/17/google-has-made-twitter-boring-heres-what-twitter-should-do-about-that/)的帖子中，Scoble 继续长篇大论地谈论 Twitter 需要立即增加哪些功能来与刚刚推出的 Google+竞争。这种咆哮中缺少的是一件非常关键的事情:自省。斯考伯没有意识到他是一个超级用户。

更直截了当地说:如果 Twitter 疯狂到足以实现 Scoble 提出的事情的一半，他们将有效地扼杀自己的产品。

当然，我们可以放心地知道 Twitter 很可能*不会*这么愚蠢。在经营了整整五年之后，他们现在必须知道，他们的优势不是模仿每一个刚刚推出的、现在炙手可热的新社交网络(在这种情况下，是 Google+)。他们的力量在于忠于让他们走到今天这一步的东西:简单。

这并不是说 Twitter 不应该增加新的功能或者改进他们的服务。他们当然应该。但是如果他们添加一些 Scoble 建议的东西，比如添加信息(真的吗？)，取消其他服务的链接推送，并为 Twitter 重新创建 Google+圈子，这是一个许多人在注册后已经不使用的产品，对大多数人来说将变得完全无法使用。

Twitter 不是 Google+。也不需要如此。如果他们试图在运行中进入 Google+,数以百万计的当前用户会理所当然地大吵一架。我觉得斯考伯也会的。

[在本周末](https://web.archive.org/web/20221007063021/https://beta.techcrunch.com/2011/07/17/is-social-in-googles-dna/)的一篇客座博文中，汤姆·安德森(人们更亲切地称其为“MySpace Tom”)提出了许多人经常忽视的关键观点——大多数服务不是被竞争对手杀死的，而是它们自己杀死的。脸书没有杀死 MySpace，MySpace 自杀了(由于管理不善)。MySpace 没有杀死 Friendster，Friendster 杀死了自己(通过缩放问题)。

事实是，Twitter *几年前也因为规模问题差点让*自杀。但不管出于什么原因，他们的竞争对手都没有能够利用 Twitter，Twitter 变得更加强大。

但是 Twitter 试图成为 Google+会导致他们的死亡。他们会自杀。虽然你可能会认为他们在 Google+面前无所作为也会导致他们自杀，但我认为这还为时过早。它假设 Twitter 没有做任何事情来改善他们的产品——这是荒谬的。

让我们一步一步来看看 Scoble 的想法。

**《初体验》**

Scoble 说，当用户第一次看到 Google+和 Twitter 时，Twitter 似乎很无聊。对一些人来说可能是这样，但对另一些人来说则相反。我和很多人谈过，他们看一眼 Google+就觉得它看起来像是内容的噩梦。

他说得对，Twitter 需要做更多的事情，让用户在第一次注册时参与进来。但是他们知道这个。这可能是他们最大的问题(嗯，也许超越了货币化)。解决方案不是从根本上改变你的产品。

对于大多数用户来说，实现 Twylah 太令人困惑，也没有必要。修改 Twitter 列表对大多数用户来说也不起作用。清除垃圾邮件一直是个问题，这是毋庸置疑的。Scoble 建议他们通过 Klout 分数过滤结果，我怀疑 Twitter 最终会做类似的事情(尽管我认为他们有自己的内部分数)。

**【图片&视频】**

嗯，Twitter 刚刚在照片空间推出了一个重要的入口。如果它被认为足够重要，我相信视频也是如此。Scoble 似乎对图片和视频没有显示在你的 Tweet 流中感到不安(相反，它们在 twitter.com 的右栏中只需点击一下)。我再次怀疑，讨厌这个特性的人和喜欢它的人一样多。

Twitter 目前的扫描能力(快速浏览内容的能力)非常棒，因为它是流线型的。相比之下，Google+就很恐怖了。几周前，迈克·蒙泰罗有效地说明了这一点。

这是一种偏好。也许 Twitter 最终会提供这个选项。但默认情况下，这会让很多当前用户想吐槽。比如像我。

**“对内容分发的控制”**

Scoble 在这里想要的是相当于 Google+圈子的东西。他希望能够一次只给几个 Twitter 用户，或者一群 Twitter 用户发消息。但这不是 Twitter 的目的。Twitter(在很大程度上)是一个公共广播系统。这让他们受益匪浅——而其他国家，比如脸书，则更多地朝着这个方向前进。

Google+对 Circles 采取了不同的方式(尽管它仍然有一个公共频道，基本上我认识的每个人似乎都在使用)。Scoble 希望 Twitter 能够改进这个很少使用的列表产品，使其更像一个圆圈。但我认为这是行不通的。[我仍然不相信圈子概念会像谷歌希望的那样长期发挥作用。现在，人们正在享受建立这些团体的便利，但是维持它们将是困难的。](https://web.archive.org/web/20221007063021/http://parislemon.com/post/7769030399/can-we-ever-digitally-organize-our-friends)

此外，正如 Twitter 自己的[郑嘉颖本周末在他的个人博客](https://web.archive.org/web/20221007063021/http://kevnull.com/2011/07/can-we-ever-digitally-organize-our-friends.html)(远在 Scoble 的帖子之前)上所言，以数字方式组织你的朋友是否会完全奏效还不清楚。社会团体通常有太多的灰色线(程称之为“软线”)。 [Fred Wilson 更进一步](https://web.archive.org/web/20221007063021/http://www.avc.com/a_vc/2011/07/explicit-groups-vs-implicit-groups.html)，他说，除了家庭和同事这两个定义明确的群体之外，所有的群体都会随着时间的推移而变化，如果没有远远超出我们现在所见的自动化，分组在很大程度上是行不通的。

谷歌可能已经解决了“没人想列清单”的问题，但是他们是否已经解决了更大的根本问题还有待观察。我持怀疑态度。虽然我喜欢 Twitter 列表(特别是为了填充 Flipboard)，但我知道大多数用户根本不会花时间去做。我也不确定他们会不会/应该这么做，即使正确的接口已经存在。

这个。是。a .纯粹。权力。用户。问题。—至少对 Twitter 来说是这样。

**“不自动推送内容”**

斯科博在推特上关注了 32297 人[。比我多 31600 人，我也是超级用户。当他抱怨“我看着 Twitter，它的很多内容都变成了无聊的 RSS 提要”时，他为什么不，你知道，取消关注几百个只是注入 RSS 内容的账户——比如 TechCrunch 账户？因为这将是一个简单的解决方案，并且他不认为他应该改变他的完全不合理的用例。](https://web.archive.org/web/20221007063021/https://twitter.com/#!/Scobleizer)

事实是，许多人使用 Twitter 就是为了这个目的。例如，我越来越多地使用 Twitter 来取代我的 RSS 阅读器。如果我不想看到推送的内容，我会取消关注那个账户。简单。

此外，如果 Google+继续流行，同样的用户不会发生的想法是可笑的。我们的几个竞争对手已经用自动推送链接覆盖了 G+。我们还没有这样做，只是因为谷歌一直说，他们更喜欢我们等到他们有自己的适当的商业帐户设置(现在任何一天，谷歌)。这是要发生的。很多。欢迎来到分发工具的世界。

**“你可以看到谁让你兴奋，谁让你生气”**

好吧，我都不知道从哪开始，所以我就长话短说。这主要是关于自我，但它也是另一个只会激怒一群普通用户的高级用户特性。

**"帖子自动流动"**

Scoble，你用 Mac，为 Mac 下载 Twitter。嘣。完成了。

不过说真的，Twitter 以前也有这个功能。当缩放成为一个问题时，他们关闭了它。他们当然可以再次打开它，但他们故意不这样做。当然，他们这样做是有原因的。他们还刚买了 TweetDeck，它也能做到这一点。所以，也许他们认为它(就像 TweetDeck 本身一样)是一个超级用户的功能。

此外，虽然 Google+确实有自动流量，但我发现它的功能现在非常糟糕。有时我的流会自动更新，有时不会。经常要刷新才能看到最新评论，+1s 等。这实际上比我发现它有用更让我生气。我相信谷歌会解决这个问题——FriendFeed 早就解决了这个问题。

**Twitter 不是 Google+**

让我总结一下这个显而易见的事实:Twitter 不是 Google+。如果它试图成为现实，这将是一个非常受欢迎的产品的自杀——一个怪异的美国总统使用的产品——[一个苹果公司刚刚把 iOS 的钥匙交给了](https://web.archive.org/web/20221007063021/https://beta.techcrunch.com/2011/06/09/twitter-ios/)。在我看来，Twitter 的崛起可以直接归因于与 Scoble 所追求的一切相反的东西:简单。

Twitter 的核心概念是过去几十年简单短消息的延伸。明信片开始短信开始即时消息状态信息[开始推特](https://web.archive.org/web/20221007063021/https://beta.techcrunch.com/2009/07/04/short-is-sweet-postcards-begat-sms-begat-twitter/)。有时最简单的想法会引起共鸣，因为它们很简单。

Twitter 在历史上有过一些时刻，他们不得不增加一个功能，使服务变得更加复杂——例如[像新型的转发——他们能够实现它，但这是一个艰难的过渡。想象一下，如果他们试图附加一些 Scoble 建议的东西会怎么样？其实不用麻烦了。因为 Twitter 不会很快自杀。](https://web.archive.org/web/20221007063021/https://beta.techcrunch.com/2009/11/11/simple-is-as-simple-does-the-risk-of-retweet/)
