微软在Release中收购了InCycle的发布管理服务，以改进其Team Foundation Server

微软已经收购了 InCycle的 发布管理解决方案 InRelease，这是一个在不同环境下自动将应用组件部署到目标服务的工具。InCycle是一家加拿大公司，专注于应用生命周期管理(ALM)和微软的发布管理解决方案。NET平台。

由于微软只是收购InRelease业务部门，公司的其余部分将继续独立运营，专注于ALM咨询。InCycle成立于2002年，去年，该公司与另一家专注于ALM的微软合作伙伴Sirk Technologies合并，该公司在旧金山和西雅图设有办事处。

InRelease支持敏捷。NET团队来自动化他们的应用程序的发布周期。它直接从微软的Team Foundation Server(微软自己的ALM和DevOps解决方案)自动部署到所有环境，包括生产环境。这确保了所有部署都以相同的方式完成。它还自动化了批准工作流和审计支持。

微软开发部门的公司副总裁S. Somasegar在今天的一份声明中表示:“随着企业被迫以越来越快的速度开发和部署高质量的应用程序，DevOps是ALM中越来越重要的一部分，也是首席信息官越来越感兴趣的一个领域。”。InRelease连续交付解决方案将使Visual Studio Team Foundation Server从开发到生产的发布过程自动化，有助于更快、更简单地交付应用程序。

InRelease已经与 Team Foundation Server 直接集成，InCycle希望这次收购能够帮助它向Team Foundation Server添加发布管理功能。微软没有具体说明除了之外它具体计划如何这样做，指出“这次收购将扩展微软在ALM和DevOps领域的产品”，并且它计划集成“从TFS到生产，都在一个解决方案中，并且都与TFS集成”
