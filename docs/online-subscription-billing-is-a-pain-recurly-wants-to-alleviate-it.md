# 在线订阅计费令人头痛。Recurly 想缓解它。| TechCrunch

> 原文：<https://web.archive.org/web/https://techcrunch.com/2009/11/17/online-subscription-billing-is-a-pain-recurly-wants-to-alleviate-it/>

![Screen shot 2009-11-17 at 12.10.36 AM](img/2727ba367b5b1c95c2acdc1b082868e9.png "Screen shot 2009-11-17 at 12.10.36 AM")大多数初创公司有大约 10 亿件事需要担心。对他们中的许多人来说，这包括执行他们的业务模型。随着在线广告网络越来越不景气，许多这样的公司开始探索基于订阅的模式。但有十几个原因可以解释为什么这会让人感到疼痛。重复进入。

Recurly 本身就是一家初创公司，今天正在进入私人测试阶段。这项服务背后的核心思想很简单:让初创公司能够提供基于订阅的服务作为一种选择，这使得*变得简单*。他们提供了一个易于使用的系统，具有良好的用户界面和良好的分析能力，这是支付网关(如 Authorize.NET)和初创公司之间必须完成的交易的基础。

还有许多其他服务提供这样的功能，但 Recurly 认为它可以在两个关键方面脱颖而出。首先，只有在您获得报酬后，他们才会获得报酬。这里没有隐藏的或每月的费用。Recurly 获取每笔交易的一定百分比。这些总量占 1%至 3%，具体取决于体积。但它也不会在您每次销售时收取费用，而是在每个月底销售完成时收取费用。

Recurly 的另一大卖点是能够为客户提供一种轻松升级或降级服务计划的方式。这意味着，如果最终用户决定他们想为某项服务转换到较低层的定价结构，Recurly 可以无缝地处理。虽然我们可能不认为这有什么大不了的，但根据联合创始人[艾萨克·霍尔](https://web.archive.org/web/20230306050618/http://www.crunchbase.com/person/isaac-hall)的说法，对于大多数公司来说，这是一个非常头痛的后端问题。

“无缝”可能是 Recurly 试图做的所有事情的关键词。例如，如果您是一家非常年轻的初创公司，甚至不想处理 API，您可以来到该网站，通过几个步骤设置一个付款表单小部件。或者你的网站需要更多的定制。有一些 API 仍然可以连接到 Recurly 的系统中，并为您提供与简单小部件相同的数据和分析。

![Screen shot 2009-11-17 at 12.12.31 AM](img/c3016eefed46d30efea8e1aeec274520.png "Screen shot 2009-11-17 at 12.12.31 AM")

那么谁是主要目标呢？主要是 SaaS 客户，联合创始人蒂姆·范·洛恩告诉我们。他认为这对他们来说是一个特别令人兴奋的时刻，对所有想使用“免费增值”商业模式的网站来说也是如此。意识到所谓的“应用经济”目前正在爆发，他在 Recurly 的未来计划中也看到了这一领域。

当然，正如我所说，竞争很激烈。其中一个大玩家是[佐拉](https://web.archive.org/web/20230306050618/http://www.zuora.com/)。但对于许多年轻的创业公司来说，他们提供的东西太复杂了。更重要的是，它也很贵。一个更接近的竞争对手可能是 [CheddarGetter](https://web.archive.org/web/20230306050618/http://cheddargetter.com/) ，一家在八月份从孵化器 [SproutBox](https://web.archive.org/web/20230306050618/http://www.sproutbox.com/) 中推出的创业公司[。这两者之间有许多相似之处，但关键的区别在于，CheddarGetter 收取固定费用(每月 39 美元起，但随着增长而上涨)，Recurly 选择使用交易费(随着增长而上涨)。这只是两种不同的做事方式。Recurly 还表示，它更侧重于企业。](https://web.archive.org/web/20230306050618/https://techcrunch.com/2009/08/13/cheddargetter-wants-to-get-your-startup-some-cheddar-invites/)

另一件值得关注的事情是 PayPal 现在对自己的 API 做了什么。但同样，霍尔确信，简单的升级/降级类型的选项是递归可以提供的，其他人将无法轻松匹配。此外，他指出，他们与 PayPal 合作，作为支付网关选项之一。

到目前为止，Recurly 一直在与少数客户进行非常小规模的封闭 alpha 测试。今天标志着他们更加开放，但仍然封闭的测试版的发布。

Recurly 还宣布， [Freestyle Capital](https://web.archive.org/web/20230306050618/http://www.freestylecapital.com/) 的联合创始人、Spinner 和 Crackle 的创始人大卫·塞缪尔将加入他们的顾问委员会。
