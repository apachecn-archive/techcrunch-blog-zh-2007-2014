# WordPress Twitter Blackbird Pie 工具带回了老派的转发方式！

在发布关于 [WordPress 实现 Twitter 的黑鸟派](https://web.archive.org/web/20221006185717/https://beta.techcrunch.com/2010/11/05/wordpress-blackbird-twitter/)工具的帖子时，我注意到了一些有趣的事情。这个工具带回了 Twitter 早已废弃的东西:老派的转发！

现在，当你在 WordPress.com 的博客文章中插入一个 Tweet URL 时，它会自动在那篇文章中生成一条漂亮的 Tweet，并带有工作链接。但它也在推文下方添加了一个“转发”链接。点击这个会把你带到 Twitter.com，在那里一条推文会自动为你填充:“RT @USERNAME TWEET HERE”。

Retweet 是 Twitter 用户在早期创建的一项功能。它基本上只是一种引用推文的方式，但使用语法“RT”让其他人知道这是其他人的推文。Twitter 最终意识到，有这么多的人在使用它，他们必须创造一些方法让所有用户(主要是新用户)更容易使用它。所以他们[创造了](https://web.archive.org/web/20221006185717/https://beta.techcrunch.com/2009/08/13/twitter-announces-a-retweeting-api/)新的转发，它不使用“RT”语法，而是将实际的推文放回关注转发者的用户的时间表中。

许多用户起初不喜欢这种新的转发方式，并对此发起了一场风暴。主要的[问题](https://web.archive.org/web/20221006185717/https://beta.techcrunch.com/2009/11/11/simple-is-as-simple-does-the-risk-of-retweet/)是新的方式不允许你给你转发的推文添加评论。Twitter 或多或少地说了句“太糟糕了”，然后继续前进。现在，新的方式几乎是公认的标准，但许多第三方服务仍然允许您使用旧的方式。

现在 Twitter 用这个工具把它带了回来(更新如下)。看起来他们不得不这样做——这几乎是唯一不需要网站安装某种插件的方法，这种插件允许 Twitter.com 上的新型转发。

欢迎回来，老派转发！
