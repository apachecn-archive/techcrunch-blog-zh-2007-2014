短信和电话的世界充满了物流障碍。总部位于旧金山的初创公司 Twilio 的目标是通过提供一个 API，让开发人员使用一些基本的和主动的命令发送文本、拨打电话和做其他常见的功能，让他们不那么令人生畏。今天，他们向公众开放了 API 的一个新部分:Twilio SMS 短代码 API。

通常用于发送大量文本信息的短码并不新鲜，但获取短码的过程可能会很漫长(而且需要长期的投入)。Twilio 希望能帮上忙。它不能让这个过程进行得更快，但一旦开发者提交了他们的申请，Twilio 将处理与运营商的所有谈判。开发人员可以开始使用标准的 10 位数字构建他们的应用程序，一旦获得批准，就可以切换到新的 5 位或 6 位短代码。

该 API 已经通过一个私人测试程序提供给开发者有一段时间了(Twilio 说它已经处理了 100 多个短代码应用程序)。但 Twilio 表示，截至今年年初，美国仍有超过 1031 个短代码可用。对于 Twilio 来说，这听起来可能不是一个巨大的机会(毕竟，Twilio 已经有[成千上万](https://web.archive.org/web/20230128183116/https://techcrunch.com/2011/06/17/charting-twilios-growth-over-the-last-year-and-the-price-drops-that-helped/)的开发者在使用它的 SMS API)，但是每一个短代码都涉及到大量的资金。

自定义代码的短代码租赁费为每月 1500 美元，随机选择的代码为 1000 美元——这些费用的很大一部分用于从公共短代码管理等组织获得许可证。用户还需为通过 API 发送的每条出站短信支付 1 美分，每条入站短信支付 0.5 美分。请记住，这些短代码是发送大量*短信的最佳选择，因此每条短信的费用可以为 Twilio 带来一大笔钱。Twilio 还允许开发商按季度租赁短代码，而不是标准的年度合同。*
