# 谷歌提供了更多关于 Chrome 网上商店 TechCrunch 的细节

> 原文：<https://web.archive.org/web/https://techcrunch.com/2010/05/20/chrome-web-store-details/>

或许谷歌 I/O 大会第一天最大的新闻是宣布推出 Chrome 网上商店。这个商店旨在突出 web 应用程序，同时为开发者提供一种更好的赚钱方式(如果他们愿意的话)。但是这家商店还没有准备好在展示，所以你可以想象，有很多关于它的问题。今天一整天，谷歌解决了其中一些问题。

在各种讨论中出现过几次的一个问题是，Chrome 网上商店是否将只有 Chrome？谷歌产品副总裁桑德尔·皮帅承认，目前会的。商店本身将仅限于(和内置)Chrome 和 Chrome OS(当它可用时)。也就是说，为网络编写的应用程序就是为网络编写的应用程序，所以它们应该可以在任何浏览器上运行。当然，那些依赖于某种 Chrome 网上商店支付结构的用户可能无法在没有商店的情况下使用(尽管谷歌[不会说](https://web.archive.org/web/20221201054736/http://groups.google.com/a/chromium.org/group/chromium-apps/browse_thread/thread/25769cab5f2945b9))。

随后的一个问题是，谷歌是否会考虑允许其他浏览器运行 Chrome 网络商店？皮查伊指出，就像谷歌所做的一切一样，他们更希望它是开放的。这不是一个真正的答案，但他说，他们正在与其他浏览器制造商就商店的可能合作伙伴关系进行谈判。也就是说，首先，它将只支持 Chrome。

一个人问谷歌收银台是否会用于通过商店支付。谷歌表示，他们还没有准备好给出细节，但用户可以期待这种结构的工作方式类似于它在 Android 上的工作方式。很有可能是谷歌结账为商店提供动力。

就收入分成而言，谷歌表示将与标准做法保持一致。皮查伊后来证实，这将是一个 70/30(开发者/谷歌)的分裂。这是安卓市场的分裂，也是苹果应用商店的分裂。

更多的问题在 Chromium 应用的谷歌群组论坛中得到解答。

有人想知道如何处理应用程序的更新。显然，它将像 Chrome 中的扩展更新一样工作。也就是说，自动地。

有趣的是，Chrome 应用程序的安全许可比 Chrome 扩展要少。谷歌人亚伦·布德曼[解释了这个基本原理:](https://web.archive.org/web/20221201054736/http://groups.google.com/a/chromium.org/group/chromium-apps/browse_thread/thread/0130f799297813a0)

> 如果你需要跨源请求，你可以把你的应用实现为 Chrome 扩展。扩展具有更高的权限，并且具有更严格的安全警告。我们希望保持 web 应用程序的良好和安全，以便安装可以非常轻量级。

是的，Flash 应用程序将[作为 Chrome 应用程序运行良好](https://web.archive.org/web/20221201054736/http://groups.google.com/a/chromium.org/group/chromium-apps/browse_thread/thread/dfdaf0007d4783a2)——《植物大战僵尸》的演示在主题演讲中用 Flash 运行。

许多问题想知道为什么如果这些新应用只是网络应用，它们需要不同的文件扩展名？Google [说](https://web.archive.org/web/20221201054736/http://groups.google.com/a/chromium.org/group/chromium-apps/browse_thread/thread/69a0ceef03105897)他们需要用最少的“胶水”包装应用程序，让它们出现在网络商店中，并且可以在 Chrome 浏览器中安装。

更多的时候，谷歌对 Chrome 网上商店问题的回答是，今天的发布只是一个预览，更多的细节将很快公布。考虑到早期的基于应用的构建已经在最新一批 Chromium 中出现，很快可能真的会很快。
