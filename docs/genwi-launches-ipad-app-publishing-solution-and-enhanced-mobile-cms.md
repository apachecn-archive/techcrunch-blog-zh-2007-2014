# GENWI 推出 iPad 应用发布解决方案和增强型移动 CMS 

> 原文：<https://web.archive.org/web/http://techcrunch.com/2011/07/13/genwi-launches-ipad-app-publishing-solution-and-enhanced-mobile-cms/>

我们第一次报道 GENWI 是在 2007 年，当时,[公司是一个社交订阅阅读器，允许用户按类别组织他们的内容订阅，并与朋友分享。自那以后，GENWI 经历了几次变化，其中最引人注目的是推出了 iSites，这是一个快速简单的 iOS 和 Android 移动应用开发平台，导致了来自 Inventus 和 Quest Venture Partners 的 100 万美元投资](https://web.archive.org/web/20230203143428/https://techcrunch.com/2007/09/01/genwi-browse-and-share-syndicated-content/)[。今天，这家初创公司宣布更名，这实际上是回归 GENWI 的名字，将 iSites 和 iSites.us 折叠到 GENWI 的保护伞下。](https://web.archive.org/web/20230203143428/https://techcrunch.com/2010/09/23/isites-raises-1-million-for-mobile-app-development-platform/)

基于云的智能手机和平板电脑应用发布平台继续在移动应用开发领域向前推进，今天推出了 iPad 应用发布解决方案和移动内容管理系统(CMS)。随着该公司的发展，它开始专注于为媒体出版商提供工具，以创建和管理 iOS 和 Android 的报纸、杂志和目录应用程序，iPad 应用程序解决方案是对之前平台迭代的合理扩展——特别是随着平板电脑的不断普及，特别是 iPad 作为内容制作工具和一些最好(和设计最优秀)的媒体应用程序的来源。

因此，GENWI 的 iPad 平台将允许出版物和媒体公司(以及二者之间的任何人)即时创建云支持的 iPad 应用程序。根据 GENWI 的联合创始人 Raju Sagiraju 的说法，该解决方案是可定制的，并为出版商提供了品牌体验。向应用程序添加新内容和更新显然对出版商极其重要，因此 GENWI 的解决方案提供了灵活的客户端架构，正如它提供了优化的模板和主题(a la WordPress)一样，使企业能够缩短应用程序设计流程和持续时间。

为此，GENWI 还提供了一个“高级选项”，允许出版商使用 HTML5 和 CSS 进行定制设计开发，从头开始创建独特的设计体验。有了这些工具，该解决方案的目标是为媒体出版商、广播公司等提供。，通过一个平台，他们可以将印刷出版物、时事通讯和播客转化为定制的应用程序。

至于新的移动内容管理系统，GENWI 称之为 mCMS(显然是在商标下)，允许出版商添加一系列不同格式的内容，包括媒体 RSS、iTunes RSS、ATOM、XML、Restful APIs、CSV、InDesign 和 PDF。然后，mCMS 实时聚合来自这些来源的内容，并以一种特定于设备的方式将它们推送到移动环境中。而且，像任何其他内容管理系统一样，GENWI 使出版商能够为每台设备管理聚合内容。

据 Sagiraju 称，GENWI 目前在 iPhone 和 Android 智能手机上提供 1500 个活跃的应用，来自 3 万多家发行商。但是，对出版公司和他们的开发者来说，最有吸引力的可能是 GENWI 应用程序不必重新提交到应用程序商店进行重新批准，用户也不必在对应用程序的设计进行小调整时升级应用程序。这在本质上类似于 SaaS 产品在网络领域的功能——毫无疑问，这是对移动应用开发领域的有益增强。

在其他功能中，GENWI 平台还包括内置的货币化功能，如广告和优惠券管理、分析以及基于 iTunes 支付系统的应用内订阅。

在成本方面，Sagiraju 表示，这将取决于使用和应用程序的交互，但一个 iPad，iPhone，Android 和 HTML5 应用程序可以以每月 500 美元的价格开发和推出。这将包括 GENWI 的 mCMS 的有限版本，具有高级功能和更高的使用水平，涉及进一步的费用。根据发行商的规模，这些费用可能会让他们有些束手束脚，但许多人可能会愿意为跨平台轻松管理和发布应用的能力付费。
