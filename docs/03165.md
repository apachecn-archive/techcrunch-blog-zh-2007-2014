# 今年九月即将来临:推特启示录！| TechCrunch

> 原文：<https://web.archive.org/web/https://techcrunch.com/2009/07/31/coming-this-september-twitpocalypse-now-redux/amp/>

 <amp-img class="vertical alignright size-full wp-image-88654 amp-wp-enforced-sizes i-amphtml-layout-intrinsic i-amphtml-layout-size-defined" title="3953" src="https://web.archive.org/web/20230403221519im_/https://techcrunch.com/wp-content/uploads/2009/07/3953.jpg" alt="3953" layout="intrinsic" i-amphtml-layout="intrinsic"><i-amphtml-sizer class="i-amphtml-sizer"></i-amphtml-sizer></amp-img> 第一个推特启示录是有一天你会告诉你的孩子的事件之一。我记得当它来袭时我在哪里:在我和一些朋友去纳帕谷的路上，我们听到汽笛声，很可能是我们所知道的推特世界的终结。

好吧，事情并没有到[那么糟](https://web.archive.org/web/20230403221519/https://techcrunch.com/2009/06/12/twitpocalypse-not-now-crisis-averted/)。但对许多第三方开发人员来说，这仍然是一个棘手的问题，尤其是苹果手机推特应用程序开发人员，他们和其他人一样，不得不在应用商店排队等待修复。现在一切又会重演！

[推特账户今天的一条推特](https://web.archive.org/web/20230403221519/http://twitter.com/twitterapi/status/3059191420)警告称，第二次推特末日比人们想象的要近。好吧，大多数人可能没有意识到它可能再次发生，但它可以，而且将会，而且它正在快速接近。

API 团队的[当前估计](Twitter%20status%20ids%20are%20fast%20approaching%20the%20maximum%2032-bit%20*unsigned*%20)是在未来 60 天的某个时候发生，可能在 9 月底。不过，他们警告称，这可能会更快发生。

那么，这意味着什么？为什么会发生这种情况？嗯，对于那些不记得的人来说，第一次推特末日发生在推特的唯一标识符达到 2147483647——32 位[有符号整数](https://web.archive.org/web/20230403221519/http://en.wikipedia.org/wiki/Integer_%28computer_science%29)的时候。这个数字导致一些第三方应用程序开始将推特标识符计算为负数，把它们搞砸了。这个新的推特包类似，只是它是针对 32 位*无符号*整数值 4，294，967，295。

如果你还没有意识到的话，这两个数字之间的鸿沟如此之快地被跨越的事实再次表明 Twitter 正在非常迅速地增长。尽管 tweet 与唯一 ID 的比例不是 1 比 1，但这个数字在短短几个月内就翻了一番(增加了 20 多亿)，这是非常巨大的。

那么你能做些什么来准备自己呢？Twitter API 团队建议开发使用 64 位整数的应用程序，这样在应用程序遇到这些整数之前，可以识别的 tweets 数量就会越来越多。
