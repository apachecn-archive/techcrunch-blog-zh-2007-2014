# Zuora 的目标是成为在线计费的销售力量；贝尼奥夫对此表示赞同

> 原文：<https://web.archive.org/web/https://techcrunch.com/2008/05/20/zuora-the-salesforce-for-online-billing-launches/amp/>

太平洋时间 2008 年 5 月 20 日上午 4 点

[<amp-img src="https://web.archive.org/web/20230404232110im_/https://techcrunch.com/wp-content/uploads/2008/05/zuora_thumb.png" class="shot2 amp-wp-enforced-sizes i-amphtml-layout-intrinsic i-amphtml-layout-size-defined" layout="intrinsic" i-amphtml-layout="intrinsic"><i-amphtml-sizer class="i-amphtml-sizer"></i-amphtml-sizer></amp-img>](https://web.archive.org/web/20230404232110/https://techcrunch.com/wp-content/uploads/2008/05/zuora_shot.png)

由来自 [Salesforce](https://web.archive.org/web/20230404232110/http://www.salesforce.com/) 和 [Webex](https://web.archive.org/web/20230404232110/http://www.webex.com/) 的业内资深人士运营的 SaaS 初创公司 Zuora ，由 SaaS 本人的代言人 [Marc Benioff](https://web.archive.org/web/20230404232110/http://www.crunchbase.com/person/marc-benioff) 支持，今天将推出其在线计费解决方案。

正如 Erick [三月份在](https://web.archive.org/web/20230404232110/https://techcrunch.com/2008/03/13/benchmark-and-benioff-put-65-million-into-zuora-to-create-a-salesforce-for-online-billing/)解释的那样，Zuora 的目标是缓解在线业务开发自己的计费系统的需求，特别是处理像订阅相关的经常性支付。

其所谓的“Z-计费平台”今天上线，处理四个主要的计费相关领域:客户账户和订阅、产品目录、计费操作和订单管理。整个产品以按需解决方案的形式提供给 la Salesforce。在线企业只需要配置他们的 Zuora 帐户，从他们的旧计费系统导入数据，并通过一组 API 插入他们的网站。在他们的网站上购买商品或订阅服务的客户将由 Zuora 处理，该公司跟踪订单、发票和付款。

自然地，Zuora 选择了一种类似于公用事业的定价模式。该公司将收取所有发票金额的 2%,随着支付金额的增加，这一比例会增加，最终会对特别昂贵的项目完全封顶。

这家初创公司已经注册了六个客户端，其中三个已经实现了该系统，但只有一个被披露: [Coremetrics](https://web.archive.org/web/20230404232110/http://www.coremetrics.com/) ，它提供 [Omniture](https://web.archive.org/web/20230404232110/http://www.omniture.com/) 般的网络分析。首席执行官 Tien Tzuo 表示，Coremetrics 很好地展示了 Zuora 计费系统的功能，因为它需要 27 种不同的定价模式，每种模式都必须得到适当的处理。

由于 Zuora 的创始人来自 SaaS，你可以预期他们最初会与其他 SaaS 公司合作。但是，平台并不局限于此类；它有可能实现从音乐流到在线约会的广泛服务。
