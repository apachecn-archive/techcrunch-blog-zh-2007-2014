# 通过 API 协议 TechCrunch，Intuit 将其 QuickBooks 会计软件与 Square 的销售点产品整合在一起

> 原文：<https://web.archive.org/web/https://techcrunch.com/2013/09/24/intuit-integrates-its-quickbooks-accounting-software-with-squares-point-of-sale-products-via-an-api-deal/>

今天早些时候， [Intuit](https://web.archive.org/web/20221208051644/http://www.intuit.com/) [宣布了](https://web.archive.org/web/20221208051644/http://about.intuit.com/about_intuit/press_room/press_release/articles/2013/IntuitLaunchestheCompletelyNewQuickBooksOnline.html)其 [QuickBooks](https://web.archive.org/web/20221208051644/http://quickbooks.intuit.com/) 会计软件的全新版本，并宣布了一个新的、颇具开创性的合作伙伴关系。它正在与移动支付巨头 Square 整合，以便使用移动支付服务的小企业可以自动将这些交易的数据输入他们的账户。

这笔交易的财务条款没有披露，但 Intuit Small Business Financial Solutions 的高级副总裁兼总经理 Dan Wernikoff 向我证实，这肯定是一项“商业安排”。整合将于 11 月 19 日正式启动，届时将公布服务的定价。这项服务目前将只在美国提供，尽管 QuickBooks 在全球的足迹很大，看看 Square 未来是否会以任何方式使用它将是一件有趣的事情。

虽然 Square 还没有宣布一个正式的 API 供第三方自由集成其服务，但这正是 Intuit 用来将 Square 集成到 QuickBooks 中的。“有一个 API，它正被用于这种集成，”Wernikoff 告诉我。

Square 的一位发言人表示，该公司目前没有透露更多关于整合的细节。“我们将在 11 月份分享更多，”她说。这是指所使用的 API 的更多细节，还是关于这项服务的定价，还不清楚。

用他的话来说，这意味着“当你用方形设备进行销售时，这种协调会自动反馈到 QuickBooks 中，”他说。“这是很好的补充；这是 QuickBooks 已经在做的事情的延伸。”

对这两家公司来说，这是一个巨大的变化。Intuit 自己开发了名为 GoPayment 的销售点产品，配有一个与移动设备相连的加密狗(右图)。

虽然这在过去被定位为 Square 的竞争对手，但今天 Wernikoff 的定位有所不同。他表示:“Go 支付在我们的支付业务中所占的比重越来越大，我预计这一趋势将会持续下去。”。“但我们所有人都处于移动支付的第一章，这是一个非常基础的阶段，重点是模仿销售点设备。第二章是我们每个人都开始朝着不同的方向前进。”

事实上，他说 Square 是 Intuit 的第一个自然的集成合作伙伴，因为两家公司实际上有很多重叠，已经有了许多相同的客户。(由于 Square 没有公开分享其数据，因此没有详细说明这需要多少客户，但他表示，目前约有 400 万家企业在使用 QuickBooks。)

就其本身而言，QuickBooks 正在走向成熟，并在寻找接触新客户的新途径。“我们一直在向一个更开放的平台发展，而不仅仅是一个应用程序，”他说。他指出，Square 是与另一家公司的金融服务的第一次大规模整合，但不会是最后一次。(其他已经存在的合作关系包括与 Salesforce 的交易。)

对于 Square 来说，加入 QuickBooks 等服务是该公司希望扩大其与消费者接触点并使其服务更有用(从而被更多人使用)的又一迹象。

“许多小企业主和 Square 客户经常谈论会计是他们经营业务的一个极其重要的部分，”Square 发言人告诉我。“这种集成使企业主可以轻松地使用 Square Register 或 Square Market 进行支付，并使用领先的小型企业会计软件进行记账。”

目前，Intuit 的服务是一次性购买 249.95 美元( [Mac 版](https://web.archive.org/web/20221208051644/http://quickbooks.intuit.com/mac/))或每月 12.95 美元起的 SaaS 式在线 QuickBooks。
