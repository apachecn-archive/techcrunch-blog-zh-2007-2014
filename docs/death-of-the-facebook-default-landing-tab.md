# 脸书时间表页面扼杀营销功能:默认登录标签 TechCrunch

> 原文：<https://web.archive.org/web/https://techcrunch.com/2012/02/29/death-of-the-facebook-default-landing-tab/>

在漂亮的封面和大头针固定的帖子中，页面重新设计的新脸书时间表有一个缺点。页面不能再显示他们的自定义应用程序作为他们的默认登陆标签，非球迷第一次看到他们的访问。这一强大的营销功能让 Pages 建立了一个欢迎应用程序，可以提供竞赛或优惠券等特殊内容，但需要用户喜欢该页面才能访问。

没有默认的登陆标签，非粉丝不得不主动点击被页面遮盖的小应用程序图标。许多人不会，这将花费营销人员的喜欢，以及电子邮件注册，竞赛报名，和其他关键绩效指标。这一改变是高尚的，它优先考虑用户体验和网站的长期健康，但我交谈过的一些营销人员已经在抱怨了。

【更新:这个功能夕阳毕竟没那么差。我刚刚从一个消息来源得知，默认的登陆标签只占页面应用总流量的 10%。90%来自已发布的链接和广告，没有默认的登录选项卡功能，它们的功能仍然相同。脸书什么时候做出了不会引起一些人抱怨的改变？

最后，这对用户体验有好处，它最多只会导致页面应用流量下降 10%，页面使用应用的大部分方式保持不变。受影响最大的将是那些依赖随机入站流量的小页面，他们可能是杜平不知情的用户，无法支付广告费用。这篇文章经过了大量的编辑，包含了这一观点。]

默认登陆标签被广泛认为是脸书页面营销策略。他们让页面完全控制潜在粉丝访问时看到的第一样东西。虽然 Timeline 的封面图片确实允许在不需要第三方应用程序的情况下增强品牌，但它也有一些限制。正如我在对 Pages 功能的所有新脸书时间轴的分类中所解释的，“封面可能不会显示行动号召或对脸书功能的引用，如“喜欢这个页面”，购买或定价信息，如“40%折扣”或“在我们的网站下载”，或联系信息，如网址。”

对于试图为其应用程序带来有机页面流量的品牌来说，现在唯一的替代方法是将他们的欢迎应用程序或其他重要应用程序交换到封面下方的一个小磁贴中，然后在他们的时间线提要顶部反复钉上该应用程序的链接。这些需要用户操作的策略不会像默认的登陆标签那样带来那么多的印象和参与。相反，它将增加对新闻源和广告的依赖，作为页面应用流量的驱动因素。

【T2![](img/9ee7a1d85a150a75d9f82e7342e670c5.png "Alternative To Default Landing Tab Done 4 5")

脸书的目标可能是让页面更多地讲述故事，而不是产品销售。而且成功了。页面看起来很漂亮，感觉不像广告牌或谄媚的营销手段。用户不会被愚弄，认为他们不能张贴到一个页面的墙上或查看它的信息或照片，除非先喜欢。他们与一个品牌的第一次互动将是看到它的有机内容和他们的朋友在说什么。这对用户来说是件好事，它会让脸书成为人们想去的地方。

在上市前夕，脸书对其用户保持忠诚是值得称赞的。但是默认登陆标签的消亡对积极的营销人员来说是不利的。这可能会证实潜在股东的担心，即脸书不会把他们放在第一位，但话说回来，这可能会激发更多广告的使用，那些寻求投资的人不会介意。
