# 网络集成媒体播放器 Songbird 发布 v0.3

[![](img/587b2dbf1fa340cb1e4b1e953d9ec77f.png)](https://web.archive.org/web/20221209134521/http://www.songbirdnest.com/)

当你用网络浏览器穿越 iTunes 时，你会得到什么？你得到了 [Songbird](https://web.archive.org/web/20221209134521/http://www.songbirdnest.com/) ，这是一个建立在 Mozilla Firefox 核心技术之上的应用程序，为那些希望与桌面媒体播放器紧密集成的媒体网站提供了一个平台。

我们上一次写 Songbird 是在一年半以前，当时发布了 0.1 版本。今晚，Songbird 发布了 0.3 版本，该公司认为它相当于 Firefox 1.0。

[![](img/3c6e46164304a5855163b0fc67f916da.png)](https://web.archive.org/web/20221209134521/https://beta.techcrunch.com/wp-content/uploads/2007/10/songbird_shot.png)

也许解释 Songbird 的最佳方式是将它与苹果公司对 iTunes 所做的进行比较。虽然 iTunes 是一个必须下载和安装的传统桌面应用程序，但 iTunes 中的 iTunes Store 实际上是一个与 iTunes 无缝(且独家)结合的网站，以至于当你寻找音乐购买时，你不会真正意识到你正在浏览一个网站。点击购买一些音乐，iTunes Store 网站与 iTunes 桌面客户端通信以触发某些功能，如用户提示和文件下载。苹果能够将整个购物体验包含在其媒体播放器中，使得新音乐的消费非常方便。

现在假设你是一家媒体公司，想要在用户的媒体播放器中建立商店，就像苹果在 iTunes 中建立商店一样。你可以尝试为 iTunes 拼凑一个插件，但你不会得到同样的结果，而且你会受到专有软件的影响。有了 Songbird，你不仅可以使用一个设备齐全的桌面媒体播放器，还可以使用一套开放的 API，你可以轻松地将现有的网站与 Songbird 结合起来。在大多数情况下，开发人员只需要对他们的网站进行编程，以利用 JavaScript 中的 Songbird DOM 对象。你也可以像为 Firefox 一样为 Songbird 构建主题和插件，因为应用程序的架构基本上是相同的。

[![](img/a50b616c6bf333ec8952113ccd7fdd96.png)](https://web.archive.org/web/20221209134521/https://beta.techcrunch.com/wp-content/uploads/2007/10/songbird_shot2.png)

Songbird 向 Mozilla 技术寻求产品，因为它想要操作系统不可知论。此外，其他可能的网络到桌面平台，如 AIR，不是用开源理念创建的，也不是为大规模项目设计的。Songbird 设想其平台不仅用于曲目下载服务，还可用于基于视频等任何类型媒体的服务。

[炒作机器](https://web.archive.org/web/20221209134521/http://hypem.com/)、 [Insound](https://web.archive.org/web/20221209134521/http://www.insound.com/) 和 [SkreemR](https://web.archive.org/web/20221209134521/http://www.skreemr.com/) 已经为 Songbird 编写了他们的网站，并打包了最新版本。我在他们网站上看到的一个播放器截图也表明一个我喜欢的整合正在进行中。

2006 年 12 月，Songbird 获得了红杉资本(Sequoia Capital)和 Atlas Venture 的 800 万美元投资。到目前为止，该软件已经被下载了大约一百万次。该公司的开发人员此前曾致力于 Winamp 和雅虎音乐引擎。
