# 朱莉·安·霍瓦特描述了她退出 GitHub 背后的性别歧视和恐吓

> 原文：<https://web.archive.org/web/https://techcrunch.com/2014/03/15/julie-ann-horvath-describes-sexism-and-intimidation-behind-her-github-exit/>

工程师[朱莉·安·霍瓦特](https://web.archive.org/web/20230406033136/https://twitter.com/nrrrdcore)退出编程网络 GitHub 引发了另一场关于科技和创业女性的讨论。她声称自己在 GitHub 面临性别歧视的内部文化，这让一些人感到惊讶，因为她曾经为这家初创公司辩护，而且[在该公司的内部工作](https://web.archive.org/web/20230406033136/http://passion-projects.github.com/)是为了提升女性在技术领域的地位。

在她离职的最初推文中，霍瓦特并没有详细说明她为什么离开这家估值很高的创业公司，或者是谁创造了导致她离开并公开批评这家公司的条件。

霍瓦特向 TechCrunch 提供了她对事件的看法，这个故事包含了对 GitHub、其内部政策和文化的严重指控。这种情况比一个人的奋斗更重要:霍瓦特的故事讲述了科技行业中许多未被充分代表的群体的感受和经历。

霍瓦特表示，她于 2012 年加入 GitHub，当时该公司“还很小”，其文化支持女性员工。她说，当时她是团队中唯一的女性设计师或开发人员。尽管 GitHub 的环境总体上对女性友好，但霍瓦特在给 TechCrunch 的一封电子邮件中写道:“我真的很难适应这种文化，在 pull 请求上咄咄逼人的沟通，以及与我一起工作的男性对我的意见的尊重和重视。”。

霍瓦特为什么为 GitHub 工作？她“喜欢 GitHub 的想法，因为它是人们为制造东西的人制造东西的地方。”

有鉴于此，霍瓦特告诉我们，她“一加入就加入了男孩俱乐部”，但当她的“角色开始在不适当的地方被讨论，如在拉请求和问题上”，情况发生了变化。

简而言之，霍瓦特表示，她觉得自己在公司内部受到了不同的待遇，仅仅是因为她的性别，而不是工作质量。她称同事们对她自己和其他 GitHub 女性员工工作的反应是一个“严重的问题”尽管 GitHub 雇佣了更多的女性开发人员，但霍瓦特说她很难感受到受欢迎。

一位创始人的妻子在邮件中没有提到她的名字，这让本已困难的局面雪上加霜。霍瓦特说，她尽力与创始人的妻子和创始人保持距离，因为害怕陷入不健康的状况。但是，正如她告诉 TechCrunch 的那样，她“行动不够迅速”

创始人的妻子邀请霍瓦特出去喝酒，她同意了。用她自己的话来说:“我当然同意，因为她是我老板的妻子，我一直在寻找可以尊敬的女性。”

据霍瓦特说:“我一见到她，我以为很随意的谈话几乎立刻变成了非常不恰当的事情。她开始告诉我她如何告知她丈夫在 GitHub 的决策，我如何最好不要离开 GitHub 并写一些关于他们的不好的东西，以及她丈夫如何告诉她，她应该干预我的关系，以确保我“非常快乐”，这样我就不会辞职并说一些关于她丈夫公司的坏话，因为“他工作非常努力”"

我们正在等待 GitHub 对这些指控的评论，GitHub 表示正在调查此事。不清楚为什么这位创始人或他的配偶似乎感觉受到了霍瓦特就业的威胁。

据霍瓦特称，这位妻子继续声称她负责 GitHub 的招聘，并要求霍瓦特向她解释她在做什么。根据霍瓦特的说法，这位妻子还声称在 GitHub 内部雇佣了“间谍”,并声称能够阅读 GitHub 员工的私人聊天室日志，而只有员工才能访问这些日志。

霍瓦特恰当地称这种情况为“香蕉”

在给 TechCrunch 的电子邮件中，霍瓦特说她“想到一个不在我公司工作的女人在幕后操纵，感到困惑和侮辱。”她还说，她觉得有人对她的个人关系和在 GitHub 的职业有明显的权力和影响力，让她感到受欺负。

回想起来，霍瓦特说她觉得她应该在这件事后递交辞呈。

然后，霍瓦特告诉她的搭档，也是 GitHub 的员工，正在发生的事情。她警告他不要接近创始人和他的妻子，并要求他不要向他们传递信息。据霍瓦特说，她的伴侣“同意这是最好的”他已经和创始人的妻子谈过了，她同意给霍瓦特一些空间。

这个问题并没有平息，霍瓦特收到了 GitHub 人力资源部的一个会议请求，并被要求“转达在办公室外进行的私人谈话的细节”霍瓦特回忆说，她“对此感到不舒服，但还是尽了最大努力。”她的搭档也被要求转述过去的事件。

据霍瓦特说，电台沉默了一个月，而有传言称，这位创始人正在询问其他员工关于她的情况，以及她与伴侣的关系。对霍瓦特来说，沉默让她认为自己是“被胁迫离开的”

在这一点上，霍瓦特说她开始感到威胁。她说，把自己的私人关系拖进工作生活，并在同事面前炫耀，这让她很不舒服。前述的妻子开始了一种消极攻击的行为模式，包括坐在霍瓦特旁边，用她告诉 TechCrunch 的话来说，“故意恐吓”她。

当创始人要求见她时，这种僵局结束了。霍瓦特说，她“不会把自己置于那样的境地，所以如果我们见面，我要求人力资源部到场。”会议进行得不顺利。

据霍瓦特说，这位创始人指责她威胁他的妻子，自从妻子请她出去喝酒后，她就没有和她互动或联系过。在这一集里，霍瓦特哭了，因为她说创始人不仅“惩罚”了她，还说她是“骗子”霍瓦特说，这位创始人在会议结束时说，与同事约会是“糟糕的判断”(指的是她与 GitHub 另一名员工的关系)，然后离开了。霍瓦特回忆说，在他离开后，他坐在那里“无法控制地哭泣和颤抖”

霍瓦特后来得知，这位创始人与她的合伙人有过类似的谈话，并要求他辞职。她的搭档还在公司。

在霍瓦特看来，她的选择是有限的，因为“人力资源和其他创始人允许这种情况发生，即使在知道他和她的行为之后。”不清楚她可以向谁求助。

在上述过程中，霍瓦特与另一名 GitHub 员工发生了她所说的尴尬、近乎挑衅的遭遇，这名员工要求自己过来“谈谈”，然后表白了自己的爱，当他被要求离开时“犹豫不决”。据霍瓦特说，霍瓦特当时正处于一段忠诚的感情中，这位员工很清楚这一点。

另一名员工的拒绝导致了 GitHub 的内部斗争。据工程师霍瓦特说，“被拒绝让我很受伤，开始被动-主动地从我们一起工作的项目中撕掉我的代码，甚至连一声 ping 或一句评论都没有。我甚至不得不恢复他的一些行为。我会做一些事情，上床睡觉，醒来发现我的工作没有任何解释。”根据霍瓦特的说法，这名员工“在 GitHub 很受欢迎”，而且“在社区中也很受欢迎”

根据霍瓦特的说法，他“对女性员工的行为，尤其是那些他认为是机会的行为令人厌恶。”

寻求从上述复杂性中创造一些积极的东西，霍瓦特决定启动[激情项目](https://web.archive.org/web/20230406033136/http://passion-projects.github.com/)，她现在声称这一倡议“不仅仅是为了解决技术问题”，也是为了“解决 GitHub 问题，并加强对可能经历类似事情的女性的支持网络”

然而，内部情况并未改善。霍瓦特称接下来的时期“更丑陋”她说，创始人的妻子继续出现在办公室，坐在她旁边，长时间地“盯着”她，“好像试图引起反应。”在一段时间后，“花了很长时间在女厕所哭泣”，霍瓦特与另一位“同情”的创始人交谈，并承诺与另一位创始人及其妻子“解决”这一情况。

第一个创始人要求见面。霍瓦特接受了。创始人道歉，并承认“不恰当地升级了局势。”他说，他的妻子从那时起将在家工作。需要说明的是，妻子并不是 GitHub 的员工。

妻子没有离开，而是在第二天出现，并且“就站在”霍瓦特的面前。霍瓦特联系了很多“很忙”但最终愿意帮忙的高管。

这种情况不仅仅是被动的痛苦；据霍瓦特说，它变得致命:

> 接下来我所知道的是，那位妻子在我的工作岗位上当着我的面对我进行口头攻击。她要求和我私下谈谈，我拒绝了。我非常平静地请她别烦我，并告诉她她让我不舒服。这一事件有目击者。
> 
> *我吓得直哆嗦，感觉自己的肾上腺素比以往任何时候都更猛。不过，我为自己没有做出反应而自豪。*
> 
> *我决定在家工作一段时间。*

人力资源部最终要求妻子不要和霍瓦特在同一层楼。但根据霍瓦特的叙述，“她继续寻找进入的方法，并把自己放在我工作的地方旁边。”这种情况一直持续到上周四她离开公司。

当霍瓦特看到男人们呆呆地看着在办公室里跳呼啦圈的女人时，她忍无可忍了。她称这一集是“非常丑陋和不恰当的一幕”她的话:

> 两个女人，其中一个是我的同事，也是我崇拜的人，还有她的一个朋友正在随着音乐跳呼啦圈。我对此没有意见。我确实有一个问题，那就是一排人坐在一张长椅上，面对着呼拉圈，呆呆地看着他们。它看起来像是从脱衣舞俱乐部出来的东西。当我向男同事提出这个问题时，他们并不觉得有什么问题。但对我来说，这感觉不安全，说实话，真的很尴尬。那一刻，我决定最终离开 GitHub。

霍瓦特告诉 TechCrunch，她认为自己“可以修复 GitHub”她现在声称她错了。

在与人力资源部合作获得并宣布简单的离职后，有人在秘密社交网络上发布了一篇关于她任期的非常不友好的帖子。当时，她的离开还是保密的。她联系了 Secret，要求撤掉这个帖子。霍瓦特声称，最初并没有撤下这个帖子。TechCrunch 已经秘密证实它最终被移除了。

秘密职位是导致霍瓦特“直言不讳”的原因霍瓦特说，她希望有人最终被追究责任，而不是寻求关注。

GitHub 表示正在调查此事:“我们正在调查此事。”
