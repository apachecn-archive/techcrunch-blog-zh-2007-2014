# 啊哦。现在你可以通过 Twitter 向 Techmeme 推荐故事

有一种“上 [Techmeme](https://web.archive.org/web/20221006151700/http://techmeme.com/) 的新方法”。它叫 [Twitter](https://web.archive.org/web/20221006151700/http://twitter.com/) ，你听说过吗？

你只需要在微分享服务上发送一条以“Tip @Techmeme”或“Tip @TechmemeFH”开头的消息。该网站从网上收集新闻故事，确定哪些故事值得在网站上展示，并检测故事之间的联系和关系，将考虑将文章发布给许多技术狂热分子和专业新闻记者。您还可以用您的 Twitter id 获得积分！

当然，这是 Techmeme 最近雇用一名人类编辑的直接结果，这本质上归结为热门网站[放弃全自动新闻](https://web.archive.org/web/20221006151700/http://www.beta.techcrunch.com/2008/12/03/techmeme-gives-up-on-fully-automated-news/)。

根据宣布新功能的博客帖子，小费将通过自动和手动方式的结合来处理，尽管我们不太确定这是什么意思。帖子中的问题部分只提到，由于有了 Twitter 搜索 API，提示会被“迅速找到”。

Techmeme 创始人 [Gabe Rivera](https://web.archive.org/web/20221006151700/http://www.crunchbase.com/person/gabe-rivera) 补充说，他们将能够获得最常见的 URL 缩短服务的目标 URL，他们将能够抵制垃圾邮件，并且提交自己的故事通常不是一个好主意(除非在不明确的、罕见的情况下)。

问题是:为什么 Techmeme 首先需要这个？它不像是一个实际发布内容的新闻网站，让某些故事自动突出的算法在过去已经证明了它的有用性，那么为什么要依赖用户提交的好东西呢？

这项服务确实偶尔会出错，但对于一个曾经以完全自动化而自豪的网站来说，众包标题收集是一个奇特的举动。

上面写着:嘿，Techmeme，给你一个[提示](https://web.archive.org/web/20221006151700/http://twitter.com/erickschonfeld/status/1156532181)。
