# 哎呦 Redux:看起来合作伙伴刚刚泄露了谷歌网站启动的+1 按钮

哦，谷歌。我不想再这样做了*，但你真的需要锁定这些合作伙伴，并让你的公共关系井然有序。*

 *在[我们从一家 NFC 合作公司](https://web.archive.org/web/20230209125038/https://techcrunch.com/2011/05/25/vivotech-google-nfc/)那里获得一个项目后的一周，几乎确认了谷歌在纽约市的活动将是关于他们的移动支付系统([谷歌钱包](https://web.archive.org/web/20230209125038/https://techcrunch.com/2011/05/26/google-wallet-offers/))，另一个合作伙伴在*另一个*声明之前已经联系了我们。这一次，一封电子邮件似乎证实了谷歌将在明天推出他们的网站+1 按钮。

事情是这样的:谷歌早些时候联系我们，看我们是否有兴趣获得一些禁运下的“重大”新闻。我们讨厌禁运，但谷歌通常擅长控制禁运，所以我们不时地合作(加上我们“有时接受禁运，有时不接受”的政策让每个人都保持警觉)。不管怎样，我们同意听听谷歌的新闻。但是他们实际上还没有告诉我们新闻是什么，因为我们的简报要到今天晚些时候才开始。所以我们不可能知道我们被禁运的原因。所以我们没有被禁运。

此外，如果合作伙伴主动给我们发送电子邮件，告诉我们将要宣布什么，那么所有的赌注都将被取消。

在这种情况下，Clearspring 的公关公司 New Venture Communications 写信给我们说:“我知道谷歌计划就他们即将推出的网站的新 Google +1 按钮与你联系。我想你可能有兴趣与 Clearspring 的首席执行官 Hooman Radfar 谈谈该公司作为 Google +1 的启动合作伙伴的角色。”

因此，我们可以期待明天会有一个+1 按钮，Clearspring 的 AddThis 工具会支持它。更多一点:

> +1 按钮旨在让用户能够+1 内容，从而帮助 publisher 的内容在搜索结果中脱颖而出。用户获得+1 内容的机会越多，出版商的 URL 就越有可能出现在带有+1 注释的搜索结果中。这有助于用户强调哪些结果是最有用的，并帮助出版商更好地吸引他们的观众。

你可能还记得，本月早些时候，[谷歌通过一个广告“意外”泄露了他们的+1 Chrome 扩展](https://web.archive.org/web/20230209125038/https://techcrunch.com/2011/05/04/google-plus-1-extension/)。该扩展仍然不可用。+1 按钮略有不同，因为出版商可能会将它嵌入到自己的网站上。鉴于这可以[改变谷歌搜索结果的提供方式](https://web.archive.org/web/20230209125038/https://techcrunch.com/2011/03/30/google-plus-one/)，这可能是一件大事。

据推测，我们将很快被禁止所有这些，所以期待明天听到更多。呵。除非有合作伙伴先给我们发邮件了解更多详情！*
