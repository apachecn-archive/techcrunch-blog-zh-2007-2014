# 游戏后期，VMware 是反对 OpenStack 还是拥抱它？

> 原文：<https://web.archive.org/web/https://techcrunch.com/2013/08/19/late-to-the-game-is-vmware-turning-against-openstack-or-embracing-it/>

VMware 面临着一个新的现实，这在几年前似乎是不可能的。虚拟化之王现在面临着一个不断增长的云公司社区，这些公司正在竞争从大大小小的 IT 商店那里赢得业务。

竞争变得如此紧迫，以至于该公司做了一些可以被视为增加任何新生运动可信度的事情:他们挑出了那些构成最大威胁的运动。3 月是亚马逊网络服务(AWS)。现在是 OpenStack，这是一个用于构建云基础设施的开源环境，CEO 帕特·基尔辛格上周在接受[网络世界](https://web.archive.org/web/20221006045657/http://www.networkworld.com/news/2013/081413-vmware-ceo-openstack-is-not-272867.html)采访时提出了这一点:

> “我们相信，OpenStack 会在哪里被采用？”盖尔辛格说。“我们没有看到它在企业中取得巨大成功，因为它是构建云的框架。人们已经在很大程度上采用并部署了非常大的 VMware，而转换成本等并不是特别有效。我们认为它在云提供商、服务提供商方面非常有效，这是 VMware 过去没有太多业务的领域，因此，我们认为我们的战略为我们开辟了一个全新的市场。”

我研究了 VMware 实际在做什么。我阅读由 [Gartner 分析师](https://web.archive.org/web/20221006045657/http://blogs.gartner.com/lydia_leong/2013/05/21/vmware-joins-the-cloud-wars-with-vcloud-hybrid-service/)撰写的帖子，以及由领域专家撰写的独立博客。虚拟化极客拥有影响者阵营中最酷的理论之一。忽略宣传——VMware 有其他意图，主要是在 OpenStack 上孤注一掷:

> 随着 Grizzly 版本 OpenStack 的发布，新的 VMware 驱动程序将支持 vCenter 和 OpenStack 之间的互操作性。[VMware/Nicira 的 Scott Lowe](https://web.archive.org/web/20221006045657/http://blog.scottlowe.org/) 演示了 OpenStack 与 vSphere 环境的集成，其中包括 ni cira 网络、VMware HA、DRS 和 vCenter。我认为视频本身就说明了 VMware 在与 OpenStack 的关系中的发展方向。
> 
> 看到这种情况，我进一步怀疑 VMware 是“all in”open stack。如果 VMware 发布它自己的 OpenStack 包——如果你愿意的话，我不会感到惊讶。为什么不是所有其他主要的 OpenStack 玩家都有自己的发行版。为什么不选择具有高度集成的 vSphere“附加组件”的 VMware 呢？理想情况下，这是销售更多“VSP here Enterprise Plus+open stack”SKU 的方法。

当然，VMware 的一群人出来说[云代工厂](https://web.archive.org/web/20221006045657/https://beta.techcrunch.com/2013/07/23/ibm-standardizes-on-cloud-foundy-the-open-source-developer-platform/ "IBM Standardizes On Cloud Foundry, The Open-Source Developer Platform")是炸弹，对于作为其开发平台的 VMware 混合云服务将是重要的。VMware 最初开发了 Cloud Foundry 平台即服务(PaaS ),然后将其分拆出来创建 Pivotal。

SAP 高管 Sanjay Poonen 一如既往地对他的新雇主 VMware 的未来感到兴奋。他建议我们下周在 VMworld 上谈。我同意。

我写博客有几个原因。最重要的是，VMware 真的来晚了，OpenStack 是由开发人员推动的，他们正在为公共和企业云服务构建基础架构。VMware 接受它是有意义的。

亚马逊网络服务已经存在七年了。VMware 正在尽一切可能阻止其客户寻求它或其他更实惠的服务，这些服务可以为开发者和 it 提供更多。

在这一点上，Gelsinger 没有太多的选择，只能保护自己的基础并质疑 OpenStack。但也许不是？他是在暗示对 OpenStack 的更深支持吗？他的客户有各种各样的选择，所以必须在一切都太晚之前采取措施。

与此同时，云基础架构的市场增长势头不减。

[MorphLabs](https://web.archive.org/web/20221006045657/http://morphlabs.com/) 已经为 OpenStack 开发了一个管理控制台，并将其大部分注意力集中在与菲律宾、印度尼西亚和新加坡的几家电信客户一起拓展亚太市场上。

这些大多是公共服务，但它标志着分析师预测将发生的转变。电信公司开始与 OpenStack 结盟，这为构建公共服务和允许企业拥有自己的虚拟基础设施创造了机会。

VMware 的混合云使该公司能够提供更加动态的服务，但它并不一定是公司如何构建新基础架构的答案。这看起来肯定不像是面向开发者的云感知环境。更重要的是，它与 OpenStack 所提供的完全不同。

相反，与 OpenStack 结盟将使该公司有机会继续成为新实体的提供商。这将使它在推动其对新的软件定义的数据中心的愿景的关注时获得立足点。
