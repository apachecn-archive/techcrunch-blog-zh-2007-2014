# 将 iOS 移植到 Android: Spotlight Mobile 分享陷阱和好处

如果你上周参加了(或观看了)我们的[移动优先的总结](https://web.archive.org/web/20230203041410/https://techcrunch.com/2011/07/30/a-look-back-on-our-mobile-first-crunchup-and-6th-annual-summer-party-at-august-capital/)，你会记得很多讨论都围绕着使用 iOS 和 Android 平台的[利弊](https://web.archive.org/web/20230203041410/https://techcrunch.com/2011/07/29/beauty-is-in-the-i-of-the-beholder/)。这场辩论特别有趣，因为它邀请了 Android 用户体验总监马蒂亚斯·杜阿尔特(Matias Duarte)，它涉及了从设计应用程序到每个平台允许开发者访问的数据的所有内容。

现在， [Spotlight Mobile](https://web.archive.org/web/20230203041410/http://www.spotlightmobile.com/) 的联合创始人尼克·法里纳(Nick Farina)写了一篇内容广泛(且及时)的文章，总结了他们将 [Meridian](https://web.archive.org/web/20230203041410/http://www.meridianapps.com/) 应用移植到安卓系统的经历(Spotlight Mobile 也开发了几款高端应用，包括 Barnes & Noble Bookstore)。这并不是第一篇探讨与跳转相关的问题的博客文章，但它比大多数文章都更全面，并且很好地解释了两种平台的优缺点(对于我们这些不是优秀开发人员的人来说，这也是完全可以理解的)。你可以在这里找到他的完整帖子[。](https://web.archive.org/web/20230203041410/http://nfarina.com/post/8239634061/ios-to-android)

法里纳帖子中的亮点包括:

*   “当你第一次开始使用时，你可能会讨厌*Eclipse(谷歌/安卓推荐的 IDE)，但过一会儿，“它基本上会为你编写代码。”*
*   Farina 发现 iOS 和 Java 端口之间的应用程序的许多基本结构非常相似。
*   Android 模拟器非常慢，以至于他建议买一部 Android 手机出去测试一下。
*   Farina 喜欢 Android 的布局系统，这更类似于为网页设计(因为它适应不同的屏幕大小和方向)，而不是 iOS 的像素完美系统。但他说，Android 在动画方面有一个严重的缺点，这可能会严重阻碍系统，因为它们不是用 GPU 渲染的。值得注意的是:我下载了 Meridian，它看起来相当不错，证明 Android 应用没有*有*难看。

同样，如果你是一名开发者，你可能会做得很好，在这里阅读整篇文章。这是一本引人入胜的读物。

Meridian，我们之前已经介绍过了，是一个可以让场馆创建定制的 GPS 向导的应用。例如，波特兰艺术博物馆(Portland Art Museum)目前使用该应用程序为用户提供每个展览周围的地图(各种展品的音频解说)，以及博物馆关键位置的地图，如洗手间。
