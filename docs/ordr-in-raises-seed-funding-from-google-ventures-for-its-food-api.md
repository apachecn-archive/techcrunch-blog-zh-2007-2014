# Ordr.in 从 Google Ventures 为其食品 API 筹集种子资金

我现在正在纽约市的 TechStars 演示日，12 家初创公司正在向一屋子的数百名投资者展示他们的产品(今天晚些时候我们将回顾一下 TechCrunch)。

这些公司中有一项名为 [Ordr.in](https://web.archive.org/web/20230203141307/http://ordr.in/) 的服务，它将食物的力量带给任何想要整合它的应用程序、服务或网站。或者，更具体地说，它将让这些服务集成在线餐厅订购。首席执行官大卫·布鲁姆刚刚宣布，该公司已经从谷歌风险投资公司获得了一轮种子资金，该公司由 Rich Miner 领导。

当然，现在已经有很多网络服务可以让你从当地餐馆订餐(我最近迷上了 SeamlessWeb)。问题是，市场是分散的。即使像 Seamless 和 Grubhub 这样最大的服务也没有覆盖整个美国，而且他们也没有在其支持的城市中拥有所有的餐厅。此外，还有许许多多覆盖当地的小型网站。

Ordr.in 的解决方案是与全美 72 家当地订餐网站(还在增加)合作，让用户从大约 7000 家餐馆订餐。然后，它将所有数据标准化，并通过 API 让其他应用程序和服务访问这些数据。

正是这个 API 让任何应用程序或服务集成了食品订购。比方说，网飞想集成一个选项，在你晚上看流媒体电影的同时订购晚餐。有了秩序，他们就能做到。而且他们有这样做的经济动机——他们从每笔交易中提成(Ordr.in 也是)。

该公司已经获得了一些主要合作伙伴，包括温德姆酒店(Wyndham Hotels)，该公司拥有 5000 家酒店，为 1.1 亿名客人提供服务，但没有客房服务。该酒店正在使用 Ordr.in API 来集成其 Wifi 欢迎屏幕上的点餐功能。已经有几十个应用程序使用该 API 实现了，包括 Hunch mashup 和 Boxee 应用程序。
