# 马克·扎克伯格:我们最大的错误是在 HTML5 上下了太多赌注

今天，马克·扎克伯格透露，脸书的移动战略过于依赖 HTML5，而不是原生应用。

这不仅是移动领域的一个大错误，扎克伯格还表示，它最大的错误是专注于 HTML5。这是脸书首席执行官首次公开承认这一点，但新的 iOS 原生应用前景看好。根据扎克伯格的说法，自从更新到新的 iOS 应用程序以来，人们正在消费两倍多的 feed 故事，这很好。

> 前半年在产品上有点慢，但在接下来的六个月里，我期待着许多真正酷的东西。

这种“非常酷的东西”可能会考虑货币化，因为很明显[移动是公司广告收入的途径](https://web.archive.org/web/20230312034420/https://techcrunch.com/2012/09/11/zuckerberg-says-on-mobile-we-are-going-to-make-a-lot-more-money-than-on-desktop/)。

对于一个公司来说，一头扎进采用一套特定的工具，然后迅速改变路线是极其困难的。我怀疑这正是发生在脸书身上的事情，事情至少正在好转。

去年，脸书当时的 CTO， [Bret Taylor 讨论了移动的未来](https://web.archive.org/web/20230312034420/https://techcrunch.com/2011/01/27/facebook-bret-taylor/)，当时很大程度上包括 HTML5。下面是这篇文章的一个片段，泰勒在其中讨论了这个平台:

> 这是否意味着远离闪存的发展？毕竟，Flash 主导了脸书所说的 HTML5 游戏类型的市场。“这很难，”泰勒特别提到了 Flash。当我笑着说他给出了外交回答时，他向我保证这是他们想了很多的事情。“我们希望走在曲线的前面，尽可能填补空白，”这是他最后的表述。
> 
> 对 HTML5 的这一立场对脸书至关重要，因为泰勒确实认为移动是未来——但就目前而言，这有点问题。“移动设备的受欢迎程度将会改变，”他说，暗示着今天占主导地位的设备在未来可能不会如此占主导地位。如果是这样的话，脸书为什么要向他们投入资源呢？如果他们只专注于 html 5——这将在未来越来越多的设备上工作，不是更容易吗？当然了。

正如泰勒所指出的，移动设备确实发生了巨大的变化，像苹果这样的公司将确保大公司正在编写本地应用程序，以充分利用设备的资源。
