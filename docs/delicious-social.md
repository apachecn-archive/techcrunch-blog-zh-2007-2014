# Delicious 增加了协作和抢先一步的隐私 Pinterest

[Delicious](https://web.archive.org/web/20230307213312/http://delicious.com/) 刚刚[宣布了 4 项新功能](https://web.archive.org/web/20230307213312/http://blog.delicious.com/)以使其栈或链接集合更加社会化。因为唯一比你最喜欢的小猫网站捆绑更好的是你和你的朋友共同创建的捆绑。现在，您可以协作构建堆栈，对整个堆栈进行评论，使用类似于 YouTube 响应的堆栈来响应堆栈，并创建私有堆栈。这些功能将允许新的用例，如秘密合作，并使 Delicious 比 [Pinterest](https://web.archive.org/web/20230307213312/https://techcrunch.com/tag/pinterest/) 更有优势，后者还没有提供私人董事会。

私人书库让美味不仅仅是一个供你品味的广播频道。你可以偷偷收藏链接，永远藏在心里，包括你不想让老板或妈妈看到的网站。

与新的协作功能相结合，您还可以默默收集并与他人讨论链接，然后决定向全世界展示这些链接。美味提供了一个完美的例子，告诉你如何“与你尚未宣布的未婚妻分享最喜欢的订婚戒指。”

Pinterest 还不提供私人董事会。它告诉一位 Quora 用户,该功能有一天可能会作为一项高级服务提供，但这不是一个优先事项，因为该网站专注于分享。这给了 Delicious 可乘之机。

协作功能允许您邀请朋友来编辑您的书库。这可以帮助团队合作进行研究，或者让朋友们围绕一个主题编辑一组喜欢的链接。

[![](img/4178558c598895483acf12ec2c0ed01a.png "Delicious Comment Link Suggestions")](https://web.archive.org/web/20230307213312/https://techcrunch.com/wp-content/uploads/2012/01/delicious-comment-link-suggestions.png)

对整个栈进行评论可以让你就链接之间的相互联系进行对话。你也可以建议一些你认为会增强其他人的链接。

[![](img/4041a5aa463c086c8e619d4385920d54.png "Stack Responses") ](https://web.archive.org/web/20230307213312/https://techcrunch.com/wp-content/uploads/2012/01/stack-responses.png) Stack responses 让你为谁能找到围绕某个主题的最佳内容而战，或者提供一个对比鲜明的集合。例如，如果你担心你的朋友真的会从他们糟糕的毛衣堆里买东西，说服他们不要以讽刺的名义用一堆时髦的毛衣来回应，从而破坏他们的形象。

Delicious 仍然不如脸书群等一些群共享平台灵活，也没有 Pinterest 的势头。为了保持相关性，它需要确定它的核心目的——稳定呈现精选链接，而基于 feed 和以图片为中心的服务并不支持这一点。
