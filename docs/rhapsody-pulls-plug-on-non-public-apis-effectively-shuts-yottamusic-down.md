# Rhapsody 拔掉了非公共 API 的插头，有效地关闭了 YottaMusic

[我们的一位读者](https://web.archive.org/web/20221209172917/http://bijansabet.com/post/22791839)今天向我们透露了一个备受推崇的音乐服务 [YottaMusic](https://web.archive.org/web/20221209172917/http://www.yottamusic.com/) 最近的消亡，在 [Rhapsody](https://web.archive.org/web/20221209172917/http://www.rhapsody.com/) 给了该服务 12 月 31 日停止使用其非公开 API 和数据的最后期限后，该服务于 12 月 30 日被关闭。

YottaMusic 是一款始于 2006 年的音乐网络应用，主要为 Rhapsody 的流媒体音乐集提供了一个高级界面，尽管它也具有社交网络功能，并与 Sonos 和 Last.fm 进行了播放列表集成。早在 2006 年，YottaMusic 被允许访问一组非公开的 Rhapsody APIs，允许它为音乐服务构建自己的播放器和界面，而不是使用 Rhapsody 自己的弹出播放器。

然而，今年 5 月，Rhapsody 联系了 YottaMusic，要求他们开始独家使用其公共 API，或者完全关闭这项服务。由于 YottaMusic 的 Rhapsody 接口依赖于非公共 API，并且由于采用标准 Rhapsody 播放器会抹去应用程序的大部分价值，这家初创公司最终感到被迫放弃并关闭了运营。

虽然创始人 Luke Matkins 表示，他理解 Rhapsody 完全有权采取这样的行动，但对于 YottaMusic 每天 1 万名访客来说，这显然是一个不幸的转变。不清楚为什么 Rhapsody 如此关注 YottaMusic 提供自己的播放器来播放流媒体音乐。今年 5 月，YottaMusic 甚至提出完全剥离其播放器，从而消除所有 YottaMusic 品牌的迹象。通过 YottaMusic 播放歌曲的用户也必须向 Rhapsody 会员付费，所以这并不是说有人在窃取音乐。然而，一些未公开的原因迫使 Rhapsody 有效地压制了其服务的一个受欢迎的访问点，这一举动在表面上看起来非常鲁莽。

我怀疑这里有相当多的政治因素在起作用，尽管我并不自称对在线音乐行业有特别敏锐的洞察力。音乐爱好者的另一个网络目的地[MOG](https://web.archive.org/web/20221209172917/http://www.beta.techcrunch.com/2007/12/11/mog-integrates-rhapsodys-streaming-music-collection-launches-redesign/)最近将 Rhapsody 整合到其服务中，现在使用 Rhapsody 自己的弹出播放器来播放歌曲。因此，Rhapsody 并没有不允许第三方应用程序传播它的服务；相反，玩家本身似乎是一个主要的症结。

YottaMusic 现在已经进入了 [TechCrunch 的死亡池](https://web.archive.org/web/20221209172917/http://www.beta.techcrunch.com/tag/deadpool/)。
