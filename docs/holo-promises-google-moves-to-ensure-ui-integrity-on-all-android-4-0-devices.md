# 赫萝承诺:谷歌将确保所有 Android 4.0 设备的用户界面完整性

谷歌在 Android 开发者博客上发布了一些新信息，这可能没有人们想象的那么严重，但仍然值得一看。该帖子详细说明了一项要求，即所有制造商在其 Android 4.0(冰淇淋三明治)设备中包含默认的赫萝主题。他们不需要使用主题，他们只需要在手机上有组成它的数据。

这与其说是针对第三方用户界面的打击，不如说是一种全面的完整性检查，确保应用和服务可以选择在许多设备上为用户提供一致的界面。

想象一下，你正在设计一个应用程序，它可以找到附近的餐馆，并从你的社交流中获取推荐——或者其他什么。你有你的默认按钮，你的文本域，你的动画，等等，你可以用它们为自己制作一个漂亮的小界面。一段时间以来，像 Sense 或 TouchWiz 这样的主题可以取代那些默认的 UI 项目，并替换为自己的项目——与主题的其他部分一致的配色方案或图标集。这种做法可能不是很常见(我没有数据支持这一点，但我没有遇到过很多)，但它对精心设计的应用程序具有潜在的破坏性。

谷歌现在将要求制造商在他们的发行版中包括原始的赫萝主题文件，不管是不是定制的。用不用由 app 开发者决定。你可以发出信号，表明你是否希望你的应用程序使用手机上默认的任何 UI 项目(因此，如果 HTC 手机要推出基于 Sense 的 ICS 设备，就在 HTC 手机上使用 Sense)，或者你可以选择强制这些元素使用谷歌提供的赫萝主题。

这使得开发人员可以完全确定他们的应用程序看起来会像他们想要的样子，同时允许制造商自由地创建定制的启动器、主题、小部件等等。它只是防止那些制造商影响和破坏(虽然，我没有听说这是一个主要问题)使用和依赖默认主题元素的应用程序。

这些都是直接的影响。但是想象你是一个开发者。没有什么理由不把要求使用赫萝的旗帜挂上去。你的应用程序可能看起来与主题设备上的其他一些应用程序略有不同，但 Android 上的用户界面已经随处可见，实际上，大多数人不会注意到(尽管我希望他们会注意到)一个应用程序中的文本字段有灰色渐变，而另一个有蓝色渐变。选择赫萝可以保证你的应用在所有兼容设备上看起来都一样。只是不同层面的一致性。

这并不意味着在安卓设备上需要某种外观，即使是为了访问核心的谷歌应用和市场。我不确定他们是否可以(启动器和主题可以很容易地运行在默认之上)。但这确实表明谷歌承认一致的用户界面是一个优势，并使其对开发者来说更容易。
