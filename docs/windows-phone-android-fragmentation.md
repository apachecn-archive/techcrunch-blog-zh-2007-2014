# 微软对 Android 的分裂伸出了一个漂亮的大中指

一个月前，在我关于 Windows Phone 7 的第一篇文章中，我对该平台持谨慎乐观态度，但对微软的最终意图有一些疑问。也就是:“*他们会用他们的力量试图让航母屈服于他们的意志吗？或者，他们会采用谷歌的方法，让他们到处发号施令的特点和臃肿的软件？*

我很高兴地告诉大家，至少在最初，看起来会是前者。

"*如果微软为你的手机发布了更新，你就能得到它。本期*，[艾德·博特为 ZDNet 撰写今日](https://web.archive.org/web/20230202220634/http://www.zdnet.com/blog/bott/microsoft-is-in-the-drivers-seat-for-windows-phone-updates/2592)。

换句话说，微软更多地采用了苹果的方式，而不是谷歌的方式。也就是说，看起来他们将负责向客户分发软件更新，而不是让运营商参与进来。

虽然谷歌一直在快速更新 Android，但运营商在向许多客户推出这些更新时却行动迟缓。他们对此有各种各样的理由。他们几乎都是胡说八道。

该公司告诉 Bott:“微软将向最终用户推送 Windows Phone 7 软件更新，所有 Windows Phone 7 设备都将有资格获得更新。他们非常直接的措辞听起来好像他们非常清楚 Android 上的碎片问题，以及这对消费者来说是多么令人头痛。事实上，我肯定他们是。

在微软重启其移动战略并推出 Windows Phone 7 之前，他们最初的移动操作系统 Windows Mobile 在很大程度上与 Android 处于同一条船上。运营商有更新的权力。顾客被骗了。

所以至少从那个方面来说，Windows Mobile 的失败可能帮助了微软。他们学会了，现在可以生产更好的产品。

谷歌在安卓之前从未开发过移动操作系统，所以也许他们不知道他们正在与运营商打交道(尽管你不得不相信安卓主管安迪·鲁宾应该知道)。苹果落入这个陷阱不仅仅是因为他们是控制狂。

谈到鲁宾，他认为运营商正在向客户学习，并最终会在发布更新时跟上该计划。“*我想随着时间的推移，他们会知道什么是好生意，什么是坏生意*，[他在 10 月](https://web.archive.org/web/20230202220634/https://techcrunch.com/2010/10/08/android-carriers/)时说。二十年的历史表明了相反的情况，但不管怎样。

微软在这方面做出了正确的决定。用鲁宾的话来说，[这个世界可能不*需要*另一个平台](https://web.archive.org/web/20230202220634/https://techcrunch.com/2010/10/08/andy-rubin-windows-phone-7/)，但拥有另一个不受运营商控制的平台肯定是件好事。
