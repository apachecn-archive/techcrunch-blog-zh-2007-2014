# JanRain 通过长尾推动 OpenID 的采用

![](img/a2fa1a4dedfd1f55a327267dd4b4ed9b.png)

一些最受欢迎的 OpenID 软件库和一个名为 [Pibb](https://web.archive.org/web/20221006141421/http://www.pibb.com/) 的类似论坛的交流工具的创造者 JanRain 已经发布了一个新的 SaaS，为那些想成为 OpenID 依赖方的网站提供服务。

我们被告知，这项服务，简称为 [RPX](https://web.archive.org/web/20221006141421/http://www.janrain.com/products/rpx) ，可以在一天内开始接受拥有 OpenID 账户的用户。这实际上是 JanRain 提供的第二个 SaaS 解决方案，第一个类似地命名为 [OPX](https://web.archive.org/web/20221006141421/http://www.janrain.com/products/opx) ，它让网站做相反的事情:向用户提供 OpenID 帐户，然后用户可以登录任何其他接受它们的网站。JanRain 还通过其 [myOpenID](https://web.archive.org/web/20221006141421/http://www.janrain.com/products/myopenid) 服务直接向用户提供 OpenID 账户。

帮助网站成为依赖合作伙伴比帮助他们成为提供合作伙伴更重要。这是因为很少有受欢迎的网站接受 OpenID，因此，消费者没有理由为自己设立 OpenID 账户。这是一个比过去几年困扰该运动的用户体验问题更大的问题。

RPX 正被推向希望提高注册对话率、从其他地方导入用户信息以及通过 oAuth 建立与其他社交服务的连接的中型网站。对于 Blogger、Plaxo 和 AOL 等大型互联网网站来说，这并不意味着什么，这些网站已经成为使用其内部技术资源的依赖方。

问题在于，OpenID 是否会通过长尾效应获得动力，或者是否会被足够数量的大玩家采用。这可能需要一些非常受欢迎的服务，如 MySpace 和脸书(通过他们各自的数据可用性和连接服务)，来普及该协议。但是一旦他们这样做了，像 RPX 这样的服务应该会帮助长尾利用它。

RPX 有两种风格:小型网站的“plus”和大型网站的“pro”。定价从固定费用开始，然后根据一年内有多少人使用 OpenID 登录您的网站而增加。
