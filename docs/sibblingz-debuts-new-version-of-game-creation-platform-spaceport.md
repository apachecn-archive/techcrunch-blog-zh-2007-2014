# Sibblingz 推出新版本的游戏创作平台 Spaceport

多设备社交游戏平台 Sibblingz 发布了其游戏创作平台 Spaceport 的新版本。

作为背景， [Spaceport](https://web.archive.org/web/20230205021009/https://techcrunch.com/2011/04/27/sibblingz-launches-new-version-of-multi-device-social-gaming-platform/) 允许游戏开发者使用 Javascript 构建一个游戏，并让它在所有智能手机操作系统上运行，具有类似原生应用的体验和快速的性能。基于网络的游戏作为混合 html 5-native 应用程序运行，游戏代码用简单的脚本语言编写一次，因此它们可以在任何设备上运行。开发者也可以使用 Spaceport 创建原生 iOS 应用。

因为 Adobe 宣布将停止进一步开发其移动 Flash 插件，Sibbling 希望填补 Flash 游戏开发者的空白。
该公司表示，Spaceport 3.0 是 Sibblingz 移动游戏开发和渲染平台迄今为止最先进的迭代。

Spaceport 3.0 允许开发人员继续使用 Adobe 基于 Flash 桌面的创作和内容创建工具编写游戏，同时使游戏可以立即在 iOS、Android 和 HTML5 上使用。该平台会自动将 Adobe tools 中创建的 Flash 动画作为“SWF”文件转换为 Spaceport 矢量图形。然后，Spaceport 中基于高性能 GPU 的渲染引擎在 iOS 和 Android 设备上渲染这些图像，而不是使用 CPU 的速度慢得多且耗费电池的 Flash 插件。

正如 Siblingz 的 Peter Relan 解释的那样: *Flash 开发人员非常擅长理解如何使用 Adobe 工具和 ActionScript 等脚本语言来构建 Flash 游戏。在迁移到移动设备的过程中，他们不得不放弃这些工具、语言和 API，学习新的工具、API 和核心语言，如 Objective C 和 Java，他们对此一无所知。Spaceport 允许他们使用脚本语言(Javascript，使用与 Adobe 的 Flash APIs 非常相似的 API)和完全相同的 Adobe 工具集开发游戏……通过 Spaceport，他们可以获得两个世界的最佳优势:Adobe 工具和 API 的舒适性，以及能够在所有移动设备上部署。*

Spaceport 3.0 使用 JavaScript 而不是 ActionScript 来支持 Android 和 iOS 上的高性能 HTML5 游戏，并在整体上提高了性能。该平台对开发者是免费的。

盈利的 Sibblingz 已经从使用其技术的游戏分享费用中获得了“每年数百万美元”的收入。该公司刚刚从 YouWeb 获得了 100 万美元的新资金。
