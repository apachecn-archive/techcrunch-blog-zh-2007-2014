# 融入潮流 TechCrunch

> 原文：<https://web.archive.org/web/https://techcrunch.com/2009/05/17/jump-into-the-stream/>

![](img/9c33791263b1075c6d544ad8cfe08423.png)

互联网又一次在我们眼前发生了变化。信息越来越多地以实时流而不是专用网页的形式分发和呈现。这种转变是显而易见的，即使只是在早期阶段。大大小小的网络公司都在拥抱这一潮流。不仅仅是推特。脸书和 Friendfeed 和 T2，美国在线和 Digg 和 Tweetdeck，Seesmic Desktop 和 Techmeme 和 Tweetmeme 和 Ustream，Qik 和 Kyte，博客和谷歌阅读器。这股潮流蜿蜒流过网络，并由 *nowness* 组织起来。

这个实时流已经建立了一段时间。它始于 RSS，但现在[变得更加强大和快捷](https://web.archive.org/web/20230205005724/http://www.techcrunchit.com/2009/05/05/rest-in-peace-rss/)，不仅包含定期新闻和思考，还包括持续的交流、状态更新、即时分享的想法、照片和视频。

这对我们将如何消费信息意味着什么？来自 [Betaworks](https://web.archive.org/web/20230205005724/http://betaworks.com/) 的 John Borthwick 认为实时网络是一个关键的投资机会(Betaworks [投资公司](https://web.archive.org/web/20230205005724/http://static.betaworks.com/work/index.html)包括 Twitter、bit.ly、Tweetdeck、Chartbeat 和 Tumblr)。他承认他和其他投资者仍然感觉一无所知，但他在一篇名为[“分销”的帖子中描述了他试图以这种方式利用的转变。。。现在":](https://web.archive.org/web/20230205005724/http://www.borthwick.com/weblog/2009/04/19/699/)

> 首先，这是一个新的隐喻——想想流和页面。
> 
> 在网站的最初设计中，阅读和写作(编辑)得到了同等的考虑——然而 15 年来，网站的主要隐喻一直是页面和阅读。我们用来限定这个可能性集合的隐喻大多来自书籍和架构(页面、浏览器、网站等)。).这些隐喻大多是静态的，单向的。蒸汽隐喻是完全不同的。它是动态的，不能很好地存在于页面中，但仍在不断发展。
> 
> 一条小溪。一个实时的、流动的、动态的信息流——我们作为用户和参与者可以随意进出，无论我们是参与其中还是只是观察我们是这个流的一部分。

从某种意义上说，他是在试图合理化自己的投资策略。但是如果他是正确的，从网页到不断扩大的信息漩涡的转变将会对许多网络商业，尤其是媒体商业产生巨大的下游影响。这股上升的潮流有可能从根本上改变网络媒体传播的轮廓。像雅虎和美国在线这样的大型目的地网站，作为分销中心已经被搜索和社交网络削弱，现在面临着被完全忽略的前景。难怪 AOL 在它服务的每一个部分都坚持流，从它的 T2 主页到 T4 的 Bebo。(雅虎也在努力应对流媒体的出现，但到目前为止，它仍然认为自己能够保持其作为中心流量和分发中心的地位。

就此而言，信息流不会取代网页或搜索，但它有潜力彻底改变它们。我们已经看到网页[采用流](https://web.archive.org/web/20230205005724/https://techcrunch.com/2009/03/04/facebooks-response-to-twitter/)作为[新的用户界面](https://web.archive.org/web/20230205005724/https://techcrunch.com/2009/04/06/new-friendfeed-simpler-faster-better-maybe-too-fast/)。网页越来越多地被设计成展示最相关的信息流的地方。随着数据流无处不在，搜索作为一种导航工具变得比以往任何时候都重要。正如博思威克指出的:

> 在这个新世界中，流量并不是均匀分布的。突然之间，人群会出现在你的网站上。

流量是突发的，这取决于人们在那一秒钟在各种服务上关注什么。有人可能会在 Twitter 上注意到一篇不知名的博客文章，它开始在 Twitter 上传播，然后转移到 FriendFeed 和脸书以及 Tweetdeck 或 Seesmic desktop 等桌面流阅读器，在你知道之前，已经有十万人在阅读这篇文章了。该流创建了一种不同形式的辛迪加，这种辛迪加不能被许可也不能被控制。

这个问题比以往任何时候都更加成为信息过载的问题。你如何避免[在洪水中淹死](https://web.archive.org/web/20230205005724/https://techcrunch.com/2008/04/17/web-30-will-be-about-reducing-the-noise%E2%80%94and-twhirl-isnt-helping/)？Borthwick 建议放弃你可以掌控数据流的想法，即使只是你自己的朋友的推文、更新、博客帖子、Flickr 照片、YouTube 视频发现等个人数据流:

> 这不是一个我们必须清空的收件箱，也不是一个我们必须找到底部的页面——这是一个我们可以随意浏览的数据流，但我们不能试图获得它的无所不包的视图。

所以跳进小溪，让它带你离开。或者你可以胆怯地站在岸边，直到你周围的人都已经跳入水中。
