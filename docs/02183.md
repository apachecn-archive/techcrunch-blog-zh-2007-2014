# 先来看看 iPhone 的谷歌移动应用:很棒但不是第二次 TechCrunch

> 原文：<https://web.archive.org/web/https://techcrunch.com/2008/11/18/first-look-at-google-mobile-app-for-iphone-great-but-not-second-coming/amp/>

好吧，它发生了。谷歌的语音识别移动应用程序[今天终于在 iPhone 应用程序商店](https://web.archive.org/web/20230118110456/http://techcrunch.com/2008/11/17/googles-voice-search-finally-hits-the-iphone/)上架。直到今天，我们所能做的只是谷歌制作的演示视频。

这个视频展示了一些简单地改变了我使用手机的方式。我不再点击虚拟键盘上的按钮来搜索网页或我的联系人，而是点击一个按钮，使用谷歌移动应用程序。它真的只是一个按钮——通过加速度计，它知道何时将手机放在耳边，何时将手机拿走。瞧啊。很酷的事情发生了。

下面是视频，由谷歌移动团队的迈克·勒博解说:

【YouTube http://www.youtube.com/watch?v=y3z7Tw1K17A&hl=en&fs=1]

让我们比较一下那个视频和我的实际结果。首先，最令人失望的是，你不能通过语音搜索联系人——你必须输入，当普通的联系人应用程序工作得一样好时，仅仅使用这款应用程序来搜索联系人是不值得的。

此外，使用该应用程序时，背景噪音非常小也很重要。离我六英尺远的电热器发出持续的嗡嗡声，干扰了扬声器上的应用程序。当然，汽车的噪音会阻止开车时使用扬声器。下面的结果是在一个安静的房间里完成的，电话放在耳边，我尽可能清晰地说话。演示结果显示在左边，我的实际结果在右边。

联系人搜索也和视频显示的一模一样，但是有点误导。**不能语音搜索联系人，只能打字。视频显示了这一点，但到了那个时候，你已经完全沉迷于语音的美妙，你并没有真正意识到这一切都是在打字，这比使用手机自带的普通联系人应用程序好不了多少。**

总的来说，除了和大乌贼的一次混乱之外，一切都很顺利。但语音识别远非完美，正如演示视频所示。联系人搜索的限制令人失望。
