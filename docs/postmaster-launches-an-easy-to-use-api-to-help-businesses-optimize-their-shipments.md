# Postmaster 推出易于使用的 API 来帮助企业优化其发货

得克萨斯州奥斯丁的一家名为 [TechStars Cloud](https://web.archive.org/web/20221006172541/http://www.techstars.com/cloud/) 的公司 Postmaster 刚刚发布了一个测试版的[新 API](https://web.archive.org/web/20221006172541/https://www.postmaster.io/docs) ，它将允许企业优化他们的小包裹运输。

您可能会认为，在这一点上，运输几乎是一个已经解决的问题。事实上，开发人员仍然很难轻松地将他们的应用程序与主要运输商的 API 挂钩，并了解谁为他们的特定需求提供了最高效和最具成本效益的运输选项。另一方面，Postmaster 为开发人员提供了一个易于集成的 REST API，允许公司快速比较 FedEx、UPS 和 USPS(更多即将推出)之间的费率，并优化其运输成本和时间。

虽然像亚马逊这样的大公司使用复杂的算法来优化他们的出货量，但大多数小企业无法获得这些数据。例如，亚马逊知道何时可以使用基本的地面运输，在两天内将包裹从一个仓库送到你的手中，而不是使用更昂贵的服务。邮局主管允许任何人智能地发送包裹。

正如 Postmaster 首席执行官兼联合创始人杰西·洛夫莱斯(Jesse Lovelace)本周早些时候告诉我的那样，与现有运输解决方案[集成的主要难点之一不仅仅是它们往往是非常过时的 API，还需要让你打印的标签得到它们的认证。这可能需要很长时间，并且会给小型企业增加不必要的复杂性。“运营商整合可能会非常麻烦。Postmaster 不仅希望使这一过程比以往任何时候都更容易，而且希望通过利用运输数据和分析，确保尽可能高效和经济地优化、路由和计费每一批货物，”Lovelace 在今天的公告中还表示。](https://web.archive.org/web/20221006172541/http://www.fedex.com/us/developer/solutions.html)

除了为企业提供访问费率和运输标签，Postmaster 还通过 SMS、电子邮件和 webhooks 提供跟踪服务，以及地址验证。该公司目前提供 Ruby、Python 和 PHP 的客户端库，带有。NET、Java 和 Go 库即将推出。

目前，这项服务完全专注于开发者，但该团队也计划很快推出一款更加以消费者为中心的产品，允许用户在不打印标签的情况下快速比较运费。
