# URL 缩短战:Twitter 为了 bit.ly 抛弃了 TinyURL

当你需要输入一个占用了你大部分空间的网址来发布你的微信时，在推特上分享链接会是一件非常痛苦的事情。这家初创公司意识到这一点，迅速自动开始缩短长网址，让用户节省 140 个字符的更新空间。

为了实现这一点，Twitter 与 TinyURL(T4)合作，这是一种显著缩短网址的服务(但不是极端)，当时已经存在多年了。它从未给出这一选择的理由，但它确实为 TinyURL 提供了大量曝光率和大量额外流量。

那些日子已经过去了。显然，Twitter 已经悄悄地用 [bit.ly](https://web.archive.org/web/20230404020310/http://www.crunchbase.com/company/bit-ly) 取代了 TinyURL 作为其默认的网址缩短服务，这是一项最近才推出的竞争服务[从几位著名的天使投资者那里筹集了 200 万美元](https://web.archive.org/web/20230404020310/https://techcrunch.com/2009/03/30/if-bitly-is-worth-8-million-tinyurl-is-worth-at-least-46-million/)。

这其实没什么好惊讶的。Twitter 相关公司 Summize ( [于 2008 年 7 月被 Twitter 收购](https://web.archive.org/web/20230404020310/https://techcrunch.com/2008/07/15/confirmed-twitter-acquires-summize-search-engine/))背后的创业加速器 Betaworks 也在 bit.ly 之后，而且它恰好也将 Twitter 早期投资者和顾问[克里斯·萨卡](https://web.archive.org/web/20230404020310/http://www.crunchbase.com/person/chris-sacca)和[罗恩·康韦](https://web.archive.org/web/20230404020310/http://www.crunchbase.com/person/ron-conway)算作自己的支持者。

这显然引发了一个不可避免的问题:此举是否意味着 Twitter 为直接收购 URL 缩短服务提供商铺平了道路？
