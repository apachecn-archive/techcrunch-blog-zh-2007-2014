# 思科的卢·塔克谈万物互联以及与以应用为中心的世界的联系

思科的 Lew Tucker 今天站在 [Cloud Connect](https://web.archive.org/web/20230217010011/http://www.cloudconnectevent.com/santaclara/) 的舞台上，推销这家网络巨头的“万物互联”，这是一个以应用为中心的世界，在未来几年将价值 14.5 万亿美元。塔克说，物联网是我们世界中的所有物体，而 IoE 是智能电网，实际上是整个供应链及其转型。

大企业公司擅长这种事情。他们谈论着巨大的市场机会和有着巨大上升空间的美好未来，但问题是，面对创新如此迅速的初创公司，他们能有多敏捷。然而，塔克因解释了以应用为中心的世界如何与软件定义的网络(SDN)联系起来，以及从传统的重量级记录系统(ERP、CRM)向参与系统(应用，提供反馈循环的轻量级服务)的转变而受到好评。

塔克引用思科自己的研究称，通过智能电网、智能工厂、智能建筑和智能城市等事物的发展，有 4.9 万亿美元的直接机会。

这是关键，也是为什么塔克说的话是正确的。我们生活在一个未来的、以应用为中心的世界，这个世界必须在一个充满越来越过时的“物理”路由器和交换机的旧世界中运行。这就是 SDN 如此重要的原因，也是思科面临新玩家挑战的原因，如[大交换机](https://web.archive.org/web/20230217010011/http://www.bigswitch.com/)和 VMware，后者[去年收购了 NIC IRA](https://web.archive.org/web/20230217010011/https://techcrunch.com/2012/07/23/vmware-buys-nicira-for-1-26-billion-and-gives-more-clues-about-cloud-strategy/)。借助 SDN，网络可以连接到应用，而不是相反。计算、存储和网络会根据客户的需求不断优化。

他的陈述反映了你从亚马逊网络服务首席技术官[沃纳·威格尔](https://web.archive.org/web/20230217010011/http://www.crunchbase.com/person/werner-vogels)那里听到的内容，他谈到了成本感知架构，这种架构使公司能够根据自己的反馈循环来管理 IT 和应用程序工作负载。数据进入数据中心，由各种应用程序和服务读取。分析结果出来后，客户会使用在交易所交易的标准、保留或现货实例进行相应的调整。

这也进一步验证了任何数量的物联网创业公司的崛起。例如， [SmartThings](https://web.archive.org/web/20230217010011/http://smartthings.com/) 已经[筹集了 300 万美元](https://web.archive.org/web/20230217010011/https://techcrunch.com/2012/12/04/smartthings-closes-3m-seed-round-led-by-first-round-capital-launches-competition-to-grow-community-of-smart-object-developers/)来开发一个将日常物品连接到互联网的平台。据我们的 [Natasha Lomas、](https://web.archive.org/web/20230217010011/https://techcrunch.com/2012/11/01/the-diy-renaissance-u-k-accelerator-springboard-launches-dedicated-bootcamp-for-hardware-startups/)伦敦和剑桥报道，总部位于英国的加速器[跳板](https://web.archive.org/web/20230217010011/http://springboard.com/)正在为硬件初创公司推出一个专门的项目，重点是物联网。

Tucker 描述了 IoE 的良性循环，这是一个以应用程序为中心的世界。物联网意味着更多的数据，这需要云基础设施。随着我们将更多带有 API 的传感器植入我们的个人物品和其他一切物品:道路、桥梁等，应用程序正在成倍增加，并将继续增加。

IoE 还为我们与这种深层的互联事物互动的方式提供了一个背景。ERP 系统将变得与公司不那么相关。相反，参与系统将把我们置于反馈环的中心，让我们能够衡量自己，并在这个过程中连接到我们生活中所有其他智能的方面。这可能是在旧金山市试图找到一个停车位或智能工厂，我们在那里订购数据生成的个人物品。

“最终真正重要的是应用程序，”塔克说。“它们改变了我们对基础设施的需求。”

我完全同意。
