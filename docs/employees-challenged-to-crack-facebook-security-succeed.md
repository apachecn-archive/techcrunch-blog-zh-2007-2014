# 员工挑战破解脸书安全，成功(更新)

显然，脸书注意到了联邦贸易委员会在六月份对推特的严厉批评，因为它*未能阻止对其系统的未经授权的管理控制*不久之后，脸书负责 SRE(现场可靠性工程)的一名高级工程师向脸书的员工发出挑战，试图通过从他那里获得的信息来危害他并进入脸书的管理系统。

他们成功了。

虽然花了几个星期。我们的消息来源称，员工可能是通过他家的无线网络进入的。细节还不完全清楚，脸书也没有说话。据我所知，他们在捕获他的 WPA 密码后，通过引诱他登录似乎是他自己路由器的流氓 WiFi SSID，拦截了他家庭网络的数据。[点击这里](https://web.archive.org/web/20230403111109/http://webcache.googleusercontent.com/search?q=cache:VArK7JzNMyUJ:www.hackforums.net/archive/index.php/thread-321253.html+hack+wpa+via+fake+ssid&cd=2&hl=en&ct=clnk&gl=us&client=safari)了解这有多简单的细节。

一旦他的家庭网络瘫痪，脸书的员工就可以监控他所有的网络活动，获取明文密码等等。

去年的推特黑客攻击始于个人邮箱账户被攻破，并由此展开。

脸书这样做绝对是明智之举，其他公司也应该这样做。但是，如果脸书的一名安全工程师受到威胁，即使他知道这种威胁即将到来，想象一下其他人也受到攻击会是多么微不足道。

现在，请原谅我要去马克·扎克伯格的后院露营一两周，并尝试设置一个流氓 WiFi SSID。祝我好运。

**更新**:挑战背后的脸书工程师 Pedram Keyani，[在评论中回应了](https://web.archive.org/web/20230403111109/https://techcrunch.com/2010/07/05/employees-challenged-to-crack-facebook-security-succeed/#IDComment85143159)。他说，这个挑战实际上证明了脸书的安全性——虽然团队可以访问他的帐户，但他们无法破坏脸书的行政/公司系统。
