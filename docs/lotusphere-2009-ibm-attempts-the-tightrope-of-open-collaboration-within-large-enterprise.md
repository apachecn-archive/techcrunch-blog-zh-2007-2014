# lotus sphere 2009:IBM 尝试在大型企业中进行开放协作

![](img/d6a04bc46b9ada57a83d928a376f459e.png "lotus live")从今天到周四，我都在【2009 年乐透世界。

今天早上的主要公告:IBM 强调了与 LinkedIn、TripIt、Skype 和 Salesforce.com 的合作关系。在内部方面，Notes crew 演示了非常实用的 Blackberry 集成，集成在 Notes 中的语音聊天，以及非常好的基于浏览器的 UI。

重点显然是社交计算和协作——包括新的[Lotus Live](https://web.archive.org/web/20221208032026/https://www.lotuslive.com/)——IBM 最新的 SaaS 成果。非常及时，考虑到我在 Lotus Notes 和 Web 2.0 世界上的有争议的帖子。

显然，IBM 认识到了社交协作的增长，他们正试图通过前门传递这一点——是的，你可以使用 Flickr、Skype 等——但一切都是为了在 Notes 平台的环境中发生。

从 IT 管理的角度来看，这种环境更容易管理。但是从用户的角度来看，[协作挤过了墙](https://web.archive.org/web/20221208032026/http://www.techcrunchit.com/2008/12/07/disruptive-knowledge-management-facebooks-private-videos-shows-social-media-is-sneaking-through-the-backdoor/)——试图做笔记让知识管理操作系统感到受限。

当这些世界观继续碰撞时，它们永远不会融合。正如今天上午来自可口可乐、汇丰银行和 NetJets 的证明所表明的，大型企业更喜欢单一软件的可伸缩性。尽管用户不断抱怨缺乏灵活性。

[的 Buzz bruggerman](https://web.archive.org/web/20221208032026/http://www.activewords.com/)和我讨论了合作关系和单一软件之间的紧张关系——他想知道为什么 IBM 不与 twitter 合作:“如果‘网络’的下一阶段是关于发现而不是搜索，那么企业领域的任何人都应该利用 Twitter 的力量。否则，他们会被落下。"
