# 谷歌打算通过取消谷歌语音、将功能整合到视频聊天工具中来巩固沟通

根据 9to5Google 的一份新报告，据说谷歌正计划将谷歌语音功能引入其跨平台通信应用 Hangouts，可能会及时用于谷歌 I/O。Hangouts 应用程序不久前获得了语音呼叫，还提供了 Android 上的[短信集成，用于向电话号码发送文本，这两项都是谷歌语音的功能。](https://web.archive.org/web/20230129063707/https://techcrunch.com/2013/10/29/google-adds-sms-support-to-its-android-messaging-app-as-facebook-axes-it/)

此举是有意义的，不仅因为已经有一些功能重叠，而且因为谷歌已经逐渐将 Hangouts 从一个简单的基于网络的视频聊天工具转变为一个多平台的聊天客户端，支持文本，视频和音频通信，以及富媒体。在最新版本的 Android (4.4 KitKat)中，Hangouts 取代了原生的 Android 短信应用程序，之前它还杀死了 Messenger 和 Talk。

根据该报告，Hangouts 可能会集成使用 VoIP 连接免费拨打电话号码的功能，就像目前在网络上从 Gmail 拨打电话一样。这是运营商可能会反对的事情，因为它绕过了他们自己的语音服务产品，但如果它获得通过，它可能会为 Android 设备铺平道路，这些设备只需一个数据计划就可以作为全功能手机运行，这似乎是谷歌语音的最初目标之一。

如果 Hangouts 最终也取代了手机应用，谷歌将在其 Android 主屏幕上有一个相当空闲的 dock，但就我们目前通常使用设备的方式而言，整合是很有意义的(语音通信越来越罕见，有时如果文本不足，就需要语音通信)。苹果也一直在转向一种模式，在这种模式下，它的更多原生电话功能都放在同一个地方，位于 FaceTime 和 iMessage/Messages.app 之间，因此谷歌也进行汇总和整理并不奇怪。
