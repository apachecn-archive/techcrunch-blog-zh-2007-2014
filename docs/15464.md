# Glympse 推出首个 API，可在任何应用或平台中共享位置跟踪技术

> 原文：<https://web.archive.org/web/https://techcrunch.com/2013/05/13/glympse-launches-its-first-api-to-put-location-sharing-into-any-app-or-platform/>

Glympse 因与福特、奔驰和宝马/Mini 等公司达成协议，将其位置共享和跟踪技术集成到联网汽车的车载系统中而成为新闻。今天，它的扩张战略更进一步，发布了一个[新软件开发包](https://web.archive.org/web/20221207045943/http://www.glympse.com/developers)，让应用程序开发者和其他人能够通过几行代码将 Glympse 支持的位置共享技术纳入他们的服务。

这一消息是在社交地图技术成为新闻的时候出现的，据报道，脸书[正在以高达 10 亿美元的价格收购 Waze，阿里巴巴](https://web.archive.org/web/20221207045943/https://beta.techcrunch.com/2013/05/09/reports-facebook-is-buying-social-mapping-and-traffic-app-waze-for-up-to-1b-to-court-more-mobile-users/)[向高德投资](https://web.archive.org/web/20221207045943/https://beta.techcrunch.com/2013/05/09/alibaba-autonavi/)近 3 亿美元，建立战略联盟，开发基于位置的商务和其他移动导航和地图服务。

虽然 Waze 已经开发了一种整理众包地图和交通数据的方法，但 Glympse 并没有自己创建地图——正如你在下面的例子中看到的那样，地图数据可以来自谷歌，也可以来自微软的 Bing、Open Streetmap 和其他公司——但其位置跟踪技术可以有效地让你创建一个实时轨迹，显示你到特定位置的路线。

Glympse 的首席执行官兼联合创始人 Bryan Trussel 表示，已经有许多公司与 Glympse 接洽，寻求将其技术集成到新应用程序中的方法——这些领域是该公司自己目前没有资源解决的。其中之一涉及到与航空旅行相关的应用集成:当一个人的飞机从 A 点飞到 B 点时，跟踪他的位置，这对等待从机场接他的人很有用。你可以看到像这样的技术如何用于休闲应用，例如游戏，但也可以用于更实际的场景，如物流跟踪。

Trussel 表示，SDK 将有效地成为 Glympse 已经向汽车公司和 Garmin 等合作伙伴提供的私有 API 的一个版本。

此时，Glympse 将继续扩大合作伙伴名单，并扩展到其他垂直市场。

他在一次采访中说，“我们每六个月就有一个重要的合作伙伴，我们计划以每两个月一个的速度进行更多的合作。”。"一些汽车合作伙伴，但大多数将在汽车领域之外."这也可能扩展到 Glympse 技术开始出现在移动设备上的许可协议。事实上，已经有一些非汽车行业的公司在使用 Glympse 的技术。其中包括 Gripwire(应用程序开发)、PetHub(宠物保护)和 Runtriz(酒店解决方案)。

Glympse 将以 Lite SDK 的形式向 300，000 或更少用户的实现免费提供 API 的使用。该免费 SDK 将包括向移动应用程序添加 Glympse 功能以及地图工具的能力，供开发人员创建和托管自定义 Glympse 地图。SDK 将允许用户添加 GPS 和位置管理、联系人集成和查看权限，以及用户界面的编码，以便用户在第三方应用程序中共享位置。

Glympse 表示，进一步的付费商业 SDK 是为预期每月活跃用户超过 300，000，或需要更多支持、用户体验流灵活性或创建更多自定义功能的开发人员和企业设计的。

那么，为什么现在才推出 API 呢？Trussel 表示，Glympse 从一开始就收到了许多使用该平台的请求，但“我们决定不首先使用该平台，因为我们希望它稳定并有记录。拥有一个 SDK 意味着处理支持和问题，我们将资源用于直接与客户合作和改进平台。现在，我们的合作伙伴正在以相同的方式使用平台，因此我们可以处理人们以许多不同方式使用的变化。时机对我们来说是正确的。”

到目前为止，Glympse 已经从包括 Menlo Ventures 和 Ignition Partners 在内的投资者那里筹集了 750 万美元。
