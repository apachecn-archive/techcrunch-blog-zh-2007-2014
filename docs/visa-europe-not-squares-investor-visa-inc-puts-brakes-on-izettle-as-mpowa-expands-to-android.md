# Visa Europe，而不是 Square 的投资者 Visa Inc，在 mPowa 扩张到 Android TechCrunch 时给 iZettle 踩了刹车

> 原文：<https://web.archive.org/web/https://techcrunch.com/2012/07/30/visa-europe-not-squares-investor-visa-inc-puts-brakes-on-izettle-as-mpowa-expands-to-android/>

今天，移动支付遭遇了一点挫折，特别是[ize ttle](https://web.archive.org/web/20221209155929/https://www.izettle.com/)——这家初创公司被塑造成一种“欧洲广场”，供商户在手机上接受卡支付。该公司今天不得不在三个市场停止接受 Visa 卡，这三个市场是丹麦、芬兰和挪威，原因似乎是该公司在其平台上使用电子签名而不是 PIN 认证用户的方式存在问题。当用户开始收到来自 iZettle 的电子邮件时，问题就暴露出来了(下面我们有一封来自一位读者的邮件)，而此时销售点移动支付公司之间的竞争正在升温，你可能更好地知道 [mPowa](https://web.archive.org/web/20221209155929/http://www.mpowa.com/) ，也就是 [Square 的对手](https://web.archive.org/web/20221209155929/https://beta.techcrunch.com/2012/07/13/mpowa-takes-down-hand-photo/)，现在将其支付服务扩展到 Android 据报道， [Square](https://web.archive.org/web/20221209155929/http://www.squareup.com/) 以 32 亿美元的估值获得了 2 亿美元的投资。

iZettle 的首席执行官雅各布·德吉尔(Jakob de Geer)将停止接受签证的必要性描述为“令人失望”、“令人恼火”和“令人困惑”，但他也指出，他认为这只是一个短期问题:“我非常有信心，我们会找到前进的道路，”他告诉 TechCrunch。“你必须重组和适应；这是游戏的一部分，也是处于颠覆性空间的一部分。”就其本身而言，Visa 欧洲发言人告诉 TechCrunch，问题源于 Visa 的“标准受理设备要求”:

> “Visa Europe 支持开发新型受理设备，供以前不受理信用卡的小商户使用。然而，Visa Europe 尚未同意在 iZettle 平台上接受 Visa 卡，因为它目前不符合我们的标准受理设备要求。”

德吉尔说，这三个国家约有 15，000 名活跃的商户受到影响，Visa Europe 和 iZettle 之间的纠纷“已经进行了一段时间”

美国的 Visa Inc .是 Square 的投资者，万事达卡(MasterCard)最近也成为了 ize ttle(T1)的投资者，因此很容易认为这个故事可能是两家更大公司之间展示实力的一个例子:或许是 Square 为欧洲推出做准备的一个迹象？

德吉尔表示，这不太可能，因为 Visa Europe 是一个由不同市场成员所有的协会，而 Visa Inc 是一家美国上市公司。“两种完全不同的生物以不同的方式工作，”德吉尔说。“我认为人们把这些联系在一起，但这与广场投资没有任何关系。”

然而，值得注意的是，在 [Visa 的公司网站](https://web.archive.org/web/20221209155929/http://corporate.visa.com/about-visa/our-business/visa-inc-and-visa-europe.shtml)上，Visa Inc .和 Visa Europe 之间的关系被描述为“……相互为对方的成功投资。这两个组织由一个共同的全球品牌联合起来，共同努力确保 Visa 的品牌完整性、互操作性、可靠性以及 Visa 产品和系统的安全性。”因此，这种紧缩政策可能有比看上去更多的原因。(谢谢，[帕特里克·斯文松](https://web.archive.org/web/20221209155929/https://twitter.com/skirtap)，关于这个细节的提示。)

然而，这无疑突出了一点，即与现有参与者的需求相联系的业务可能是一个混乱和令人困惑的业务。德吉尔说，iZettle 已经被万事达卡、美国运通、大来俱乐部和芯片卡的全球标准 EMV 批准。

(这也潜在地指出了这些不同支付服务的另一个问题:它们都过于依赖现有的大型参与者。瑞士的金融服务专家 Yann Ranchere 指出,[这是 PayPal 或 Dwolla 脱颖而出的地方，也是它们更具破坏性和成功的潜在可能。)](https://web.archive.org/web/20221209155929/https://twitter.com/tek_fin/status/229879865703268352)

Visa 欧洲的发言人不会告诉 TechCrunch iZettle 没有满足哪些要求，TechCrunch 知道这可能与 ize ttle 平台上的支付认证方式有关:消费者使用卡上的芯片支付——不同于美国卡上最常见的磁条——但不输入 PIN 进行认证；相反，他在设备的屏幕上签名。德吉尔声称，这在北欧是一种可接受的支付形式，手机上的 PIN 不太安全，但他承认，无论如何，Visa 可能希望将 PIN 元素纳入平台(iZettle 还指出，其方法得到了 EMV 的批准)。

令人困惑的是，德吉尔告诉我们，在瑞典，iZettle 有一项商业服务在运行，不必关闭其签证支付选项。在英国，iZettle 正在进行一项试验，从未将 Visa 作为其服务的一部分——可能是因为这个持续的问题。

Visa Europe 不愿透露它是否正在向其他移动支付提供商发出类似的订单，但提供与 iZettle 和 Square 类似产品的 mPowa 确认了这一点。

mPowa 首席执行官丹·瓦格纳(Dan Wagner)对 TechCrunch 表示:“我们没有问题，因为我们不是芯片和签名。事实上，该公司还没有推出芯片和密码服务。他说，这些将在两个月后到来。该公司最近将其加密狗服务扩展到安卓设备和 iOS 系统。它还计划在“适当的时候”升级 Windows。他表示，该公司目前正接受 NDA 与“主要银行和电信公司”的委托，在不同市场推出 mPowa 支付服务的白标版本。

iZettle 也在其[博客](https://web.archive.org/web/20221209155929/https://www.izettle.com/2012/07/30/letter-to-izettlers-in-denmark-finland-and-norway/)上发布了这条消息。

**更新** : iZettle 希望进一步澄清它与 mPowa 的区别。“我们是‘一站式商店’，而使用 mPowa，您需要单独注册才能获得银行的商户账户，甚至能够获得第一笔付款(获得商户账户通常是一个漫长的过程)，”一位发言人指出。“我们也为个人提供 iZettle 的个人账户，但 mPowa 的服务只针对商家。”
