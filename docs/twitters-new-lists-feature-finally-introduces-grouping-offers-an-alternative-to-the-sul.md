# Twitter 的新“列表”功能最终引入了分组，为 SUL 提供了另一种选择

Twitter 刚刚[宣布了](https://web.archive.org/web/20230404082416/http://blog.twitter.com/2009/09/soon-to-launch-lists.html)一项新功能，这肯定会让那些一直在等待该网站提供其推荐用户列表替代品的人兴奋不已。这个新功能被恰当地称为“列表”，它将允许任何人列出其他 Twitter 用户的列表，并对其进行适当的标记(例如，我可以列出一个名为“TC Staff”的列表)，然后与其他成员分享该列表。Twitter 写道，该功能仍在有限的测试中，但最终将向所有用户推出。

默认情况下，您创建的任何列表都是公开的，尽管您也可以隐藏它们。如果你选择公开他们，其他 Twitter 用户将能够点击一个按钮“关注这个列表”,这样他们就可以一次添加所有人。这是一件大事——到目前为止，在 Twitter 上开始大规模关注人们的唯一便捷方式是使用自己策划的 SUL。如果我们看到一些用户争相成为最好的“名单制造者”，提供名人、新闻门户、博客等全面的名单，我不会感到惊讶。看看 Twitter 是否汇总了最全面的列表，并将其作为注册过程的一部分(这实际上只是 SUL 的一种不同形式)，也将是一件有趣的事情。

Twitter 的帖子描述了这项新功能，但它似乎比 discovery 有更大的影响——它也可能被用于*分组*，这是一些第三方应用程序已经提供的功能，但尚未得到 Twitter 的官方支持。简而言之，这将让你把你关注的人分成不同的列表(比如，一个是新闻，一个是密友，等等)，然后在他们之间快速跳转。Twitter 将通过其 API 支持新的列表功能，因此我们可以预计这一功能将扩展到大多数尚不支持分组的第三方应用程序。

同样值得注意的是:分组的逻辑扩展将是支持组内搜索，这可以帮助用户清除垃圾邮件。当然，我们谈论的是 Twitter，所以在我们看到这种情况发生之前，可能还有很长一段时间。

“列表”可能会与像 [WeFollow](https://web.archive.org/web/20230404082416/http://www.wefollow.com/) 这样的网站竞争，这些网站专门列出各种类别的顶级 Twitter 用户(另一方面，如果这些网站成为寻找最全面列表的最佳地点，它们实际上可能会变得更受欢迎)。它也有可能取代 [TweepML](https://web.archive.org/web/20230404082416/http://tweepml.org/) ，这是 Twitter 用户群共享的开放标准格式。
