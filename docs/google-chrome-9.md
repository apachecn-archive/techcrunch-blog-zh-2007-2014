# 谷歌发布 Chrome 9，并感谢 Reddit 帮助修复

> 原文：<https://web.archive.org/web/http://techcrunch.com/2011/02/03/google-chrome-9/>

# 谷歌发布了 Chrome 9，并感谢 Reddit 对其修复的帮助

谷歌最近大谈不要在 Chrome 的版本号上小题大做。“这只是一个数字”是这些天的基本观点。事实上，有时他们甚至懒得让 T2 宣布这些消息。但今天，他们确实花了一些时间来承认最新版本——以及一个使这一切成为可能的有趣的合作伙伴。

谷歌已经正式推出 Chrome 9，这意味着该浏览器的稳定版本已经更新到该迭代。他们没有在[博客帖子](https://web.archive.org/web/20230203044226/http://chrome.blogspot.com/2011/02/dash-of-speed-3d-and-apps.html)中提到任何关于更新的数字，但是相信我，这是第 9 版。相反，谷歌专注于这一版本中所有人都可以使用的三个新的主要功能:WebGL、Chrome Instant 和 Chrome 网络商店。

许多用户已经熟悉了所有这些，因为他们已经在 Chrome 的测试版上呆了一段时间，而开发版在此之前更久。但是，每一个都值得注意，因为这三个现在都是 Chrome 体验中不可或缺的一部分。WebGL 带来了硬件加速的 3D 图形支持。Chrome Instant 为 Omnibox 带来了谷歌即时支持。Chrome 网络商店是谷歌进一步推动网络应用计划的一部分。

但是更幽默的部分来自于谷歌 Chrome 发布的博客文章。这篇文章详细介绍了 Chrome 9 中的错误修复。在这篇文章中，谷歌对 Reddit 社区指出了一个严重的错误给予了特别的表扬。正如谷歌指出的:

> 特别感谢 Reddit 社区，他们玩了这么多游戏“Z-Type ”,发现了一个 Chromium 音频 bug——见下文！

下面，他们详细描述了这个错误:

> [ [69195](https://web.archive.org/web/20230203044226/http://code.google.com/p/chromium/issues/detail?id=69195) ]音频处理中的临界竞争条件。红迪网的玩家们！

所以你看，虽然这不是一个“[精英](https://web.archive.org/web/20230203044226/https://techcrunch.com/2011/01/12/chrome-elite-bug/)”的错误，甚至不是一个“ [l33t](https://web.archive.org/web/20230203044226/https://techcrunch.com/2010/03/19/google-chrome-1337/) ”的错误，但看到谷歌因为指出另一个服务的缺陷而给予它信任是很有趣的。
