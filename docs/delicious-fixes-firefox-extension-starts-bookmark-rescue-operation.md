# 美味修复火狐扩展，开始“书签拯救行动”

[AVOS](https://web.archive.org/web/20230314130530/http://www.crunchbase.com/company/avos) 在[最近推出](https://web.archive.org/web/20230314130530/https://techcrunch.com/2011/09/27/you-can-now-get-a-taste-of-the-new-delicious-screenshots-video/)[全新美味](https://web.archive.org/web/20230314130530/http://www.delicious.com/)的后，一直受到前[美味](https://web.archive.org/web/20230314130530/http://www.crunchbase.com/company/delicious)用户和粉丝的[整](https://web.archive.org/web/20230314130530/http://www.readwriteweb.com/archives/new_delicious_is_a_bitter_dissapointment.php) [批](https://web.archive.org/web/20230314130530/http://www.zdnet.com/blog/violetblue/avos-delicious-disaster-lessons-from-a-complete-failure/705)。当然，在发布前再等几个月或许会更好。

但是 AVOS 很快着手解决一些问题，而[宣布](https://web.archive.org/web/20230314130530/http://deliciousengineering.blogspot.com/2011/09/welcome-to-delicious-beta-status-blog.html)正在解决其他问题。

在“Delicious beta status blog”上的一个新帖子中，AVOS 刚刚[宣布](https://web.archive.org/web/20230314130530/http://deliciousengineering.blogspot.com/2011/09/firefox-extension-fixed-after-long.html)现在已经修复了 Firefox 扩展的一个问题，更重要的是，启动了一个“书签拯救行动”来安抚那些丢失了旧的 Delicious 书签的前用户:

> 对于那些错过了选择加入警告的人，雅虎！在过去的几个月里，我们一直在努力制定一个解决方案来挽救您的书签。我们已经设计了一个巧妙的解决方案，明天应该会有更多的消息。敬请关注…

正如我在新的 Delicious 推出时所写的，技术人群是否(再次)爱上这项服务并不重要，重要的是它是否能吸引大众。

也就是说，我认为这可能是一个好主意，让 AVOS 证明他们关心并愿意解决老用户的担忧。
